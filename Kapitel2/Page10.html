Grid Search & Hyperparameter-Tuning

Was passiert, wenn wir mehr als einen Hyperparameter haben? Dann wird aus einer Validierungskurve eine mehrdimensionale Fehlerfläche. GridSearchCV sucht das Optimum dieser Fläche systematisch – und liefert die beste Hyperparameter-Kombination.

Fehlerfläche param_grid GridSearchCV .best_params_ .best_estimator_ Abschluss

Lernfokus dieser Seite

Du kannst GridSearchCV einsetzen – von der param_grid-Definition über .fit() bis zur Nutzung von .best_params_ und .best_estimator_. Du verstehst, warum Grid Search bei mehreren Hyperparametern nötig ist und was eine mehrdimensionale Fehlerfläche bedeutet.

10.1 Grid Search – Grundkonzept

Validierungs- und Lernkurven haben uns einen Einblick in das Trainieren von ML-Algorithmen verschafft – aber in der Praxis haben Modelle mehr als einen Hyperparameter.

Vom Kurven- zum Flächen-Problem

  • Validierungs- und Lernkurven geben Einblick in ML-Training
  • In der Praxis: ML-Modelle haben mehr als einen Hyperparameter
  • Dadurch werden aus Validierungskurven mehrdimensionale „Flächen"
  • Daher: in der Praxis nur den Validierungs-Score untersuchen
  • Optimum = globales Maximum des Validierungs-Scores

Warum heißt es „Grid Search"?

  • Der Parameterraum wird als Gitter (Grid) aufgespannt
  • Jeder Gitterpunkt = eine Hyperparameter-Kombination
  • An jedem Punkt wird ein Modell trainiert und validiert
  • Am Ende: der Punkt mit dem höchsten Validierungs-Score = Best Model
  • Brute-Force: alle Kombinationen werden durchsucht

10.2 GridSearchCV – Definition & Implementation

Man definiert vorab einen param_grid mit den zu suchenden Hyperparameter-Bereichen. GridSearchCV übernimmt dann den Rest.

sklearn API: GridSearchCV

  • GridSearchCV(model, param_grid, cv=k)
  • Beim Grid Search gibt man vorab bestimmte Bereiche der Hyperparameter vor
  • Diese Bereiche werden „abgefahren" / „gesweept" / „getuned": für jede Parameterkombination wird ein Modell trainiert
  • Jedes dieser Modelle resultiert in einen Validierungs-Score
  • Aufgabe: Optimum / Maximum des Validierungs-Scores in Abhängigkeit von der Parametereinstellung finden

Schritt 1 — Hilfsfunktionen vorbereiten

Wir benötigen wieder unsere Hilfsfunktionen PolynomialRegression() und make_data() aus Seite 9. Hier zur Vollständigkeit nochmals definiert.

Python – Setup (Imports & Hilfsfunktionen) 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression

def PolynomialRegression(degree=2, **kwargs):
    return make_pipeline(
        PolynomialFeatures(degree),
        LinearRegression(**kwargs))

def make_data(N, err=1.0, rseed=1):
    rng = np.random.RandomState(rseed)
    X = rng.rand(N, 1) ** 2
    y = 10 - 1. / (X.ravel() + 0.1)
    if err > 0:
        y += err * rng.randn(N)
    return X, y

X, y = make_data(50)
Erklärung
  • PolynomialRegression(degree) = Pipeline aus PolynomialFeatures + LinearRegression
  • make_data(50): 50 synthetische Datenpunkte
  • Pipeline-Parameter: stepname__param (doppelter Unterstrich)

Schritt 2 — param_grid definieren & GridSearchCV erstellen

Das param_grid-Dictionary enthält alle zu suchenden Hyperparameter-Bereiche. Die Keys folgen der Pipeline-Konvention: stufenname__parametername (doppelter Unterstrich).

Python – param_grid & GridSearchCV 
from sklearn.model_selection import GridSearchCV

# Hyperparameter-Bereiche definieren
param_grid = {
    'polynomialfeatures__degree':
        np.arange(21),              # 0 bis 20
    'linearregression__fit_intercept':
        [True, False],              # 2 Optionen
}

# GridSearchCV Instanz erstellen
grid = GridSearchCV(
    PolynomialRegression(),
    param_grid,
    cv=7)

# Anzahl Kombinationen berechnen
n = 21 * 2
print(f"Anzahl Kombinationen: {n}")
print(f"Anzahl CV-Splits:     7")
print(f"Anzahl Trainings:     {n * 7}")
Ausgabe
Anzahl Kombinationen: 42
Anzahl CV-Splits: 7
Anzahl Trainings: 294
  • 21 Degree-Werte × 2 fit_intercept-Werte = 42 Kombinationen
  • Für jede → 7-Fold CV → 294 Trainings!
  • → Das ist der „Preis" des Grid Search

Alternativ: GridSearchCV mit expliziter Pipeline (Folie 47, Teil 2)

Man kann GridSearchCV auch direkt mit einer expliziten sklearn Pipeline und allen 3 Parametern (inkl. fit_intercept) definieren. Beide Varianten führen zum gleichen Ergebnis.

Python – GridSearchCV mit Pipeline (Folie 47) 
from sklearn.pipeline import Pipeline

# Explizite Pipeline-Variante
grid_pipeline = GridSearchCV(
    cv=7,
    estimator=Pipeline(steps=[
        ('polynomialfeatures', PolynomialFeatures()),
        ('linearregression',   LinearRegression())
    ]),
    param_grid={
        'polynomialfeatures__degree': np.arange(21),
        'linearregression__fit_intercept': [True, False]
    }
)

print("param_grid keys:")
for k in grid_pipeline.param_grid:
    print(" ", k)
Ausgabe
param_grid keys:
  polynomialfeatures__degree
  linearregression__fit_intercept
  • Beide Varianten (PolynomialRegression()-Shortcut vs. explizite Pipeline) sind äquivalent
  • Pipeline-Benennung: stepname__param
  • Stufenname = lowercase Klassenname

10.3 GridSearchCV – Anwendung & Ergebnisse

Wie ein normales ML-Modell: GridSearchCV per .fit() anpassen, dann .best_params_ und .best_estimator_ nutzen.

Schritt 3 — .fit() ausführen & .best_params_ ausgeben

grid.fit(X, y) startet den kompletten Grid Search – alle Kombinationen werden per Cross-Validation evaluiert. Danach stehen .best_params_ und .best_score_ bereit.

Python – grid.fit() + best_params_ 
# Grid Search ausführen (dauert etwas!)
grid.fit(X, y)

# Beste Hyperparameter ausgeben
print("Best Params:")
print(grid.best_params_)

# Bester CV-Score
print("\nBest CV-Score:", grid.best_score_.round(4))
Ausgabe (Folie 48)
Best Params:
{'linearregression__fit_intercept': False,
 'polynomialfeatures__degree': 4}

Best CV-Score: 0.9894
  • Optimaler Polynomgrad: degree=4
  • fit_intercept=False ist besser für diese Daten
✅ grid.best_params_ – Beste Hyperparameter-Kombination
'linearregression__fit_intercept' False Achsenabschnitt nicht fitten
'polynomialfeatures__degree' 4 Optimaler Polynomgrad

Schritt 4 — .best_estimator_ nutzen & plotten

grid.best_estimator_ gibt das komplett konfigurierte Modell mit den besten Hyperparametern zurück. Wir fitten es auf den gesamten Datensatz und visualisieren den resultierenden Fit.

Python – best_estimator_ nutzen & plotten 
# Bestes Modell extrahieren
model = grid.best_estimator_

# Scatter + Fit plotten
plt.scatter(X.ravel(), y, label='Daten')
lim = plt.axis()

X_new = np.linspace(0, 1, 100).reshape(-1, 1)
y_new = model.fit(X, y).predict(X_new)

plt.plot(X_new, y_new, color='blue',
         label='Best Model (degree=4)')
plt.axis(lim)
plt.legend()
plt.xlabel('x'); plt.ylabel('y')
plt.title('GridSearchCV – Bestes Modell')
plt.show()
Beobachtung
  • 📊 Scatter mit blauer Kurve (degree=4)
  • Glatter, realistischer Fit – kein Overfitting
  • Kurve folgt dem allgemeinen Trend der Daten
  • .best_estimator_ = voll konfiguriertes Modell
  • Kann direkt wie ein normales sklearn-Modell verwendet werden

GridSearchCV als normales Modell

  • Wie ein normales ML-Modell auch: GridSearchCV auf Daten anpassen mit .fit(X, y)
  • .best_params_ → beste Hyperparameter-Kombination als Dictionary
  • .best_estimator_ → konfiguriertes Modell mit Best Params
  • .best_score_ → bester CV-Score

Zusammenfassung – Grid Search Prozess

  • Schritt 1: param_grid Dictionary definieren
  • Schritt 2: GridSearchCV(model, param_grid, cv=k)
  • Schritt 3: grid.fit(X, y) – alle Kombinationen testen
  • Schritt 4: grid.best_params_ und grid.best_estimator_ nutzen
💡 So what?

GridSearchCV automatisiert das Hyperparameter-Tuning komplett. Nach dem .fit() verhält es sich wie ein normales sklearn-Modell. Rechenintensiv bei vielen Kombinationen – aber vollständig automatisiert.
