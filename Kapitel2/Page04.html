4.1 Clustering

Ein typisches Unsupervised-Learning-Problem: ungelabelte Daten werden diskreten Gruppen zugeordnet. Das Modell entdeckt die Struktur selbst – eng verwandt mit der Dichteschätzung der zugrundeliegenden Daten.

💬 Denkfrage 2 – aus der Vorlesung

„Was denken Sie? Welches Problem sehen Sie hier?" (Hinweis: Wie viele Cluster gibt es wirklich?)

Clustering – Das Konzept

  • Ein typisches Unsupervised Learning Problem
  • Ungelabelte Daten werden diskreten Gruppen bzw. Labels zugeordnet
  • Eng verwandt mit der Dichteschätzung der zugrundeliegenden Daten
  • Neue Datenpunkte können durch Abstandsbetrachtungen einem bestimmten Cluster zugewiesen werden
  • Ergebnis: diskrete Cluster-IDs (0, 1, 2, ...) statt kontinuierlicher Labels

Das offene Problem: Wie viele Cluster?

  • Im Beispiel: 3 oder 4 Cluster? → nicht eindeutig erkennbar
  • Die Wahl von k (Anzahl Cluster) ist ein zentrales Problem
  • Das Abstandsmaß bestimmt, welche Cluster das Modell „sieht"
  • Typische Maße: Euklidischer Abstand, Manhattan, Kosinus-Ähnlichkeit
  • Lösungsansätze: Elbow Plot, Silhouette Score → ausführlich in Kapitel 4 (Woche 5)
💡 So what?

Beim Clustering gibt es keine „richtige Antwort" im Sinne eines bekannten Labels – das Modell entscheidet selbst, welche Gruppen sinnvoll sind. Das Abstandsmaß und die Anzahl der Cluster k sind entscheidende Hyperparameter, die man als Mensch festlegt.

4.2 Dimensionsreduktion

Dimensionsreduktion – Das Ziel

  • Ein weiteres Problem des Unsupervised Learnings
  • Ziel: eine niedrigdimensionale Repräsentation der Daten, die die meisten/wichtigsten Eigenschaften erhält
  • Im Beispiel: 2D-Daten auf einer Spirale → eigentlich nur 1 Parameter nötig
  • Das Modell entdeckt diese innere Struktur selbst
  • Anwendungen: Visualisierung, Datenkompression, Feature-Engineering

Das Spiralen-Beispiel: Intrinsisch eindimensional

  • Die 2D-Daten liegen auf einer Spirale – visuell erkennbar
  • Obwohl 2 Features (x, y) vorhanden: die Spirale ist intrinsisch eindimensional (nur 1 Koordinate entlang der Spirale nötig)
  • Die Farbe in der rechten Grafik = die entdeckte latente Variable (Position entlang der Spirale)
  • Das Modell hat eine neue Koordinatenachse entdeckt – die nicht in den originalen Features existiert

Latente Variable – Eine neue Koordinatenachse

  • Die Farbe in der Grafik (Folie 17 rechts) entspricht einer sog. latenten Variable
  • Latent = verborgen – sie war nicht in den originalen Features (Feature 1, Feature 2) vorhanden
  • Das Modell entdeckt diese neue Koordinatenachse, die die Datenstruktur beschreibt
  • Im Spiralen-Beispiel: die latente Variable = Position entlang der Spirale (von Anfang = gelb bis Ende = dunkelviolett)
  • Farbskala rechts neben der Grafik: zeigt den Wertebereich der gelernten latenten Variable
💡 So what?

Dimensionsreduktion ist mächtig, weil sie versteckte Struktur in Daten aufdeckt. Wenn 2D-Daten auf einer Spirale liegen, brauchen wir nur 1 Zahl, um jeden Punkt eindeutig zu beschreiben – das spart Rechenaufwand und hilft bei der Visualisierung. Die latente Variable ist das, was das Modell dabei lernt.

Zusammenfassung: UL – Clustering vs. Dimensionsreduktion

Beide sind Unsupervised-Learning-Methoden ohne Labels – aber sie lösen unterschiedliche Probleme.

Aspekt 🟢 Clustering 🔵 Dimensionsreduktion
Ziel Datenpunkte in diskrete Gruppen einteilen Daten in weniger Dimensionen darstellen
Output Diskrete Cluster-Labels (0, 1, 2, ...) Latente Variablen / reduzierte Koordinaten
Kernidee Abstandsbetrachtung + Dichteschätzung Intrinsische Struktur entdecken
Beispiel Grüne/gelbe/blaue Cluster im Scatter-Plot Spirale → 1D latente Variable (Farbe)
Problem Wie viele Cluster k? Wie viele Dimensionen bleiben?

💡 Gesamtfazit – Unsupervised Learning Konzepte

Unsupervised Learning ist mächtig, weil es ohne Labels auskommt – aber genau das macht es auch schwieriger zu validieren. Clustering liefert diskrete Gruppen, Dimensionsreduktion liefert kompaktere Repräsentationen. In beiden Fällen gilt: model.fit(X) – kein y! Das Modell entdeckt die Struktur selbst.
