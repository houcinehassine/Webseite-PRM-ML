💡 So what?

sklearn hat einen sehr zugänglichen und konsistenten Aufbau – jeder ML-Algorithmus hat denselben API-Aufbau. Das bedeutet: wenn du einen Algorithmus kennst, kennst du im Prinzip alle – zumindest was die Syntax angeht.

5.2 Generischer 4-Schritte-Prozess

Jeder ML-Algorithmus in sklearn folgt demselben Ablauf. Dieser Prozess ist universell – für Supervised und Unsupervised Learning gleichermaßen.

1
Wähle Modell & importiere entsprechende Klasse aus sklearn
  • Importiere das gewünschte Modell aus dem entsprechenden sklearn-Modul: from sklearn.X import Y
  • Beispiele: from sklearn.linear_model import LinearRegression | from sklearn.cluster import KMeans
  • Das Modul (sklearn.X) gibt den Bereich an: linear_model, cluster, svm, ...
2
Wähle Hyperparameter durch entsprechende Instanziierung
  • Erstelle eine Instanz der Modellklasse mit den gewünschten Hyperparametern: model = Y(hyperparameter=wert)
  • Hyperparameter = werden vor dem Training festgelegt (z.B. n_clusters=3, max_depth=5)
  • Im Gegensatz zu Parametern = werden durch das Training gelernt
3
Trainiere/fitte das Modell via .fit()
  • Supervised: model.fit(X, y) – Modell lernt aus Feature-Matrix + Labels
  • Unsupervised: model.fit(X) – nur Feature-Matrix, kein y
  • Nach .fit(): Modellparameter sind gelernt (im Modell-Objekt gespeichert)
4
Wende das Modell auf neue Daten an
a) Supervised Learning
  • Labels für neue Daten vorhersagen
  • y_pred = model.predict(X_new)
b) Unsupervised Learning
  • Transformieren oder Strukturen zuweisen
  • .transform(X) oder .predict(X)

5.3 Datenformat – Wie werden die Daten erwartet?

sklearn erwartet die Feature-Matrix und den Label-Vektor in einem spezifischen Format – dem Format, das wir bereits aus Kapitel 1 kennen.

Feature Matrix X

  • Liegt üblicherweise als NumPy-Array oder Pandas DataFrame vor
  • Shape: (n_samples, n_features) = (N, T)
  • Jede Zeile = eine Beobachtung (Sample)
  • Jede Spalte = ein Feature
  • sklearn erwartet immer 2D: auch bei nur 1 Feature → reshape(-1, 1)

Target Vector y

  • Liegt als NumPy-Array oder Pandas Series vor
  • Shape: (n_samples,) = (N,) – 1D-Array
  • Diskret (Klassen) bei Klassifikation
  • Kontinuierlich (Zahlen) bei Regression
  • Nur bei Supervised Learning nötig – UL: kein y!
💡 So what?

sklearn erwartet die Feature-Matrix und den zugehörigen Target-Vektor im beschriebenen Format – das Format, das wir aus Kapitel 1 schon kennen. Die Feature Matrix liegt üblicherweise als np.ndarray oder pd.DataFrame vor, der Target-Vektor als np.ndarray oder pd.Series.

5.4 Exemplarisches Beispiel: Linear Regression in sklearn

Wir haben die einfache lineare Regression schon aus dem Data Understanding Block kennengelernt. Hier gehen wir den typischen sklearn-Prozess an einem vollständigen Beispiel durch.

Das Szenario

  • Wir generieren synthetische Daten: 50 Samples, 1 Feature
  • Labels: y = 2 · X + Rauschen – eine lineare Beziehung mit Noise
  • Ziel: LinearRegression-Modell trainieren und Regressionsgerade einzeichnen
  • Schritte: Daten generieren → Importieren → Fit → Predict + Plot

Schritt 1 — Daten generieren & visualisieren

Wir erstellen einen reproduzierbaren Zufallszahlen-Generator (RandomState(42)) und generieren 50 Datenpunkte mit einer linearen Beziehung plus zufälligem Rauschen. Der Scatter-Plot zeigt die generierten Daten.

Python – Daten generieren 
import numpy as np
import matplotlib.pyplot as plt

# Reproduzierbarer Zufallszahlen-Generator
rng = np.random.RandomState(42)

# Feature-Matrix X: 50 Samples, 1 Feature
X = 10 * rng.rand(50, 1)

# Labels y = 2*X + Rauschen
y = 2 * X + rng.rand(50, 1)

# Scatter-Plot der Daten
plt.scatter(X, y)
plt.xlabel("X")
plt.ylabel("y")
plt.title("Generierte Daten: y = 2X + noise")
plt.show()
Beobachtung
  • 📊 Scatter-Plot: X-Achse 0–10, Y-Achse 0–20
  • Blaue Datenpunkte zeigen deutlich eine lineare Beziehung
  • Zufälliges Rauschen via rng.rand(50,1)
  • RandomState(42) garantiert Reproduzierbarkeit
  • X hat Shape (50, 1) – korrekt für sklearn
X.shape → (50, 1)
y.shape → (50, 1)
y ≈ 2*X + noise (0–1)

Schritt 2 — Modell importieren & .fit() – Training

Wir importieren LinearRegression aus sklearn.linear_model, erstellen eine Instanz (keine Hyperparameter nötig bei einfacher LR) und trainieren das Modell mit .fit(X, y). Nach dem Fit sind die Koeffizienten im Modell gespeichert.

Python – Schritt 1 & 2: Import & Training 
# Schritt 1: Modell importieren
from sklearn.linear_model import LinearRegression

# Schritt 2: Instanz erstellen (keine Hyperparameter)
model = LinearRegression()

# Schritt 3: Training (.fit)
model.fit(X, y)

# Gelernte Parameter ausgeben
print("Steigung (coef_):", model.coef_)
print("Achsenabschnitt (intercept_):", model.intercept_)
Ausgabe & Erklärung
Steigung (coef_): [[2.00...]}
Achsenabschnitt (intercept_): [0.5...]
  • model.coef_ ≈ 2.0 → gelernte Steigung
  • model.intercept_ ≈ 0.5 → Achsenabschnitt
  • Diese Werte sind die gelernten Modellparameter
  • Learning = Optimierung dieser Koeffizienten

Schritt 3 — .predict() & Regressionsgerade visualisieren

Mit .predict() berechnen wir die vorhergesagten Labels für neue Eingabewerte. Wir erstellen dazu ein feines Gitter von X-Werten (linspace), lassen das Modell vorhersagen und zeichnen die Regressionsgerade ein.

Python – Schritt 4: Predict & Plot 
# Feines X-Gitter für die Regressionsgerade
X_new = np.linspace(0, 10, 100).reshape(-1, 1)

# Schritt 4: Vorhersage für neue X-Werte
y_new = model.predict(X_new)

# Scatter-Plot + Regressionsgerade
plt.scatter(X, y, label="Daten")
plt.plot(X_new, y_new, color='red',
         label="Regression")
plt.xlabel("X")
plt.ylabel("y")
plt.legend()
plt.title("LinearRegression: Fit")
plt.show()
Beobachtung
  • 📊 Scatter-Plot: blaue Datenpunkte + rote Regressionsgerade
  • Die Gerade folgt klar dem Trend der Daten
  • reshape(-1, 1): 1D → 2D-Array (sklearn erwartet 2D)
  • Demo: „Nutzen Sie .predict() um die Regressionsgerade einzuzeichnen"
  • Frage aus der Vorlesung: „Sind wir schon fertig?" → Nein! → Model Validation!
💬 Denkfrage aus der Vorlesung

„Was denken Sie? Jetzt haben wir unser erstes exemplarisches Modell trainiert – sind wir schon fertig?"

Der vollständige sklearn-Prozess

  • Import: from sklearn.linear_model import LinearRegression
  • Instanziierung: model = LinearRegression()
  • Training: model.fit(X, y)
  • Vorhersage: y_pred = model.predict(X_new)

Gelernte Parameter nach .fit()

  • model.coef_ → gelernte Steigung(en)
  • model.intercept_ → gelernter Achsenabschnitt
  • Diese Parameter wurden aus den Trainingsdaten gelernt
  • Bei anderen Modellen: andere Attribute, aber gleiches Muster
  • → Learning is finding parameters!
