💡 So what?

Im Machine Learning bringen wir Algorithmen bei, wissenschaftlich zu denken – d.h. aus Daten zu lernen und dieses Wissen auf neue, unseen Daten anzuwenden. Das Fundament aus Kapitel 1 (Feature-Matrix, Feature-Raum) ist genau das Format, das ML-Algorithmen als Eingabe erwarten.

1.4 Was ist Machine Learning?

Die Kerndefinition und die drei zentralen Konzepte, auf denen alles aufbaut.

Machine Learning == „Building Models of Data"

Machine Learning zeichnet sich dadurch aus, dass mathematische Modelle entstehen, um zugrundeliegende Daten zu verstehen.

  • Diese Modelle werden an die Daten angepasst, damit sie diese so optimal wie möglich repräsentieren
  • Diese Anpassung kann stattfinden, da ML-Modelle „tunable parameters“ haben → das Modell lernt aus den Daten
  • Nach dem Training können Modelle Labels ungesehener Daten vorhersagen
🔧
Tunable Parameters
Im Gegensatz zu klassischen Algorithmen hat ein ML-Modell Parameter, die nicht vom Entwickler gesetzt werden – sie werden durch das Training aus den Daten gelernt. Der Optimierungsprozess passt diese Parameter so an, dass das Modell die Daten möglichst gut beschreibt.
📚
Training / Lernen
Beim Training wird dem Modell ein Trainingsdatensatz (Feature-Matrix X + Label-Vektor y) präsentiert. Das Modell schätzt die Abhängigkeit zwischen Features und Labels und speichert dieses Wissen in seinen Parametern.
🎯
Prediction / Scoring
Nach dem Training kann das Modell auf neue, ungesehene Daten angewendet werden. Neue Beobachtungen werden dem Modell übergeben – es weist ihnen Labels zu. Dieser Prozess heißt Scoring (bei Regression) oder Classification (bei Klassifikation).

  • Zuerst das Konzept verstehen, dann die formale Notation und den sklearn-Code

💡 So what?

Wenn du ein Modell anwenden kannst – kannst du jedes anwenden, zumindest bezüglich der Syntax. Das generische Muster in sklearn ist immer: import → instanziieren → fit → predict. Das gilt für lineare Regression genauso wie für Random Forest oder k-Means.
