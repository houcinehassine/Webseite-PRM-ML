Supervised Learning: Klassifikation & Regression

Wie lernt ein Modell eine Trennlinie? Wie wendet es das Gelernte auf neue Daten an – und was ändert sich bei kontinuierlichen Labels? Diese Seite erklärt Klassifikation und Regression konzeptuell, mit dem allgemeinen Scoring-Konzept als rotem Faden.

Trainingsdatensatz Trennlinie Modellparameter Scoring Prediction Regression Kontinuierliche Labels

Lernfokus dieser Seite

Du kannst den Trainingsprozess bei Klassifikation (Trennlinie finden, Koeffizienten als Parameter) von Scoring/Prediction klar unterscheiden. Du weißt, wie sich Regression von Klassifikation unterscheidet (kontinuierlich vs. diskrete Labels) und was es bedeutet, eine Ebene auf Daten zu fitten.

3.1 Klassifikation – Das Konzept

Gegeben ein gelabelter Trainingsdatensatz: Das Modell sucht eine Trennlinie, die die Klassen aufteilt. Die Koeffizienten dieser Geraden sind die Modellparameter – und diese werden aus den Daten gelernt.

Input Data — Gelabelter Trainingsdatensatz
Trainiertes Modell — Trennlinie gefunden

Aufgabe bei der Klassifikation

  • Gelabelter Trainingsdatensatz liegt vor (X + y)
  • Daran trainieren wir unser ML-Modell
  • Modell sucht eine Trennlinie, die die Klassen aufteilt
  • Neue Daten: anhand der Trennlinie bewerten → blaue oder rote Kategorie

Modellparameter = Koeffizienten

  • Die Trennlinie ist mathematisch eine Gerade (2D-Fall)
  • Beschrieben durch Koeffizienten – Steigung & Achsenabschnitt
  • Diese Koeffizienten = die Modellparameter
  • Sie werden aus den Daten gelernt → Training = Parameter optimieren
💬 Denkfrage aus der Vorlesung

„Was denken Sie? Worin steckt das Gelernte beim Model Learning – und wie wird ein Modell aussehen?"

„So what? Learning is finding parameters."
Prof. · Folie 11 · Grüne Box · Kernaussage
💡 So what?

Training beim Supervised Learning bedeutet immer: Modellparameter werden optimiert, sodass das Modell die Trainingsdaten möglichst gut beschreibt. Bei der Klassifikation sind das die Koeffizienten der Trennlinie – bei anderen Modellen (SVM, Random Forest) sind es andere Parameter. Das Prinzip bleibt identisch.

3.2 Scoring & Prediction – Das allgemeine Konzept

Nach dem Training kann das gelernte Wissen auf neue, vorher ungesehene Daten angewendet werden. Diese Zuweisung von Labels heißt Scoring bzw. Prediction.

Scoring bzw. Prediction – Definition

  • Ziel: gelerntes Wissen auf neue, vorher ungesehene Daten anwenden
  • Man führt dem Modell neue Beobachtungen zu
  • Das Modell bewertet diese und weist Labels zu
  • Diese Zuweisung nennt man Scoring (auch: Prediction / Inferenz)

In sklearn

  • Trainieren: .fit(X_train, y_train)
  • Scoring: y_pred = .predict(X_new)
  • Rückgabe: Array mit vorhergesagten Labels ŷ pro Datenpunkt
  • Gilt für Klassifikation (diskret) und Regression (kontinuierlich)
  • Generalisierung = Performance auf ungesehenen Daten
💡 So what?

Das Scoring-Konzept ist universell: ob Klassifikation oder Regression – das Schema ist immer dasselbe: Trainiere auf (X, y) → Wende an auf X_neu → Erhalte ŷ_neu. In sklearn immer: .fit(X, y) dann .predict(X_new).

3.3 Klassifikation Scoring – konkret

Neue, unbekannte Datenpunkte werden mit der gelernten Geraden verglichen. Ihre Position relativ zur Trennlinie entscheidet, welches diskrete Label (rot oder blau) sie erhalten.

Unknown Data — Neue, ungelabelte Punkte
Predicted Labels — Links / Rechts der Geraden

Scoring: Links vs. Rechts der Geraden

  • Neue Daten werden mit dem Modell (der gelernten Geraden) verglichen
  • Je nachdem auf welcher Seite der Geraden ein Punkt liegt → entsprechendes Label
  • Blau = Links/Oben der Grenze  |  Rot = Rechts/Unten der Grenze
  • Gestrichelte Linien = Klassengrenzen (Entscheidungsregionen)
  • Ergebnis: diskrete Labels – jeder Punkt bekommt genau eine Klasse

In sklearn: .predict()

  • Trainieren: model.fit(X_train, y_train)
  • Scoring: y_pred = model.predict(X_new)
  • Rückgabe: Array diskrete Labels z.B. [0, 1, 0, 1, 1, ...]
  • Klassengrenze = gelernte Trennlinie (Hyperebene im allgemeinen Fall)
  • Optional: .predict_proba(X) liefert Wahrscheinlichkeiten je Klasse

3.4 Regression – Das Konzept

Im Gegensatz zur Klassifikation haben wir bei der Regression kontinuierliche Labels – also einen fließenden Farbverlauf statt diskreter Klassen. Das Label wird als dritte Dimension im 2D Feature-Raum vorgestellt.

Schritt 1 — Labelled Data (continuous)

Datenpunkte verteilt mit kontinuierlichem Farbverlauf (kein Sprung zwischen Klassen):

Kleiner Label-Wert (Gelb) Großer Label-Wert (Violett)
→
Schritt 2 — Label als 3. Dimension vorstellen
  • Im 2D Feature-Raum (Feature 1, Feature 2) stellen wir uns das Label als dritte Dimension (Z-Achse = Höhe) vor
  • 3D-Koordinatensystem:
    X-Achse = Feature 1 · Y-Achse = Feature 2 · Z-Achse = label
  • Datenpunkte in 3D: ihre Höhe entspricht ihrem Label-Wert
  • Einfache Regression: versuche eine Ebene auf diese 3D-Daten zu fitten
  • Parameter der Ebene (a₀, a₁, a₂, ...) werden gelernt
3D: (Feature 1, Feature 2, label)
Ebene: z = a₀ + a₁·x₁ + a₂·x₂

Schritt 3 — Input Data with Linear Fit (Folie 14 · Heatmap rechts)

Feature 2 ↑
Hoher Label-Wert (Violett)
Feature 1 → Niedriger Label-Wert (Gelb)

Die gefittete 3D-Ebene wird als 2D-Heatmap dargestellt. Datenpunkte sind farblich gemäß ihrem tatsächlichen Label-Wert eingefärbt und auf die Heatmap überlagert – man sieht, wie gut die Ebene die Daten beschreibt.

Klassifikation vs. Regression

  • Klassifikation: diskrete Labels (0, 1, 2, ...) → Trennlinie finden
  • Regression: kontinuierliche Labels (15.7, 23.4, ...) → Ebene fitten
  • Feature-Raum ist identisch – nur der Label-Typ unterscheidet sich
  • In sklearn: gleiche API – .fit(X, y) und .predict(X_new)

Ebene auf Daten fitten

  • Die Ebene ist das Regressionsmodell im 3D-Raum
  • Parameter (a₀, a₁, a₂, ...) werden aus den Daten gelernt
  • Ziel: Ebene passt sich so an, dass sie Daten optimal beschreibt
  • Im 2D-Fall (1 Feature): eine Gerade statt einer Ebene
  • In sklearn: LinearRegression()
💡 So what?

Regression und Klassifikation unterscheiden sich nur in der Art des Labels. Bei der Regression lernt das Modell eine Funktion (Gerade/Ebene), die kontinuierliche Werte vorhersagt – statt Klassen durch eine Trennlinie zu separieren.

3.5 Regression Scoring – Projektion auf die Ebene

Beim Scoring im Regressionsfall werden neue, unbekannte Daten auf die geschätzte Ebene projiziert – das liefert die vorhergesagten, kontinuierlichen Labels.

Unknown Data — Neue, ungelabelte Punkte
Predicted Labels — Kontinuierlicher Farbverlauf

Regression Scoring: Projektion auf die Ebene

  • Unbekannte Daten werden auf die geschätzte Ebene projiziert
  • Dies liefert die neuen, kontinuierlichen Labels
  • Keine diskrete Entscheidung (links/rechts) – sondern ein kontinuierlicher Wert (die Höhe der Ebene an diesem Punkt)
  • Farbverlauf: Gelb (niedriger Wert) → Violett (hoher Wert)

💡 So what? – Zusammenfassung Seite 3

Klassifikation und Regression folgen dem gleichen Ablauf: Training auf (X, y) → Modell lernt Parameter → Scoring auf X_neu → ŷ_neu. Der Unterschied liegt nur im Typ der Labels – diskret vs. kontinuierlich. Die sklearn-API ist in beiden Fällen identisch: .fit(X, y) dann .predict(X_new).
