9.1 Polynomiale Regression – Unser Modell-Vertreter

Wir nutzen die Polynomiale Regression als konkreten Repräsentanten eines ML-Modells – der Polynomgrad ist der Hyperparameter, der die Modellkomplexität repräsentiert.

f(x) = aₙxⁿ + … + a₁x + a₀
f(x) = a₁x + a₀   (degree=1, gerade Linie)   ·   f(x) = a₂x² + a₁x + a₀   (degree=2, Parabel)   ·  …
Polynomial Fits: degree=1 (blau) · degree=3 (grün) · degree=5 (rot)

Polynomgrad = Hyperparameter = Modellkomplexität

  • Unser ML-Modell: eine Polynomfunktion beliebigen Grades
  • Diese wird per Polynomial Regression an die Daten angepasst
  • Durch die Anpassung werden die Koeffizienten geschätzt (a₀, a₁, ...)
  • Hyperparameter = Anzahl Summanden / Grad des Polynoms → repräsentiert die Modellkomplexität
  • Werden getuned / geswept: erst degree=1, dann 2, 3, ...

High-Bias vs. High-Variance bei Polynomen

  • degree=1 → gerade Linie → High-Bias (zu simpel)
  • degree=3 → sanfte Kurve → Guter Fit
  • degree=5+ → komplex/wellig → High-Variance (Overfitting)
  • Ziel: optimalen Grad finden → validation_curve()

9.2 PolynomialFeatures & LinearRegression in sklearn

Um eine Polynomiale Regression durchzuführen, kombinieren wir PolynomialFeatures (erweitert die Feature-Matrix) mit LinearRegression (fittet das lineare Modell auf die erweiterten Features) in einer Pipeline.

📄 sklearn · PolynomialFeatures – Docstring
Generate polynomial and interaction features.

Generate a new feature matrix consisting of all polynomial
combinations of the features with degree less than or equal
to the specified degree.

For example, if an input sample is two dimensional and of
the form [a, b], the degree-2 polynomial features are:
[1, a, b, a², ab, b²].
📄 sklearn · LinearRegression – Docstring
LinearRegression fits a linear model with coefficients
w = (w1, ..., wp) to minimize the residual sum of squares
between the observed targets in the dataset, and the targets
predicted by the linear approximation.

Schritt 1 — Hilfsfunktionen definieren

Wir definieren zwei Hilfsfunktionen: PolynomialRegression() baut eine sklearn-Pipeline aus PolynomialFeatures + LinearRegression, make_data() generiert einen synthetischen Datensatz.

Python – Hilfsfunktionen 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression

# PolynomialRegression: Pipeline aus PF + LR
def PolynomialRegression(degree=2, **kwargs):
    return make_pipeline(
        PolynomialFeatures(degree),
        LinearRegression(**kwargs))

# make_data: synthetische Daten generieren
def make_data(N, err=1.0, rseed=1):
    rng = np.random.RandomState(rseed)
    X = rng.rand(N, 1) ** 2
    y = 10 - 1. / (X.ravel() + 0.1)
    if err > 0:
        y += err * rng.randn(N)
    return X, y
Erklärung
  • make_pipeline(PolynomialFeatures(degree), LinearRegression()) = Pipeline
  • Pipeline: Step 1 = Feature-Erweiterung, Step 2 = lin. Regression
  • param_name='polynomialfeatures__degree' (double-underscore für Pipelines)
  • make_data(N): Kurve y = 10 - 1/(x+0.1) + Rauschen
  • rseed=1 → reproduzierbar

9.3 validation_curve in sklearn

Wir sweepen den Polynomgrad von 0 bis 20 und berechnen für jeden Grad den Trainings- und Validierungs-Score mit 7-Fold Cross-Validation. Werkzeug: validation_curve().

sklearn API

  • validation_curve(model, X, y, param_name=..., param_range=..., cv=...)
  • Führt für jeden Wert in param_range eine Cross-Validation durch
  • Gibt zurück: train_scores, val_scores als Arrays (Shape: n_params × n_cv_folds)
  • Zum Plotten: np.median(scores, 1) oder np.mean(scores, 1) (aggregiert über Folds)

Schritt 3 — validation_curve berechnen

Wir sweepen den Polynomgrad von 0 bis 20. Der Parameter heißt in der Pipeline polynomialfeatures__degree (sklearn-Konvention: Stufenname + doppelter Unterstrich + Parametername).

Python – validation_curve berechnen 
from sklearn.model_selection import validation_curve

degree = np.arange(0, 21)  # degree 0 bis 20

train_score, val_score = validation_curve(
    PolynomialRegression(), X, y,
    param_name='polynomialfeatures__degree',
    param_range=degree,
    cv=7)

print("train_score.shape:", train_score.shape)
print("val_score.shape:  ", val_score.shape)
Ausgabe
train_score.shape: (21, 7)
val_score.shape: (21, 7)
  • 21 Degree-Werte × 7 CV-Folds
  • Median über Folds aggregieren

Schritt 4 — Validierungskurve plotten

Wir plotten den Median der Scores über alle 7 Folds. Trainings-Score (blau) und Validierungs-Score (rot) gegen den Polynomgrad.

Python – Validierungskurve plotten 
plt.plot(degree, np.median(train_score, 1),
         color='blue', label='training score')
plt.plot(degree, np.median(val_score, 1),
         color='red', label='validation score')
plt.legend(loc='lower center')
plt.ylim(0, 1)
plt.xlabel('degree')
plt.ylabel('score')
plt.title('Validierungskurve – Polynomiale Regression')
plt.show()
Beobachtung
  • Blaue Kurve: Training-Score bleibt konstant hoch (~0.98+)
  • Rote Kurve: steigt → Peak → fällt wieder
  • Optimum: degree ≈ 3 (Maximum der roten Kurve)
  • Ab degree > 5: High-Variance → deutliche Divergenz

9.5 Learning Curves – Konzept

Die Validierungskurve hängt nicht nur von der Modellkomplexität ab, sondern auch von der Datenmenge. Mit mehr Daten kann man eine höhere Komplexität verwenden.

Schritt 5 — Mehr Daten generieren (N=200)

Wir generieren 200 statt 50 Datenpunkte – und schauen, wie sich die Validierungskurve verändert.

Python – Mehr Daten + Validierungskurve 
# 200 Datenpunkte generieren
X2, y2 = make_data(200)
plt.scatter(X2.ravel(), y2, color='blue', s=8)
plt.title('Mehr Daten: N=200')
plt.xlabel('x'); plt.ylabel('y')
plt.show()
Beobachtung
  • 📊 Scatter: 200 Punkte (statt 50)
  • Gleiche Kurvenform, aber dichter besetzt
  • Mit mehr Daten → höhere Komplexität möglich
Python – Validierungskurven Vergleich (N=50 vs N=200) 
degree = np.arange(21)

# N=50
train_s, val_s = validation_curve(
    PolynomialRegression(), X, y,
    param_name='polynomialfeatures__degree',
    param_range=degree, cv=7)

# N=200
train_s2, val_s2 = validation_curve(
    PolynomialRegression(), X2, y2,
    param_name='polynomialfeatures__degree',
    param_range=degree, cv=7)

plt.plot(degree, np.median(val_s2, 1),
         color='red', label='val (N=200)')
plt.plot(degree, np.median(val_s, 1),
         color='red', alpha=0.3, linestyle='--',
         label='val (N=50)')
plt.legend(loc='lower center')
plt.ylim(0, 1); plt.xlabel('degree')
plt.ylabel('score')
plt.title('Mehr Daten → höhere Komplexität möglich')
plt.show()
Beobachtung (Folie 43)
  • Rote Kurve (N=200) liegt höher als gestrichelt (N=50)
  • Mit mehr Daten: Optimum verschiebt sich nach rechts
  • → Höhere Komplexität wird möglich
  • → Validierungskurve ≠ nur Funktion der Komplexität
  • → auch Funktion der Datenmenge!

Learning Curves – Was sie zeigen

  • Mittels Learning Curves: Einfluss der Datensatzgröße auf Model-Performance untersuchen
  • Man wählt eine Modellkomplexität und trägt Validierungs- und Trainings-Score gegen die Datensatzgröße auf
  • Frage: ab wie vielen Daten bringt mehr kein Mehrwert mehr?

Verbindung zur Validierungskurve

  • Validierungskurve (Page08): X-Achse = Modellkomplexität
  • Learning Curve: X-Achse = Datensatzgröße
  • Beide Achsen beeinflussen die Performance!
  • → Aus der Untersuchung des Datensatzgrößen-Einflusses resultieren die Learning Curves

9.6 Learning Curves – Schema & Charakteristiken

Das allgemeine Schema der Learning Curve: Training-Score startet bei 1.0 und fällt, während Validation-Score steigt und sich annähert.

Learning Curve Schema (Folie 44)

Allgemeine Charakteristiken (Folie 44)

  • Bei gegebener Modellkomplexität und kleinem Datensatz: Overfitting (großer Gap zwischen Kurven)
  • Bei gegebener Modellkomplexität und großem Datensatz: Underfitting (Kurven konvergieren, aber zu niedrig)
  • Der Validierungs-Score wird nie (außer durch Zufall) größer als der Trainings-Score

Konvergenz – So What?

  • Mit zunehmender Datensatzgröße nähern sich die Kurven an
  • Konvergenz = Good Fit Zone
  • Ab einem gewissen Punkt bringt die Vergrößerung des Datensatzes keinen Mehrwert mehr!
  • Das Learning-Curve-Bild sagt uns, ob wir mehr Daten oder ein komplexeres Modell brauchen

9.7 Learning Curves in sklearn

sklearn stellt learning_curve() bereit. Wir vergleichen degree=2 (gutes Modell) vs. degree=9 (zu komplex).

sklearn API

  • learning_curve(model, X, y, cv=..., train_sizes=...)
  • train_sizes = Array von Trainingsgrößen (z.B. np.linspace(0.3, 1, 25))
  • Gibt zurück: N, train_scores, val_scores (absolute Trainingsgrößen + Score-Arrays)
  • Zum Plotten: np.mean(scores, 1) über CV-Folds aggregieren

Schritt 6 — Learning Curves berechnen: degree=2 vs. degree=9

Wir berechnen und plotten Learning Curves für zwei verschiedene Polynomgrade. Vertikales Layout (figsize=(6,8), subplot(2,1)) um die Graphen gut vergleichen zu können.

Python – Learning Curves (degree=2 vs degree=9) 
from sklearn.model_selection import learning_curve

# Vertikales Layout: (2,1) statt (1,2)
fig, axes = plt.subplots(2, 1, figsize=(6, 8))

for i, degree in enumerate([2, 9]):
    N, train_lc, val_lc = learning_curve(
        PolynomialRegression(degree), X, y,
        cv=7,
        train_sizes=np.linspace(0.3, 1, 25))

    axes[i].plot(N, np.mean(train_lc, 1),
                 color='blue',
                 label='training score')
    axes[i].plot(N, np.mean(val_lc, 1),
                 color='red',
                 label='validation score')
    axes[i].hlines(0.9, N[0], N[-1],
                   linestyle='--', alpha=0.3)
    axes[i].set_ylim(0, 1)
    axes[i].set_xlabel('training size')
    axes[i].set_ylabel('score')
    axes[i].set_title(f'degree = {degree}')
    axes[i].legend(loc='lower right')

plt.tight_layout()
plt.show()
Beobachtung (Folie 45)
  • degree=2 (oben): Val-Score steigt schnell auf ~0.9, konvergiert mit Training → gutes Modell
  • degree=9 (unten): Val-Score bleibt niedrig bei ~0.8, großer Gap zu Training → Overfitting
  • degree=9: mehr Daten würden helfen (Kurven konvergieren langsam)
  • degree=2: bereits bei ~25 Samples gut, mehr Daten kaum Mehrwert
  • Vertikales Layout erlaubt direkten optischen Vergleich

„Was zeigt uns das Konvergenzverhalten? Was bedeutet das für unsere Datenstrategie?"

degree=2 – Gutes Modell

  • Training-Score startet hoch, fällt leicht ab
  • Validation-Score steigt schnell an
  • Kurven konvergieren bei ~30 Samples
  • → Mehr Daten bringen kaum Mehrwert

degree=9 – Overfitting

  • Training-Score bleibt nahe 1.0
  • Validation-Score bleibt niedrig, steigt langsam
  • Großer Gap zwischen Training und Validation
  • → Mehr Daten würden helfen
  • → Oder: Komplexität reduzieren (degree=3)
