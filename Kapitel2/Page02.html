Kategorien des ML

Machine Learning teilt sich in drei Hauptkategorien – Supervised, Unsupervised und Reinforcement Learning. Diese Seite klärt, wie sich diese unterscheiden, wie die Daten jeweils aussehen und was die mathematische Notation dahinter bedeutet.

Supervised Learning Unsupervised Learning Reinforcement Learning Feature-Matrix X Label-Vektor y Trainingsdatensatz

Lernfokus dieser Seite

Du kannst die drei ML-Kategorien klar voneinander abgrenzen, weißt wie die Daten bei Supervised vs. Unsupervised Learning strukturiert sind (Feature-Matrix X, Label-Vektor y) und verstehst die mathematische Notation – von der einzelnen Beobachtung (xi, yi) bis zum Trainingsdatensatz.

2.1 Die drei Kategorien des Machine Learning

Alle ML-Verfahren lassen sich in eine von drei Hauptkategorien einordnen – der entscheidende Unterschied liegt darin, ob Labels vorhanden sind.

📊
Supervised Learning
  • Modellierung der Abhängigkeiten zwischen Features und Labels
  • Jede Beobachtung hat ein bekanntes Label y
  • Nach dem Training: Labels für neue Daten vorhersagen
  • „Den Datensatz für das Modell sprechen lassen"
Classification Regression
🔍
Unsupervised Learning
  • Modellierung der Eigenschaften/Struktur von Daten
  • Keine Labels – Struktur ist nicht markiert
  • Erkenntnisse durch Heuristiken gewinnen
  • „Den Datensatz für sich sprechen lassen"
Clustering Density Estimation Dimension Reduction
🎮
Reinforcement Learning
  • Agent lernt durch Interaktion mit einer Umgebung
  • Feedback in Form von Belohnungen / Strafen
  • Ziel: Strategie (Policy) optimieren
  • Beispiele: Spiele, Robotik, autonomes Fahren
Policy Optimization Q-Learning
💡 So what?

Der entscheidende Unterschied: Supervised braucht Labels y zum Training, Unsupervised hat keine Labels und entdeckt Struktur selbst. In diesem Kapitel fokussieren wir Supervised und Unsupervised – Reinforcement Learning ist ein eigenes Themengebiet.

Farbe = Label – Klassifikation (diskret)

  • Labels haben diskrete Ausprägungen (Klassen)
  • Jede Farbe entspricht einer Klasse: 🔵 Klasse A, 🟡 Klasse B, 🟢 Klasse C
  • Ziel: Trennlinie / Entscheidungsgrenze im Feature-Raum finden
  • Beispiele: Spam-Detektion, Bilderkennung, Krankheitsdiagnose
  • Label-Typ: int (0, 1, 2, ...) oder str ("A", "B")

Farbe = Label – Regression (kontinuierlich)

  • Labels haben kontinuierliche Ausprägungen
  • Farbverlauf statt diskreter Farben: Gelb → Türkis → Violett
  • Ziel: Label als 3. Dimension modellieren (Ebene / Hyperebene)
  • Beispiele: Hauspreisprognose, Aktienkurs, Stromverbrauch
  • Label-Typ: float (z.B. 15.7, 23.4, ...)

2.3 Supervised Learning – Mathematische Notation

Die formale Sprache hinter der Datensicht – wie wir Feature-Matrix, Label-Vektor und Trainingsdatensatz schreiben.

X
Feature-Matrix
  • Dimensionen: N × T
  • N = Anzahl Beobachtungen (Zeilen)
  • T = Anzahl Features (Spalten)
  • Jede Zeile = ein Featurevektor xi
X ∈ ℝ^(N×T)
y
Label-Vektor
  • N-dimensionaler Spaltenvektor
  • Ein Eintrag pro Beobachtung (Zeile von X)
  • Diskret → Klassifikation
  • Kontinuierlich → Regression
y ∈ ℝ^N (Regression) oder y ∈ {0,1,...,K} (Klasse)
(xi, yi)
Eine Beobachtung
  • xi = Featurevektor der i-ten Beobachtung (Zeile i von X)
  • yi = zugehöriges Label der i-ten Beobachtung
  • Paar aus Features und Label
  • i = 1, 2, ..., N
x_i ∈ ℝ^T, y_i ∈ ℝ (oder Klasse)
Trainingsdatensatz
Definition
  • Menge aller beschrifteten Beobachtungen
  • Daran wird das Modell trainiert (gefittet)
  • Abhängigkeit Labels ↔ Features wird geschätzt
  • Basis für alle späteren Vorhersagen
D = {(x₁,y₁), (x₂,y₂), ..., (xN,yN)}

Trainingsprozess – Supervised Learning

  • Im Trainingsdatensatz existiert zu jeder Beobachtung (Zeile der Feature-Matrix) ein Eintrag im Label-Vektor
  • Das ML-Modell betrachtet die Feature-Matrix als eine Menge gestapelter Zeilenvektoren xi – mit der zugehörigen Assoziation (xi, yi)
  • Ziel: Die Funktion f(X) ≈ y lernen, sodass das Modell auf neuen, ungesehenen Daten Xneu die Labels yneu vorhersagen kann
💡 So what?

Die Notation ist konsistent in sklearn: X ist immer die Feature-Matrix, y immer der Label-Vektor. Egal ob Klassifikation oder Regression – model.fit(X, y) erwartet immer genau dieses Format.

Unsupervised == Keine Labels

  • Beim Unsupervised Learning liegen keine Labels vor
  • Die Feature-Matrix ist für uns nicht eingefärbt
  • Wir haben eine „nicht eingefärbte" Feature-Matrix
  • Durch UL-Modelle versuchen wir, die Einfärbung zu schätzen
  • Typisch: Datenpunkte scheinen in Clustern zu liegen – aber ohne bekannte Labels

Grobe Unterkategorien

  • Clustering & Dichteschätzung – Gruppen / Muster finden
  • Dimensionsreduktion – niedrigdimensionale Repräsentation der Daten finden
  • Ziel: Die zugrundeliegende Struktur der Daten verstehen
  • Labels können nach dem Training zugeordnet werden (z.B. Cluster-IDs)

2.5 Unsupervised Learning – Mathematische Notation

Im Unsupervised Fall gibt es kein y – nur die Feature-Matrix X. Das Modell entdeckt selbst Strukturen.

Nur X – Kein y-Vektor!

  • Im Unsupervised Learning haben wir zuerst nur die Feature-Matrix X
  • Das ML-Modell versucht, in diesem Fall Strukturen bzw. Muster in den Daten zu entdecken
  • Wir können diese Muster dann auch mit Labels y versehen (z.B. Cluster-ID = 0, 1, 2)
  • Nachdem das Modell trainiert wurde, können auch im UL-Fall neue, ungesehene Daten den entdeckten Strukturen zugeordnet werden

Unterschied zur mathematischen Notation

  • Supervised: Input = (X, y) → Modell schätzt Funktion f: X → y
  • Unsupervised: Input = nur X → Modell entdeckt Struktur in X
  • Das .fit() in sklearn akzeptiert entsprechend:
  • Supervised: model.fit(X, y)
  • Unsupervised: model.fit(X) — kein y!

Trainingsprozess – Unsupervised Learning

  • Obere Grafik (Folie 10 links): Feature-Matrix X mit Dimensionen (N × T) – keine Farbcodierung, kein y-Vektor daneben
  • Pfeil nach unten: Zahnrad-Symbol (Machine Learning Modell) in der Mitte
  • Untere Grafik: Feature-Matrix X mit Farbcodierung (vier Farbbänder: blau, grün, grün/türkis, gelb)
  • Gelber Balken unten rechts = symbolisiert eine einzelne zugeordnete Klasse/Cluster
💡 So what?

Auch beim Unsupervised Learning können Vorhersagen gemacht werden – nur eben nicht für Labels y, sondern für Strukturzugehörigkeit (z.B. Cluster-ID oder niedrigdimensionale Koordinate). Die Methode: .transform() oder .predict() – je nach Algorithmus.

💡 So what?

Das Kernprinzip ist in beiden Fällen gleich: X geht rein, das trainierte Modell kann neue Beobachtungen bewerten. Der einzige Unterschied: beim Supervised Learning kennen wir die „richtige Antwort" (y) und lehren das Modell diese – beim Unsupervised Learning überlassen wir dem Modell die Entdeckung.
