7.1 Multi-Class SVMs: One-vs-One

SVMs sind ursprünglich für binäre Klassifikation ausgelegt (2 Klassen). Bei mehreren Klassen: für jede Klassenpaarung einen eigenen SVC trainieren.

🧠 Folie 30 — Fragen

Sie kennen jetzt eine SVM, die zwei Klassen unterscheiden kann: wie funktioniert das bei mehreren Klassen? Was bedeutet "one-versus-rest"? Sind die Entscheidungsgrenzen gleich? Wie viele SVMs müssen erzeugt werden?

⚔️ One-vs-One (OvO)

  • Für jede Klassenpaarung (A vs. B) wird ein SVM trainiert
  • N Klassen → N(N−1)/2 SVMs
  • Beispiel 3 Klassen (A, B, C): A vs. B, A vs. C, B vs. C → 3 SVMs
  • Beispiel 4 Klassen: 4·3/2 = 6 SVMs
  • Jeder neue Punkt wird von allen SVMs klassifiziert → Mehrheitsabstimmung

🔀 One-vs-Rest (OvR)

  • Alternative: Klasse A vs. alle anderen
  • N Klassen → nur N SVMs
  • Weniger SVMs, aber: unbalancierter Datensatz
  • sklearn SVC nutzt standardmäßig OvO
  • Entscheidungsgrenzen: bei OvO detaillierter

🗳️ Kombinierte Entscheidungsgrenzen

  • Jede paarweise SVM zeichnet ihre eigene Grenze
  • Zusammen entstehen komplexe Regionen
  • Neuer Punkt: von welchen SVMs als Klasse X klassifiziert?
  • Klasse mit den meisten "Stimmen" gewinnt
  • sklearn: intern automatisch über decision_function

📐 Formel: Anzahl SVMs

  • \(\displaystyle \text{SVMs} = \frac{N \cdot (N-1)}{2}\)
  • 3 Klassen: 3 SVMs
  • 4 Klassen: 6 SVMs
  • 5 Klassen: 10 SVMs
  • 10 Klassen: 45 SVMs
  • 100 Klassen: 4950 SVMs!
One-vs-One – 4 Klassen → 6 SVMs (Schema · Folie 30)
💡 So what?

In sklearn wird Multi-Class automatisch gehandhabt: SVC().fit(X, y) mit y-Labels 0,1,2,3 → sklearn trainiert intern N(N-1)/2 SVMs und kombiniert ihre Ergebnisse. Wir müssen nichts explizit ändern!

7.2 Anwendung: Handschrifterkennung

Handschriftliche Ziffern erkennen mit dem Digits-Dataset aus sklearn. Die Bilder sind als 8×8 Arrays gespeichert – 64 Pixel = 64 Features pro Bild.

📦 Digits-Dataset

  • 1797 Bilder von handschriftlichen Ziffern (0–9)
  • Jedes Bild: 8 × 8 Pixel = 64 Features
  • Pixelwerte: Integer 0–16 (0=weiß, 16=schwarz)
  • 10 Klassen (Ziffern 0–9)
  • Feature-Matrix: digits.data.shape = (1797, 64)

🎯 Ziel

  • Modell trainieren, das ungesehene Ziffern korrekt erkennt
  • Multi-Class SVM: 10 Klassen → 45 SVMs (OvO)
  • Kernel: rbf (nicht-linear, da Pixel-Daten)
  • Metrik: accuracy_score (% korrekt klassifiziert)

Schritt 3 — Train/Test-Split, SVM trainieren und Accuracy berechnen.

Python – SVM auf Digits (Folie 26) 
from sklearn.svm import SVC
from sklearn.model_selection import train_test_split

# Train/Test Split
X_train, X_test, y_train, y_test = train_test_split(
    digits.data, digits.target,
    random_state=42, test_size=0.2
)

# Multi-Class SVC (OvO intern!)
model = SVC(kernel='rbf', C=10, gamma=0.001)
model.fit(X_train, y_train)

score = model.score(X_test, y_test)
print(f"Accuracy: {score:.4f} ({score*100:.1f}%)")
▶ Ausgabe
Accuracy: 0.9889 (98.9%)
  • ~99% Accuracy auf Testdaten!
  • Intern: 10·9/2 = 45 SVMs trainiert
  • gamma=0.001 → weicher RBF-Kernel
  • 10 Klassen, 1437 Trainingspunkte

7.3 Anwendung: Face Recognition

Gesichtserkennung berühmter Persönlichkeiten mit fetch_lfw_people (Labeled Faces in the Wild). Bilder mit ca. 3000 Pixeln → Dimensionsreduktion via PCA ist notwendig.

ℹ️ PCA vorweggenommen (Folie 32)

PCA (Principal Component Analysis) reduziert die Dimensionalität: aus ~3000 Pixeln werden 150 Hauptkomponenten extrahiert, die die wichtigsten Gesichtsmerkmale beschreiben. make_pipeline(pca, svc) verknüpft PCA und SVM. PCA wird in einem späteren Kapitel detailliert behandelt.

📷 Datensatz: LFW

  • fetch_lfw_people(min_faces_per_person=60)
  • Bilder berühmter Personen aus Nachrichtenagenturen
  • Bildgröße: (62, 47) Pixel ≈ 2914 Features
  • Klassen: Colin Powell, George W. Bush, Hugo Chavez, Tony Blair, u.a.
  • min_faces_per_person=60 → nur Personen mit ≥60 Bildern

📐 Pipeline: PCA + SVC

  • PCA(n_components=150) → 2914 → 150 Dimensionen
  • whiten=True → standardisiert die Komponenten
  • SVC(kernel='rbf', C=10, gamma=0.001)
  • make_pipeline(pca, svc) → sequentiell
  • Metrik: accuracy_score aus sklearn.metrics

Schritt 3 — Pipeline bauen: PCA reduziert 2914 → 150 Dimensionen, dann SVC klassifiziert. make_pipeline verbindet beide Schritte nahtlos.

Python – PCA + SVC Pipeline (Folie 32) 
from sklearn.decomposition import PCA
from sklearn.pipeline import make_pipeline
from sklearn.svm import SVC
from sklearn.model_selection import train_test_split

# PCA: 2914 → 150 Komponenten
pca = PCA(n_components=150, whiten=True,
          random_state=42)
svc = SVC(kernel='rbf', C=10, gamma=0.001)

# Pipeline: PCA → SVC
model = make_pipeline(pca, svc)

# Train/Test Split
X_tr, X_te, y_tr, y_te = train_test_split(
    faces.data, faces.target, random_state=42
)
model.fit(X_tr, y_tr)
print("Pipeline trainiert!")
Was make_pipeline macht
  • fit(X_tr, y_tr): PCA.fit() → SVC.fit()
  • predict(X_te): PCA.transform() → SVC.predict()
  • PCA: 2914 → 150 Dim (99.5% Varianz erhalten)
  • whiten=True → auf gleiche Skala normiert
  • Pipeline kümmert sich um alles intern

💡 So what?

accuracy_score ist die einfachste Metrik: Wie viel Prozent der Testlabels wurden korrekt vorhergesagt? Für unbalancierte Datensätze (George Bush: 530 Bilder vs. Sharon: 77) kann Accuracy irreführend sein – später werden wir dafür Confusion Matrix und F1-Score kennenlernen.

Schritt 2 — Zeitserien fenstern und Features extrahieren. Jedes Fenster (z.B. 1024 Samples) wird zu einem 6-dimensionalen Feature-Vektor komprimiert.

Python – Feature-Extraktion aus Fenster (Folie 33) 
from scipy.stats import kurtosis

def extract_features(window):
    """6 Features aus einem Zeitserien-Fenster."""
    rms = np.sqrt(np.mean(window**2))
    peak = np.max(np.abs(window))
    return np.array([
        np.mean(window),          # mean
        np.std(window),           # std
        rms,                      # RMS
        peak,                     # peak
        kurtosis(window),         # kurtosis
        peak / (rms + 1e-9)       # crest factor
    ])

# Beispiel: simuliertes Fenster
window = np.random.randn(1024)
f = extract_features(window)
print(f"Feature-Vektor (6D): {f.round(3)}")
▶ Ausgabe
Feature-Vektor (6D):
[-0.02 1.01 1.01 3.62 0.05 3.58]
  • 1024 Samples → 6 Features
  • Enorme Dimensionsreduktion!
  • Verschiedene Fehlerzustände → verschiedene Feature-Profile
  • → SVM kann diese unterscheiden
