8.1 SVR: Das Konzept

Support Vector Regression kehrt die SVC-Logik um: statt Punkte außerhalb des Margins zu fordern, wollen wir möglichst viele Punkte innerhalb eines ε-Tubes.

🪟 Whiteboard Time! — Folie 34

Warum benötigen wir nun ein Epsilon? Wie könnte nun unsere Kostenfunktion aussehen? Was ist der clevere Trick, der uns von einer Klassifikation zu einer Regression bringt?

🎯 SVC – Klassifikation
  • Punkte sollen außerhalb des Margins sein
  • Margin-Verletzungen = Punkte im Margin → Strafe
  • Ziel: breiter Margin mit wenig Verletzungen
  • Output: Klassen-Label (+1 oder −1)
⇄
📈 SVR – Regression
  • Punkte sollen innerhalb des ε-Tubes sein
  • Tube-Verletzungen = Punkte außerhalb → Strafe
  • Ziel: passender Tube mit wenig Ausreißern
  • Output: kontinuierlicher Wert ŷ
Der ε-Tube: Innen = kostenlos, Außen = Strafe (Folie 34)
SVR – Kostenfunktion (Folie 34)
\[ \min_{\mathbf{w},b}\;\frac{1}{2}\|\mathbf{w}\|^2 + C \cdot \sum_{i=1}^N \max\!\left\{0,\; |y_i - (\langle\mathbf{w}, x_i\rangle + b)| - \varepsilon\right\} \]
\(\varepsilon\) Tube-Breite: Punkte INNERHALB ±ε haben Kosten 0. Je größer ε, desto mehr Punkte sind kostenfrei → glatteres Modell
\(|y_i - \hat{y}_i| - \varepsilon\) Überschuss über den Tube: 0 wenn Punkt innen, positiver Wert wenn außen (Abstand − ε)
\(C\) Regularisierung: wie stark werden Tube-Verletzungen bestraft? Gleiches Prinzip wie in SVC

⚖️ Effekt von ε

  • Großes ε → breiter Tube → viele Punkte kostenlos → glattere Kurve
  • Kleines ε → schmaler Tube → wenig Punkte kostenlos → engerer Fit
  • ε = 0: maximale Sensitivität
  • ε zu groß: Modell ignoriert alle Punkte
  • → Hyperparameter via Cross-Validation tunen!

⚙️ Effekt von C

  • Großes C: Verletzungen teuer → enger Fit → Overfitting-Risiko
  • Kleines C: Verletzungen günstig → smoothere Kurve → Underfitting-Risiko
  • Dasselbe Prinzip wie bei SVC!
  • C und ε interagieren: beide zusammen tunen
💡 So what?

SVR ist das natürliche Gegenstück zu SVC: dieselbe Mathematik, umgekehrte Logik. Statt Punkte außerhalb zu fordern, tolerieren wir Punkte innerhalb des Tubes kostenfrei. Außenliegende Punkte werden mit C bestraft. Auch hier: Kernel-Trick möglich → nicht-lineare Regression!

8.2 SVR in sklearn

sklearn bietet zwei SVR-Klassen: LinearSVR für lineare Probleme und SVR(kernel=...) für nicht-lineare Probleme. Code aus 03_core.ipynb (Cells 18–27).

LinearSVR

  • Für lineare Regressionsprobleme
  • Parameter: epsilon, C
  • Schneller als SVR(kernel='linear')
  • Aus: from sklearn.svm import LinearSVR

SVR(kernel=...)

  • Für nicht-lineare Regressionsprobleme
  • Parameter: kernel, C, epsilon, gamma
  • Kernel: 'rbf', 'poly', 'linear'
  • Aus: from sklearn.svm import SVR

A) LinearSVR — ε=1.5 vs. ε=0.5

Schritt 1 — Linearen Regressionsdatensatz + Hilfsfunktionen erstellen. Die Funktion plot_svm_regression() zeichnet Kurve, ε-Tube und Ausreißer.

Python – Daten & Hilfsfunktion (Cells 18–20) 
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)
m = 50
X = 2 * np.random.rand(m, 1)
y = (4 + 3 * X + np.random.randn(m, 1)).ravel()

def find_support_vectors(svm, X, y):
    """Punkte außerhalb des ε-Tube."""
    off_margin = np.abs(y - svm.predict(X)) >= svm.epsilon
    return np.argwhere(off_margin)

def plot_svm_regression(svm, X, y, ax, xlim):
    xs = np.linspace(xlim[0], xlim[1], 100).reshape(-1,1)
    yp = svm.predict(xs)
    ax.plot(xs, yp, 'k-', lw=2, label='ŷ')
    ax.plot(xs, yp + svm.epsilon, 'k--', alpha=0.7)
    ax.plot(xs, yp - svm.epsilon, 'k--', alpha=0.7,
            label=f'±ε={svm.epsilon}')
    sv_idx = find_support_vectors(svm, X, y)
    ax.scatter(X[sv_idx], y[sv_idx], s=180,
               facecolors='#FFAAAA', edgecolors='red',
               zorder=5, label='Außerhalb ε')
    ax.plot(X, y, 'bo', ms=4, alpha=0.7)
Hilfsfunktionen erklärt
  • find_support_vectors: Punkte mit |y−ŷ| ≥ ε → außerhalb Tube
  • plot_svm_regression: Kurve + ε-Tube + Ausreißer
  • Blaue Punkte = korrekt innerhalb Tube
  • Rote Kreise = Ausreißer außerhalb Tube

Schritt 2 — Zwei LinearSVR-Modelle mit ε=1.5 und ε=0.5 vergleichen. Vertikal gestapelt (2×1). Rote Kreisringe = Punkte außerhalb des Tubes.

Python – LinearSVR ε-Vergleich vertikal (Folie 34) 
from sklearn.svm import LinearSVR

reg1 = LinearSVR(epsilon=1.5, random_state=42)
reg2 = LinearSVR(epsilon=0.5, random_state=42)
reg1.fit(X, y); reg2.fit(X, y)

# Vertikal: 2 Zeilen, 1 Spalte
fig, axes = plt.subplots(2, 1, figsize=(5, 9),
                          sharey=True)

for ax, model in zip(axes, [reg1, reg2]):
    plot_svm_regression(model, X, y, ax, [0, 2])
    ax.set_xlim(0, 2); ax.set_ylim(3, 11)
    ax.set_title(f'LinearSVR: ε = {model.epsilon}',
                 fontsize=12)
    ax.legend(fontsize=8)
    ax.set_xlabel('x₁'); ax.set_ylabel('y')

plt.tight_layout(); plt.show()
▶ Ausgabe
  • 📊 Oben (ε=1.5): breiter Tube → wenige rote Punkte außen
  • 📊 Unten (ε=0.5): schmaler Tube → viele rote Punkte außen
  • Beide: gerade Linie (linear!)
  • Rote Kreise: außerhalb Tube → Strafe in Kostenfunktion
  • Unterschied: Tube-Breite, nicht Linienneigung

B) SVR Poly-Kernel — C=100 vs. C=0.01

Schritt 3 — Nicht-linearen Datensatz (Parabel + Rauschen) erstellen.

Python – Nicht-linearer Datensatz (Folie 35) 
np.random.seed(42)
m = 100
X_nl = 2 * np.random.rand(m, 1) - 1
y_nl = (0.2 + 0.1*X_nl + 0.5*X_nl**2
        + np.random.randn(m, 1)/10).ravel()

fig, ax = plt.subplots(figsize=(5, 4))
ax.scatter(X_nl, y_nl, s=20, alpha=0.7,
           color='steelblue')
ax.set_title('Nicht-linearer Datensatz (Parabel)')
ax.set_xlabel('x₁'); ax.set_ylabel('y')
plt.tight_layout(); plt.show()
▶ Ausgabe
  • 📊 100 Punkte mit quadratischem Trend (Parabel)
  • Minimum bei x₁ ≈ 0
  • Linear nicht zu fitten → Kernel-Trick!
  • np.random.randn()/10 = geringes Rauschen

Schritt 4 — SVR(kernel='poly') mit C=100 (enger Fit) vs. C=0.01 (glatte Kurve). Vertikal (2×1).

Python – SVR poly: C=100 vs C=0.01 (Folie 35) 
from sklearn.svm import SVR

reg_ph = SVR(kernel='poly', degree=2,
             C=100, epsilon=0.1, gamma='scale')
reg_pl = SVR(kernel='poly', degree=2,
             C=0.01, epsilon=0.1, gamma='scale')
reg_ph.fit(X_nl, y_nl)
reg_pl.fit(X_nl, y_nl)

fig, axes = plt.subplots(2, 1, figsize=(5, 9))
x_r = np.linspace(-1, 1, 200).reshape(-1, 1)

for ax, (m, lbl) in zip(axes, [
    (reg_ph, 'poly, C=100'),
    (reg_pl, 'poly, C=0.01')
]):
    y_r = m.predict(x_r)
    ax.scatter(X_nl, y_nl, s=20, alpha=0.5,
               color='steelblue')
    ax.plot(x_r, y_r, 'k-', lw=2)
    ax.fill_between(x_r.ravel(),
                    y_r-m.epsilon, y_r+m.epsilon,
                    alpha=0.2, color='orange')
    ax.set_title(f'SVR: {lbl}', fontsize=12)
plt.tight_layout(); plt.show()
▶ Ausgabe
  • 📊 Oben (C=100): stark angepasste Parabel
  • 📊 Unten (C=0.01): glattere, einfachere Kurve
  • Orange Band = ±ε Tube
  • C=100: High Variance (mehr Overfitting)
  • C=0.01: High Bias (mehr Underfitting)

Schritt 5 — SVR(kernel='rbf') mit C=100 vs. C=0.01. RBF-Kernel passt sich flexibler an als Poly.

Python – SVR rbf: C=100 vs C=0.01 (Folie 35) 
reg_rh = SVR(kernel='rbf', C=100, epsilon=0.1)
reg_rl = SVR(kernel='rbf', C=0.01, epsilon=0.1)
reg_rh.fit(X_nl, y_nl)
reg_rl.fit(X_nl, y_nl)

fig, axes = plt.subplots(2, 1, figsize=(5, 9))
x_r = np.linspace(-1, 1, 200).reshape(-1, 1)

for ax, (m, lbl) in zip(axes, [
    (reg_rh, 'rbf, C=100'),
    (reg_rl, 'rbf, C=0.01')
]):
    y_r = m.predict(x_r)
    ax.scatter(X_nl, y_nl, s=20, alpha=0.5,
               color='steelblue')
    ax.plot(x_r, y_r, 'k-', lw=2)
    ax.fill_between(x_r.ravel(),
                    y_r-m.epsilon, y_r+m.epsilon,
                    alpha=0.2, color='teal')
    ax.set_title(f'SVR: {lbl}', fontsize=12)
plt.tight_layout(); plt.show()
▶ Ausgabe
  • 📊 Oben (C=100): sehr enger Fit an die Daten
  • 📊 Unten (C=0.01): fast konstante Linie (extreme Regularisierung)
  • RBF: flexibler als Poly bei hohem C
  • Türkises Band = ±ε Tube
  • → C muss via Cross-Validation gewählt werden!
💡 So what?

SVR und SVC teilen dieselbe API: SVR(kernel=..., C=..., epsilon=...). Der Kernel-Trick funktioniert genauso. Der einzige Unterschied: SVR hat zusätzlich den ε-Parameter für die Tube-Breite. Auch hier gilt: ε und C via Cross-Validation optimieren.

8.3 Vorteile, Nachteile & No-Free-Lunch

Was können SVMs gut, was nicht? Ein ehrlicher Blick auf Stärken und Schwächen – und das wichtige No-Free-Lunch Theorem.

👍 Vorteile (Pros)
  • Sehr effektiv bei hochdimensionalen Datensätzen
  • Funktioniert gut wenn Dimensionen > Datenpunkte
  • Komputativ effizient durch Sparsity (nur Support Vectors zählen)
  • Erlaubt geometrische Interpretation: Hyperebene, Margin, Support Vectors
  • Flexibel durch Kernel-Trick: linear und nicht-linear
  • Gutes Verhalten auf kleinen und mittelgroßen Datensätzen
👎 Nachteile (Cons)
  • Probleme bei sehr großen Datensätzen (Kernel-Matrix wächst quadratisch)
  • Schlecht bei stark überlappenden Klassen oder viel Rauschen
  • Keine probabilistische Interpretation möglich
  • Hyperparameter C (und γ, ε) müssen sorgfältig gewählt werden
  • Schwer zu interpretieren bei nicht-linearem Kernel
  • Training kann langsam sein für große N
🎫
NO FREE LUNCH
Folie 31 · Wichtigste Lektion des Kapitels
Es gibt keinen universell besten Algorithmus. SVMs sind hervorragend für hochdimensionale, kleine bis mittelgroße Datensätze – aber Random Forests, neuronale Netze oder Naïve Bayes können in anderen Szenarien besser abschneiden. Das No-Free-Lunch Theorem besagt: Kein Algorithmus kann auf allen möglichen Problemen gleichzeitig optimal sein. Deshalb: immer mehrere Modelle ausprobieren, mit Cross-Validation vergleichen, und den Datensatz kennen!

Komplette Parameter-Referenz: SVC & SVR

🔧 C — Regularisierung

  • Gilt für SVC und SVR
  • Groß: wenig Regularisierung, enger Fit, Overfitting-Risiko
  • Klein: mehr Regularisierung, glattere Entscheidung, Underfitting-Risiko
  • Default: C=1.0

🔧 epsilon — Tube-Breite (SVR)

  • Nur für SVR / LinearSVR
  • Groß: breiter Tube, glattes Modell
  • Klein: enger Tube, sensitives Modell
  • Default: epsilon=0.1

🔧 kernel — Kernel-Typ

  • 'linear': gerade Hyperebene
  • 'rbf': Radial Basis Function (Default)
  • 'poly': polynomialer Kernel (Grad via degree)
  • 'sigmoid': neuronales Netz ähnlich

🔧 gamma — Kernel-Breite

  • Nur für RBF/Poly-Kernel
  • Steuert den Einflussbereich jedes Support Vectors
  • Groß: enger Einfluss → komplexe Grenze
  • Klein: weiter Einfluss → glattere Grenze
  • Default: gamma='scale' (automatisch)
