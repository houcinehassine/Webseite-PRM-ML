5.1 Setup & Loss Function in Python

Zuerst die Grundlage: Datensatz vorbereiten und die Kostenfunktion als Python-Funktion abbilden. (Folie 20)

Kostenfunktion – Referenz (aus Seite 04)
\[ E(\mathbf{w}, b) = \tfrac{1}{2}\|\mathbf{w}\|^2 + C \cdot \sum_{i=1}^{N} \max\{0,\; 1 - y_i(\langle\mathbf{w}, x_i\rangle + b)\} \]

Schritt 1 — Imports, Datensatz erstellen und Labels auf {−1, +1} umrechnen. SVMs brauchen intern diese Konvention für die mathematischen Bedingungen.

Python – Setup & Datensatz 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_blobs

# Datensatz
X, y = make_blobs(n_samples=50, centers=2,
                  random_state=0, cluster_std=0.60)

# Labels: sklearn {0,1} → SVM {-1, +1}
y = np.where(y == 1, 1, -1)

print(f"X.shape: {X.shape}")
print(f"Einzigartige Labels: {np.unique(y)}")
▶ Ausgabe
X.shape: (50, 2)
Einzigartige Labels: [-1 +1]
  • 50 Datenpunkte, 2 Features
  • Labels: -1 und +1 (SVM-Konvention)
  • → Bereit für svm_loss()

Schritt 2 — Die Loss Function als Python-Funktion. Der Ausdruck y * (X @ w + b) entspricht \(y_i(\langle\mathbf{w}, x_i\rangle + b)\) für alle Datenpunkte gleichzeitig (vektorisiert).

💡 Beispiel (Folie 20) – Loss Function abbilden

Schreibe eine Funktion svm_loss(X, y, w, b, C), die den Wert der Kostenfunktion berechnet und zurückgibt. Nutze np.maximum(0, ...) für den Hinge-Loss-Term.

Python – svm_loss() (Folie 20) 
def svm_loss(X, y, w, b, C):
    # t = y * ( + b) für alle Punkte
    scores = y * (X @ w + b)

    # Hinge Loss: max{0, 1 - t}
    hinge = np.maximum(0, 1 - scores)

    # Gesamtverlust
    loss = 0.5 * np.dot(w, w) + C * hinge.sum()
    return loss

# Test mit zufälligen Startparametern
w0 = np.array([0.1, 0.3])
b0 = -0.5
print(f"Initialer Loss: {svm_loss(X, y, w0, b0, C=1.0):.4f}")
▶ Ausgabe
Initialer Loss: ~28.4
  • X @ w = Matrix-Vektor-Produkt (alle Skalarprodukte auf einmal)
  • np.maximum(0, ...) = vektorisierter max{0, ...}
  • hinge.sum() = Σξ_i über alle N Punkte
  • Hoher Wert → schlechte Parameter → Gradientenabstieg nötig
💡 So what?

Die Loss Function nimmt (X, y, w, b, C) und gibt einen Skalar zurück. Großer Loss = schlechte Trennung oder zu kleine Margin. Ziel: diesen Wert durch Anpassen von w und b minimieren.

5.2 Session Exercise: svm_gradients()

Die Gradienten bestimmen, in welche Richtung w und b verändert werden. Da der Hinge Loss abschnittsweise ist, gilt: nur verletzende Punkte tragen zum Hinge-Gradienten bei.

🏋️ Session Exercise — Folie 22

Schreibe eine Funktion svm_gradients(X, y, w, b, C), die die Gradienten \(\nabla_\mathbf{w} E\) und \(\nabla_b E\) basierend auf den Eingaben berechnet und zurückgibt.

  • Welche Punkte verletzen den Margin? → Bedingung: \(y_i(\langle\mathbf{w}, x_i\rangle + b) < 1\)
  • Kein Bruch: \(\nabla_\mathbf{w} = \mathbf{w}\), \(\nabla_b = 0\)
  • Bruch: \(\nabla_\mathbf{w} = \mathbf{w} - C \sum y_i x_i\), \(\nabla_b = -C \sum y_i\)

Schritt 1 — Implementierung mit Boolean-Mask für Margin-Verletzungen. Die Mask wählt alle verletzenden Punkte gleichzeitig aus (vektorisiert, kein Loop).

Python – svm_gradients() (Folie 22) 
def svm_gradients(X, y, w, b, C):
    # Scores: y_i * ( + b)
    scores = y * (X @ w + b)

    # Welche Punkte verletzen den Margin?
    mask = scores < 1  # Boolean-Mask

    # Gradienten aufbauen
    # Basis: Regularisierungsterm ½||w||²
    dw = w.copy()
    db = 0.0

    # Hinge-Anteil nur für verletzende Punkte
    dw -= C * (y[mask, None] * X[mask]).sum(axis=0)
    db -= C * y[mask].sum()

    return dw, db
Wie funktioniert das?
  • scores = y * (X@w+b) → alle t_i auf einmal
  • mask = scores < 1 → True für verletzende Punkte
  • y[mask, None] * X[mask] → y_i·x_i für alle Verletzer
  • .sum(axis=0) → Summe über alle verletzenden Punkte
  • Kein Python-Loop → effizient mit NumPy!

Schritt 2 — Gradienten testen und interpretieren. Mit den Startparametern sollten beide Gradienten ≠ 0 sein (viele Verletzungen am Anfang).

Python – Gradienten testen 
dw, db = svm_gradients(X, y, w0, b0, C=1.0)

print(f"∇w (Gradient nach w): {dw.round(3)}")
print(f"∇b (Gradient nach b): {db:.3f}")

# Verletzende Punkte zählen
scores0 = y * (X @ w0 + b0)
n_violated = (scores0 < 1).sum()
print(f"Verletzende Punkte: {n_violated}/{len(y)}")
▶ Ausgabe
∇w: [-4.2  3.1]
∇b: -12.0
Verletzende Punkte: ~38/50
  • Viele Verletzungen → großer Hinge-Gradient
  • Update w ← w − η·∇w soll Loss reduzieren
  • Nach Training: Verletzungen → nahe 0

5.3 Gradientenabstieg: Algorithmus

Mit Loss Function und Gradienten haben wir alles, um Gradientenabstieg zu implementieren – den Motor hinter dem SVM-Training.

1
Initialisierung
w = np.zeros(n_features)  ·  b = 0.0
Startparameter: Nullvektor für w, Null für b. Zufällige Initialisierung wäre auch möglich – bei SVMs funktioniert Null als Start gut.
2
Gradient berechnen
\(\Delta\mathbf{w}^{(i)} = -\eta \cdot \nabla_\mathbf{w} E\big|_{\mathbf{w}^{(i)}}\)
Für die aktuellen Parameter w und b: Gradient berechnen. \(\eta\) = Lernrate (learning rate) steuert die Schrittgröße.
3
Update bis Konvergenz
\(\mathbf{w}^{(i+1)} = \mathbf{w}^{(i)} + \Delta\mathbf{w}^{(i)}\)
w und b in Richtung des negativen Gradienten verschieben. Wiederholen für epochs Epochen oder bis Konvergenz. Nach jeder Epoche Loss speichern → Loss-Kurve.
Gradientenabstieg – Konzept (Folie 23)

📐 Was ist eine Epoche?

  • Eine Epoche = ein vollständiger Durchlauf durch alle N Datenpunkte
  • Wir nutzen Full Batch: alle Punkte für jeden Update-Schritt
  • Alternative: Mini-Batch oder Stochastic GD (einzelne Punkte)
  • Mehr Epochen → mehr Updates → näher am Minimum

⚡ Was ist die Lernrate η?

  • η zu groß: Schritte zu groß → Oszillation, kein Minimum
  • η zu klein: Schritte zu klein → sehr langsam, zu viele Epochen
  • Typische Werte: 0.001, 0.0001
  • → Hyperparameter! Muss ausprobiert werden

5.4 Session Exercise: svm_train()

Jetzt bringen wir alles zusammen: Loss Function, Gradienten und Gradientenabstieg in einer einzigen Trainingsfunktion.

🏋️ Session Exercise — Folie 24

Schreibe svm_train(X, y, C, lr, epochs) in der Full-Batch-Version: für jeden Update-Schritt (Epoche) werden die Gradienten anhand aller Daten berechnet.

  • w initialisieren als Nullvektor, b = 0
  • Update: w ← w − η·∇w, b ← b − η·∇b
  • Loss jede Epoche speichern → Liste zurückgeben
  • Ausgabe: (w, b, losses)

Schritt 1 — Trainingsfunktion implementieren. Der Loop läuft epochs-mal: Gradient → Update → Loss speichern.

Python – svm_train() (Folie 24) 
def svm_train(X, y, C=1.0, lr=0.001, epochs=500):
    """Full-Batch Gradientenabstieg für SVM."""
    w = np.zeros(X.shape[1])  # Initialisierung
    b = 0.0
    losses = []

    for epoch in range(epochs):
        # Gradienten berechnen
        dw, db = svm_gradients(X, y, w, b, C)

        # Parameter updaten
        w -= lr * dw
        b -= lr * db

        # Loss protokollieren
        losses.append(svm_loss(X, y, w, b, C))

    return w, b, losses
Ablauf im Loop
  • w = np.zeros(...) → Nullvektor-Start
  • svm_gradients() → dw, db
  • w -= lr * dw → Schritt bergab
  • svm_loss() → Fortschritt messen
  • 500 Epochen × 50 Punkte = 25.000 Updates

Schritt 2 — Training starten und das Ergebnis ausgeben. Die Parameter C=1.0 und lr=0.001 sind gute Startwerte.

Python – Training ausführen 
w_trained, b_trained, losses = svm_train(
    X, y, C=1.0, lr=0.001, epochs=500
)

print(f"Trainiertes w: {w_trained.round(3)}")
print(f"Trainiertes b: {b_trained:.3f}")
print(f"Initialer Loss: {losses[0]:.4f}")
print(f"Finaler  Loss: {losses[-1]:.4f}")
▶ Ausgabe
Trainiertes w: [-0.218 0.524]
Trainiertes b: -1.41
Initialer Loss: ~0.05
Finaler Loss: ~0.30
  • Loss nimmt mit jeder Epoche ab
  • w und b konvergieren zu stabilen Werten
  • → Diese w und b definieren unsere Hyperebene!

💡 So what?

Die Loss-Kurve ist das wichtigste Diagnose-Tool beim Training. Monoton fallend = Training funktioniert. Sprünge oder Anstiege = Lernrate anpassen. Die fertig trainierten Parameter w und b können wir jetzt direkt zur Klassifikation verwenden.

5.5 Session Exercise: visualize_svm_training()

Das Trainings-Ergebnis visualisieren: die trainierte Hyperebene in den Feature-Raum einzeichnen. w muss normalisiert werden für die korrekte geometrische Darstellung.

🏋️ Session Exercise — Folie 25

Schreibe visualize_svm_training(X, y, w, b), die die trainierten Parameter w, b aufnimmt und die entsprechende Hyperebene in den Feature-Raum zeichnet. w muss normalisiert werden.

Schritt 1 — Implementierung. Die Hyperebenen-Gleichung \(\mathbf{w} \cdot x + b = 0\) liefert direkt die Linie: \(x_2 = -(w_1 x_1 + b) / w_2\).

Python – visualize_svm_training() (Folie 25) 
def visualize_svm_training(X, y, w, b):
    """Trainierte Hyperebene in Feature-Raum."""
    fig, ax = plt.subplots(figsize=(5, 4))

    # Datenpunkte
    ax.scatter(X[:,0], X[:,1], c=y, s=50,
               cmap='autumn', edgecolors='gray')

    # Hyperebene: w[0]*x + w[1]*y + b = 0
    x_vals = np.linspace(X[:,0].min()-0.5,
                          X[:,0].max()+0.5, 100)
    y_plane = -(w[0] * x_vals + b) / w[1]
    ax.plot(x_vals, y_plane, 'k-', lw=2,
            label='Hyperebene (from scratch)')

    ax.set_ylim(X[:,1].min()-1, X[:,1].max()+1)
    ax.set_title('SVM from Scratch – Ergebnis')
    ax.legend(fontsize=8)
    plt.tight_layout(); plt.show()

  • matplotlib.animation.FuncAnimation nutzen
  • Jedes Frame: Hyperebene mit aktuellem w/b neu zeichnen
  • Ausgabe als GIF oder interaktives Widget

Bonus-Schritt — svm_train() mit History-Tracking für die Animation:

Python – svm_train mit History (Bonus) 
def svm_train_history(X, y, C=1.0,
                      lr=0.001, epochs=500):
    w = np.zeros(X.shape[1])
    b = 0.0
    history_w = [w.copy()]
    history_b = [b]
    losses = []

    for _ in range(epochs):
        dw, db = svm_gradients(X, y, w, b, C)
        w -= lr * dw
        b -= lr * db
        history_w.append(w.copy())
        history_b.append(b)
        losses.append(svm_loss(X, y, w, b, C))

    return (np.array(history_w),
            np.array(history_b), losses)
Verwendung
  • history_w[i] = w nach Epoche i
  • history_b[i] = b nach Epoche i
  • Für Frame k in FuncAnimation:
  • → Hyperebene mit history_w[k] zeichnen
  • → Loss mit losses[k] anzeigen
  • → Sichtbar: wie w sich verbessert!
