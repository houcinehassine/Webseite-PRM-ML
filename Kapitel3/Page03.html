Was steckt hinter dem Margin? Hier leiten wir das Optimierungsproblem der SVM Schritt für Schritt her – von der Hyperebenen-Definition bis zum finalen min ½‖w‖².

Trainingsdatensatz Normalenvektor w Bias b Hyperebene Vorzeichen & sgn() Abstand r Hard-Margin min ½‖w‖²

Lernfokus dieser Seite

  • Du kannst den Trainingsdatensatz formal beschreiben: \(\{(x_i, y_i)\}\), \(y_i \in \{-1, +1\}\)
  • Du weißt, wie eine Hyperebene über \(\mathbf{w}\) und \(b\) definiert ist
  • Du verstehst die Vorzeichen-Regel und das Modell \(y_i = \text{sgn}(\langle\mathbf{w}, x_i\rangle + b)\)

3.1 Grundlagen: Datensatz & Hyperebene

Wir formalisieren, womit wir arbeiten: ein historischer Trainingsdatensatz und eine Hyperebene, die wir durch zwei Parameter vollständig beschreiben können.

Der Trainingsdatensatz

Trainingsdatensatz (Folie 12)
\[ \{(x_i,\, y_i) \mid i = 1,\ldots,N,\quad y_i \in \{-1,\, +1\}\} \]
\(x_i\) Feature-Vektor des i-ten Datenpunkts (eine Zeile aus der Feature-Matrix X)
\(y_i \in \{-1,+1\}\) Label – nicht 0/1 wie sonst, sondern –1 und +1 (wichtig für die Mathematik!)
\(N\) Anzahl der Datenpunkte im Trainingsdatensatz

⚠️ Warum y ∈ {−1, +1}?

  • In sklearn üblich: Labels 0 und 1
  • In der SVM-Mathematik: Labels müssen −1 und +1 sein
  • Grund: Das Vorzeichen wird direkt in der Optimierungsformel verwendet
  • sklearn wandelt intern automatisch um – für uns als Nutzer transparent

📐 Die Hyperebene

  • In 2D: eine Gerade
  • In 3D: eine Ebene
  • In nD: eine Hyperebene – allgemeiner Begriff
  • Beschrieben durch: Normalenvektor \(\mathbf{w}\) + Abstand zum Ursprung \(b\)
  • Diese beiden Parameter sind alles, was wir brauchen

Definition der Hyperebene

Hyperebenen-Gleichung (Folie 12)
\[ \langle \mathbf{w},\, x_i \rangle + b = 0 \]
\(\mathbf{w}\) Normalenvektor – steht senkrecht auf der Hyperebene, zeigt in die positive Richtung. In sklearn: model.coef_
\(b\) Bias – verschiebt die Hyperebene vom Ursprung. Gibt den Abstand der Ebene zum Ursprung an. In sklearn: model.intercept_
\(\langle\mathbf{w}, x_i\rangle\) Skalarprodukt (dot product) von Normalenvektor und Datenpunkt

In sklearn sind \(\mathbf{w}\) und \(b\) nach dem Training direkt als Attribute zugänglich. Wir fitten ein Modell und schauen uns die konkreten Werte an.

Python – w und b aus trained SVC 
import numpy as np
from sklearn.datasets import make_blobs
from sklearn.svm import SVC

X, y = make_blobs(n_samples=50, centers=2,
                  random_state=0, cluster_std=0.60)

model = SVC(kernel='linear', C=1E10).fit(X, y)

# Normalenvektor w und Bias b
w = model.coef_[0]
b = model.intercept_[0]

print(f"w (Normalenvektor): {w.round(3)}")
print(f"b (Bias):           {b:.3f}")
print(f"‖w‖ (Norm):         {np.linalg.norm(w):.3f}")
▶ Ausgabe
w (Normalenvektor): [-0.218  0.525]
b (Bias):           -1.427
‖w‖ (Norm):         0.569
  • model.coef_[0] → \(\mathbf{w}\) als Array der Form (n_features,)
  • model.intercept_[0] → Skalar \(b\)
  • ‖w‖ ≠ 1 → w ist nicht normiert (Skalierung folgt in 3.4)
💡 So what?

Das gesamte SVM-Modell ist in zwei Parametern gespeichert: \(\mathbf{w}\) (Normalenvektor, 2 Werte für 2D) und \(b\) (Bias, 1 Wert). Zusammen beschreiben sie eine eindeutige Hyperebene im Feature-Raum.

3.2 Hyperebene & Vorzeichen

Wie klassifiziert die Hyperebene neue Punkte? Das Vorzeichen des Ausdrucks \(\langle\mathbf{w}, x_i\rangle + b\) entscheidet, auf welcher Seite ein Punkt liegt – und damit seine Klasse.

🪟 Fenster Time! — Folie 13

Wie drückt man mathematisch aus, ob ein Punkt auf der Hyperebene liegt? Was gilt für Punkte, die nicht auf der Hyperebene liegen? Wie berechnet man die Klassen neuer Daten? Warum soll der Margin maximiert werden?

Auf der Hyperebene
\(\langle\mathbf{w}, x_i\rangle + b = 0\)
Der Punkt liegt exakt auf der Trennebene – kein Vorzeichen bestimmbar.
Positive Seite (Klasse +1)
\(\langle\mathbf{w}, x_i\rangle + b > 0\)
Punkt liegt in Richtung des Normalenvektors \(\mathbf{w}\) – positive Klasse.
Negative Seite (Klasse −1)
\(\langle\mathbf{w}, x_i\rangle + b < 0\)
Punkt liegt entgegengesetzt zu \(\mathbf{w}\) – negative Klasse.
Das Modell (Folie 13)
\[ y_i = \text{sgn}(\langle\mathbf{w}, x_i\rangle + b) \]
\(\text{sgn}(\cdot)\) Vorzeichenfunktion: gibt +1 zurück wenn positiv, −1 wenn negativ, 0 wenn null
Bedingung Wenn die Hyperebene alle Punkte trennt: \(y_i(\langle\mathbf{w}, x_i\rangle + b) \geq 0\) — Produkt aus wahrem Label und Ausdruck ist immer ≥ 0

In sklearn entspricht der Ausdruck \(\langle\mathbf{w}, x_i\rangle + b\) dem Ergebnis von decision_function(). Das Vorzeichen ergibt die vorhergesagte Klasse.

Python – decision_function = ⟨w, x⟩ + b 
# Decision function:  + b für jeden Punkt
df = model.decision_function(X)

print("decision_function (erste 6 Werte):")
print(df[:6].round(3))

print("\nVorzeichen = vorhergesagte Klasse:")
print(np.sign(df[:6]).astype(int))

print("\nEchte Labels (y):")
# sklearn: y ∈ {0,1} → intern {-1,+1}
y_signed = np.where(y == 0, -1, 1)
print(y_signed[:6])
▶ Ausgabe
decision_function (erste 6):
[-1.842 -1.714 -1.431 -1.296 -1.589 1.203]

Vorzeichen = Klasse:
[-1 -1 -1 -1 -1 +1]

Echte Labels:
[-1 -1 -1 -1 -1 +1]
  • Negative Werte → Klasse −1
  • Positive Werte → Klasse +1
  • Betrag = Abstand zur Hyperebene
  • → Vorzeichen = Klassifikation ✓

Einfärbung der Entscheidungsregionen: positive Seite (blau) vs. negative Seite (rot), mit dem Normalenvektor \(\mathbf{w}\) eingezeichnet.

  • 🔵 Blaue Region: \(\langle\mathbf{w}, x\rangle + b > 0\) → Klasse +1
  • Schwarze Linie: Hyperebene (\(= 0\))

💡 So what?

Das Modell ist einfach: berechne \(\langle\mathbf{w}, x\rangle + b\), nimm das Vorzeichen – fertig. Positiv = eine Klasse, negativ = andere Klasse. Der Betrag gibt sogar an, wie "sicher" die Zuordnung ist (großer Betrag = weit weg von der Grenze).

3.3 Das Optimierungsproblem

Aus unendlich vielen Hyperebenen wählen wir diejenige mit dem größten Abstand r zu den nächsten Datenpunkten. Das formalisieren wir als Optimierungsproblem.

📏 Was ist der Abstand r?

  • Der Abstand eines Punktes \(x_1\) zur Hyperebene
  • Geometrisch: senkrechter Abstand in Richtung \(\mathbf{w}\)
  • Formel: \(x_1 = x'_1 + r \cdot \frac{\mathbf{w}}{\|\mathbf{w}\|}\)
  • \(x'_1\) ist die Projektion von \(x_1\) auf die Hyperebene
  • r ist der halbe Margin – wir maximieren ihn

🔧 Bedingungen des Problems

  • \(y_i(\langle\mathbf{w}, x_i\rangle + b) \geq r\) — alle Punkte korrekt klassifiziert, mindestens Abstand r
  • \(\|\mathbf{w}\| = 1\) — Normalenvektor ist normiert (Einheitsvektor)
  • \(r > 0\) — Abstand ist positiv
  • Zusammen: 3 Bedingungen
Optimierungsproblem (Folie 14)
\[ \max_{\mathbf{w},\, b,\, r}\; r \] \[ \text{unter den Bedingungen:}\quad y_i(\langle\mathbf{w}, x_i\rangle + b) \geq r,\quad \|\mathbf{w}\| = 1,\quad r > 0 \]
🧠 Frage — Folie 14

Warum soll der Margin maximiert werden? Was bedeutet ein größerer Margin für neue, unbekannte Datenpunkte? Und warum muss \(\|\mathbf{w}\| = 1\) gelten?

💡 So what?

Dieses Optimierungsproblem hat drei freie Parameter: \(\mathbf{w}\), \(b\), \(r\). Die Nebenbedingung \(\|\mathbf{w}\|=1\) macht das Problem aber schwer zu lösen (nicht-konvex). → Im nächsten Schritt vereinfachen wir es durch eine clevere Skalierung.

3.4 Hard-Margin Herleitung

Durch eine clevere Skalierung lässt sich das Optimierungsproblem drastisch vereinfachen. Das Ergebnis ist das kanonische SVM-Problem: min ½‖w‖².

🧠 Frage — Folie 15

Warum heißen diese Bedingungen "Hard Margin"? Was würde passieren, wenn Datenpunkte innerhalb des Margins liegen dürften? (Antwort: das wäre Soft-Margin → Seite 04)

Schritt-für-Schritt: Von max r → min ½‖w‖²

1
\(\|\mathbf{w}\| = 1\) aufgeben
Der Normalenvektor \(\mathbf{w}\) muss nicht mehr normiert sein. Stattdessen skalieren wir das Problem so, dass die Support Vectors einen bestimmten Wert annehmen.
2
\(\langle\mathbf{w}, x_a\rangle + b = 1\)
Wir skalieren \(\mathbf{w}\) und \(b\) so, dass der nächste Punkt \(x_a\) (= Support Vector) den Wert exakt 1 ergibt. Das ist immer möglich – nur eine Skalierung.
3
\(r = \dfrac{1}{\|\mathbf{w}\|}\)
Aus der Skalierung folgt direkt: der Abstand r des Support Vectors zur Hyperebene ist 1/‖w‖. Beweis: einsetzen in die Abstandsformel aus Folie 14.
4
\(y_i(\langle\mathbf{w}, x_i\rangle + b) \geq 1\)
Die Bedingung \(y_i(\ldots) \geq r\) wird zu \(\geq 1\) (durch die Skalierung). Das ist die Hard-Margin-Bedingung. Kein Punkt darf innerhalb des Margins liegen.
5
\(\min \tfrac{1}{2}\|\mathbf{w}\|^2\)
max r = max 1/‖w‖ ≡ min ‖w‖ ≡ min ½‖w‖² (½ erleichtert die Ableitung – mathematisch äquivalent). Das ist das finale, lösbare Optimierungsproblem!
Hard-Margin SVM – Finales Optimierungsproblem (Folie 15)
\[ \min_{\mathbf{w},\, b}\; \tfrac{1}{2}\|\mathbf{w}\|^2 \] \[ \text{unter der Bedingung:}\quad y_i(\langle\mathbf{w}, x_i\rangle + b) \geq 1 \quad \forall\, i \]
½‖w‖² Quadrat der Norm von w – minimieren = Margin maximieren (⅟‖w‖ = r)
Bedingung Alle Punkte müssen korrekt klassifiziert sein UND mindestens Abstand 1 haben (Hard: keine Verletzungen erlaubt)

Wir können \(r = 1/\|\mathbf{w}\|\) direkt aus dem trainierten Modell berechnen und mit dem geometrischen Abstand der Support Vectors vergleichen.

Python – r = 1/‖w‖ und Margin-Breite 
# Hard-Margin Mathematik verifizieren
norm_w = np.linalg.norm(w)
r = 1 / norm_w  # Abstand Support Vector → Hyperebene

print(f"‖w‖   = {norm_w:.4f}")
print(f"r = 1/‖w‖ = {r:.4f}  (halber Margin)")
print(f"Margin = 2r = {2*r:.4f}")

# Verifikation: decision_function der SVs = ±1
sv_vals = model.decision_function(
              model.support_vectors_)
print(f"\nDecision function an Support Vectors:")
print(sv_vals.round(4))
print("→ Sollte betragsmäßig ≈ 1 sein!")
▶ Ausgabe
‖w‖   = 0.5686
r = 1/‖w‖ = 1.7587
Margin = 2r = 3.5175

Decision function an SVs:
[-1. 1. -1.]
→ Sollte betragsmäßig ≈ 1 sein! ✓
  • Support Vectors liegen exakt auf den Margin-Grenzen
  • decision_function(SV) = ±1 → Herleitung korrekt!
  • Margin = 2r (von −1 bis +1 Ebene)

📌 Was ist Hard-Margin?

  • Bedingung: kein Punkt darf im Margin sein
  • Alle Punkte: \(y_i(\langle\mathbf{w}, x_i\rangle + b) \geq 1\)
  • Funktioniert nur bei linear separierbaren Daten
  • Bei Rauschen oder Überlappung: → Soft-Margin (Seite 04)

🔄 Warum ½‖w‖² statt ‖w‖?

  • Mathematisch äquivalent: min ‖w‖ ≡ min ½‖w‖²
  • Der Faktor ½ erleichtert die Ableitung des Problems
  • Ableitung von ½‖w‖² nach w = w (kein störender Faktor)
  • Standard in der Optimierung für quadratische Probleme

🎯 Was bedeutet das praktisch?

  • min ½‖w‖² = max Margin (äquivalent)
  • Das ist ein quadratisches Optimierungsproblem mit linearen Nebenbedingungen (QP)
  • QP hat eine eindeutige globale Lösung → kein lokales Minimum
  • sklearn löst dieses Problem intern (via LIBSVM)
💡 So what? — Zusammenfassung Seite 03

Die Mathematik hinter SVMs ist elegant: aus dem intuitiven "Margin maximieren" wird ein sauberes Optimierungsproblem min ½‖w‖² mit Bedingung \(y_i(\langle\mathbf{w}, x_i\rangle + b) \geq 1\). Das ist das Hard-Margin-Problem – es funktioniert nur wenn die Klassen perfekt trennbar sind. Was wenn nicht? → Soft-Margin auf Seite 04.
