Soft-Margin & Optimierung

Reale Daten sind selten perfekt trennbar. Der C-Parameter kontrolliert, wie viel Toleranz wir zulassen – von hartem Hard-Margin bis zu flexiblem Soft-Margin. Dazu kommt der Hinge Loss als elegante kontinuierliche Kostenfunktion.

4.1 Hard- vs. Soft-Margin: Einführung

Bei SVMs kann man den Margin entweder starr (hard) oder weich (soft) definieren – je nachdem, ob Datenpunkte innerhalb des Margins erlaubt sind.

😤 Das Problem mit Hard-Margin

  • In der Praxis sind Klassen selten perfekt trennbar
  • Bei Rauschen oder Überlappung: Hard-Margin schlägt fehl
  • Sehr enger Margin → Overfitting-Risiko
  • Kleine Störungen in den Daten → komplett andere Trennlinie
  • → Hard-Margin ist zu rigide für reale Daten

✅ Die Lösung: Soft-Margin

  • Soft-Margin erlaubt Punkte innerhalb des Margins
  • Leitfrage: wie viele und zu welchem Ausmaß?
  • Kontrolliert durch den Hyperparameter C
  • Große C → Hard-Margin (wenig Toleranz)
  • Kleine C → Soft-Margin (viel Toleranz)
  • Optimalen C-Wert: via Cross-Validation finden

C-Parameter Spektrum

← C klein: Soft-Margin (breiter Margin, viele SVs) C groß: Hard-Margin (enger Margin, wenige SVs) →
C = 0.1 → Soft-Margin
  • Breiter Margin
  • Viele Support Vectors
  • Mehr Punkte im Margin erlaubt
  • Höhere Regularisierung
  • Eher Underfitting-Risiko
C = 10 → Hard(er)-Margin
  • Enger Margin
  • Wenige Support Vectors
  • Kaum Punkte im Margin erlaubt
  • Wenig Regularisierung
  • Eher Overfitting-Risiko
💡 So what?

Der optimale Wert von C hängt vom Datensatz ab und muss via z.B. Cross-Validation bestimmt werden. Es gibt kein universell bestes C – das ist das No-Free-Lunch-Theorem in der Praxis.

4.2 Soft-Margin: Mathematik

Wir erweitern das Hard-Margin-Problem durch die Slack-Variable ξ_i – sie quantifiziert, wie stark ein Punkt den Margin verletzt.

🧠 Frage — Folie 17

Was ändern wir an unseren Formulierungen aus der Hard-Margin-Herleitung? Wie können wir Verletzungen des Margins quantifizieren – und wie bestrafen wir sie?

📏 Was ist die Slack-Variable ξ_i?

  • \(\xi_i = 0\): Punkt liegt korrekt außerhalb des Margins
  • \(0 < \xi_i \leq 1\): Punkt liegt im Margin (richtige Seite)
  • \(\xi_i > 1\): Punkt liegt auf der falschen Seite (Fehler)
  • Jeder Datenpunkt hat seine eigene Slack-Variable
  • → Sie misst die Margin-Verletzung

⚖️ Der Trade-Off

  • Ohne C: alle ξ_i werden 0 gesetzt → nur Hard-Margin möglich
  • Mit C: wir bestrafen Verletzungen mit Gewicht C
  • Großes C → Verletzungen teuer → Modell versucht Hard-Margin
  • Kleines C → Verletzungen günstig → Modell akzeptiert weichen Margin
  • → C heißt auch Regularisierungsparameter
Soft-Margin SVM – Optimierungsproblem (Folie 17)
\[ \min_{\mathbf{w},\, b,\, \boldsymbol{\xi}}\; \tfrac{1}{2}\|\mathbf{w}\|^2 + C \cdot \sum_{i=1}^N \xi_i \]
unter den Bedingungen:
\[ y_i(\langle\mathbf{w}, x_i\rangle + b) \geq 1 - \xi_i \qquad \text{und} \qquad \xi_i \geq 0 \quad \forall\, i \]
\(\xi_i \geq 0\) Slack-Variable: misst die Margin-Verletzung des i-ten Punkts (0 = keine Verletzung)
\(C \cdot \sum \xi_i\) Summe aller Verletzungen, gewichtet mit C – dieser Term bestraft Verletzungen
\(\geq 1 - \xi_i\) Relaxierte Bedingung: statt ≥ 1 ist jetzt ≥ 1 − ξ_i erlaubt (Hard: ξ_i = 0)
💡 So what?

Das Soft-Margin-Problem ist eine direkte Erweiterung des Hard-Margin: wir addieren den Straf-Term C·Σξ_i. Mit C → ∞ erhalten wir Hard-Margin zurück (alle ξ_i = 0 erzwungen). C heißt Regularisierungsparameter, weil er den Trade-Off zwischen Margin-Größe und Fehlertoleranz kontrolliert.

4.4 Hinge Loss Function

Die Slack-Variable lässt sich eliminieren – durch den Hinge Loss. Das ergibt eine elegante, kontinuierlich differenzierbare Kostenfunktion, die mit Gradientenabstieg optimiert werden kann.

❌ Warum nicht Accuracy?

  • Accuracy (0-1-Loss) wäre die naheliegendste Verlustfunktion
  • Problem: nicht differenzierbar (Sprünge bei ganzzahligen Fehlern)
  • Gradientenabstieg braucht aber einen glatten Verlauf
  • → Ein kombinatorisches Optimierungsproblem – nicht handhabbar

✅ Warum Hinge Loss?

  • Kontinuierlich: keine Sprünge
  • Linear für t < 1 → Ableitung ist konstant
  • Null für t ≥ 1 → korrekt klassifizierte Punkte außerhalb des Margins = 0 Kosten
  • Direkte Verbindung zur Slack-Variable: \(\xi_i = \max\{0, 1-t_i\}\)
Hinge Loss (Folie 19)
\[ L(t) = \max\{0,\; 1 - t\} \qquad \text{mit} \qquad t = y \cdot f(x) = y(\langle\mathbf{w}, x\rangle + b) \]
\(t = y\cdot f(x)\) Wenn t > 1: korrekt & außerhalb Margin → L = 0 (kein Verlust). Wenn t < 1: Margin-Verletzung → L > 0
\(\max\{0, 1-t\}\) Für t ≥ 1: immer 0. Für t < 1: wächst linear. Entspricht exakt der Slack-Variable ξ_i

Vergleich von 0-1-Loss (nicht differenzierbar, orange gestrichelt) und Hinge Loss (durchgezogen, orange) – beide erfassen dasselbe Prinzip, aber der Hinge Loss ist optimierbar.

Python – Hinge Loss visualisieren (Folie 19) 
t = np.linspace(-0.5, 2.5, 300)

# Hinge Loss: max(0, 1-t)
hinge = np.maximum(0, 1 - t)

# 0-1 Loss (vereinfacht)
zero_one = (t < 1).astype(float)

fig, ax = plt.subplots(figsize=(5, 4))
ax.plot(t, zero_one, 'k--', lw=2,
        label='0-1 Loss (nicht differenzierbar)')
ax.plot(t, hinge, color='darkorange', lw=2.5,
        label='Hinge Loss: max{0, 1−t}')

ax.axvline(x=1, color='gray', ls=':', alpha=0.6)
ax.axhline(y=0, color='gray', alpha=0.3)
ax.set_xlabel('t = y · f(x)')
ax.set_ylabel('L(t)')
ax.set_title('Hinge Loss vs. 0-1 Loss')
ax.set_ylim(-0.1, 1.5)
ax.legend(fontsize=9)
plt.tight_layout(); plt.show()
▶ Ausgabe
  • 📊 X-Achse: t = y·f(x) – das "Score × Label"
  • Bei t=1: Knick-Punkt (Support Vector liegt hier!)
  • Links von t=1: Hinge Loss > 0 (Verlust)
  • Rechts von t=1: Hinge Loss = 0 (kein Verlust)
  • 0-1 Loss: Sprung → nicht differenzierbar
  • Hinge Loss: stetige Kurve → differenzierbar
Finales Soft-Margin-Problem mit Hinge Loss (Folie 19)
\[ \min_{\mathbf{w},\, b}\; \tfrac{1}{2}\|\mathbf{w}\|^2 + C \cdot \sum_{i=1}^{N} \max\{0,\; 1 - y_i(\langle\mathbf{w}, x_i\rangle + b)\} \]
Die Slack-Variablen ξ_i wurden durch den Hinge-Loss-Term ersetzt. Die Formulierung ist jetzt vollständig vereinfacht und direkt per Gradientenabstieg optimierbar.
💡 So what? — Regularisierung = Margin-Maximierung

Der Term ½‖w‖² ist gleichzeitig Regularisierung (verhindert Overfitting) und Margin-Maximierung (‖w‖ klein → 1/‖w‖ groß → Margin groß). SVMs vereinen also Klassifikation und Regularisierung in einem Optimierungsproblem.

4.5 Gradienten der Loss Function

Um Gradientenabstieg anzuwenden, brauchen wir die Gradienten der Kostenfunktion nach w und b. Da der Hinge Loss einen Knick bei t=1 hat, sind die Gradienten abschnittsweise definiert.

🔀 Warum abschnittsweise?

  • Der Hinge Loss hat bei t=1 einen Knick (nicht glatt, aber differenzierbar)
  • Links: \(L(t) = 1-t\) → Steigung = −1
  • Rechts: \(L(t) = 0\) → Steigung = 0
  • → Gradient hängt davon ab, ob ein Punkt den Margin verletzt oder nicht

📋 Die zwei Fälle

  • Kein Margin-Bruch: \(y_i(\langle\mathbf{w}, x_i\rangle+b) \geq 1\)
  • → Hinge Loss = 0, nur der Regularisierungsterm ½‖w‖² trägt bei
  • Margin-Bruch: \(y_i(\langle\mathbf{w}, x_i\rangle+b) < 1\)
  • → Hinge Loss > 0, beide Terme tragen zum Gradienten bei
✅ Kein Margin-Bruch
\(y_i(\langle\mathbf{w}, x_i\rangle + b) \geq 1\)
\(\nabla_{\mathbf{w}}\, E = \mathbf{w}\)
\(\nabla_{b}\, E = 0\)
Nur der Regularisierungsterm ½‖w‖² trägt bei. Gradient von ½‖w‖² nach w ist einfach w – kein Einfluss auf b.
⚠️ Margin-Bruch
\(y_i(\langle\mathbf{w}, x_i\rangle + b) < 1\)
\(\nabla_{\mathbf{w}}\, E = \mathbf{w} - C \sum_i y_i \cdot x_i\)
\(\nabla_{b}\, E = -C \sum_i y_i\)
Regularisierungs- und Hinge-Loss-Term tragen bei. Der Hinge-Loss zieht w und b in Richtung korrekterer Klassifikation.
Gradientenabstieg – Update-Regel
\[ \mathbf{w}^{(t+1)} = \mathbf{w}^{(t)} - \eta \cdot \nabla_{\mathbf{w}} E \] \[ b^{(t+1)} = b^{(t)} - \eta \cdot \nabla_b E \]
\(\eta\) Lernrate (learning rate) – kontrolliert die Schrittgröße beim Update
\(E\) Kostenfunktion: min ½‖w‖² + C·Σmax{0, 1-y_i(⟨w,x_i⟩+b)}

Die Gradienten können wir direkt in Python übersetzen – genau das tun wir auf Seite 05 (SVM from Scratch). Hier als Vorschau die Logik in Pseudocode:

Python – Gradient Logik (Vorschau auf Seite 05) 
def compute_gradient(X, y, w, b, C):
    """Abschnittsweise Gradienten des Hinge Loss."""
    dw = np.zeros_like(w)
    db = 0.0

    for xi, yi in zip(X, y):
        # yi ∈ {-1, +1}: intern umrechnen
        yi_signed = 1 if yi == 1 else -1

        # Bedingung prüfen
        margin = yi_signed * (np.dot(w, xi) + b)

        if margin >= 1:
            # Kein Margin-Bruch
            dw += w
            db += 0
        else:
            # Margin-Bruch
            dw += w - C * yi_signed * xi
            db += -C * yi_signed

    return dw / len(X), db / len(X)
Was dieser Code macht
  • Schleife über alle Punkte (xi, yi)
  • Bedingung: margin = yi·(⟨w,xi⟩+b)
  • ≥ 1: nur Regularisierungs-Gradient
  • < 1: Regularisierung + Hinge-Gradient
  • Division durch N → Mittelwert über Batch
  • → Vollständige Implementierung auf Seite 05
💡 So what? — Zusammenfassung Seite 04

Soft-Margin SVMs sind eine Erweiterung des Hard-Margin-Problems durch die Slack-Variable ξ_i. Der Hinge Loss fasst das elegant zusammen: er ist 0 für korrekt klassifizierte Punkte außerhalb des Margins und wächst linear für Verletzungen. Die abschnittsweisen Gradienten ermöglichen Gradientenabstieg. → Alles zusammen → SVM from Scratch auf Seite 05.
