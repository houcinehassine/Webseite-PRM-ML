Nicht-lineare Probleme & Kernel-Trick

Was passiert, wenn Klassen gar nicht linear trennbar sind? SVMs haben eine elegante Antwort: Daten in höhere Dimensionen projizieren – und das so effizient, dass die Projektion nie wirklich berechnet wird.

make_circles linear nicht trennbar Radial Basis Function r = exp(−‖x‖²) 2D→3D Projektion Kernel-Trick kernel='rbf' Elliptische Grenze

Lernfokus dieser Seite

  • Du erkennst ein nicht-lineares Problem und weißt, warum linearer SVC scheitert
  • Du verstehst Radial Basis Functions (RBF): warum \(r = e^{-\|x\|^2}\) funktioniert
  • Du weißt, wie 2D→3D Projektion lineare Separierbarkeit herstellt
  • Du kennst den Kernel-Trick: implizite Projektion mit kernel='rbf'
  • Du kannst das Ergebnis: elliptische Entscheidungsgrenze interpretieren

6.1 Das Problem: Nicht-linear separierbar

Bisher haben wir angenommen, dass eine gerade Hyperebene die Klassen trennen kann. Was wenn das grundsätzlich unmöglich ist?

❌ Das Problem — Folie 27

Konzentrische Ringe: Klasse 2 (gelb, innen) ist vollständig von Klasse 1 (rot, außen) umschlossen. Keine gerade Linie kann diese Klassen jemals trennen – egal wo wir sie ziehen. Das ist ein nicht-linear separierbares Problem.

Schritt 1 — Datensatz erstellen und linearen SVC versuchen. make_circles() erzeugt zwei konzentrische Ringe. Der lineare SVC zeichnet nur eine gerade Linie – er scheitert hier fundamental.

Python – make_circles + Linear SVC (Folie 27, Cell 9) 
import matplotlib.pyplot as plt
import numpy as np
from sklearn.datasets import make_circles
from sklearn.svm import SVC

# Konzentrische Ringe: linear nicht trennbar!
X, y = make_circles(100, factor=0.1, noise=0.1)

# Linearer SVC – wird scheitern
clf_linear = SVC(kernel='linear').fit(X, y)

fig, ax = plt.subplots(figsize=(5, 4))
ax.scatter(X[:, 0], X[:, 1], c=y, s=50,
           cmap='autumn', edgecolors='gray')
plot_svc_decision_function(clf_linear, ax,
                            plot_support=False)
ax.set_title('Linear SVC: Grundlegendes Versagen!')
plt.tight_layout(); plt.show()
▶ Ausgabe
  • 📊 Konzentrische Ringe: gelb (innen), rot (außen)
  • Gerade Linie schneidet beide Klassen
  • Egal wie wir die Linie legen: immer falsch
  • factor=0.1: kleiner Innenring
  • noise=0.1: etwas Rauschen
  • → Kein linearer Klassifikator kann das lösen!

🔍 Was denken Sie? — Folie 27

  • Wie würden Sie dieses Problem lösen?
  • Welche geometrische Form würde die Klassen trennen?
  • Warum versagt eine Gerade hier fundamental?
  • → Idee: die Daten in einen höheren Raum projizieren!

💡 Die Schlüsselidee

  • In 2D: Klassen sind nicht linear trennbar
  • Was wenn wir eine 3. Dimension hinzufügen?
  • In 3D könnten die Klassen auf unterschiedlicher Höhe liegen
  • Dann wäre eine horizontale Ebene die Trennlinie!
  • → Lineare Separierbarkeit durch Dimensionserhöhung
💡 So what?

SVMs sind in der Lage, nicht-linear separierbare Probleme zu lösen. Die Lösung: Projektion der Daten in einen höherdimensionalen Raum, wo lineare Separierbarkeit wiederhergestellt wird. Wie? → Radial Basis Functions.

6.2 Lösung: RBF & 3D-Projektion

Eine Möglichkeit, eine neue Dimension hinzuzufügen: Radial Basis Functions (RBFs) verwenden. Sie transformieren jeden 2D-Punkt in einen 3D-Punkt – und die neue Dimension macht die Klassen trennbar.

Start

(x₁, x₂) für jeden Punkt.

Nicht trennbar!
→
RBF
📐
r = exp(−‖x‖²)
Neue Dimension basierend

→
Ergebnis
🧊
3D-Datensatz
(x₁, x₂, r) für jeden Punkt.
Innen-Punkte oben (r≈1).
Linear trennbar!
Radial Basis Function (Folie 28)
\[ r = e^{-\|x\|^2} = e^{-(x_1^2 + x_2^2)} \]
\(\|x\|^2\) Quadratischer Abstand vom Ursprung: x₁² + x₂² (je weiter weg, desto größer)
\(e^{-\|x\|^2}\) Exponential: nah am Ursprung → r nahe 1. Weit weg → r nahe 0. Nur vom Abstand abhängig!
Innenring Punkte nah am Ursprung → ‖x‖² klein → r ≈ 1 (hoher r-Wert)
Außenring Punkte weit vom Ursprung → ‖x‖² groß → r ≈ 0 (tiefer r-Wert)
🧠 Frage — Folie 28

Warum hängt r nur vom Abstand zum Ursprung ab? Was passiert geometrisch, wenn wir r als dritte Achse einführen? Welches Problem sehen wir generell bei dieser manuellen Projektion?

Schritt-für-Schritt: Die r-Formel verstehen

Schritt 1 — Rohdaten und quadrierte Features inspizieren. Jedes Feature wird einzeln quadriert: x₁² und x₂².

Python – Rohdaten & x² (Cells 11–12)

print(X[:5].round(3))

print("\n(X**2)[:5] — Quadriert (x₁², x₂²):")

Schritt 2 — Quadrate summieren = ‖x‖² (quadratischer Abstand vom Ursprung) und den RBF-Wert r berechnen.

Python – ‖x‖² summieren & r berechnen (Cells 13, 10) 
# Summe der Quadrate = ||x||² = x₁² + x₂²
dist_sq = (X**2).sum(axis=1)
print("||x||² (erste 5):")
print(dist_sq[:5].round(3))

# RBF: r = exp(-||x||²)
r = np.exp(-dist_sq)
print("\nr = exp(-||x||²) (erste 5):")
print(r[:5].round(3))
print(f"\nMin r: {r.min():.3f}, Max r: {r.max():.3f}")
▶ Ausgabe
||x||²:
[0.004 0.052 0.892 1.117 0.944]

r:
[0.996 0.949 0.410 0.327 0.389]

Min: 0.007  Max: 0.998
  • axis=1 = Summe pro Zeile (über Features)
  • Innenring: ‖x‖² klein → r ≈ 1 (hoher Wert)
  • Außenring: ‖x‖² groß → r ≈ 0 (tiefer Wert)
  • → Die zwei Klassen sind jetzt auf verschiedener Höhe!

Schritt 3 — Die dritte Dimension r anhängen und den 3D-Datensatz inspizieren. Jetzt hat jeder Punkt 3 Features: (x₁, x₂, r).

Python – 3D-Datensatz (Cell 15) 
# r als neue Dimension anhängen
X_3d = np.concatenate((X, r.reshape(-1, 1)),
                       axis=1)

print(f"2D Form: {X.shape}")
print(f"3D Form: {X_3d.shape}")
print("\nErste 3 Punkte (x₁, x₂, r):")
print(X_3d[:3].round(3))
▶ Ausgabe

3D Form: (100, 3)

Punkte (x₁, x₂, r):

💡 So what?

Im 3D-Raum sind die Klassen linear separierbar! Eine horizontale Ebene bei z ≈ 0.5 trennt sie perfekt. Das Problem: Wir haben die Projektion manuell und explizit berechnet. Was wenn wir 10, 100 oder 1000 neue Dimensionen brauchen? → Der Kernel-Trick löst das elegant!

6.3 Der Kernel-Trick

Der Kernel-Trick treibt die Idee der Projektion auf die Spitze: Man kann implizit an jeden der N Datenpunkte eine zusätzliche Dimension anfügen – ohne diese Projektion wirklich durchzuführen.

❌ Linear (kernel='linear')
  • Sucht eine gerade Hyperebene
  • Funktioniert nur für linear trennbare Daten
  • Entscheidungsgrenze: gerade Linie
  • Versagt bei konzentrischen Ringen
vs.
✅ RBF (kernel='rbf')
  • Projiziert implizit in N-dimensionalen Raum
  • Findet dort eine gerade Hyperebene
  • Entscheidungsgrenze: elliptische/kreisförmige Kurve
  • Löst konzentrische Ringe perfekt!

🔮 Was ist der Kernel-Trick genau?

  • Statt Φ(x) zu berechnen (teuer!), nutzt man die Kernel-Funktion
  • RBF-Kernel: \(K(x_i, x_j) = e^{-\gamma\|x_i - x_j\|^2}\)
  • K berechnet den Skalar-Wert, den ⟨Φ(xᵢ), Φ(xⱼ)⟩ hätte – ohne Φ zu berechnen
  • → Implizit: jeder Datenpunkt in einem N-dim. Raum
  • → Explizit: nur eine Zahl (Skalar) wird berechnet

⚙️ In sklearn: kernel='rbf'

  • Hyperparameter: kernel='rbf' → RBF-Kernel aktivieren
  • Weiterer Parameter: C → Regularisierung (wie bisher)
  • gamma → steuert die "Breite" der RBF (hier: automatisch)
  • Ergebnis: nicht-lineare Entscheidungsgrenze im 2D-Raum
  • → Im Original-Raum: elliptische/kreisförmige Kurve

Schritt 1 — SVC(kernel='rbf') fitten. C=1E6 für sehr harten Margin (Daten sind perfekt trennbar). Das ist Cell 17 aus dem Notebook.

Python – SVC(kernel='rbf') (Folie 29, Cell 17) 
# RBF Kernel – Kernel-Trick!
clf = SVC(kernel='rbf', C=1E6)
clf.fit(X, y)

print(f"Kernel:             {clf.kernel}")
print(f"C:                  {clf.C}")
print(f"Support Vectors:    {len(clf.support_vectors_)}")
print(f"Klassen:            {clf.classes_}")
▶ Ausgabe

  • Modell trainiert in Sekunden
  • ~17 Support Vectors (an der Grenze)

Schritt 2 — Das Ergebnis visualisieren. Die Entscheidungsgrenze ist jetzt kreisförmig/elliptisch – sie umschließt die inneren gelben Punkte perfekt. (Cell 18)

Python – Elliptische Grenze (Cell 18) 
fig, ax = plt.subplots(figsize=(5, 4))

# Scatter + Entscheidungsgrenze
ax.scatter(X[:, 0], X[:, 1], c=y, s=50,
           cmap='autumn', edgecolors='gray')
plot_svc_decision_function(clf, ax)

# Support Vectors hervorheben
ax.scatter(clf.support_vectors_[:, 0],
           clf.support_vectors_[:, 1],

           facecolors='none',

▶ Ausgabe
  • 📊 Scatter: gelb (innen), rot (außen)
  • Kreisförmige Linie umschließt die gelben Punkte
  • Zwei gestrichelte Kreise = Margin-Grenzen
  • Blaue Kreise = Support Vectors
  • → Perfekte Trennung trotz nicht-linearer Struktur!
  • Lineare Hyperebene im N-dim. Raum = Kreis im 2D-Raum

Schritt 3 — Direktvergleich: Linear SVC (scheitert) vs. RBF SVC (löst das Problem). Vertikal gestapelt.

Python – Vergleich linear vs. rbf (vertikal) 
fig, axes = plt.subplots(2, 1, figsize=(5, 9))

for ax, (kernel, C) in zip(axes, [
    ('linear', 1.0),
    ('rbf',    1E6)
]):
    clf_cmp = SVC(kernel=kernel, C=C).fit(X, y)
    ax.scatter(X[:,0], X[:,1], c=y, s=40,
               cmap='autumn', edgecolors='gray')
    plot_svc_decision_function(clf_cmp, ax)
    label = '❌ Versagt' if kernel=='linear' else '✅ Perfekt!'
    ax.set_title(f"kernel='{kernel}' {label}")

plt.tight_layout(); plt.show()
▶ Ausgabe
  • 📊 Oben: linear – gerade Linie, trennt schlecht
  • 📊 Unten: rbf – Kreislinie, trennt perfekt
  • figsize=(5, 9) → vertikal gestapelt
  • Selber Datensatz, anderer Kernel
  • → Der Kernel macht den Unterschied!
💡 So what? — Zusammenfassung Seite 06

Der Kernel-Trick ist das eleganteste Konzept der SVMs: durch kernel='rbf' wird jeder Datenpunkt implizit in einen N-dimensionalen Raum projiziert – ohne diese Projektion je explizit berechnen zu müssen. Im Original-2D-Raum erscheint die Entscheidungsgrenze als nicht-lineare Kurve (Ellipse/Kreis). Als nächstes: Multi-Class SVMs & Praxis-Anwendungen → Seite 07.
