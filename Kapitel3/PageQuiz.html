🟦 Konzeptfragen
8 Fragen
K1
Konzept Was sind Support Vectors? Warum sind sie so wichtig für das SVM-Modell?
Support Vectors sind die Datenpunkte, die am nächsten an der Entscheidungsgrenze liegen – exakt auf den Margin-Grenzen.

Warum wichtig? Das gesamte SVM-Modell ist durch nur diese Punkte definiert. Alle anderen Trainingspunkte können entfernt werden, ohne das Modell zu verändern. In sklearn: model.support_vectors_

Wert der decision function an SVs: ±1 (exakt auf Margin-Grenze)
K2
Konzept Was bedeutet es, dass SVMs "diskriminative" Klassifikatoren sind?
Diskriminativ bedeutet: SVMs modellieren direkt die Grenze zwischen Klassen, ohne die Datenverteilung selbst zu modellieren.

Im Gegensatz dazu stehen generative Modelle (z.B. GMM, Naïve Bayes), die die Verteilung P(x|y) modellieren.

SVMs fragen: "Wo liegt die optimale Trennlinie?"
Generative Modelle fragen: "Wie wurden die Daten erzeugt?"
K3
Konzept Erkläre den Unterschied zwischen One-vs-One (OvO) und One-vs-Rest (OvR) für Multi-Class SVMs.
OvO (One-vs-One): Für jede Paarung zweier Klassen wird eine SVM trainiert. N Klassen → N(N−1)/2 SVMs. sklearn SVC nutzt OvO.

OvR (One-vs-Rest): Für jede Klasse vs. alle anderen wird eine SVM trainiert. N Klassen → N SVMs.

OvO: mehr SVMs, aber jede auf einem ausgeglicheneren Datensatz.
OvR: weniger SVMs, aber unbalancierte Daten pro SVM.
K4
Konzept Warum eignen sich SVMs besonders gut für hochdimensionale Daten?
3 Gründe:

1. SVMs sind sparse: Nur die Support Vectors definieren das Modell – nicht alle N Punkte.
2. SVMs funktionieren gut, wenn Dimensionen > Datenpunkte (im Gegensatz zu z.B. k-NN).
3. Durch den Kernel-Trick können sie in noch höherdimensionalen Räumen operieren.

Nachteil bei sehr großen N: Die Kernel-Matrix (N×N) wird zu groß – dann sind andere Modelle besser.
K5
Konzept Was bewirkt eine Verdoppelung der Datenpunkte (von N=60 auf N=120) bei SVMs?
Hinweis: gleiche Datenverteilung, random_state=0
Nahezu keine Änderung! Die Trennlinie und die Support Vectors bleiben identisch.

Warum? Weil SVMs nur von den Support Vectors (Punkte an der Margin-Grenze) abhängen. Die neuen 60 Punkte liegen weiter weg von der Grenze und beeinflussen das Modell nicht.

Das zeigt die Robustheit von SVMs gegenüber "irrelevanten" Datenpunkten. Mehr Daten helfen nur, wenn neue Punkte nah an der Entscheidungsgrenze liegen.
K6
Konzept Erkläre die Slack-Variable ξ_i. Welche Werte kann sie annehmen und was bedeuten sie?
Die Slack-Variable ξ_i misst die Margin-Verletzung des i-ten Datenpunkts:

ξ_i = 0: Punkt korrekt außerhalb des Margins → keine Verletzung
0 < ξ_i ≤ 1: Punkt im Margin, aber richtige Seite
ξ_i > 1: Punkt auf der falschen Seite (Missklassifikation)

Im Soft-Margin-Problem: y_i(⟨w,x_i⟩+b) ≥ 1 − ξ_i mit ξ_i ≥ 0. Je größer C, desto teurer sind Verletzungen → weniger ξ_i > 0.
K7
Konzept Warum ist SVR die "umgekehrte Optimierung" von SVC?
SVC: Sucht eine Hyperebene, außerhalb der sich Punkte befinden sollen. Margin-Verletzungen = Punkte die sich INNEN befinden = werden bestraft.

SVR: Sucht eine Regressionsgerade + ε-Tube, INNEN deren sich Punkte befinden sollen. Tube-Verletzungen = Punkte die sich AUSSEN befinden = werden bestraft.

Schlüsselunterschied: SVC will Punkte außen → SVR will Punkte innen. Deshalb: "umgekehrte Optimierungsbedingung".
K8
Konzept Was ist das No-Free-Lunch-Theorem und warum ist es für SVMs relevant?
Das No-Free-Lunch-Theorem besagt: Kein Algorithmus ist auf allen möglichen Problemen gleichzeitig optimal.

Für SVMs bedeutet das:
• SVMs sind exzellent für hochdimensionale, kleine Datensätze
• Für große N oder sehr verrauschte Daten können andere Modelle besser sein
• Der optimale Hyperparameter C (und γ, ε) ist immer datensatzspezifisch

Prüfungsrelevanz: Immer Cross-Validation nutzen, immer mehrere Modelle vergleichen, nie blind auf "besten" Algorithmus vertrauen.
🟣 Formelfragen
6 Fragen
F1
Formel Vervollständige: Das Hard-Margin-Optimierungsproblem lautet min _____ unter der Bedingung _____.
min ½‖w‖²   unter der Bedingung   y_i(⟨w, x_i⟩ + b) ≥ 1 ∀ i

Warum ½? Für einfachere Ableitung: d/dw (½‖w‖²) = w (kein störender Faktor).
Äquivalent zu: max r = max 1/‖w‖ = min ‖w‖.
F2
Formel Ein trainiertes SVM hat ‖w‖ = 2.5. Wie groß ist der Margin r?
r = 1/‖w‖ = 1/2.5 = 0.4

Das ist der Abstand von der Hyperebene bis zur Margin-Grenze. Der gesamte Margin (von +1 Ebene bis −1 Ebene) ist 2r = 0.8.

Zu minimieren ist ½‖w‖² = ½·6.25 = 3.125.
F3
Formel Berechne den Hinge Loss für t = 0.3 und für t = 1.8.
Hinge Loss: L(t) = max{0, 1−t}

t = 0.3: L(0.3) = max{0, 1−0.3} = max{0, 0.7} = 0.7 (Verlust!)
t = 1.8: L(1.8) = max{0, 1−1.8} = max{0, −0.8} = 0 (kein Verlust)

t = y·f(x). Bei t = 0.3: Punkt auf richtiger Seite, aber noch im Margin. Bei t = 1.8: Punkt außerhalb Margin → kein Verlust.
F4
Formel Schreibe den Gradienten ∇_w E für den Fall eines Margin-Bruchs bei Punkt (x_i, y_i). C = 2, Summe nur über diesen einen Punkt.
Bei Margin-Bruch: ∇_w E = w − C · y_i · x_i

Mit C = 2, und dem einen verletzenden Punkt:
∇_w E = w − 2 · y_i · x_i

Update: w_neu = w − η · (w − 2 · y_i · x_i)

Für ∇_b: ∇_b E = −C · y_i = −2 · y_i
F5
Formel Für N = 7 Klassen: Wie viele SVMs trainiert sklearn intern mit OvO?
N(N−1)/2 = 7·6/2 = 21 SVMs

Alle Paare: (1,2), (1,3), (1,4), (1,5), (1,6), (1,7), (2,3), ... (6,7).

Merke: N=2 → 1 SVM | N=3 → 3 | N=4 → 6 | N=5 → 10 | N=10 → 45 | N=100 → 4950
F6
Formel Ein Punkt x hat ‖x‖² = 0. Was ist der RBF-Wert r? Ein anderer Punkt hat ‖x‖² = 100. Was ist r?
r = exp(−‖x‖²)

‖x‖² = 0 (Ursprung): r = exp(0) = 1 (maximaler Wert, Innenring)
‖x‖² = 100 (weit weg): r ≈ 3.7·10⁻⁴⁴ ≈ 0 (nahezu null, Außenring)

Das ist genau der Trick: Innenring hat r ≈ 1 (oben im 3D-Raum), Außenring r ≈ 0 (unten) → linear trennbar!

C1
Code Was gibt dieser Code aus?
model = SVC(kernel='linear', C=1E10).fit(X, y) print(len(model.support_vectors_)) print(model.decision_function(model.support_vectors_).round(2))
Zeile 1: Anzahl der Support Vectors (typisch: 3 für make_blobs mit random_state=0, cluster_std=0.6)
Zeile 2: [-1. 1. -1.] (oder ähnlich) – Support Vectors liegen auf den Margin-Grenzen, decision_function = ±1

Schlüsselwissen: model.support_vectors_ enthält die Koordinaten der Support Vectors. decision_function für diese Punkte ist immer betragsmäßig ≈ 1.
C2
Code Was macht dieser Code? Was wird unterschiedlich sein bei ε=1.5 vs ε=0.5?
from sklearn.svm import LinearSVR reg1 = LinearSVR(epsilon=1.5, random_state=42).fit(X, y) reg2 = LinearSVR(epsilon=0.5, random_state=42).fit(X, y) off1 = (np.abs(y - reg1.predict(X)) >= reg1.epsilon).sum() off2 = (np.abs(y - reg2.predict(X)) >= reg2.epsilon).sum() print(off1, off2)
Was macht der Code: Zählt die Punkte außerhalb des ε-Tubes (= "Support Vectors" bei SVR)

Erwartetes Ergebnis: off1 < off2
ε=1.5 hat weniger Punkte außerhalb (breiterer Tube → mehr Punkte innen → off1 klein)
ε=0.5 hat mehr Punkte außerhalb (schmaler Tube → wenige innen → off2 groß)

Merke: Großes ε → wenig Verletzungen, kleines ε → viele Verletzungen.
C3
Code Was macht r = np.exp(-(X**2).sum(axis=1))? Was bedeutet axis=1?
Schritt für Schritt:
1. X**2: Quadriert jedes Element (x₁², x₂²) – Form: (N, 2)
2. .sum(axis=1): Summiert über Spalten (Features) → ‖x‖² = x₁² + x₂² – Form: (N,)
3. np.exp(-...): Berechnet r = e^(−‖x‖²) für jeden Datenpunkt

axis=1: "summiere entlang der Feature-Achse" = Summe pro Datenpunkt (Zeile). axis=0 würde über Datenpunkte summieren (Spaltensumme).
C4
Code Was passiert bei diesem Code? Welchen Score erwarten Sie?
from sklearn.datasets import make_circles from sklearn.svm import SVC X, y = make_circles(100, factor=0.1, noise=0.1) clf_lin = SVC(kernel='linear').fit(X, y) clf_rbf = SVC(kernel='rbf', C=1E6).fit(X, y) print(clf_lin.score(X, y), clf_rbf.score(X, y))
Ergebnis: ~0.55 1.0

linear: ~55% – schlechter als Zufall bei diesem Problem! Kann konzentrische Ringe nicht trennen.
rbf mit C=1E6: ≈100% – RBF-Kernel projiziert implizit in höheren Raum, findet kreisförmige Grenze.

Wichtig: C=1E6 = sehr harter Margin. Bei diesem synthetischen Datensatz ohne Rauschen OK, aber in der Praxis via Cross-Validation wählen!
C5
Code Ergänze den fehlenden Code für svm_gradients (Margin-Bruch Fall):
def svm_gradients(X, y, w, b, C): scores = y * (X @ w + b) mask = scores < 1 dw = w.copy() # Kein Bruch: nur w db = 0.0 dw -= C * (________) # Ergänze hier db -= C * y[mask].sum() return dw, db
(y[mask, None] * X[mask]).sum(axis=0)

Vollständig: dw -= C * (y[mask, None] * X[mask]).sum(axis=0)

Erklärung:
• y[mask, None]: Labels der verletzenden Punkte als Spaltenvektor (N_v, 1)
• X[mask]: Feature-Vektoren der verletzenden Punkte (N_v, 2)
• Elementweises Produkt: y_i · x_i für jeden verletzenden Punkt
• .sum(axis=0): Summiert über alle verletzenden Punkte → Vektor der Form (2,)
C6
Code Was muss man vor diesem Code tun, damit er korrekte Ergebnisse liefert? Warum?
clf = SVC(kernel='rbf', C=10, gamma='scale') X_train, X_test, y_train, y_test = train_test_split(X, y) clf.fit(X_train, y_train) print(clf.score(X_test, y_test))
Skalierung (StandardScaler) fehlt!

from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test = scaler.transform(X_test) # NICHT fit_transform!

Warum wichtig? RBF-Kernel berechnet ‖x_i − x_j‖². Features mit unterschiedlichen Skalen (z.B. Feature 1 im Bereich [0,1], Feature 2 im Bereich [0,1000]) würden Feature 2 völlig dominieren.

Achtung: Scaler immer nur auf Trainingsdaten fitten, dann auf Test anwenden!
🟠 Fallen
6 Fallen-Fragen
🪤
Falle „Ein SVM mit C=1E10 funktioniert auf jedem Datensatz und gibt immer den besten Margin." Wahr oder falsch?
FALSCH!

C=1E10 erzwingt Hard-Margin: kein Punkt darf im Margin liegen. Bei nicht-linear separierbaren Daten oder Daten mit Rauschen/Überlappung: sklearn findet keinen validen Hard-Margin oder produziert ein sehr schlecht generalisierendes Modell.

Richtig: C via Cross-Validation wählen. C=1E10 nur bei synthetischen, perfekt separierbaren Daten sinnvoll.
🪤
Falle „model.coef_ gibt den Normalenvektor w zurück, der immer normiert ist (‖w‖=1)." Wahr oder falsch?
FALSCH!

model.coef_ gibt w zurück, aber nicht normiert (‖w‖ ≠ 1 im Allgemeinen)!

Die Normierungsbedingung ‖w‖=1 galt nur im ursprünglichen Optimierungsproblem (Folie 14). Nach der Skalierung (Folie 15) hat w einen beliebigen Betrag – der Margin ist r = 1/‖w‖.

Prüfen: np.linalg.norm(model.coef_[0]) → meist ≠ 1.
🪤
Falle „Bei SVR bedeutet ε=0 das beste Modell, da keine Toleranz erlaubt wird." Wahr oder falsch?
FALSCH!

ε=0 macht SVR zum normalen Regressionsmodell mit maximaler Sensitivität – jeder minimale Abstand wird bestraft. Das führt zu Overfitting!

Richtig: ε kontrolliert den Trade-off zwischen Modellkomplexität und Genauigkeit. Ein sinnvolles ε (z.B. 0.1–1.5) erlaubt kleine Residuen ohne Strafe → bessere Generalisierung. ε via Cross-Validation bestimmen.
🪤
Falle Labels y bei SVM müssen {0, 1} sein (sklearn-Standard). Wahr oder falsch für die Mathematik?
TEILWEISE FALSCH – es kommt auf den Kontext an!

In sklearn: SVC akzeptiert {0, 1} oder jede ganze Zahl. sklearn konvertiert intern automatisch.

In der Mathematik (Vorlesung): Labels müssen {−1, +1} sein! Die Formel y_i(⟨w,x_i⟩+b) ≥ 1 funktioniert nur mit {−1, +1}.

In from-Scratch-Code: Labels müssen manuell umgerechnet werden: y = np.where(y == 1, 1, -1)
🪤
Falle „kernel='rbf' ist immer besser als kernel='linear' für nicht-lineare Daten." Wahr oder falsch?
FALSCH! → No-Free-Lunch

RBF kann bei falschen Hyperparametern C und gamma schlechter als linear sein! Beispiel: Bei sehr hohem C (C=1E10) und kleinem gamma: Overfitting.

Richtige Aussage: kernel='rbf' kann nicht-lineare Probleme lösen, die linear nicht lösbar sind. Aber nur mit richtig gewählten C und gamma via Cross-Validation → GridSearchCV oder cross_val_score.
🪤
Falle Im Gradientenabstieg: Wenn decision_function(x) = 0.3, wird ∇_w = w oder ∇_w = w − C·y·x genutzt?
Zuerst prüfen: Ist y·f(x) < 1? (Margin-Bruch-Bedingung)

decision_function(x) gibt f(x) = ⟨w,x⟩+b zurück. Wir brauchen das Label y!

Fall 1: y = +1, f(x) = 0.3: t = y·f(x) = +1·0.3 = 0.3 < 1 → ∇_w = w − C·y·x (Margin-Bruch!)
Fall 2: y = −1, f(x) = 0.3: t = −1·0.3 = −0.3 < 1 → ∇_w = w − C·y·x (Auch Bruch!)

Merke: Die Bedingung ist y·f(x) < 1, nicht nur f(x) < 1! Ein Punkt auf der falschen Seite (y=−1 aber f(x)=0.3>0) hat IMMER einen Margin-Bruch.
