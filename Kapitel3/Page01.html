1.1 SVM: Wo befinden wir uns?

SVMs sind ein Supervised-Learning-Verfahren – sie können sowohl für Klassifikation (SVC) als auch für Regression (SVR) eingesetzt werden. Das folgende Diagramm zeigt die Einordnung in die ML-Taxonomie.

🤖 Machine Learning
└─ 📊 Supervised Learning Wir sind hier └─ 🔍 Unsupervised Learning └─ 🎮 Reinforcement Learning
├─ 🏷️ Klassifikation → SVC – Support Vector Classifier └─ 📈 Regression → SVR – Support Vector Regressor
💡 So what?

SVMs sind flexibel: Dasselbe Grundprinzip (Margin-Maximierung) ermöglicht sowohl Klassifikation (SVC) als auch Regression (SVR). In diesem Kapitel liegt der Fokus auf dem Classifier – SVR kommt auf Seite 08.

1.3 SVM: Allgemein

Was sind SVMs – und warum sind sie relevant? Vier Kernaussagen aus der Vorlesung auf einen Blick.

🏷️ Bekanntheit & Einsatzbereich

  • Einer der bekanntesten Algorithmen im Machine Learning
  • Besonders stark bei klein- und mittelgroßen Datensätzen
  • Weniger geeignet bei sehr großen Datensätzen (→ Kernel-Matrix wird zu groß)

🎯 Diskriminativer Klassifikator

  • SVMs sind diskriminativ: sie modellieren direkt die Grenze zwischen Klassen
  • Im Gegensatz zu generativen Modellen (z.B. GMM), die die Datenverteilung modellieren
  • Frage: „Wo liegt die Trennlinie?" – nicht „Wie entstehen die Daten?"

🔧 Flexibilität

  • SVMs können für Klassifikation (SVC) eingesetzt werden
  • … und für Regression (SVR)
  • Durch den Kernel-Trick auch für nicht-lineare Probleme geeignet

❓ Das Ausgangsproblem

  • Gegeben: Zwei Klassen im 2D-Feature-Raum
  • Gesucht: Eine optimale Trennlinie (Hyperebene in höheren Dimensionen)
  • Problem: Es gibt unendlich viele mögliche Trennlinien – welche ist die beste?
  • → Antwort gibt der Margin!

Ausgangssituation: Zwei Klassen im 2D-Featureraum

Wir erstellen einen einfachen Datensatz mit make_blobs() und visualisieren ihn – genau das ist unser Startpunkt für das gesamte Kapitel.

Python – Datensatz erstellen & plotten 
import matplotlib.pyplot as plt
import numpy as np
from sklearn.datasets import make_blobs
import seaborn as sns; sns.set()

# Datensatz: 2 Klassen, 50 Punkte
X, y = make_blobs(n_samples=50, centers=2,
                  random_state=0, cluster_std=0.60)

fig, ax = plt.subplots(figsize=(5, 4))
ax.scatter(X[:, 0], X[:, 1], c=y, s=50,
           cmap='autumn', edgecolors='gray')
ax.set_xlabel('Feature 1')
ax.set_ylabel('Feature 2')
ax.set_title('Ausgangssituation: 2 Klassen')
plt.tight_layout()
plt.show()
▶ Ausgabe
  • 📊 Scatter-Plot: 50 Punkte, 2 Klassen (rot / gelb)
  • random_state=0 → reproduzierbar
  • cluster_std=0.60 → geringe Streuung
  • Klassen sind linear trennbar
  • Frage: Welche Trennlinie ist optimal?
💡 So what?

Dieses einfache Szenario – zwei trennbare Klassen – ist unser Startpunkt. Beliebig viele Geraden könnten die Klassen trennen. SVMs wählen die beste durch Margin-Maximierung. Was genau das bedeutet, sehen wir auf Seite 02.

Hilfsfunktion: plot_svc_decision_function()

Diese Hilfsfunktion aus Cell 2 zeichnet die Entscheidungsgrenze und die Margin-Linien eines trainierten SVM-Modells. Sie wird in fast jeder Code-Zelle des Notebooks wiederverwendet.

Python – Hilfsfunktion (Cell 2) 
def plot_svc_decision_function(model,
                               ax=None,
                               plot_support=True):
    """Entscheidungsgrenze für 2D-SVC plotten."""
    if ax is None:
        ax = plt.gca()
    xlim = ax.get_xlim()
    ylim = ax.get_ylim()

    # Gitter für das Modell erstellen
    x = np.linspace(xlim[0], xlim[1], 30)
    y = np.linspace(ylim[0], ylim[1], 30)
    Y, X = np.meshgrid(y, x)
    xy = np.vstack([X.ravel(), Y.ravel()]).T
    P = model.decision_function(xy).reshape(X.shape)

    # Trennlinie + Margin-Linien (-1, 0, +1)
    ax.contour(X, Y, P, colors='k',
               levels=[-1, 0, 1], alpha=0.5,
               linestyles=['--', '-', '--'])

    # Support Vectors einzeichnen
    if plot_support:
        ax.scatter(model.support_vectors_[:, 0],
                   model.support_vectors_[:, 1],
                   s=300, linewidth=1,
                   facecolors='none')
    ax.set_xlim(xlim)
    ax.set_ylim(ylim)
Was diese Funktion macht
  • decision_function() → Abstand jedes Gitterpunkts zur Hyperebene
  • levels=[-1, 0, 1] → 3 Linien: –1 (neg. Margin), 0 (Trennlinie), +1 (pos. Margin)
  • linestyles=['--', '-', '--'] → Margin gestrichelt, Trennlinie durchgezogen
  • plot_support=True → Support Vectors als leere Kreise einzeichnen
  • Wird auf jeder Folgeseite genutzt → einmal definieren, immer verwenden
