Ausgangslage: Beyond k-means & DBSCAN

Schritt 1 · Imports & Datenerzeugung

Python 
import matplotlib.pyplot as plt
import seaborn as sns; sns.set()
import numpy as np

from sklearn.datasets import make_blobs

# Synthetischer Datensatz: 400 Punkte, 4 Cluster
X, y_true = make_blobs(n_samples=400, centers=4,

✅ X.shape = (400, 2) – 400 Datenpunkte mit 2 Features. 4 gut separierte Cluster mit geringer Streuung (cluster_std=0.60).

Schritt 2 · k-means anwenden & plotten

Python 
from sklearn.cluster import KMeans

kmeans = KMeans(4, random_state=0)
labels = kmeans.fit(X).predict(X)

plt.scatter(X[:, 0], X[:, 1], c=labels, s=40, cmap='viridis')
plt.title("k-means Clustering – 4 Cluster")
plt.show()
Output

🎨 Scatter-Plot mit 4 farbcodierten Clustern. Bei „well-behaved" Daten trennt k-means die Cluster sauber – aber ohne jede Aussage über Zugehörigkeitswahrscheinlichkeiten.

Problem 1 – Non-Probabilistisch

  • k-means ist non-probabilistisch und basiert auf einfachen Abstandsbetrachtungen
  • → einfach verständlich, aber fragliche Anwendbarkeit in realen Situationen
  • Kein Cluster-Algorithmus liefert eine Aussage darüber, wie wahrscheinlich ein Datenpunkt zu einem Cluster gehört
  • → ob sich der Algorithmus evtl. sogar unsicher ist in der Zuweisung – bleibt unbekannt

Was war bei DBSCAN?

  • Bei DBSCAN hatten wir erste Möglichkeiten kennengelernt:
  • → Gruppierung in Core-, Density-Reachable- und Noise-Punkte
  • Jedoch: eine kontinuierliche Wahrscheinlichkeitsaussage war auch hier nicht wirklich drin
  • → Dieses Problem gehen wir nun an mit GMM

Improving k-means

Um die Schwäche von k-means zu verstehen, stellen wir uns das Modell geometrisch vor: Jeder Cluster wird durch eine Hypersphäre repräsentiert. Was passiert, wenn die Daten nicht sphärisch sind?

Schritt 3 · Hilfsfunktion plot_kmeans definieren

Die Funktion zeichnet neben den Datenpunkten auch die Hypersphären um jeden Cluster – der maximale Abstand vom Centroid zum Cluster-Rand bestimmt den Radius.

Python 
from sklearn.cluster import KMeans
from scipy.spatial.distance import cdist

def plot_kmeans(kmeans, X, n_clusters=4, rseed=0, ax=None):
    labels = kmeans.fit_predict(X)
    ax = ax or plt.gca()
    ax.axis('equal')
    ax.scatter(X[:, 0], X[:, 1], c=labels, s=40,
               cmap='viridis', zorder=2)
    centers = kmeans.cluster_centers_
    # Radius = max. Abstand vom Centroid zu jedem Punkt im Cluster
    radii = [cdist(X[labels == i], [center]).max()
             for i, center in enumerate(centers)]
    for c, r in zip(centers, radii):
        ax.add_patch(plt.Circle(c, r, fc='gray',
                                lw=3, alpha=0.5, zorder=1))
Output

✅ Funktion definiert – bereit zum Aufruf.

Was sie tut:
Fittet k-means, plottet die Punkte farbkodiert und zeichnet graue Kreise (Hypersphären) um jeden Cluster.

Schritt 4 · Reguläre vs. gestreckte Daten (vertikal vergleichen)

Wir erzeugen mit np.dot(X, rng.randn(2,2)) einen elliptisch gestreckten Datensatz und vergleichen beide Fälle vertikal. Die Hypersphären-Visualisierung zeigt sofort das Problem: sphärische Cluster passen nicht mehr.

Python 
# Vertikaler Vergleich: 2 Zeilen, 1 Spalte
fig, (ax1, ax2) = plt.subplots(2, 1, figsize=(6, 10))

# Oben: Reguläre Daten
kmeans = KMeans(n_clusters=4, random_state=0)
plot_kmeans(kmeans, X, ax=ax1)
ax1.set_title("Reguläre Daten – Hypersphären passen")

# Unten: Gestreckte Daten
rng = np.random.RandomState(13)
X_stretched = np.dot(X, rng.randn(2, 2))
kmeans = KMeans(n_clusters=4, random_state=0)
plot_kmeans(kmeans, X_stretched, ax=ax2)
ax2.set_title("Gestreckte Daten – Hypersphären versagen")

plt.tight_layout()
plt.show()
Output

Oben: 4 saubere Kreise – k-means funktioniert bei sphärischen Daten einwandfrei.

Unten: Die Kreise überlappen massiv – bei elliptischen Daten kommt es zur Vermischung der Cluster-Zuordnung.

Hypersphären – das geometrische Modell

  • k-means spannt um jeden Cluster Hypersphären, die Cluster-Zugehörigkeit im Datenraum repräsentieren
  • Vergrößert man die Hypersphären immer weiter entstehen lineare Trennhyperebenen
  • → Die Cluster-Modelle müssen sphärisch sein
  • → keine Möglichkeit, elliptische bzw. nicht-konvexe Strukturen zu modellieren

Problem 2 – Sphärische Annahme

  • Bei nicht-sphärischen Datensätzen → Vermischung der Clusterzuordnung
  • Kreise passen nicht in elliptische Strukturen – sie müssen sich zu stark überdehnen
  • → Cluster-Grenzen werden an falschen Stellen gezogen
  • Reale Daten sind selten schön sphärisch – das ist das fundamentale Problem

⚠️ So What? – Die zwei Schwächen von k-means:

  • Non-probabilistisches Verhalten – kein Maß für Unsicherheit oder Zugehörigkeitswahrscheinlichkeit
  • Starke Annahmen über Cluster-Strukturen – sphärische Form ist in echten Daten die Ausnahme, nicht die Regel

→ Beide Probleme löst das Gaussian Mixture Model.

Towards GMMs

Das Gaussian Mixture Model kombiniert mehrere Gauß'sche Wahrscheinlichkeitsdichten zu einem flexiblen Modell – und behebt damit beide Schwächen von k-means.

Was ist ein GMM?

  • Ein Gaussian Mixture Model (GMM) versucht mittels der Kombination mehrerer, mehrdimensionaler Normalverteilungen beliebig strukturierte Datensätze zu modellieren
  • Die Grundidee: Jeder Cluster wird durch eine eigene Gauß'sche Wahrscheinlichkeitsdichte repräsentiert
  • → Elliptische, beliebig ausgerichtete Cluster sind damit möglich

sklearn API

  • Die zugehörige Klasse aus sklearn: GaussianMixture()
  • Modul: sklearn.mixture
  • Wichtigster Parameter: n_components – Anzahl der Gauß-Komponenten (≙ Anzahl der Cluster)
  • Verwendung ähnlich wie k-means: .fit(X) → .predict(X)

Schritt 5 · Erster Einsatz von GaussianMixture()

Zunächst verwenden wir das GMM auf die gleiche Weise wie k-means – mit .fit(X).predict(X). Das visuelle Ergebnis ist ähnlich, aber dahinter steckt ein probabilistisches Modell.

Python 
from sklearn.mixture import GaussianMixture

gmm = GaussianMixture(n_components=4).fit(X)
labels = gmm.predict(X)

plt.scatter(X[:, 0], X[:, 1], c=labels, s=40, cmap='viridis')
plt.title("GMM Clustering – gleiche Daten wie k-means")
plt.show()
Output

🔵 Visuell ähnlich wie k-means – 4 Cluster farbkodiert.

Der entscheidende Unterschied liegt nicht im Plot, sondern im Modell: GMM liefert zusätzlich Wahrscheinlichkeiten für die Cluster-Zugehörigkeit jedes Punktes.

💭 Was bedeutet es, Daten mittels Wahrscheinlichkeiten zu präsentieren?

Statt einem Datenpunkt hart einem Cluster zuzuweisen (→ k-means), berechnet das GMM für jeden Punkt die Wahrscheinlichkeit, zu jedem Cluster zu gehören. Das ist die Grundlage von .predict_proba() – Thema der nächsten Seite.
