7.1 Datensätze & ihre Charakteristiken

Bevor der Code läuft: Was sind die Eigenschaften der sechs Testdatensätze – und was erwarten wir?

①

Noisy Circles make_circles

  • Zwei konzentrische Kreise – ein kleiner im inneren eines großen
  • Nicht-linear separierbar – k-means und lineare Verfahren versagen
  • Dichtebasierte und graphenbasierte Methoden notwendig
  • Parameter: factor=0.5, noise=0.05
②

Noisy Moons make_moons

  • Zwei ineinandergreifende Halbmonde
  • Nicht-linear separierbar – bekannt von Seite 5
  • Spectral Clustering und DBSCAN funktionieren hier gut
  • Parameter: noise=0.05
③

Varied Variance make_blobs

  • Drei Cluster mit stark unterschiedlicher Streuung
  • cluster_std=[1.0, 2.5, 0.5] – ein enger, ein weiter, ein mittlerer Cluster
  • Problem für Algorithmen, die gleich große Cluster bevorzugen
  • k-means hat hier Schwierigkeiten
④

Anisotropic make_blobs + Transformation

  • Normale Blobs werden durch eine lineare Transformation gestreckt/rotiert
  • Entstehen: elongierte, schräg liegende Cluster
  • Transformation: [[0.6, -0.6], [-0.4, 0.8]]
  • Verfahren mit euklidischer Distanz haben Probleme
⑤

Blobs make_blobs

  • Der „Idealfall" – kompakte, klar trennbare Cluster
  • Nahezu alle Algorithmen sollten hier gut funktionieren
  • Guter Referenzpunkt für Vergleiche
⑥

No Structure np.random.rand

  • Vollständig zufällig gleichverteilte Datenpunkte
  • Es gibt keine echten Cluster
  • Teste: Welche Algorithmen erkennen „kein Cluster"? Welche erzwingen trotzdem Cluster?
Datensatz Cluster-Form k vorab bekannt Schwierigkeit
Noisy Circles Konzentrisch k=2 ⚠️ Hoch – nicht-linear
Noisy Moons Halbmonde k=2 ⚠️ Hoch – nicht-linear
Varied Variance Elliptisch, unterschiedlich k=3 ⚠️ Mittel – unterschiedl. Dichte
Anisotropic Elongiert, rotiert k=3 ⚠️ Mittel – gestreckte Cluster
Blobs Kompakt, rund k=3 ✅ Gering – Idealfall
No Structure Keine – 🔵 Sonderfall – kein echtes Cluster

7.2 Die 11 Algorithmen im Überblick

Bevor der Code läuft: was macht jeder Algorithmus – und was sind seine Stärken und Grenzen?

MiniBatchKMeans

  • k-means auf zufälligen Datenteilmengen (Mini-Batches)
  • Deutlich schneller als normales k-means
  • Leicht schlechtere Qualität – für große Datensätze ideal
  • Gleiche Einschränkung: nur lineare Cluster

AffinityPropagation

  • Bestimmt k automatisch über „Message Passing"
  • Datenpunkte wählen sich gegenseitig als Repräsentanten
  • Hyperparameter: damping, preference
  • ⚠️ Langsam bei großen Datensätzen

MeanShift

  • Findet Moden (lokale Dichtegipfel) im Datensatz
  • k wird automatisch aus den Daten bestimmt
  • Hyperparameter: bandwidth (Fenstergröße)
  • ⚠️ Empfindlich gegenüber Bandbreite-Wahl

Spectral Clustering

  • Graphenbasiert → beliebige Formen
  • Projiziert via Eigenvektoren in neuen Raum
  • Kein .predict() – nur .fit_predict()
  • ⚠️ Langsam bei großen Daten

Ward / Agglomerative

  • Hierarchisches Clustering – bottom-up
  • Ward: minimiert Varianz-Zuwachs beim Zusammenführen
  • Average Linkage: Manhattan-Distanz, city-block
  • Connectivity-Matrix steuerbar

DBSCAN

  • Dichtebasiert, beliebige Formen
  • k wird automatisch bestimmt
  • Noise-Klasse (-1) eingebaut
  • ⚠️ Probleme bei sehr unterschiedlichen Dichten

HDBSCAN

  • Hierarchisches DBSCAN – robuster gegenüber unterschiedlichen Dichten
  • Findet Cluster auf verschiedenen Dichteniveaus
  • Parameter: min_cluster_size, min_samples
  • Robusterer als DBSCAN bei komplexen Daten

OPTICS

  • Ähnlich zu DBSCAN, aber kein festes eps
  • Erstellt Erreichbarkeits-Plot über alle Dichtestufen
  • Parameter: min_samples, xi, min_cluster_size
  • Gut bei variierenden Dichten

BIRCH

  • Clustering via CF-Baum (Clustering Feature Tree)
  • Sehr speicher- und laufzeiteffizient
  • Gut für sehr große Datensätze
  • Nur konvexe Cluster

GaussianMixture

  • Modelliert Daten als Mischung von Gauß-Verteilungen
  • Weiche Zuordnung (Wahrscheinlichkeit pro Cluster)
  • Kann elliptische Cluster verschiedener Größe erkennen
  • Parameter: covariance_type='full'

7.3 Vergleich – Vollständige Implementierung

Der komplette sklearn-Vergleichscode – strukturiert in Datengenerierung, Parameter-Setup und Vergleichsschleife.

Was macht der Code? Er erstellt eine 6 × 11 Subplot-Matrix: 6 Zeilen (Datensätze) × 11 Spalten (Algorithmen). Jede Zelle zeigt das Clustering-Ergebnis des Algorithmus auf dem jeweiligen Datensatz, inklusive Laufzeit.

Schritt 1 · Imports & Datensätze generieren

Python 
import time
import warnings
from itertools import cycle, islice

import matplotlib.pyplot as plt
import numpy as np
from sklearn import cluster, datasets, mixture
from sklearn.neighbors import kneighbors_graph
from sklearn.preprocessing import StandardScaler

n_samples = 500
seed = 30

# ① Konzentrische Kreise
noisy_circles = datasets.make_circles(
    n_samples=n_samples, factor=0.5,
    noise=0.05, random_state=seed)

# ② Halbmonde
noisy_moons = datasets.make_moons(
    n_samples=n_samples, noise=0.05,
    random_state=seed)

# ③ Kompakte Blobs (Referenz)
blobs = datasets.make_blobs(
    n_samples=n_samples, random_state=seed)

# ④ Keine Struktur (Rauschen)
rng = np.random.RandomState(seed)
no_structure = rng.rand(n_samples, 2), None

# ⑤ Anisotropisch – gestreckte Cluster
random_state = 170
X, y = datasets.make_blobs(
    n_samples=n_samples, random_state=random_state)
transformation = [[0.6, -0.6], [-0.4, 0.8]]
X_aniso = np.dot(X, transformation)
aniso = (X_aniso, y)

# ⑥ Verschiedene Varianzen
varied = datasets.make_blobs(
    n_samples=n_samples,
    cluster_std=[1.0, 2.5, 0.5],
    random_state=random_state)
Datensätze
  • ① noisy_circles – konzentrisch
  • ② noisy_moons – halbmondförmig
  • ③ blobs – Referenz
  • ④ no_structure – kein Cluster
  • ⑤ aniso – elongiert
  • ⑥ varied – unterschiedl. Dichte
  • Jeder Datensatz: 500 Punkte

Schritt 2 · Default- & datensatzspezifische Parameter

Python 
# Standard-Hyperparameter für alle Algorithmen
default_base = {
    "quantile": 0.3,       # MeanShift Bandbreite
    "eps": 0.3,            # DBSCAN Radius
    "damping": 0.9,        # AffinityPropagation
    "preference": -200,    # AffinityPropagation
    "n_neighbors": 3,      # Connectivity-Graph
    "n_clusters": 3,       # k für k-basierte Verfahren
    "min_samples": 7,      # DBSCAN / OPTICS
    "xi": 0.05,            # OPTICS
    "min_cluster_size": 0.1,
    "allow_single_cluster": True,
    "hdbscan_min_cluster_size": 15,
    "hdbscan_min_samples": 3,
    "random_state": 42,
}

# Datensatzspezifische Überschreibungen
datasets_list = [
    (noisy_circles,
     {"damping": 0.77, "preference": -240,
      "quantile": 0.2, "n_clusters": 2,
      "min_samples": 7, "xi": 0.08}),
    (noisy_moons,
     {"damping": 0.75, "preference": -220,
      "n_clusters": 2, "min_samples": 7,
      "xi": 0.1}),
    (varied,
     {"eps": 0.18, "n_neighbors": 2,
      "min_samples": 7, "xi": 0.01,
      "min_cluster_size": 0.2}),
    (aniso,
     {"eps": 0.15, "n_neighbors": 2,
      "min_samples": 7, "xi": 0.1,
      "min_cluster_size": 0.2}),
    (blobs,
     {"min_samples": 7, "xi": 0.1,
      "min_cluster_size": 0.2}),
    (no_structure, {}),
]
Parameter-Logik
  • default_base – gilt für alle Datensätze als Basis
  • Datensatzspezifische Parameter überschreiben die Defaults mit .update(algo_params)
  • noisy_circles und noisy_moons: n_clusters=2
  • varied und aniso: kleinere eps nötig
  • no_structure: keine Änderung notwendig

Schritt 3 · Algorithmen instanziieren & Vergleichsschleife

Python 
plt.figure(figsize=(9 * 2 + 3, 13))
plt.subplots_adjust(left=0.02, right=0.98,
    bottom=0.001, top=0.95,
    wspace=0.05, hspace=0.01)
plot_num = 1

for i_dataset, (dataset, algo_params) in \
        enumerate(datasets_list):

    params = default_base.copy()
    params.update(algo_params)
    X, y = dataset

    # Standardisieren – gleiche Skala für alle
    X = StandardScaler().fit_transform(X)

    # Bandbreite für MeanShift schätzen
    bandwidth = cluster.estimate_bandwidth(
        X, quantile=params["quantile"])

    # Connectivity-Matrix für Ward / Average
    connectivity = kneighbors_graph(
        X, n_neighbors=params["n_neighbors"],
        include_self=False)
    connectivity = 0.5 * (connectivity +
                          connectivity.T)

    # ── Alle 11 Algorithmen instanziieren ──────
    clustering_algorithms = (
        ("MiniBatch\nKMeans",
         cluster.MiniBatchKMeans(
             n_clusters=params["n_clusters"],
             random_state=params["random_state"])),

        ("Affinity\nPropagation",
         cluster.AffinityPropagation(
             damping=params["damping"],
             preference=params["preference"],
             random_state=params["random_state"])),

        ("MeanShift",
         cluster.MeanShift(
             bandwidth=bandwidth,
             bin_seeding=True)),

        ("Spectral\nClustering",
         cluster.SpectralClustering(
             n_clusters=params["n_clusters"],
             eigen_solver="arpack",
             affinity="nearest_neighbors",
             random_state=params["random_state"])),

        ("Ward",
         cluster.AgglomerativeClustering(
             n_clusters=params["n_clusters"],
             linkage="ward",
             connectivity=connectivity)),

        ("Agglomerative\nClustering",
         cluster.AgglomerativeClustering(
             linkage="average",
             metric="cityblock",
             n_clusters=params["n_clusters"],
             connectivity=connectivity)),

        ("DBSCAN",
         cluster.DBSCAN(eps=params["eps"])),

        ("HDBSCAN",
         cluster.HDBSCAN(
             min_samples=params[
                 "hdbscan_min_samples"],
             min_cluster_size=params[
                 "hdbscan_min_cluster_size"],
             allow_single_cluster=params[
                 "allow_single_cluster"])),

        ("OPTICS",
         cluster.OPTICS(
             min_samples=params["min_samples"],
             xi=params["xi"],
             min_cluster_size=params[
                 "min_cluster_size"])),

        ("BIRCH",
         cluster.Birch(
             n_clusters=params["n_clusters"])),

        ("Gaussian\nMixture",
         mixture.GaussianMixture(
             n_components=params["n_clusters"],
             covariance_type="full",
             random_state=params["random_state"])),
    )

    # ── Für jeden Algorithmus: fitten & plotten ──
    for name, algorithm in clustering_algorithms:
        t0 = time.time()

        with warnings.catch_warnings():
            warnings.filterwarnings("ignore")
            algorithm.fit(X)

        t1 = time.time()

        # Labels holen (.labels_ oder .predict())
        if hasattr(algorithm, "labels_"):
            y_pred = algorithm.labels_.astype(int)
        else:
            y_pred = algorithm.predict(X)

        plt.subplot(len(datasets_list),
                    len(clustering_algorithms),
                    plot_num)

        # Spaltentitel nur in erster Zeile
        if i_dataset == 0:
            plt.title(name, size=18)

        # Farben für Cluster + schwarz für Noise
        colors = np.array(list(islice(cycle([
            "#377eb8", "#ff7f00", "#4daf4a",
            "#f781bf", "#a65628", "#984ea3",
            "#999999", "#e41a1c", "#dede00"
        ]), int(max(y_pred) + 1))))
        colors = np.append(colors, ["#000000"])

        plt.scatter(X[:, 0], X[:, 1],
                    s=10, color=colors[y_pred])
        plt.xlim(-2.5, 2.5)
        plt.ylim(-2.5, 2.5)
        plt.xticks(())
        plt.yticks(())

        # Laufzeit anzeigen
        plt.text(0.99, 0.01,
                 ("%.2fs" % (t1 - t0)).lstrip("0"),
                 transform=plt.gca().transAxes,
                 size=15,
                 horizontalalignment="right")
        plot_num += 1

plt.show()
Code-Erklärung
  • StandardScaler – alle Datensätze werden normalisiert
  • kneighbors_graph – Connectivity-Matrix für Ward/Average
  • hasattr(algorithm, "labels_") – unterscheidet Algorithmen mit .labels_ vs. .predict()
  • colors[-1] = "#000000" – Schwarz für Noise (−1)
  • Laufzeit wird rechts unten in jedem Subplot angezeigt
  • Ergebnis: 6 × 11 = 66 Plots in einem Figure
