Scaling Distanzmaß DBSCAN eps min_samples Core Region Noise Hyperparameter-Tuning

Lernfokus dieser Seite

Du kannst die Scaling-Entscheidung begründen, verstehst den DBSCAN-Algorithmus von Core über Border bis Noise und kennst die Bedeutung der Hyperparameter eps und min_samples.

6.1 Skalierung – Wann und Warum?

Die Frage „Soll ich skalieren?" lässt sich auf eine tiefere Frage zurückführen.

Die eigentliche Frage

  • Skalieren oder nicht = Frage nach dem geeigneten Distanzmaß
  • Clustering basiert auf Distanzen – Features mit größeren Wertebereichen dominieren automatisch
  • Umformulierung: „In welcher Situation liegt mir ein geeignetes Distanzmaß vor?"

Entscheidungsregel

  • Zwei Features – eines mit großen, eines mit kleinen Differenzen:
  • Größere Differenz = mehr Gewicht gewollt?
    → Kein Scaling durchführen
  • Alle Features gleich gewichten?
    → Scaling durchführen (z. B. StandardScaler, MinMaxScaler)
Merksatz: Skalieren bedeutet, allen Features das gleiche Gewicht im Distanzmaß zu geben. Ob das sinnvoll ist, hängt vom fachlichen Kontext ab – nicht von einer pauschalen Regel. Ein Praxisbeispiel folgt auf Seite 8.1.

6.2 Was ist Density-based Clustering?

Ein fundamental anderer Ansatz – Cluster als dichte Regionen, nicht als Gruppen um Centroids.

Grundidee

  • Anstatt Cluster über ein Abstandsmaß zu Centroids zu definieren: Abgrenzung dichter von weniger dichten Regionen
  • Cluster = zusammenhängende dichte Datenwolke, unabhängig von ihrer Form
  • Voller Name: Density-Based Spatial Clustering of Applications with Noise

Großer Vorteil

  • Beliebige Cluster-Formen können erkannt werden
  • k muss nicht vorab bestimmt werden
  • Ausreißer werden als Noise explizit klassifiziert – nicht einem Cluster zugeordnet
  • Nur zwei Hyperparameter: eps und min_samples

6.3 DBSCAN Algorithmus

Schritt für Schritt: wie DBSCAN jeden Datenpunkt klassifiziert.

1

Epsilon-Nachbarschaft bestimmen

Für jeden Datenpunkt wird gezählt, wie viele andere Punkte sich innerhalb einer kleinen Distanz eps befinden.
eps = Hyperparameter Epsilon-Nachbarschaftsgröße (Radius des Suchkreises).

A

Core Region (A)

Ein Punkt gilt als Core Point, wenn sich mindestens min_samples Datenpunkte in seiner eps-Nachbarschaft befinden.

  • Bildet das Herzstück eines Clusters
  • Verknüpfte Core Points → ein zusammenhängender Cluster
B/C

Density-reachable (B & C)

Punkte, die innerhalb der eps-Nachbarschaft eines Core Points liegen, aber selbst nicht genug Nachbarn haben – also selbst kein Core Point sind.

  • Gehören zum Cluster des Core Points
  • Sind nicht vollständig deterministisch zugeordnet – bei Überlappung zweier Cluster kann die Zuordnung variieren
N

Noise (N) – Label: −1

Alle Punkte, die weder Core noch density-reachable sind, werden als Rauschen (Noise) klassifiziert.

  • Label in sklearn: -1
  • Werden keinem Cluster zugeordnet
  • Entsprechen Ausreißern / Anomalien
Hyperparameter Bedeutung Effekt wenn zu klein Effekt wenn zu groß
eps Radius der Epsilon-Nachbarschaft Zu viele Noise-Punkte, viele kleine Cluster Alles wird ein Cluster
min_samples Min. Punkte für Core Region Zu viele Core Points, instabil Zu viele Noise-Punkte

6.4 DBSCAN in sklearn

Wir wenden DBSCAN auf das make_moons-Problem aus Seite 5 an – und sehen, was schlechte Hyperparameter anrichten.

Schritt 1 · DBSCAN mit schlechtem eps=0.05

Python 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_moons
from sklearn.cluster import DBSCAN

# Daten
X, y = make_moons(n_samples=1000,
                  noise=0.05, random_state=42)

# DBSCAN mit eps zu klein
dbscan = DBSCAN(eps=0.05, min_samples=5)
dbscan.fit(X)

# Erste Labels ansehen
print(dbscan.labels_[:10])
# → [ 0  2 -1 -1  1  0  0  0  2  5]

# Alle einzigartigen Labels
print(np.unique(dbscan.labels_))
# → [-1  0  1  2  3  4  5  6]
Output 
[ 0  2 -1 -1  1  0  0  0  2  5]
[-1  0  1  2  3  4  5  6]
  • -1 = Noise – keinem Cluster zugeordnet
  • 7 Cluster-Labels (0–6) gefunden – viel zu viele!
  • eps=0.05 ist zu klein → Nachbarschaft zu eng → Punkte werden isoliert

Schritt 2 · Ergebnis visualisieren (eps=0.05)

Python 
# Clustering mit eps=0.05 plotten
plt.scatter(X[:, 0], X[:, 1],
            c=dbscan.labels_, s=50, cmap='viridis')
plt.title("DBSCAN – eps=0.05 (zu klein!)")
plt.colorbar(label='Cluster Label')
plt.show()
Beobachtung
  • ❌ 7 Farben statt 2
  • Die Halbmonde werden in viele kleine Fragmente zerrissen
  • Viele Punkte als Noise (−1) klassifiziert
  • eps muss größer gewählt werden

6.5 Hyperparameter-Tuning

Die Lösung: eps auf einen geeigneten Wert erhöhen – und verstehen, warum .predict() fehlt.

Das Problem

  • Hyperparameter-Optimierung in einem zweidimensionalen Raum: eps × min_samples
  • Beide Parameter beeinflussen sich gegenseitig
  • Es gibt keine automatische Methode wie den Elbow Plot – Tuning erfordert Domänenwissen oder Grid Search

Warum kein .predict()?

  • Die DBSCAN-Klasse in sklearn hat bewusst kein .predict()
  • Der Nutzer soll frei wählen können, wie neue Punkte zugeordnet werden
  • Eine Möglichkeit: k-Nearest-Neighbors Classifier auf den DBSCAN-Labels trainieren
    → Details auf Seite 8.2

Lösung · eps=0.20 – korrektes Ergebnis

Python 
# DBSCAN mit korrektem eps
dbscan = DBSCAN(eps=0.20, min_samples=5)
dbscan.fit(X)

# Wie viele Cluster jetzt?
print(np.unique(dbscan.labels_))

# Ergebnis plotten
plt.scatter(X[:, 0], X[:, 1],
            c=dbscan.labels_, s=50, cmap='viridis')
plt.title("DBSCAN – eps=0.20 (korrekt)")
plt.colorbar(label='Cluster Label')
plt.show()
Ergebnis 
[-1  0  1]
  • ✅ 2 Cluster (Labels 0 und 1) + wenige Noise-Punkte (−1)
  • Jeder Halbmond ist eine Farbe
  • DBSCAN erkennt die nicht-lineare Struktur korrekt – genauso wie Spectral Clustering
  • eps=0.20 passt zur Punktdichte der Halbmonde
Einstellung Gefundene Cluster Noise-Punkte Ergebnis
eps=0.05, min_samples=5 7 (Labels 0–6) Viele ❌ Zu viele Fragmente
eps=0.20, min_samples=5 2 (Labels 0, 1) Wenige ✅ Korrekt
