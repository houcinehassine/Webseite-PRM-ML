DBSCAN + k-Nearest-Neighbors

DBSCAN hat kein .predict() – wie ordnet man neue Datenpunkte trotzdem zu? Lösung: den k-Nearest-Neighbors Klassifikator auf den DBSCAN-Labels trainieren.

DBSCAN components_ core_sample_indices_ KNeighborsClassifier Majoritätsvotum make_moons

Lernfokus dieser Seite

Du verstehst, warum DBSCAN kein .predict() hat, weißt was components_ und core_sample_indices_ bedeuten und kannst einen kNN-Klassifikator auf DBSCAN-Ergebnissen aufsetzen.

8.2.1 Das Problem: Neue Punkte ohne .predict()

DBSCAN klassifiziert bekannte Daten hervorragend – aber was passiert, wenn neue Datenpunkte ankommen?

Warum fehlt .predict()?

  • DBSCAN wurde bewusst ohne .predict() implementiert
  • Neue Punkte könnten als Noise oder als Cluster-Mitglied klassifiziert werden – je nach gewählter Strategie
  • Der Nutzer soll frei entscheiden, wie er neue Punkte zuordnet – sklearn erzwingt keine Strategie

Lösung: kNN als Klassifikator

  • Idee: DBSCAN auf bekannte Daten anwenden → Labels erhalten
  • k-Nearest-Neighbors Klassifikator auf diesen Labels trainieren
  • Neue Punkte werden durch Majoritätsvotum der k nächsten Nachbarn klassifiziert
  • Aus sklearn.neighbors: KNeighborsClassifier

8.2.2 k-Nearest-Neighbors – On-the-fly

Der intuitivste Klassifikator überhaupt – hier kurz erklärt, bevor wir ihn einsetzen.

1

Training = Datensatz abspeichern

kNN hat kein echtes Training im klassischen Sinn. Der .fit()-Schritt speichert lediglich den gesamten Datensatz mit seinen Labels in einer effizienten Datenstruktur (z. B. KD-Baum). Es wird keine Parameteroptimierung durchgeführt – daher auch als lazy learner bezeichnet.

2

Vorhersage = Majoritätsvotum

Für einen neuen Punkt werden die k nächsten Nachbarn im Trainingsdatensatz gesucht. Das häufigste Label unter diesen k Punkten wird als Klasse vergeben. Bei k=50 wird also das Label der Mehrheit unter 50 nächsten Nachbarn zurückgegeben.

3

Labels kommen von DBSCAN

Wir trainieren den kNN nicht auf den Rohdaten, sondern auf den components_ (= Core-Punkte) mit ihren DBSCAN-Labels aus core_sample_indices_. So lernt der kNN die räumliche Struktur der bekannten Cluster.

dbscan.components_

  • Enthält alle Core-Punkte des DBSCAN-Clusterings
  • Das sind die Datenpunkte, die mindestens min_samples Nachbarn in ihrer eps-Umgebung haben
  • Shape: (n_core_samples, n_features)
  • Noise-Punkte (Label −1) sind nicht enthalten

dbscan.core_sample_indices_

  • Enthält die Indizes der Core-Punkte im originalen Datensatz
  • Damit können die Labels der Core-Punkte aus dbscan.labels_ extrahiert werden:
  • dbscan.labels_[dbscan.core_sample_indices_]
  • So wird sichergestellt, dass nur sichere, gut platzierte Punkte zum Training genutzt werden

8.2.3 Vollständige Pipeline

Von den Rohdaten bis zur Vorhersage neuer Punkte – Schritt für Schritt.

Schritt 1 · Daten erzeugen

Python 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_moons
from sklearn.cluster import DBSCAN
from sklearn.neighbors import KNeighborsClassifier

# Zwei Halbmonde – bekannt aus Seite 5 und 6
X, y = make_moons(n_samples=1000,
                  noise=0.05,
                  random_state=42)
Info
  • 1000 Datenpunkte, 2 Halbmonde
  • y wird nicht verwendet – wir tun so, als wäre es unbekannt
  • noise=0.05 – geringe Streuung, klar trennbar

Schritt 2 · DBSCAN fitten & visualisieren

Python 
# DBSCAN mit kalibriertem eps=0.20
dbscan = DBSCAN(eps=0.20, min_samples=5)
dbscan.fit(X)

# Ergebnis plotten
plt.scatter(X[:, 0], X[:, 1],
            c=dbscan.labels_, s=50, cmap='viridis')
plt.title("DBSCAN – eps=0.20, min_samples=5")
plt.colorbar(label='Cluster Label (-1 = Noise)')
plt.show()

# Überblick
print("Unique Labels:", np.unique(dbscan.labels_))
print("Anzahl Core-Punkte:", len(dbscan.core_sample_indices_))
print("Shape components_:", dbscan.components_.shape)
Output 
Unique Labels: [-1  0  1]
Anzahl Core-Punkte: 952
Shape components_: (952, 2)
  • ✅ 2 Cluster (0 und 1) + wenige Noise-Punkte (−1)
  • 952 von 1000 Punkten sind Core-Punkte
  • components_ = diese 952 Core-Punkte als Array

Schritt 3 · kNN auf DBSCAN-Labels trainieren

Python 
# kNN Klassifikator instanziieren
knn = KNeighborsClassifier(n_neighbors=50)

# Training:
# X-Daten  → nur Core-Punkte (components_)
# y-Labels → DBSCAN-Labels der Core-Punkte
knn.fit(
    dbscan.components_,                            # (952, 2)
    dbscan.labels_[dbscan.core_sample_indices_]    # (952,) – nur Core-Labels
)

print("kNN trainiert auf", len(dbscan.components_),
      "Core-Punkten")
Erklärung
  • dbscan.components_ → die Koordinaten der Core-Punkte
  • dbscan.labels_[dbscan.core_sample_indices_] → die zugehörigen Cluster-Labels
  • Noise-Punkte (Label −1) werden bewusst ausgeschlossen
  • kNN speichert nur die Daten – kein echtes Training
  • n_neighbors=50 – stabiles Mehrheitsvotum über viele Nachbarn

Schritt 4 · Neue Datenpunkte zuordnen

Python 
# Zwei neue unbekannte Datenpunkte
new_point_1 = [[-0.5, 0.5]]   # Liegt im linken Halbmond
new_point_2 = [[2, -0.5]]     # Liegt weit außerhalb

# Vorhersage durch kNN
pred_1 = knn.predict(new_point_1)
pred_2 = knn.predict(new_point_2)

print("new_point_1 →  Cluster:", pred_1)
print("new_point_2 →  Cluster:", pred_2)

# Visualisierung inkl. neuer Punkte
plt.scatter(X[:, 0], X[:, 1],
            c=dbscan.labels_, s=30,
            cmap='viridis', alpha=0.5,
            label='Trainingsdaten')

plt.scatter(*new_point_1[0], c='red', s=200,
            marker='*', zorder=5,
            label=f'Punkt 1 → Cluster {pred_1[0]}')
plt.scatter(*new_point_2[0], c='orange', s=200,
            marker='*', zorder=5,
            label=f'Punkt 2 → Cluster {pred_2[0]}')

plt.legend()
plt.title("DBSCAN + kNN – Neue Punkte zugeordnet")
plt.show()
Output 
new_point_1 → Cluster: [0]
new_point_2 → Cluster: [1]
  • [-0.5, 0.5] → liegt geometrisch im linken Halbmond → korrekt Cluster 0
  • [2, -0.5] → liegt außerhalb des Datensatzes, aber nächste Nachbarn gehören zu Cluster 1
  • ⚠️ kNN kann nie „Noise" vorhersagen – jeder neue Punkt bekommt ein Cluster-Label

8.2.4 Übungsaufgabe

Im Code steht ein Kommentar: „How to do it with noise data points? → exercise"

Aufgabe: Im obigen Beispiel wird der kNN nur auf den Core-Punkten trainiert – Noise-Punkte (Label −1) werden ausgeschlossen.

Was würde passieren, wenn man auch Noise-Punkte einbezieht? Wie würde man das implementieren? Welche Vor- und Nachteile hätte das?

Variante A – Noise ausschließen

(aktueller Ansatz) 

# Nur Core-Punkte
knn.fit(
  dbscan.components_,
  dbscan.labels_[
    dbscan.core_sample_indices_]
)
  • ✅ Trainiert nur auf sicheren, dichten Punkten
  • ✅ Labels sind eindeutig und stabil
  • ⚠️ Noise-Bereiche der Daten nicht repräsentiert

Variante B – Alle Punkte inkl. Noise

(Übungsaufgabe) 

# Alle Punkte außer Noise
mask = dbscan.labels_ != -1
knn.fit(
  X[mask],
  dbscan.labels_[mask]
)
  • ✅ Mehr Trainingsdaten – auch Border-Punkte
  • ✅ Bessere Abdeckung der Cluster-Ränder
  • ⚠️ Border-Punkte können unsichere Labels haben

Pipeline-Überblick · DBSCAN + kNN

Schritt Was passiert sklearn-Methode Ergebnis
1. DBSCAN fitten Cluster auf Trainingsdaten X bestimmen dbscan.fit(X) labels_, components_
2. Core-Punkte extrahieren Nur sichere, dichte Punkte als Trainingsbasis components_, core_sample_indices_ X_core, y_core
3. kNN trainieren Labels und Koordinaten abspeichern knn.fit(X_core, y_core) kNN-Modell gespeichert
4. Neue Punkte zuordnen Majoritätsvotum der k nächsten Nachbarn knn.predict(new_points) Cluster-Label für neuen Punkt
