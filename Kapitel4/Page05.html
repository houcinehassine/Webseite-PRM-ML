K-means Variationen

K-means versagt bei nicht-linear separierbaren Problemen. Spectral Clustering, k-means++ und k-Median zeigen, wie man diese Grenzen überwindet.

Nicht-lineare Probleme Kernel-Trick Spectral Clustering k-means++ k-Median Manhattan Distanz

Lernfokus dieser Seite

Du verstehst, warum k-means bei komplexen Geometrien versagt, kennst den Kernel-Trick als Lösung und weißt, welche Variationen für welche Probleme geeignet sind.

5.1 Nicht-lineare Probleme

Sehen wir uns an, wo k-means strukturell scheitert – am Beispiel eines make_moons-Datensatzes.

Szenario: Zwei halbmondförmige Cluster – nicht-linear separierbar, aber für das menschliche Auge klar erkennbar. Was macht k-means daraus?

Schritt 1 · Datensatz erzeugen

Python 
import matplotlib.pyplot as plt
from sklearn.datasets import make_moons
from sklearn.cluster import KMeans

# Zwei halbmondförmige Cluster
X, y = make_moons(200, noise=.05, random_state=0)

# Rohdaten – ohne Labels
plt.scatter(X[:, 0], X[:, 1], s=50)
plt.title("make_moons – Rohdaten")
plt.show()
Info
  • make_moons – zwei ineinandergreifende Halbmonde
  • noise=0.05 – geringe Streuung
  • 200 Datenpunkte, 2 echte Cluster
  • Cluster sind nicht linear trennbar

Schritt 2 · Wie performt k-means hier?

Python 
# k-means mit k=2 auf make_moons anwenden
labels = KMeans(2, random_state=0).fit_predict(X)

# Ergebnis plotten
plt.scatter(X[:, 0], X[:, 1],
            c=labels, s=50, cmap='viridis')
plt.title("k-means – k=2 auf make_moons")
plt.show()
Beobachtung
  • ❌ k-means teilt die Daten horizontal – nicht entlang der Halbmonde
  • Jeder Halbmond wird in zwei Hälften gerissen
  • Die Cluster-Grenze ist eine gerade Linie

Warum versagt k-means?

  • K-means ist nur für linear-separierbare Probleme geeignet
  • Die Heuristik „Datenpunkte sind näher am eigenen Centroid" führt bei komplexen Geometrien zu falschen Zuordnungen
  • Die Cluster-Grenze ist immer eine lineare Funktion der Features – das ist strukturell bedingt

Lösung: Kernel-Trick

  • Aus SVMs bekannt: Daten in einen höherdimensionalen Raum projizieren
  • Im höherdimensionalen Raum werden die Daten linear separierbar
  • Dort kann k-means mit linearen Grenzen korrekte Cluster finden
  • Konkrete Umsetzung → Spectral Clustering

5.2 Spectral Clustering

Der Kernel-Trick für k-means – via Graphentheorie in den höherdimensionalen Raum.

Funktionsweise

  • Mittels graphentheoretischer Maße werden die Daten in einen höherdimensionalen Raum projiziert
  • Dazu wird ein Ähnlichkeitsgraph (z. B. k-nearest-neighbors) über die Datenpunkte aufgebaut
  • Die Spektralzerlegung (Eigenvektoren der Laplace-Matrix) liefert die neue Repräsentation
  • In diesem neuen Raum wird normales k-means ausgeführt – mit linearen Grenzen, die im Original nicht-lineare Cluster trennen

Vergleich zu k-means

  • K-means: Arbeitet direkt im Feature-Raum → nur lineare Grenzen möglich
  • Spectral: Transformiert erst, dann k-means → beliebige Formen möglich
  • affinity='nearest_neighbors' – definiert den Ähnlichkeitsgraph
  • assign_labels='kmeans' – k-means im transformierten Raum

Spectral Clustering auf make_moons

Python 
from sklearn.cluster import SpectralClustering

# Spectral Clustering Modell
model = SpectralClustering(
    n_clusters=2,
    affinity='nearest_neighbors',  # Ähnlichkeitsgraph
    assign_labels='kmeans'         # k-means im neuen Raum
)

# Fitten und Labels bestimmen
labels = model.fit_predict(X)

# Ergebnis plotten
plt.scatter(X[:, 0], X[:, 1],
            c=labels, s=50, cmap='viridis')
plt.title("Spectral Clustering – k=2 auf make_moons")
plt.show()
Ergebnis
  • ✅ Jeder Halbmond ist eine Farbe
  • Die Cluster folgen der natürlichen Struktur der Daten
  • Spectral Clustering erkennt, was k-means nicht kann
  • Kein .predict() für neue Punkte verfügbar (→ siehe Page08_2 mit kNN)
Eigenschaft K-means Spectral Clustering
Clusterform Nur konvex / linear ✅ Beliebig
Grundlage Euklidische Distanz Graphentheorie + Eigenwerte
.predict() ✅ Ja ❌ Nein (nur .fit_predict())
Laufzeit ✅ Schnell ⚠️ Langsamer bei großen Daten
Hyperparameter n_clusters n_clusters, affinity, assign_labels

5.3 K-means++

Gleiche Ergebnisse wie k-means – aber schnellere Konvergenz durch intelligentere Initialisierung.

Das Problem mit zufälliger Initialisierung

  • Standard k-means wählt Centroids vollständig zufällig
  • Schlechte Startwerte → viele Iterationen oder schlechtes lokales Optimum
  • Mehrfache Initialisierungen nötig (n_init) – zeitaufwändig

K-means++ Lösung

  • Centroids werden nach einer gezielten Heuristik initialisiert
  • Die Wahrscheinlichkeit, als Centroid gewählt zu werden, ist proportional zur Distanz zum nächsten bereits gewählten Centroid
  • → Centroids starten verteilt, nicht geclustert
  • Konvergiert schneller bei gleicher Ergebnisqualität
  • In sklearn: init='k-means++' (ist der Default!)
Python 
from sklearn.cluster import KMeans

# k-means++ ist der Default in sklearn
kmeans_pp = KMeans(
    n_clusters=4,
    init='k-means++',   # Default – hier explizit gesetzt
    n_init=10,          # 10 Initialisierungen
    random_state=0
)
kmeans_pp.fit(X)

# Zum Vergleich: zufällige Initialisierung
kmeans_rand = KMeans(
    n_clusters=4,
    init='random',      # Vollständig zufällig
    n_init=10,
    random_state=0
)
kmeans_rand.fit(X)

print("k-means++ Inertia:", kmeans_pp.inertia_)
print("k-means rand Inertia:", kmeans_rand.inertia_)
print("k-means++ Iterationen:", kmeans_pp.n_iter_)
print("k-means rand Iterationen:", kmeans_rand.n_iter_)
Ergebnis
  • Beide liefern ähnliche Inertia
  • k-means++ braucht deutlich weniger Iterationen bis zur Konvergenz
  • init='k-means++' ist deshalb der sklearn-Default

5.4 K-Median

Eine robustere Variante von k-means – tauscht Mittelwert gegen Median und euklidische gegen Manhattan-Distanz.

Unterschiede zu K-means

  • Distanzmaß: Manhattan-Distanz statt euklidischer Distanz
    \(d(a, b) = \sum_i |a_i - b_i|\) statt \(\sqrt{\sum_i (a_i - b_i)^2}\)
  • M-Schritt: Median statt arithmetisches Mittel
  • Der Centroid ist daher immer ein echter Datenpunkt aus dem Datensatz

Wann k-Median nutzen?

  • Robustheit gegenüber Ausreißern – der Median ist stabiler als der Mittelwert
  • Bei Daten mit starken Ausreißern oder schiefen Verteilungen
  • Wenn der Centroid ein interpretierbares Objekt sein soll (z. B. repräsentativer Kunde)
  • Höhere Rechenkosten als k-means (Median-Berechnung)
