Funktionsweise, Heuristik und vollständige Implementierung in sklearn – von den Rohdaten bis zu den gefärbten Clustern mit eingezeichneten Centroids.

Funktionsweise Heuristik KMeans in sklearn .fit() / .predict() cluster_centers_

Lernfokus dieser Seite

Du verstehst, wie k-means intern arbeitet, welche zwei Heuristiken den Algorithmus definieren und wie du ihn vollständig in sklearn von Problem bis Visualisierung umsetzt.

2.1 Funktionsweise

Was macht k-means – und was braucht er als Eingabe?

Aufgabe des Algorithmus

Der k-means Algorithmus sucht nach einer vorab bestimmten Anzahl an Clustern in einem nicht gelabelten Datensatz. Er versucht, die Datenpunkte so in k Gruppen aufzuteilen, dass die Punkte innerhalb einer Gruppe möglichst nah beieinander liegen.

Wichtige Voraussetzung

Die Anzahl der Cluster k muss vor dem Training festgelegt werden – der Algorithmus lernt sie nicht selbst aus den Daten. Die richtige Wahl von k ist daher entscheidend und wird durch Validierungsmaße wie den Elbow Plot oder Silhouette Score unterstützt.

2.2 Heuristik des K-means Algorithmus

Zwei fundamentale Annahmen definieren, was ein „gutes" Cluster ist.

①

Centroid = Arithmetisches Mittel

Der Cluster-Mittelpunkt (Centroid) ist definiert als das arithmetische Mittel aller Punkte des jeweiligen Clusters. Er liegt also im geometrischen Zentrum der zugehörigen Datenpunkte – und wird nach jeder Iteration neu berechnet.

②

Jeder Punkt liegt näher am eigenen Centroid

Jeder Datenpunkt ist näher an seinem eigenen Centroid als an jedem anderen Centroid. Diese Bedingung wird nach jeder Zuweisung neu überprüft – und ist das Konvergenzkriterium des Algorithmus.

Zusammenspiel: Beide Heuristiken zusammen ergeben den iterativen Expectation-Maximization-Ablauf von k-means – mehr dazu auf Seite 3: EM-Algorithmus.

2.3 K-means in Sklearn

Die sklearn-Implementierung folgt dem bekannten fit → predict-Schema aller Modelle in sklearn.

API

Gleiche Struktur wie alle sklearn-Modelle

In sklearn weist der k-means Algorithmus die gleiche Struktur wie andere Modelle auf – instanziieren, fitten, vorhersagen. Als wichtiger Zusatz bei der Instanziierung muss die Anzahl der Cluster n_clusters angegeben werden.

.predict()

.predict() – Zugehörigkeit von Datenpunkten

Die Zugehörigkeit der Datenpunkte zu den jeweiligen Centroids – auch für neue, ungesehene Datenpunkte – erhalten wir über die .predict() Methode. Sie gibt für jeden Punkt den Index des nächstgelegenen Centroids zurück.

attr

.cluster_centers_ – Die erlernten Centroids

Die erlernten Centroids nach dem Training stecken im Attribut .cluster_centers_ des Modells. Es handelt sich um ein Array der Form (n_clusters, n_features) – also k Punkte im Feature-Raum.

Element Beschreibung Beispiel
KMeans(n_clusters=k) Modell instanziieren mit k Clustern KMeans(n_clusters=4)
.fit(X) Modell auf Daten trainieren – Centroids lernen kmeans.fit(X)
.predict(X) Cluster-Label für jeden Datenpunkt bestimmen kmeans.predict(X)
.fit_predict(X) fit + predict in einem Schritt kmeans.fit_predict(X)
.cluster_centers_ Koordinaten der erlernten Centroids kmeans.cluster_centers_
.inertia_ Summe der quadratischen Abstände (SSE) – Gütemaß kmeans.inertia_

2.4 Problem Beispiel

Wir erzeugen einen synthetischen Datensatz mit 4 Clustern und visualisieren ihn zunächst ohne Labels.

Szenario: 300 Datenpunkte werden mit make_blobs um 4 Cluster-Zentren generiert. Die Cluster sind kompakt (cluster_std=0.60). Ziel: k-means soll diese 4 Gruppen automatisch erkennen – ohne die wahren Labels y_true zu kennen.

Schritt 1 · Datensatz erzeugen und visualisieren

Python 
import matplotlib.pyplot as plt
import seaborn as sns
sns.set()  # Plot-Styling aktivieren

from sklearn.datasets import make_blobs

# Synthetischen Datensatz mit 4 Clustern erzeugen
X, y_true = make_blobs(
    n_samples=300,
    centers=4,
    cluster_std=0.60,
    random_state=0
)

# Datenpunkte ohne Labels plotten (unsupervised → keine Farben)
plt.scatter(X[:, 0], X[:, 1], s=50)
plt.title("Rohdaten – keine Labels bekannt")
plt.xlabel("Feature 1")
plt.ylabel("Feature 2")
plt.show()
Output

📊 Scatter-Plot mit 300 grauen Punkten – die Clusterstruktur ist erkennbar, aber noch nicht gelabelt.

make_blobs

  • n_samples=300 – Anzahl der Datenpunkte
  • centers=4 – Anzahl der Cluster-Zentren
  • cluster_std=0.60 – Streuung um das Zentrum
  • random_state=0 – Reproduzierbarkeit

Warum ungelabelt?

Im unüberwachten Lernen stehen uns keine Labels zur Verfügung. y_true wird hier nur für den Vergleich nach dem Clustering verwendet – im echten Anwendungsfall ist es nicht vorhanden.

2.5 K-means Lösung

Jetzt wenden wir k-means an und visualisieren Cluster-Zugehörigkeiten und Centroids.

Schritt 2 · K-means fitten und Labels bestimmen

Python 
from sklearn.cluster import KMeans

# Modell mit k=4 Clustern instanziieren
kmeans = KMeans(n_clusters=4)

# Modell auf die Daten fitten → Centroids lernen
kmeans.fit(X)

# Cluster-Labels für jeden Datenpunkt bestimmen
y_kmeans = kmeans.predict(X)

print("Cluster-Labels (erste 10):", y_kmeans[:10])
print("Anzahl Punkte pro Cluster:", 
      {i: (y_kmeans == i).sum() for i in range(4)})
Output 
Cluster-Labels (erste 10): [1 3 2 0 3 1 2 0 1 3]
Anzahl Punkte pro Cluster: {0: 75, 1: 75, 2: 75, 3: 75}

Schritt 3 · Cluster und Centroids visualisieren

Python 
import matplotlib.pyplot as plt
import seaborn as sns
sns.set()

# Datenpunkte eingefärbt nach Cluster-Zugehörigkeit
plt.scatter(
    X[:, 0], X[:, 1],
    c=y_kmeans,       # Farbe = Cluster-Label
    s=50,
    cmap='viridis'
)

# Erlernten Centroids aus dem Modell holen
centers = kmeans.cluster_centers_   # Shape: (4, 2)

# Centroids als große schwarze Punkte einzeichnen
plt.scatter(
    centers[:, 0], centers[:, 1],
    c='black',
    s=200,
    alpha=0.5,
    label='Centroids'
)

plt.title("K-means Clustering – k=4")
plt.xlabel("Feature 1")
plt.ylabel("Feature 2")
plt.legend()
plt.show()
Output

📊 Scatter-Plot mit 4 farbigen Cluster-Gruppen (viridis). 4 schwarze Punkte markieren die Centroids im Zentrum jedes Clusters.

c=y_kmeans

Der Parameter c=y_kmeans in plt.scatter() weist jedem Datenpunkt eine Farbe basierend auf seinem Cluster-Label zu. Mit cmap='viridis' werden die Labels 0–3 auf eine Farbskala gemappt.

cluster_centers_

kmeans.cluster_centers_ ist ein Array mit den Koordinaten der k erlernten Centroids. Sie werden separat mit größeren schwarzen Punkten (s=200) eingezeichnet, um sie visuell hervorzuheben.

Was sehen wir?

Vier klar abgegrenzte Farbgruppen – jede mit einem zentralen schwarzen Centroid. Der Algorithmus hat die 4 natürlichen Cluster korrekt erkannt, obwohl er y_true nie gesehen hat.

Vollständiger Code auf einen Blick

Python – Komplett 
import matplotlib.pyplot as plt
import seaborn as sns
sns.set()
from sklearn.datasets import make_blobs
from sklearn.cluster import KMeans

# 1. Daten erzeugen
X, y_true = make_blobs(n_samples=300, centers=4,
                       cluster_std=0.60, random_state=0)

# 2. K-means fitten
kmeans = KMeans(n_clusters=4)
kmeans.fit(X)
y_kmeans = kmeans.predict(X)

# 3. Cluster plotten
plt.scatter(X[:, 0], X[:, 1], c=y_kmeans, s=50, cmap='viridis')

# 4. Centroids plotten
centers = kmeans.cluster_centers_
plt.scatter(centers[:, 0], centers[:, 1], c='black', s=200, alpha=0.5)

plt.title("K-means – k=4")
plt.show()
Output

📊 Finales Clustering – 4 farbige Gruppen, schwarze Centroids in der Mitte jedes Clusters.
