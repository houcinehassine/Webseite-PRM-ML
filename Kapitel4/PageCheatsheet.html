Alle Formeln auf einen Blick

Dunn-Index

\[DI_m = \frac{\min\limits_{1 \leq i < j \leq m} \delta(C_i, C_j)} {\max\limits_{1 \leq k \leq m} \Delta_k}\]
  • Je größer, desto besser
  • Zähler = kleinste Inter-Cluster-Distanz
  • Nenner = größte Intra-Cluster-Distanz

Davies-Bouldin Index

\[DB = \frac{1}{m} \sum_{i=1}^{m} \max_{i \neq j} \left(\frac{s_i + s_j}{d(C_i, C_j)}\right)\]
  • Je kleiner, desto besser
  • \(s_i\) = Streuung innerhalb Cluster \(i\)
  • sklearn: davies_bouldin_score(X, y)

Silhouette Score S(o)

\[S(o) = \frac{\text{dist}(B,o) - \text{dist}(A,o)} {\max\{\text{dist}(A,o),\,\text{dist}(B,o)\}}\]
  • Wertebereich −1 bis +1, höher = besser
  • \(\text{dist}(A,o)\) = Ø Abstand zum eigenen Cluster
  • \(\text{dist}(B,o)\) = Ø Abstand zum nächsten fremden Cluster
  • sklearn: silhouette_score(X, labels)

SSE / Inertia

\[\text{SSE} = \sum_{i=1}^{k} \sum_{x \in C_i} \|x - \mu_i\|^2\]
  • Je kleiner, desto kompakter die Cluster
  • Sinkt immer mit steigendem \(k\) – allein nicht aussagekräftig
  • sklearn: kmeans.inertia_

Manhattan-Distanz (k-Median)

\[d(a, b) = \sum_i |a_i - b_i|\]
  • Robuster gegenüber Ausreißern als euklidisch
  • Wird im M-Schritt von k-Median verwendet

Intra-/Inter-Cluster

\[\text{dist}(A,o) = \frac{1}{n_A-1}\sum_{a \in A, a\neq o}\text{dist}(a,o)\]
\[\text{dist}(B,o) = \min_{C \neq A}\frac{1}{n_C}\sum_{c \in C}\text{dist}(c,o)\]

sklearn API – Alle wichtigen Befehle

KMeans

from sklearn.cluster import KMeans

km = KMeans(
    n_clusters=k,
    init='k-means++',  # Default
    n_init=10,
    random_state=0
)
labels = km.fit_predict(X)

km.inertia_           # SSE
km.cluster_centers_   # Centroids
km.labels_            # Cluster-Labels
km.n_iter_            # Iterationen

DBSCAN

from sklearn.cluster import DBSCAN

db = DBSCAN(
    eps=0.20,         # Radius
    min_samples=5     # Min. Nachbarn
)
db.fit(X)

db.labels_              # -1 = Noise
db.core_sample_indices_ # Indizes Core
db.components_          # Core-Punkte

Spectral Clustering

from sklearn.cluster import SpectralClustering

sc = SpectralClustering(
    n_clusters=2,
    affinity='nearest_neighbors',
    assign_labels='kmeans',
    random_state=0
)
labels = sc.fit_predict(X)
# Kein .predict() verfügbar!

Validierungsmetriken

from sklearn.metrics import (
    silhouette_score,
    silhouette_samples,
    davies_bouldin_score
)

# Durchschnittlicher Score
s = silhouette_score(X, labels)

# Pro Datenpunkt
s_all = silhouette_samples(X, labels)

# Davies-Bouldin (kleiner = besser)
db = davies_bouldin_score(X, labels)

DBSCAN + kNN Pipeline

from sklearn.neighbors import (
    KNeighborsClassifier
)

# kNN auf Core-Punkten trainieren
knn = KNeighborsClassifier(n_neighbors=50)
knn.fit(
    db.components_,
    db.labels_[db.core_sample_indices_]
)

# Neuen Punkt zuordnen
knn.predict([[x1, x2]])

Scaling

from sklearn.preprocessing import (
    StandardScaler,
    MinMaxScaler
)

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Für neue Punkte (KEIN fit_transform!)
X_new_scaled = scaler.transform(X_new)

Entscheidungsregeln für die Prüfung

Welchen Algorithmus, welches Maß – wann?

?

Welcher Clustering-Algorithmus?

  • Konvexe, kompakte Cluster → k-means / k-means++
  • Beliebige Formen, nicht-linear → DBSCAN oder Spectral Clustering
  • Unterschiedliche Dichten → HDBSCAN oder OPTICS
  • Ausreißer explizit erkennen → DBSCAN (Noise-Label −1)
  • k unbekannt → DBSCAN, MeanShift, AffinityPropagation
  • Sehr große Datensätze → MiniBatchKMeans oder BIRCH
📏

Skalieren oder nicht?

  • Frage: Soll das Feature mit größerem Wertebereich auch mehr Gewicht bekommen?
  • Ja → kein Scaling
  • Nein / alle gleich gewichten → StandardScaler anwenden
  • Faustregel: Bei physikalisch unterschiedlichen Einheiten (z. B. Vibration vs. Temperatur) meist skalieren
k?

Optimales k finden

  • Elbow Plot: SSE gegen k – Knick = optimales k
  • Silhouette Score: Maximum = optimales k
  • Beide Methoden kombinieren für robuste Entscheidung
  • Domänenwissen (z. B. 6 Weihenstephaner Zustände) als Startpunkt nutzen
✓

Silhouette-Plot lesen

  • Gut: alle Balken rechts der roten Linie, ähnliche Breiten, keine negativen Werte
  • Schlecht: viele Balken links der roten Linie, negative Scores, stark unterschiedliche Breiten
  • Rote Linie = Durchschnittlicher Silhouette Score

Typische Fallen & Missverständnisse

Was in der Prüfung gerne falsch gemacht wird.

🪤 k-means auf nicht-linearen Daten

k-means versagt bei Halbmonden, Kreisen etc. – die Cluster-Grenzen sind immer linear. Lösung: Spectral Clustering oder DBSCAN.

🪤 SSE / Inertia als alleiniges Maß

Inertia sinkt immer mit steigendem k – auch wenn die Cluster schlechter werden. Immer mit Silhouette kombinieren!

🪤 DBSCAN: fit_transform vs. transform

Scaler muss auf Trainingsdaten mit .fit_transform() angepasst werden. Für neue Punkte nur .transform() – sonst werden die Statistiken neu berechnet!

🪤 DBSCAN Label −1 vergessen

dbscan.labels_ enthält −1 für Noise-Punkte. Beim Zählen der Cluster mit np.unique() enthält das Ergebnis auch −1 – nicht als Cluster zählen! 

# Korrekt: Noise herausfiltern
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)

🪤 components_ vs. labels_

dbscan.components_ enthält nur die Core-Punkte (Koordinaten), nicht alle Datenpunkte. dbscan.labels_ enthält Labels für alle Datenpunkte inkl. Border und Noise.

🪤 Dunn vs. Davies-Bouldin Richtung

Dunn-Index: größer = besser
Davies-Bouldin: kleiner = besser
Beide verwechseln führt zu falschen Schlüssen!

🪤 kNN trainiert auf falschen Labels

Beim DBSCAN+kNN Pattern: kNN muss auf db.labels_[db.core_sample_indices_] trainiert werden – nicht auf db.labels_ direkt, da components_ nur Core-Punkte enthält.

🪤 Spectral Clustering: kein predict()

SpectralClustering hat kein .predict() – nur .fit_predict(). Für neue Punkte → kNN auf den gelernten Labels trainieren.

Merksätze – Ein Satz pro Konzept

Für schnelle Wiederholung kurz vor der Prüfung.

1

k-means teilt Daten in k Gruppen – durch abwechselndes Zuordnen (E-Schritt) und Centroid-Neuberechnen (M-Schritt), bis Konvergenz.

2

Validierung: Gut = Intra-Cluster-Ähnlichkeit hoch und Inter-Cluster-Ähnlichkeit gering.

3

Elbow Plot: SSE gegen k – der Knick zeigt, wo weitere Cluster kaum noch Varianz erklären.

4

Silhouette Score nahe +1: Punkt liegt gut im Cluster; nahe 0: zwischen zwei Clustern; negativ: falsches Cluster.

5

DBSCAN findet Cluster als dichte Regionen – beliebige Formen, automatisches k, Noise wird explizit erkannt (Label −1).

6

Spectral Clustering nutzt den Kernel-Trick – Daten werden via Graphentheorie transformiert, sodass k-means beliebige Formen trennen kann.

7

Scaling-Entscheidung: Sollen alle Features gleich gewichtet werden? → Ja: StandardScaler. Nein: kein Scaling.

8

kNN hat kein echtes Training – er speichert nur den Datensatz und klassifiziert per Majoritätsvotum der k nächsten Nachbarn.
