EM-Algorithmus in K-means

Die Heuristik von k-means wird zum formalen Rezept: der Expectation-Maximization-Algorithmus – mit mathematischer Herleitung und vollständiger From-Scratch-Implementierung in Python.

EM-Algorithmus E-Schritt M-Schritt Mathematik From Scratch Konvergenz

Lernfokus dieser Seite

Du verstehst, warum k-means ein Spezialfall des allgemeinen EM-Algorithmus ist, kennst die mathematischen Schritte und kannst den Algorithmus von Grund auf implementieren.

3.1 Was ist der EM-Algorithmus?

Die Heuristik von k-means lässt sich in ein allgemeines „Rezept" überführen – den Expectation-Maximization-Algorithmus.

Eigenschaften

  • Findet in vielen ML-Modellen Anwendung
  • Ist ein iterativer Algorithmus
  • Wechselt zwischen zwei Schritten bis zur Konvergenz
  • Garantiert lokale, aber nicht globale Optima

Generelle Idee

  • Init: Zufällige Konfiguration des Modells
  • E-Schritt – Zuordnung der Daten zu Modellteilen
  • M-Schritt – Parameter an neue Zuordnung anpassen
  • Konvergenz: Keine wesentliche Verbesserung → Abbruch

3.2 Analoge Schritte bei K-means

Wie übersetzt sich der allgemeine EM-Ablauf konkret auf k-means?

0

Initialisierung

Allgemein: Initialisiere eine zufällige Konfiguration des Modells.
Bei k-means: Wähle k Centroids zufällig aus dem Datensatz.

E

E(xpectation)-Schritt

Allgemein: Zuordnung der Daten zu den einzelnen Teilen des Modells.
Bei k-means: Weise jeden Datenpunkt dem nächstgelegenen Centroid zu.

→ „Unsere Erwartung updaten, zu welchem Cluster die Datenpunkte gehören."

M

M(aximization)-Schritt

Allgemein: Parameter an die neueste Zuordnung anpassen.
Bei k-means: Berechne neue Centroids als arithmetisches Mittel der zugeordneten Datenpunkte.

→ „Ein Gütemaß wird maximiert – durch Berechnung des Mittelwertes."

↻

Wiederholen bis Konvergenz

E- und M-Schritt werden abwechselnd wiederholt, bis keine wesentliche Verschiebung der Centroids mehr stattfindet (deviation ≤ threshold).

EM (Allgemein) K-means (Konkret)
Zufällige Modell-Initialisierung k Centroids zufällig wählen
E-Schritt: Daten zuordnen Jeden Punkt dem nächsten Centroid zuweisen
M-Schritt: Parameter updaten Centroids als Mittelwert neu berechnen
Konvergenz prüfen Centroid-Verschiebung < threshold?

3.3 Input & Output

Was geht in den k-means-Algorithmus hinein – und was kommt heraus?

📥 Input

  • X – Data matrix, Dimensionen \(N \times T\)
  • N – Anzahl der Beobachtungen (Datenpunkte)
  • T – Anzahl der Features (z. B. \(\bar{x}, \sigma^2, \ldots\))
  • k – Anzahl der zu extrahierenden Cluster (muss vorab angegeben werden!)

📤 Output

  • cz – Cluster-Centroids für \(z = 1, \ldots, k\)
    → Koordinaten der k Cluster-Mittelpunkte
  • \(\hat{z}_t\) – Zugewiesener Cluster-Index für Datenpunkt \(\mathbf{x}_t\)
    → Label-Vektor der Länge N

3.4 Mathematische Ausführung

Die formale Beschreibung der drei Kernschritte des EM-Algorithmus in k-means.

1

Initialisierung

Wähle k Centroids \(\mathbf{c}_z\) zufällig aus dem Datensatz.

E

Expectation-Schritt – Zuweisung zum nächsten Centroid

Jeden Datenpunkt dem nächstgelegenen Centroid zuweisen:

\[ \hat{z}_t = \arg\min_z \; d(\mathbf{c}_z,\, \mathbf{x}_t) \]
  • \(\hat{z}_t\) – zugewiesener Cluster-Index von Punkt \(\mathbf{x}_t\)
  • d(·,·) – Distanzfunktion (Standard: euklidische Distanz)
M

Maximization-Schritt – Neuen Centroid berechnen

Centroid als gewichtetes Mittel der zugehörigen Punkte:

\[ \mathbf{c}_z \leftarrow \frac{\displaystyle\sum_{t=1}^{T} u(t,z,\hat{z}_t)\,\mathbf{x}_t} {\displaystyle\sum_{t=1}^{T} u(t,z,\hat{z}_t)} \]

mit der Indikatorfunktion:

\[ u(t,z,\hat{z}_t) = \begin{cases} 1, & \text{wenn } \hat{z}_t = z \\ 0, & \text{sonst} \end{cases} \]
  • u(·) – wählt nur Punkte aus, die zu Cluster z gehören
  • Ergebnis = arithmetisches Mittel aller Cluster-Punkte
↻

Wiederholen

Schritte E und M werden abwechselnd ausgeführt, bis die Centroid-Verschiebung unter einen Schwellwert threshold fällt → Konvergenz.

3.5 From-Scratch Implementierung

Der k-means Algorithmus ist so anschaulich, dass wir ihn vollständig von Hand implementieren können – ohne sklearn's KMeans-Klasse.

Aufgabe: Schreibe eine Funktion find_clusters(X, num_clusters), die als Eingabe die Datenmatrix und die Anzahl der Cluster nimmt – und Centroids + Labels zurückgibt.

Schritt 1 · Daten vorbereiten

Python 
from sklearn.datasets import make_blobs

# Datensatz: 300 Punkte, 4 Cluster
X, y_true = make_blobs(
    n_samples=300,
    centers=4,
    cluster_std=0.60,
    random_state=0
)
Info
  • Gleicher Datensatz wie auf Seite 2
  • X.shape → (300, 2)
  • y_true wird nicht verwendet

Schritt 2 · find_clusters() Funktion implementieren

Python 
import numpy as np
from scipy.spatial.distance import cdist

def find_clusters(X, num_clusters, threshold=0.0001):
    """
    K-means from Scratch.
    Returns: centroids, labels
    """

    # 1. Zufällige Initialisierung der Centroids
    initial_centroid_indices = np.random.permutation(
        X.shape[0])[:num_clusters]
    centroids = X[initial_centroid_indices]

    deviation = threshold + 1  # Startwert > threshold

    while deviation > threshold:

        # 2a. E-Schritt: Label = nächster Centroid
        distances = cdist(X, centroids)
        labels = np.argmin(distances, axis=1)

        # 2b. M-Schritt: Neue Centroids = Mittelwert
        new_centers = np.array([
            X[labels == n_cluster].mean(0)
            for n_cluster in range(num_clusters)
        ])

        # 2c. Konvergenz prüfen
        deviation = np.sum((new_centers - centroids) ** 2)
        centroids = new_centers

    return centroids, labels
Code-Erklärung
  • np.random.permutation – zufällige Reihenfolge → erste k als Centroids
  • cdist(X, centroids) – Abstands-Matrix: jeder Punkt zu jedem Centroid
  • np.argmin(..., axis=1) – Index des nächsten Centroids pro Punkt
  • .mean(0) – Mittelwert entlang der Punkte-Achse (M-Schritt)
  • deviation – quadratische Verschiebung der Centroids

Schritt 3 · Funktion aufrufen & Ergebnis plotten

Python 
import matplotlib.pyplot as plt

# Funktion ausführen – k=4 Cluster
centroids, labels = find_clusters(X, 4)

# Ergebnis plotten
plt.figure(figsize=(4, 3))
plt.scatter(X[:, 0], X[:, 1],
            c=labels, s=50, cmap='viridis')
plt.xlabel('Feature 1')
plt.ylabel('Feature 2')
plt.title('K-means from Scratch – k=4')
plt.show()
Output

📊 Identisches Ergebnis wie sklearn.KMeans – 4 farbige Cluster ohne schwarze Centroid-Punkte (da wir sie separat plotten müssten).

3.6 Nachteile des EM-Algorithmus

Zwei fundamentale Schwächen – und wie sklearn damit umgeht.

⚠️ Kein globales Optimum

  • EM verbessert sich von Iteration zu Iteration
  • Konvergenz ins globale Optimum ist nicht garantiert
  • Abhängig von den Initialbedingungen (zufällige Centroid-Wahl)
  • Lösung: Algorithmus mehrfach starten mit verschiedenen Startbedingungen → bestes Ergebnis wählen
sklearn-Lösung: n_init – steuert die Anzahl der Initialisierungen (default: n_init=10). sklearn führt dies automatisch durch.

⚠️ k muss vorgegeben werden

  • k-means lernt die Clusteranzahl nicht eigenständig aus den Daten
  • Problem betrifft viele Unsupervised-Algorithmen
  • Jede beliebige Zahl k kann übergeben werden – auch eine falsche
  • Man benötigt ein Validierungskriterium zur Entscheidung
Lösung: Seite 4 zeigt Elbow Plot, Silhouette Score und weitere Validierungsmaße.

Beispiel · Was passiert bei falschem k? (k=6 statt k=4)

Python 
from sklearn.datasets import make_blobs
from sklearn.cluster import KMeans
import matplotlib.pyplot as plt

# Datensatz hat eigentlich 4 Cluster
X, y_true = make_blobs(n_samples=300, centers=4,
                       cluster_std=0.60, random_state=0)

# Aber wir geben k=6 an → zu viele Cluster!
kmeans = KMeans(6, random_state=0)
labels = kmeans.fit_predict(X)

plt.scatter(X[:, 0], X[:, 1],
            c=labels, s=50, cmap='viridis')
plt.title("k=6 auf einem 4-Cluster-Datensatz")
plt.show()
Beobachtung
  • 6 Farben sichtbar – natürliche Cluster werden zerschnitten
  • k-means erzwingt 6 Gruppen, obwohl nur 4 vorhanden sind
  • → Validierung ist notwendig! Ohne Validierung ist das Ergebnis nicht interpretierbar
