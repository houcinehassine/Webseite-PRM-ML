Validierung von Clustering

Wie bewertet man, ob ein Clustering „gut" ist – ohne Labels? Von einfachen Heuristiken über Dunn-Index und Silhouette Score bis zum Elbow Plot.

Inertia Dunn-Index Davies-Bouldin Silhouette Score Elbow Plot SSE

Lernfokus dieser Seite

Du kennst die vier Arten der Cluster-Validierung, kannst Silhouette-Plots lesen und interpretieren und findest mit dem Elbow Plot das optimale k.

4.1 Was bedeutet Validierung?

Beim unüberwachten Lernen gibt es keine Labels – wie beurteilt man dann die Qualität des Ergebnisses?

Definition

  • Validierung misst die Güte eines Clusterings
  • Bewertet, ob die Gruppen natürliche Strukturen der Daten widerspiegeln
  • Da keine Labels vorliegen → braucht man spezielle Kriterien
  • Kernfrage: Sind Punkte im gleichen Cluster ähnlich und verschiedene Cluster verschieden?

Grundprinzip – So what?

  • Inter-Cluster-Ähnlichkeit → so gering wie möglich
    d.h. Cluster sollen weit voneinander entfernt sein
  • Intra-Cluster-Ähnlichkeit → so hoch wie möglich
    d.h. Punkte innerhalb eines Clusters sollen nah beieinander sein

Erste Intuition · kmeans.inertia_

Python 
from sklearn.cluster import KMeans
import matplotlib.pyplot as plt

# k-means mit k=4 fitten
kmeans = KMeans(4, random_state=0)
labels = kmeans.fit_predict(X)

# Clustering visualisieren
plt.scatter(X[:, 0], X[:, 1],
            c=labels, s=50, cmap='viridis')
plt.title("k=4 Clustering")
plt.show()

# Inertia ausgeben (= SSE)
print("Inertia:", kmeans.inertia_)
Output 
Inertia: 170.729...
  • .inertia_ = Summe der quadratischen Abstände aller Punkte zu ihrem Centroid
  • Je kleiner, desto kompakter die Cluster
  • Allein nicht aussagekräftig – sinkt immer mit steigendem k

4.2 Arten der Cluster-Validierung

Vier grundlegende Ansätze – je nach verfügbaren Informationen.

IN

Internal Evaluation

Das Clustering wird durch ein Gütemaß bewertet, das nur die Daten selbst und die Cluster-Zuordnung verwendet – ohne externe Information. Beispiele: Silhouette Score, Dunn-Index, Davies-Bouldin-Index.

EX

External Evaluation

Das Clustering wird mit einer vorhandenen „Ground Truth" an Labels verglichen. Nur möglich, wenn echte Labels bekannt sind – was beim echten Unsupervised Learning meist nicht der Fall ist.

MA

Manual Evaluation

Manuelle Inspektion durch einen Fachexperten. Zeitaufwändig, aber oft unverzichtbar in der Praxis – besonders wenn Domänenwissen entscheidend ist.

IND

Indirect Evaluation

Evaluation durch die Anwendung des Clustering-Ergebnisses in einem nachgelagerten Task. Das Clustering gilt als gut, wenn es die Downstream-Aufgabe (z. B. Klassifikation) verbessert.

4.3 Dunn-Index

Das intuitivste Gütemaß – basierend auf dem Verhältnis von Inter- zu Intra-Cluster-Distanz.

Idee

  • Verhältnis aus Inter-Cluster-Distanz \(\delta(C_i, C_j)\) zu Intra-Cluster-Distanz \(\Delta_k\)
  • Inter-Cluster-Distanz → Abstände zwischen den Centroids
  • Intra-Cluster-Distanz → Maximum der Abstände innerhalb eines Clusters
  • Je größer der Dunn-Index, desto besser das Clustering

Formel

\[ DI_m = \frac{ \min\limits_{1 \leq i < j \leq m} \delta(C_i,\, C_j) }{ \max\limits_{1 \leq k \leq m} \Delta_k } \]
  • m – Anzahl der Cluster
  • \(\delta(C_i, C_j)\) – kleinste Inter-Cluster-Distanz (Zähler)
  • \(\Delta_k\) – größte Intra-Cluster-Distanz (Nenner)
  • Ziel: Zähler groß, Nenner klein → \(DI_m\) groß

4.4 Davies-Bouldin Index

Weniger intuitiv als der Dunn-Index, aber in sklearn direkt verfügbar.

Beschreibung

  • Bewertet die Trennung zwischen Clustern
  • Und die Streuung innerhalb der Cluster
  • s_i = Streuung der Abstände aller Punkte zum Centroid in Cluster \(i\)
  • Je kleiner der DB-Index, desto besser das Clustering (Gegenteil zum Dunn-Index!)
  • In sklearn: davies_bouldin_score(X, y) aus sklearn.metrics

Formel

\[ DB = \frac{1}{m} \sum_{i=1}^{m} \max_{i \neq j} \left( \frac{s_i + s_j}{d(C_i,\, C_j)} \right) \]
  • s_i + s_j – Summe der Streuungen beider Cluster (Zähler)
  • d(C_i, C_j) – Abstand zwischen Centroids (Nenner)
  • max wählt den schlechtesten Nachbarn für jeden Cluster
  • Mittelwert davon über alle Cluster
Intuition für min/max: max – sucht den ähnlichsten (schlechtesten) Nachbar-Cluster. min im Dunn-Index – sucht den am weitesten entfernten Cluster-Abstand.

4.5 Silhouette Score

Das vielseitigste Maß – bewertet jeden einzelnen Datenpunkt und liefert ein globales sowie clusterweises Gütemaß.

Beschreibung

  • Bewertet, wie ähnlich ein Punkt seinem eigenen Cluster ist
  • Und wie unähnlich er zu allen anderen Clustern ist
  • Wertebereich: −1 bis +1

Interpretation

  • Nahe +1 – Punkt liegt gut in seinem Cluster
  • Um 0 – Punkt liegt zwischen zwei Clustern
  • Negativ (< 0) – Punkt liegt näher an einem fremden Cluster → Clustering verbesserungswürdig
  • Viele hohe Silhouette-Scores → gutes Clustering
a(o)

Intra-Cluster-Distanz dist(A, o)

Mittlerer Abstand von Punkt \(o\) zu allen anderen Punkten im eigenen Cluster \(A\):

\[ \text{dist}(A, o) = \frac{1}{n_A - 1} \sum_{\substack{a \in A,\, a \neq o}} \text{dist}(a,\, o) \]
b(o)

Nächster-Cluster-Abstand dist(B, o)

Kleinster mittlerer Abstand zu allen Punkten der fremden Cluster \(C \neq A\):

\[ \text{dist}(B,\, o) = \min_{C \neq A} \underbrace{ \left( \frac{1}{n_C} \sum_{c \in C} \text{dist}(c,\, o) \right) }_{=\,\text{dist}(C,\, o)} \]
S(o)

Silhouette Score für Datenpunkt o

\[ S(o) = \begin{cases} 0 & \text{wenn } o \text{ einziges Element von } A \\[6pt] \dfrac{\text{dist}(B,o) - \text{dist}(A,o)} {\max\{\text{dist}(A,o),\;\text{dist}(B,o)\}} & \text{sonst} \end{cases} \]

Der Mittelwert von \(S(o)\) über alle Datenpunkte ist das globale Gütemaß für den gesamten Clustering-Vorgang.

4.6 Silhouette-Plot – Übung

Cluster-Validierung kann tückisch sein – wir sehen uns das in einer vollständigen Übungsaufgabe an.

Szenario: Ein Datensatz mit 4 Clustern – davon ein deutlicher und drei nah beieinanderliegende. Wir testen k = 2 bis 10 und bewerten jedes Clustering mit einem Silhouette-Plot.

Schritt 1 · Datensatz erzeugen

Python 
from sklearn.datasets import make_blobs
import numpy as np
import matplotlib.pyplot as plt
import matplotlib.cm as cm
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score, silhouette_samples

# Datensatz: 4 Cluster, aber ungleich verteilt
X, y = make_blobs(
    n_samples=500,
    n_features=2,
    centers=4,
    cluster_std=1,
    center_box=(-10.0, 10.0),
    shuffle=True,
    random_state=1
)

# k-Werte, die wir testen wollen
range_n_clusters = range(2, 11)
Parameter
  • n_samples=500 – 500 Datenpunkte
  • centers=4 – 4 echte Cluster
  • cluster_std=1 – mittlere Streuung
  • random_state=1 – Reproduzierbarkeit
  • Test für k = 2 bis 10
Was macht die folgende Schleife? Für jedes k wird ein k-means Clustering durchgeführt. Dann werden pro Durchlauf zwei Plots erstellt:
  • Links: Silhouette-Plot – zeigt den Score jedes einzelnen Datenpunkts, gruppiert nach Cluster. Die rote gestrichelte Linie = Durchschnitt.
  • Rechts: Scatter-Plot – die tatsächliche Cluster-Zuordnung mit nummerierten Centroids.
So kann man visuell beurteilen, welches k das Clustering am besten beschreibt.

Schritt 2 · Silhouette-Analyse Schleife (k = 2 bis 10)

Python 
for n_clusters in range_n_clusters:

    fig, (ax1, ax2) = plt.subplots(1, 2)
    fig.set_size_inches(18, 7)

    # ── Linker Plot: Silhouette ──────────────────
    ax1.set_xlim([-0.1, 1])
    ax1.set_ylim([0, len(X) + (n_clusters + 1) * 10])

    # k-means fitten
    clusterer = KMeans(n_clusters=n_clusters,
                       random_state=10)
    cluster_labels = clusterer.fit_predict(X)

    # Durchschnittlicher Silhouette Score
    silhouette_avg = silhouette_score(X, cluster_labels)
    print(f"k={n_clusters} → avg silhouette: "
          f"{silhouette_avg:.4f}")

    # Score pro Datenpunkt
    sample_silhouette_values = silhouette_samples(
        X, cluster_labels)

    y_lower = 10
    for i in range(n_clusters):
        # Scores für Cluster i sortieren
        ith_vals = sample_silhouette_values[
            cluster_labels == i]
        ith_vals.sort()

        size_cluster_i = ith_vals.shape[0]
        y_upper = y_lower + size_cluster_i

        color = cm.nipy_spectral(
            float(i) / n_clusters)
        ax1.fill_betweenx(
            np.arange(y_lower, y_upper),
            0, ith_vals,
            facecolor=color, edgecolor=color,
            alpha=0.7)
        ax1.text(-0.05,
                 y_lower + 0.5 * size_cluster_i,
                 str(i))
        y_lower = y_upper + 10

    ax1.set_title("Silhouette Plot")
    ax1.set_xlabel("Silhouette Coefficient")
    ax1.set_ylabel("Cluster Label")
    ax1.axvline(x=silhouette_avg,
                color="red", linestyle="--")
    ax1.set_yticks([])
    ax1.set_xticks(
        [-0.1, 0, 0.2, 0.4, 0.6, 0.8, 1])

    # ── Rechter Plot: Cluster Scatter ───────────
    colors = cm.nipy_spectral(
        cluster_labels.astype(float) / n_clusters)
    ax2.scatter(X[:, 0], X[:, 1],
                marker='.', s=30, lw=0, alpha=0.7,
                c=colors, edgecolor='k')

    centers = clusterer.cluster_centers_
    ax2.scatter(centers[:, 0], centers[:, 1],
                marker='o', c="white",
                alpha=1, s=200, edgecolor='k')
    for i, c in enumerate(centers):
        ax2.scatter(c[0], c[1],
                    marker=f'${i}$',
                    alpha=1, s=50, edgecolor='k')

    ax2.set_title("Clustered Data")
    ax2.set_xlabel("Feature 1")
    ax2.set_ylabel("Feature 2")

    plt.suptitle(
        f"Silhouette Analysis – n_clusters = "
        f"{n_clusters}",
        fontsize=14, fontweight='bold')
    plt.show()

4.7 Silhouette-Plot lesen & interpretieren

Wie erkennt man auf einen Blick, ob das Clustering gut oder schlecht ist?

Gutes Clustering

  • Alle Cluster-Balken liegen rechts der roten Linie (über dem Durchschnitt)
  • Balken haben ähnliche Breite (vergleichbare Clustergrößen)
  • Keine oder wenige Punkte mit negativem Score
  • Rote Linie liegt weit rechts (hoher Durchschnitt)

Schlechtes Clustering

  • Viele Punkte liegen links der roten Linie (unter dem Durchschnitt)
  • Negative Silhouette-Scores → Punkte im falschen Cluster
  • Stark unterschiedliche Balkenbreiten → unnatürliche Clustergrößen
  • Rote Linie liegt weit links (niedriger Durchschnitt)
Heuristik: Natürliche Cluster haben oft vergleichbare Größen. Starke Unterschiede in den Balkenbreiten → evtl. schlechtes Clustering. Dies ist jedoch keine universelle Regel.
Was tun wir mit diesen Werten? Der Silhouette Score ist mächtig, aber aufwändig. Eine einfachere, schnelle Alternative zur k-Wahl bietet der Elbow Plot auf Basis der Inertia: → direkt nach unten

4.8 SSE & Elbow Plot

Die schnellste Methode zur Wahl des optimalen k – visuell und intuitiv.

SSE / Inertia

  • Sum of Squared Errors (SSE) = Summe der quadratischen Abstände aller Datenpunkte zu ihrem Centroid
  • In sklearn: kmeans.inertia_
  • Sinkt immer mit steigendem k (trivialerweise)
  • Ab einem gewissen k bringt ein weiterer Cluster kaum noch Verbesserung

Elbow Plot / Knee Plot

  • SSE wird für verschiedene k in einem Linienplot aufgetragen
  • Der Graph macht an der optimalen Stelle einen „Knick" (Elbow)
  • Ab diesem Punkt erklären weitere Cluster zu wenig Varianz
  • Auch bekannt als: Knee Plot, Scree Plot

Schritt 1 · Datensatz mit 4 bekannten Clustern

Python 
from sklearn.datasets import make_blobs
import matplotlib.pyplot as plt

# Datensatz: 4 kompakte Cluster (Wahrheit bekannt)
X, y_true = make_blobs(
    n_samples=3000,
    centers=4,
    cluster_std=0.30,
    random_state=0
)

# Rohdaten ansehen
plt.figure()
plt.scatter(X[:, 0], X[:, 1], s=10, alpha=0.5)
plt.title("Rohdaten – 4 Cluster")
plt.show()
Info
  • 3000 Punkte, sehr kompakte Cluster (std=0.30)
  • Wir tun so, als wüssten wir nicht, dass k=4
  • Elbow-Plot soll k=4 bestätigen

Schritt 2 · SSE für k = 2 bis 10 berechnen & Elbow Plot

Python 
from sklearn.cluster import KMeans

# SSE für k = 2 bis 10 sammeln
list_of_inertias = []

for n_clusters in range(2, 11):
    kmeans = KMeans(n_clusters)
    kmeans.fit(X)
    list_of_inertias.append(kmeans.inertia_)

# Elbow Plot
plt.figure()
plt.plot(range(2, 11), list_of_inertias, 'go--')
plt.ylabel('Sum of Squared Distances (SSE)')
plt.xlabel('k (Anzahl Cluster)')
plt.title('Elbow Plot')
plt.xticks(range(2, 11))
plt.show()
Ergebnis
  • 📊 Linienplot mit grünen Punkten
  • Starker Abfall von k=2 → k=3 → k=4
  • Bei k=4 macht die Kurve einen Elbow (Knick)
  • Ab k=5 nur noch geringer Rückgang
  • → Optimales k = 4 ✓ bestätigt
