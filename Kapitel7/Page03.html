ROC Curves in sklearn

Von der Theorie zur Implementierung: roc_curve(), RocCurveDisplay, das mathematische Fundament des Decision Thresholds – und ein Gefühl für ROC-Kurven durch Variation der Streuung.

roc_curve RocCurveDisplay SVC.decision_function Threshold-Integral cluster_std variieren

Lernfokus dieser Seite

Du kannst eine ROC-Kurve mit sklearn berechnen und plotten, verstehst was y_score ist, und weißt, warum Gaußsche Dichten und Integrale hinter dem Threshold stecken.

3.1 sklearn ROC-API – Überblick

sklearn stellt zwei Wege bereit, um ROC-Kurven zu erzeugen.

roc_curve(y_true, y_score)

  • Gibt fpr, tpr und thresholds zurück
  • y_score: Ausgabe der .decision_function() oder .predict_proba() – kein binäres Label!
  • Nur für binäre Klassifikation direkt verwendbar
  • from sklearn.metrics import roc_curve

RocCurveDisplay.from_estimator(model, X, y)

  • Convenience-Funktion: berechnet ROC und plottet direkt
  • Zeigt die AUC im Plot an
  • Braucht das trainierte Modell, X-Testdaten und y-Testlabels
  • from sklearn.metrics import RocCurveDisplay
Wichtig: y_score ist kein Vorhersage-Label (0 oder 1), sondern ein kontinuierlicher Wert – z.B. der Abstand zur Entscheidungsgrenze oder eine Wahrscheinlichkeit. Erst durch Anlegen eines Thresholds auf diesen Wert entsteht die binäre Vorhersage. Das ist der Schlüssel zur ROC-Kurve.

3.2 Erstes Beispiel – SVC mit roc_curve

Synthetischen Datensatz erzeugen, SVC trainieren, ROC-Kurve berechnen und plotten.

Schritt 1 · Daten erzeugen und Modell trainieren

Python 
from sklearn.datasets import make_blobs
from sklearn.model_selection import train_test_split
from sklearn.svm import SVC
from sklearn.metrics import roc_curve, RocCurveDisplay
import matplotlib.pyplot as plt

# Synthetischen Datensatz mit 2 Klassen erzeugen
X, y = make_blobs(n_samples=400, centers=2,
                  cluster_std=.90, random_state=0)
X_train, X_test, y_train, y_test = train_test_split(X, y)

# SVM trainieren (linear)
model = SVC(kernel='linear').fit(X_train, y_train)

# Score-Werte berechnen (kein Label, sondern Abstand)
y_score = model.decision_function(X_test)
Info
  • make_blobs: 400 Punkte, 2 Cluster, cluster_std=0.90
  • decision_function(): liefert den vorzeichenbehafteten Abstand zur Entscheidungsgrenze
  • Positive Werte → Modell tendiert zu Klasse 1
  • Negative Werte → Modell tendiert zu Klasse 0

Schritt 2 · ROC-Kurve berechnen und plotten

Python 
# Methode 1: Manuell berechnen
fpr, tpr, thresholds = roc_curve(y_test, y_score)

print("FPR:", fpr[:5])
print("TPR:", tpr[:5])
print("Thresholds:", thresholds[:5])

# Methode 2: Convenience-Funktion (plottet direkt + zeigt AUC)
RocCurveDisplay.from_estimator(model, X_test, y_test)
Output (vereinfacht) 
FPR: [0.   0.   0.   0.   0.01 ...]
TPR: [0.   0.02 0.04 0.06 0.08 ...]
Thresholds: [3.12 2.87 2.54 2.21 1.97 ...]
  • Jeder Eintrag = ein Threshold-Wert
  • Je niedriger der Threshold, desto liberaler → TPR und FPR steigen
  • RocCurveDisplay plottet Kurve inkl. AUC-Wert

3.3 Ein Gefühl entwickeln – Streuung variieren

Was passiert mit der ROC-Kurve, wenn die Klassen zunehmend überlappen?

Idee: Wir erzeugen denselben Datensatz mehrfach mit steigender cluster_std. Dadurch überlappen die Klassen mehr und mehr – der Klassifikator hat es schwerer, und das sieht man an der ROC-Kurve.
Python 
import numpy as np

for std in np.linspace(0.2, 2, 5):
    X, y = make_blobs(n_samples=400, centers=2,
                      cluster_std=std, random_state=0)
    X_train, X_test, y_train, y_test = train_test_split(X, y)
    model = SVC(kernel='linear').fit(X_train, y_train)
    y_score = model.decision_function(X_test)

    # Datenpunkte plotten
    plt.figure()
    plt.scatter(X_train[:, 0], X_train[:, 1],
                c=y_train, s=40, cmap='viridis', edgecolors='gray')
    plt.title(f"Daten – std={std:.2f}")

    # ROC-Kurve plotten
    plt.figure()
    RocCurveDisplay.from_estimator(model, X_test, y_test)
    plt.title(f"ROC – std={std:.2f}")
Beobachtung
  • std = 0.2: Klar getrennte Cluster → ROC-Kurve nahe am Ideal, AUC ≈ 1.0
  • std = 1.1: Leichte Überlappung → ROC-Kurve bewegt sich Richtung Diagonale
  • std = 2.0: Starke Überlappung → ROC-Kurve fast auf der Diagonalen, AUC ≈ 0.5
  • Die AUC sinkt mit zunehmender Überlappung
Kernaussage: Die ROC-Kurve und die AUC sind ein Maß dafür, wie gut ein Klassifikator die beiden Klassen grundlegend voneinander trennen kann – unabhängig vom gewählten Threshold.

3.4 Mathematisches Fundament: Decision Threshold als Integral

Warum entstehen ROC-Kurven, wenn der Threshold variiert? Die formale Herleitung über Wahrscheinlichkeitsdichten.

Das Modell als Zufallsvariable

Betrachte den Klassifikator als etwas, das für jeden Datenpunkt einen Score \(x\) ausgibt. Datenpunkte der Klasse „positiv" folgen einer Wahrscheinlichkeitsdichte \(f_1(x)\), Punkte der Klasse „negativ" folgen \(f_0(x)\).

Der Threshold T

Ein Datenpunkt wird als „positiv" klassifiziert, wenn \(x \geq T\). Je nach Lage von \(T\) werden unterschiedlich viele Positive und Negative auf der richtigen oder falschen Seite eingeordnet.

Formale Definition von TPR und FPR über Integrale

Rate Integralformel Bedeutung
TPR \[\text{TPR}(T) = \int_T^{\infty} f_1(x)\, dx\] Anteil der Positiven, die über dem Threshold liegen
FPR \[\text{FPR}(T) = \int_T^{\infty} f_0(x)\, dx\] Anteil der Negativen, die fälschlicherweise über dem Threshold liegen
TNR \[\text{TNR}(T) = 1 - \text{FPR}(T)\] Anteil der Negativen unter dem Threshold
FNR \[\text{FNR}(T) = 1 - \text{TPR}(T)\] Anteil der Positiven unter dem Threshold

Wie entsteht die ROC-Kurve?

Indem der Threshold \(T\) von \(-\infty\) bis \(+\infty\) variiert wird, entstehen für jeden \(T\)-Wert ein (FPR, TPR)-Wertepaar. Die Menge aller dieser Wertepaare ergibt die ROC-Kurve.

  • \(T \to +\infty\): Alles unter dem Threshold → TPR = 0, FPR = 0 (Punkt links unten)
  • \(T \to -\infty\): Alles über dem Threshold → TPR = 1, FPR = 1 (Punkt rechts oben)

Gaußsche Spezialfälle

Wenn man annimmt, dass \(f_1(x)\) und \(f_0(x)\) Gaußsche Verteilungen sind, kann man ROC-Kurven analytisch herleiten – aus dem Abstand der Mittelwerte und der gemeinsamen Standardabweichung. In der Praxis werden die Kurven numerisch berechnet.
