Precision-Recall & F1-Score

Wenn True Negatives fehlen – wie bei Object Detection – brauchen wir eine Alternative zur ROC-Kurve. Die Precision-Recall-Kurve und der F1-Score lösen dieses Problem.

Precision (PPV) Recall (TPR) precision_recall_curve F1-Score Object Detection

Lernfokus dieser Seite

Du weißt, wann die Precision-Recall-Kurve der ROC-Kurve vorzuziehen ist, kannst Precision berechnen und den F1-Score als harmonisches Mittel erklären.

6.1 Wann ist die ROC-Kurve ungeeignet?

Die ROC-Kurve setzt voraus, dass True Negatives existieren und zählbar sind. Bei manchen Use Cases ist das nicht der Fall.

Das Problem: Fehlende True Negatives

In der Object Detection zum Beispiel: Das Modell soll erkennen, ob und wo Objekte (z.B. Autos, Fußgänger) im Bild sind.

  • Ein Nicht-Objekt = alles, was kein Auto / Fußgänger ist – das sind potentiell Millionen von Regionen im Bild
  • True Negatives wären astronomisch groß und nicht sinnvoll zählbar
  • Der FPR wird damit zu einem nichtssagenden Wert

Die Lösung: Precision statt FPR

Statt dem FPR (der True Negatives braucht) verwenden wir Precision (auch PPV = Positive Predictive Value).

Precision beantwortet: „Wie viele meiner Erkennungen waren tatsächlich richtig?" – ohne True Negatives im Nenner.

\[\text{Precision} = \text{PPV} = \frac{TP}{TP + FP}\]
Prüfungs-Merksatz: Bei stark unbalancierten Klassen (sehr viele Negative) oder wenn True Negatives nicht sinnvoll definiert sind (Object Detection, Information Retrieval): → Precision-Recall-Kurve statt ROC-Kurve verwenden.

6.2 Precision und Recall im Detail

Precision (PPV)

\[\text{Precision} = \frac{TP}{TP + FP}\]

Frage: Welcher Anteil meiner positiven Vorhersagen war korrekt?

  • Fokus auf die Qualität der Erkennungen
  • Hohe Precision → wenige Fehlalarme
  • Niedrige Precision → viele Fehlalarme unter den Erkennungen

Recall (TPR)

\[\text{Recall} = \text{TPR} = \frac{TP}{P} = \frac{TP}{TP + FN}\]

Frage: Welcher Anteil der echten Positiven wurde erkannt?

  • Fokus auf die Vollständigkeit der Erkennung
  • Hoher Recall → wenige verpasste Positive
  • Niedriger Recall → viele Positive werden übersehen
Maß Formel Nenner enthält Messung
Precision (PPV) TP / (TP + FP) Alle positiven Vorhersagen Präzision der Erkennungen
Recall (TPR) TP / (TP + FN) Alle echten Positiven (P) Vollständigkeit der Erkennung
FPR FP / (FP + TN) Alle echten Negativen (N) Fehlerrate bei Negativen – braucht TN!
!

Precision-Recall Trade-off

Genau wie bei TPR und FPR gibt es einen Trade-off: Erhöht man die Precision (weniger Fehlalarme), sinkt der Recall (mehr Misses). Die Precision-Recall-Kurve zeigt alle Kompromisse. Ein guter Klassifikator bleibt bei hohem Recall auch mit hoher Precision oben rechts in der Kurve.

6.3 Precision-Recall-Kurve in sklearn

Python – Einzelne Kurve 
from sklearn.metrics import precision_recall_curve
import matplotlib.pyplot as plt

# Precisions und Recalls für alle Thresholds
precisions, recalls, thresholds = precision_recall_curve(y_test, y_score)

# Plot
plt.plot(recalls, precisions)
plt.xlabel('Recall')
plt.ylabel('Precision')
plt.title('Precision-Recall-Kurve')
plt.grid(True, alpha=0.3)
plt.xlim([0, 1])
plt.ylim([0, 1])
Interpretation
  • X-Achse: Recall (= TPR) – von 0 bis 1
  • Y-Achse: Precision – von 0 bis 1
  • Bessere Klassifikatoren → Kurve rechts oben
  • Große Fläche unter der Kurve = gute Klassifikation
  • Vergleiche mit ROC: Achsen anders, aber ähnliche Logik
Python – Variation der Streuung 
import numpy as np
from sklearn.datasets import make_blobs
from sklearn.model_selection import train_test_split
from sklearn.svm import SVC

for std in np.linspace(0.2, 2, 5):
    X, y = make_blobs(n_samples=400, centers=2,
                      cluster_std=std, random_state=0)
    X_train, X_test, y_train, y_test = train_test_split(X, y)
    model = SVC(kernel='linear').fit(X_train, y_train)
    y_score = model.decision_function(X_test)

    # Precision-Recall-Kurve berechnen
    precisions, recalls, thresholds = precision_recall_curve(y_test, y_score)

    # Plot
    plt.figure()
    plt.plot(recalls, precisions, label=f'std={std:.2f}')
    plt.xlabel('Recall')
    plt.ylabel('Precision')
    plt.title(f'Precision-Recall-Kurve – std={std:.2f}')
Beobachtung
  • std klein → Kurve hoch rechts oben (hohe Precision bei hohem Recall)
  • std groß → Kurve sinkt (muss bei Recall erhöhen viel Precision opfern)
  • Verhalten analog zur ROC: mehr Überlappung = schlechtere Kurve

6.4 F1-Score – Precision und Recall in einem Wert

Definition

Der F1-Score ist das harmonische Mittel aus Precision und Recall – er fasst beide in einer einzigen Zahl zusammen.

\[F_1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}\]
  • Wertebereich: 0 (schlechtester) bis 1 (bester Wert)
  • Harmonisches (nicht arithmetisches) Mittel → bestraft extreme Werte
  • Nur hoch, wenn beide Maße hoch sind

F1 in sklearn

Binäre Klassifikation
from sklearn.metrics import f1_score

f1 = f1_score(y_test, y_predicted)
print(f"F1-Score: {f1:.3f}")
Multi-Class Klassifikation
# average='macro': ungewichteter Durchschnitt
f1_macro = f1_score(y_test, y_pred, average='macro')

# average='weighted': gewichtet nach Klassengröße
f1_weighted = f1_score(y_test, y_pred, average='weighted')
Warum harmonisches Mittel? Das arithmetische Mittel (P+R)/2 würde auch dann hoch sein, wenn eine der Größen perfekt und die andere miserabel ist. Das harmonische Mittel 2·P·R/(P+R) ist hingegen klein, sobald eines der beiden Maße niedrig ist. Beispiel: Precision=1.0, Recall=0.01 → arithm. Mittel = 0.505, F1 = 0.02.

Alle Evaluierungsmaße im Vergleich

Maß Formel Braucht TN? sklearn-Funktion Verwendung
TPR / Recall TP / P Nein recall_score() Wie viele Positive erkannt?
FPR FP / N Ja (für N) via roc_curve() Wie viele Fehlalarme bei Negativen?
Precision TP / (TP+FP) Nein precision_score() Wie präzise sind meine Erkennungen?
F1-Score 2·P·R/(P+R) Nein f1_score() Balance Precision+Recall
AUC Fläche unter ROC Ja (für FPR) roc_auc_score() Gesamtgüte eines Klassifikators
