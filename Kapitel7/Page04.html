Decision Threshold

Mit dem Decision Threshold kannst du ein Modell an den spezifischen Use Case anpassen – ohne es neu zu trainieren. Krankheitserkennung vs. Predictive Maintenance, und die vollständige adjusted_predict()-Funktion.

adjusted_predict() TPR-Ziel FPR-Ziel Krankheitserkennung Predictive Maintenance

Lernfokus dieser Seite

Du kannst entscheiden, in welche Richtung du den Threshold für einen konkreten Use Case verschieben sollst, und die adjusted_predict()-Funktion aus dem Gedächtnis implementieren.

4.1 Was ist der Decision Threshold?

Das Kriterium, das bestimmt, ab wann ein kontinuierlicher Score als „positiv" gilt.

Definition

Jedes ML-Modell gibt intern einen kontinuierlichen Score aus (Abstand zur Entscheidungsgrenze, Wahrscheinlichkeit, etc.). Erst durch das Anlegen eines Schwellwerts T (Decision Threshold) wird dieser Score in eine binäre Vorhersage (0 oder 1) umgewandelt.

\[\hat{y} = \begin{cases} 1 & \text{wenn } \text{score}(x) \geq T \\ 0 & \text{sonst} \end{cases}\]

Standard-Threshold

Standardmäßig verwendet sklearn einen Threshold von 0 (für decision_function()) bzw. 0.5 (für predict_proba()). Das ist jedoch nicht immer optimal – der Use Case entscheidet.

Mit roc_curve() kannst du alle möglichen Threshold-Werte und die dazugehörigen (FPR, TPR)-Paare berechnen.

4.2 Use Cases – Wohin verschiebt man den Threshold?

Der optimale Threshold hängt davon ab, welcher Fehler teurer ist: ein verpasster Positiver (FN) oder ein falscher Alarm (FP).

🏥 Krankheitserkennung

Use Case: Anhand von Bildgebungsdaten wird klassifiziert, ob ein Patient eine bestimmte Krankheit hat.

  • Was ist schlimmer? Ein kranker Patient wird nicht erkannt (FN) – das kann lebensbedrohlich sein.
  • Ein gesunder Patient wird fälschlich als krank eingestuft (FP) – führt zu weiteren Tests, aber ist beherrschbar.
→ Threshold senken (liberaler) → TPR maximieren (hohe Sensitivity)
Akzeptiert höheren FPR als Preis für weniger verpasste Fälle.

⚙️ Predictive Maintenance (Kugellager)

Use Case: Anhand von Vibrationsdaten wird vorhergesagt, ob ein Lager beschädigt ist. Jeder Alarm wird dem Operator gemeldet.

  • Was ist schlimmer? Zu viele Fehlalarme (FP) – der Operator ignoriert das System irgendwann.
  • Ein beschädigtes Lager wird übersehen (FN) – führt zu Maschinenausfall, aber seltener als Fehlalarme.
→ Threshold erhöhen (konservativer) → FPR minimieren (hohe Specificity)
Akzeptiert niedrigere TPR für weniger Fehlalarme.
Situation Teuerster Fehler Threshold-Richtung Ziel
Krankheit nicht erkennen ist fatal FN (Miss) Threshold senken TPR maximieren
Fehlalarme sind teuer / nervig FP (Falscher Alarm) Threshold erhöhen FPR minimieren
Ausgewogen TP + FP gleichgewichtig Youden-Index TPR − FPR maximieren

4.3 Die adjusted_predict()-Funktion

Implementierung einer Funktion, die Vorhersagen mit einem selbstgewählten TPR- oder FPR-Ziel zurückgibt – und den gewählten Punkt auf der ROC-Kurve markiert.

Aufgabe (Cheap & Expensive Bearings): Erstelle eine Funktion adjusted_predict(model, Xtest, ytest, tpfp_value, tp=True), die:
1. Die ROC-Kurve berechnet
2. Den Threshold findet, der am nächsten am Ziel-TPR oder Ziel-FPR liegt
3. Den Punkt auf der ROC-Kurve markiert
4. Die angepassten binären Vorhersagen zurückgibt
→ Vollständig gelöst auf der Beispiel-Seite
Python – Vollständige Lösung 
from sklearn.metrics import roc_curve
import matplotlib.pyplot as plt
import numpy as np

def adjusted_predict(model, Xtest, ytest, tpfp_value, tp=True):
    """
    Gibt angepasste Vorhersagen zurück, die einen gewünschten
    TPR- oder FPR-Wert auf der ROC-Kurve treffen.

    Parameter:
        model       : trainiertes sklearn-Modell
        Xtest       : Test-Features
        ytest       : wahre Testlabels
        tpfp_value  : Zielwert für TPR (wenn tp=True) oder FPR (wenn tp=False)
        tp          : True → Ziel ist TPR, False → Ziel ist FPR

    Rückgabe:
        ypredicted_new : Angepasste binäre Vorhersagen
    """
    # Schritt 1: Score-Werte berechnen
    yscore = model.decision_function(Xtest)

    # Schritt 2: ROC-Kurve berechnen
    fpr, tpr, thresholds = roc_curve(ytest, yscore)

    # Schritt 3: Wähle TPR oder FPR als Ziel
    if tp:
        tpfpr = tpr   # Suche Threshold der nächsten am Ziel-TPR liegt
    else:
        tpfpr = fpr   # Suche Threshold der nächsten am Ziel-FPR liegt

    # Schritt 4: Index des nächsten Werts finden
    index_of_tpfp_value = np.argmin(np.abs(tpfpr - tpfp_value))
    threshold = thresholds[index_of_tpfp_value]

    # Schritt 5: ROC-Kurve plotten mit markiertem Punkt
    plt.plot([0, 1], [0, 1], linestyle='--', label='No Skill')
    plt.plot(fpr, tpr, marker='.', label='Model')
    plt.scatter(fpr[index_of_tpfp_value],
                tpr[index_of_tpfp_value],
                marker='o', color='black', s=100, zorder=5,
                label=f'Gewählter Punkt')
    plt.xlabel('False Positive Rate')
    plt.ylabel('True Positive Rate')
    plt.title('ROC-Kurve mit angepasstem Threshold')
    plt.legend()

    # Schritt 6: Angepasste Vorhersagen zurückgeben
    return yscore > threshold


# Anwendung: FPR-Ziel von ~2.3%
ypredicted_new = adjusted_predict(model, Xtest, ytest,
                                  tpfp_value=0.023, tp=False)

# Anwendung: TPR-Ziel von ~97%
ypredicted_new = adjusted_predict(model, Xtest, ytest,
                                  tpfp_value=0.97, tp=True)
Erklärung: Schritt für Schritt
  • decision_function() → kontinuierlicher Score (kein Label)
  • roc_curve() → Array aller (fpr, tpr, threshold)-Tripel
  • np.argmin(np.abs(...)) → Index des nächsten Werts zum Ziel
  • thresholds[index] → der konkrete Schwellwert
  • yscore > threshold → angepasste binäre Vorhersagen (True/False)
  • Der schwarze Punkt auf der ROC-Kurve markiert die neue Position

tp=False → FPR-Ziel

adjusted_predict(model, Xtest, ytest, 0.023, tp=False)
→ Sucht den Threshold, bei dem FPR ≈ 0.023 (2.3 % Fehlalarme bei negativen Lagern). Teuer Lager werden selten fälschlich ausgetauscht.

tp=True → TPR-Ziel

adjusted_predict(model, Xtest, ytest, 0.97, tp=True)
→ Sucht den Threshold, bei dem TPR ≈ 0.97 (97 % der beschädigten Lager erkannt). Günstige Lager werden zuverlässig zum richtigen Zeitpunkt ausgetauscht.

4.4 Was darf man anpassen – und was nicht?

✅ Was sich verändert

  • Der Threshold-Wert, ab dem der Score als „positiv" gilt
  • Die TPR und FPR in der Vorhersage
  • Die Confusion Matrix der Vorhersagen
  • Die Anzahl der False Positives und False Negatives

❌ Was sich NICHT verändert

  • Die ROC-Kurve selbst – die beschreibt die grundlegende Modellgüte
  • Die AUC – invariant gegenüber Threshold-Wahl
  • Das trainierte Modell und seine Parameter
  • Die Möglichkeiten des Modells – nur Kompromiss-Punkt auf der Kurve
Prüfungs-Falle: Das Anpassen des Decision Thresholds verbessert nicht die AUC! Es verschiebt nur den Kompromiss zwischen TPR und FPR. Wenn die AUC schlecht ist, hilft auch kein Threshold-Tuning – dann muss das Modell selbst verbessert werden.
