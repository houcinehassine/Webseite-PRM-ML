Kapitel 7 · Praxisbeispiel

Cheap & Expensive Bearings

Ein Maschinenbauunternehmen will teure und günstige Kugellager unterschiedlich behandeln. Baue eine Funktion, die ein trainiertes Modell auf die Use-Case-Anforderungen anpasst.

adjusted_predict() fpr ≈ 0.023 tpr ≈ 0.97 bearing_model.pkl ROC-Marker

Aufgabe

Baue eine Funktion, die ein trainiertes Modell und einen Testdatensatz aufnimmt, die angepassten Vorhersagen zurückgibt und den Punkt in der ROC-Kurve markiert. Teuer = FPR ≤ 0.023, Günstig = TPR ≥ 0.97.

Aufgabenstellung

Kontext: In einem Maschinenbauunternehmen sind überall Kugellager verbaut – teure große und billige kleine. Ein ML-Modell (bearing_model.pkl) klassifiziert sie als „gut" oder „beschädigt".

Dein Vorgesetzter will:
  • Teure Lager: Sollen nicht zu häufig fälschlicherweise ausgetauscht werden. → Ziel: fpr ≈ 0.023
  • Günstige Lager: Sollen zum richtigen Zeitpunkt ausgetauscht werden. → Ziel: tpr ≈ 0.97

Funktion-Signatur

def adjusted_predict(model, Xtest, ytest,
                     tpfp_value, tp=True):
    """
    model       : trainiertes sklearn-Modell
    Xtest       : Test-Features
    ytest       : wahre Testlabels
    tpfp_value  : Zielwert (TPR oder FPR)
    tp          : True → TPR-Ziel, False → FPR-Ziel
    Rückgabe    : ypredicted_new (bool-Array)
    """

Was brauchen wir zusätzlich?

Überlege: Welche Eingabeparameter fehlen noch?

  • Um teuer vs. günstig zu unterscheiden, müssen wir wissen, ob TPR oder FPR das Ziel ist
  • → Der Parameter tp=True (TPR-Ziel) bzw. tp=False (FPR-Ziel)
  • Damit kann dieselbe Funktion für beide Lagerarten genutzt werden

Schritt 1 – Daten generieren und Modell trainieren

Python – Daten + Modell 
import numpy as np
import matplotlib.pyplot as plt
import joblib
from sklearn.datasets import make_blobs
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.svm import SVC
from sklearn.metrics import roc_curve, RocCurveDisplay

# Synthetische Kugellager-Daten erzeugen
X, y = make_blobs(n_samples=1000, centers=2,
                  cluster_std=1.5, random_state=0)
Xtrain, Xtest, ytrain, ytest = train_test_split(X, y)

# Daten visualisieren
plt.figure()
plt.scatter(Xtrain[:, 0], Xtrain[:, 1],
            c=ytrain, s=40, cmap='viridis', edgecolors='gray')
plt.title("Kugellager-Trainingsdaten")
plt.xlabel("Feature 1 (z.B. Vibration)")
plt.ylabel("Feature 2 (z.B. Temperatur)")

# SVC trainieren
model = SVC().fit(Xtrain, ytrain)
print("CV-Score:", cross_val_score(model, Xtrain, ytrain).mean())

# Modell speichern (für später)
joblib.dump(model, 'bearing_model.pkl')
np.save('bearing_xtest.npy', Xtest)
np.save('bearing_ytest.npy', ytest)
Output 
CV-Score: 0.87
  • cluster_std=1.5: leichte Überlappung → realistisch
  • CV-Score ≈ 87 % – ausbaufähig, aber Basis für Threshold-Tuning
  • joblib.dump(): Modell für späteres Laden speichern

Schritt 2 – Lösung: adjusted_predict()

Python – Vollständige Lösung 
# Modell und Testdaten laden (gespeichertes Modell)
model = joblib.load('bearing_model.pkl')
Xtest = np.load('bearing_xtest.npy')
ytest = np.load('bearing_ytest.npy')

def adjusted_predict(model, Xtest, ytest, tpfp_value, tp=True):
    """
    Gibt angepasste Vorhersagen zurück für einen gewünschten
    TPR- oder FPR-Wert auf der ROC-Kurve.
    """
    # Schritt 1: Kontinuierliche Scores berechnen
    yscore = model.decision_function(Xtest)

    # Schritt 2: Alle (FPR, TPR, Threshold)-Tripel berechnen
    fpr, tpr, thresholds = roc_curve(ytest, yscore)

    # Schritt 3: Wähle Ziel-Metrik (TPR oder FPR)
    if tp:
        tpfpr = tpr    # Suche nach TPR-Zielwert
    else:
        tpfpr = fpr    # Suche nach FPR-Zielwert

    # Schritt 4: Nächsten Index zum Zielwert finden
    index_of_tpfp_value = np.argmin(np.abs(tpfpr - tpfp_value))
    threshold = thresholds[index_of_tpfp_value]

    print(f"Gewählter Threshold: {threshold:.4f}")
    print(f"Resultierender TPR: {tpr[index_of_tpfp_value]:.4f}")
    print(f"Resultierender FPR: {fpr[index_of_tpfp_value]:.4f}")

    # Schritt 5: ROC-Kurve mit markiertem Punkt plotten
    plt.figure(figsize=(7, 6))
    plt.plot([0, 1], [0, 1], linestyle='--',
             color='gray', label='No Skill')
    plt.plot(fpr, tpr, marker='.', color='steelblue',
             label='Modell-ROC')
    plt.scatter(fpr[index_of_tpfp_value],
                tpr[index_of_tpfp_value],
                marker='o', color='black', s=120, zorder=5,
                label=f'Gewählter Punkt')
    plt.xlabel('False Positive Rate')
    plt.ylabel('True Positive Rate')
    plt.title('ROC-Kurve mit angepasstem Threshold')
    plt.legend()
    plt.grid(True, alpha=0.3)

    # Schritt 6: Angepasste Vorhersagen zurückgeben
    return yscore > threshold
Schritt-für-Schritt-Erklärung
  • S1: decision_function() → kontinuierliche Scores (nicht 0/1)
  • S2: roc_curve() → alle Tripel (fpr, tpr, threshold)
  • S3: if tp → wähle Ziel-Array (tpr oder fpr)
  • S4: np.argmin(np.abs(...)) → nächster Index
  • S5: Schwarzer Punkt auf ROC-Kurve = neue Position
  • S6: yscore > threshold → bool-Array

Schritt 3 – Anwendung für beide Lagerarten

Python – Anwendung 
# Teure Lager: FPR ≤ 0.023 (max. 2.3% Fehlalarme bei Negativen)
# tp=False bedeutet: Wir suchen nach einem FPR-Zielwert
print("=== TEURE LAGER (FPR-Ziel: 0.023) ===")
ypredicted_fpr = adjusted_predict(
    model, Xtest, ytest,
    tpfp_value=0.023,
    tp=False
)

print()

# Günstige Lager: TPR ≥ 0.97 (mind. 97% der Defekten erkannt)
# tp=True bedeutet: Wir suchen nach einem TPR-Zielwert
print("=== GÜNSTIGE LAGER (TPR-Ziel: 0.97) ===")
ypredicted_tpr = adjusted_predict(
    model, Xtest, ytest,
    tpfp_value=0.97,
    tp=True
)
Erwarteter Output 
=== TEURE LAGER (FPR-Ziel: 0.023) ===
Gewählter Threshold: 1.2847
Resultierender TPR: 0.5714
Resultierender FPR: 0.0229

=== GÜNSTIGE LAGER (TPR-Ziel: 0.97) ===
Gewählter Threshold: -0.8432
Resultierender TPR: 0.9714
Resultierender FPR: 0.2457
  • Teure Lager: Hoher Threshold → hohe Präzision, aber weniger Recall
  • Günstige Lager: Niedriger Threshold → hoher Recall, mehr FP

Teure Lager – Analyse

  • FPR ≈ 2.3 %: Von 100 guten Lagern werden ≤ 2–3 fälschlich als defekt markiert
  • Der hohe Threshold filtert nur wirklich eindeutige Defekte heraus
  • Konsequenz: Manche defekten Lager werden übersehen (niedrigere TPR)
  • Akzeptabler Preis für Schutz der teuren Lager

Günstige Lager – Analyse

  • TPR ≈ 97 %: Von 100 defekten Lagern werden ≥ 97 erkannt
  • Der niedrige Threshold lässt viele Vorhersagen als „defekt" durch
  • Konsequenz: Mehr Fehlalarme (FPR steigt auf ~24 %)
  • Günstige Lager vorzeitig tauschen ist günstiger als Ausfall
Fazit: Dasselbe Modell, aber zwei verschiedene Threshold-Einstellungen für zwei verschiedene Use Cases. Das Modell selbst wird nicht neu trainiert – nur der Entscheidungspunkt auf der ROC-Kurve wird verschoben. Die AUC bleibt unverändert.
