📊 Die Confusion Matrix auf einen Blick

Vier Felder, zwei Klassen – die komplette Übersicht aller Ergebnistypen.

Zeile = Wahrer Zustand  |  Spalte = Vorhersage des Modells

Predicted: Positiv
Predicted: Negativ
Actual: Positiv
TP True Positive Richtig erkannt
FN False Negative Verpasst (Typ-II-Fehler)
Actual: Negativ
FP False Positive Falscher Alarm (Typ-I)
TN True Negative Richtig abgelehnt
TPR / Recall / Sensitivity
\( \text{TPR} = \dfrac{TP}{P} = \dfrac{TP}{TP + FN} = 1 - \text{FNR} \)
FPR = 1 − Specificity
\( \text{FPR} = \dfrac{FP}{N} = \dfrac{FP}{FP + TN} = 1 - \text{TNR} \)
Precision / PPV
\( \text{PPV} = \dfrac{TP}{TP + FP} \)
Anteil richtig positiver Calls
Accuracy
\( \text{Acc} = \dfrac{TP + TN}{P + N} \)
Täuscht bei Klassenungleichgewicht!

📈 ROC-Kurve & Decision Threshold

TPR(T)
\( \text{TPR}(T) = \displaystyle\int_T^{+\infty} f_1(x)\,dx \)
\(f_1\) = Dichtefunktion der positiven Klasse
FPR(T)
\( \text{FPR}(T) = \displaystyle\int_T^{+\infty} f_0(x)\,dx \)
\(f_0\) = Dichtefunktion der negativen Klasse
ROC Kurve
Parametrische Kurve: \(T\) von \(-\infty\) bis \(+\infty\) durchlaufen → Punkte \((\text{FPR}(T),\;\text{TPR}(T))\) bilden die Kurve
Prediction
\( \hat y = 1 \iff \text{score}(x) \geq T \)

📏 AUC & Youden-Index

AUC
\( \text{AUC} = \displaystyle\int_0^1 \text{TPR}(\text{FPR}^{-1}(u))\,du \)
Fläche unter ROC-Kurve · 0.5 = Zufall · 1.0 = Perfekt
Youden J
\( J = \text{TPR} - \text{FPR} \)
\( T^* = \arg\max_T\;J(T) \)
Optimale Balance zwischen TP- und FP-Rate
Idealer Klass.
AUC = 1.0  ·  Zufallsklassifikator: AUC = 0.5
Threshold-inv.
AUC ist unabhängig vom gewählten Threshold → robusteres Maß

🎯 Precision-Recall & F1-Score

Precision
\( P = \text{PPV} = \dfrac{TP}{TP + FP} \)
Welcher Anteil meiner Entdeckungen war richtig?
Recall
\( R = \text{TPR} = \dfrac{TP}{TP + FN} \)
Wie viele tatsächlich positive Fälle fand ich?
F1-Score
\( F_1 = 2 \cdot \dfrac{P \cdot R}{P + R} = \dfrac{2\,TP}{2\,TP + FP + FN} \)
Harmonisches Mittel · Wertebereich: 0 (schlecht) – 1 (perfekt)
Wann PR?
Wenn True Negatives nicht verfügbar / bedeutungslos (z.B. Object Detection, stark unbalancierte Daten)

🔢 Multi-Class ROC (OvR)

Mikro-Avg
Alle TP, FP, FN gemeinsam aggregieren: roc_curve(y_test.ravel(), y_score.ravel())
Bevorzugt häufige Klassen
Makro-Avg
Einzelne ROC-Kurven interpolieren & mitteln: np.interp(all_fpr, fpr[i], tpr[i])
Alle Klassen gleichgewichtig
OvO vs OvR
OvO: \(\binom{k}{2}\) Kurven · OvR: \(k\) Kurven (eine pro Klasse vs. alle anderen)
Binarisierung
label_binarize(y, classes=[0,1,2]) → One-Hot-Matrix, dann OneVsRestClassifier

adjusted_predict() – Vollständiger Code

def adjusted_predict(model, Xtest, ytest, tpfp_value, tp=True):
    """Passe den Decision Threshold an gewünschte TPR oder FPR an."""
    yscore = model.decision_function(Xtest)
    fpr, tpr, thresholds = roc_curve(ytest, yscore)

    tpfpr = tpr if tp else fpr                # tp=True → TPR-Ziel, tp=False → FPR-Ziel
    index = np.argmin(np.abs(tpfpr - tpfp_value))
    threshold = thresholds[index]

    # ROC-Kurve mit markiertem Punkt plotten
    plt.plot(fpr, tpr, marker='.', label='Model')
    plt.scatter(fpr[index], tpr[index], c='black', label='searched')
    plt.xlabel('FPR'); plt.ylabel('TPR'); plt.legend()

    return yscore > threshold                 # Boolean-Array als neue Predictions

# Verwendung:
ypred_tp97 = adjusted_predict(model, Xtest, ytest, 0.97, tp=True)   # TPR ≥ 97 %
ypred_fp023 = adjusted_predict(model, Xtest, ytest, 0.023, tp=False)# FPR ≤ 2.3 %

Youden-Index – Optimalen Threshold finden

fpr, tpr, thresholds = roc_curve(y_test, y_score)

J = tpr - fpr                                # Youden J für jeden Threshold
best_idx = np.argmax(J)                      # Index mit maximalem J
best_threshold = thresholds[best_idx]
best_tpr = tpr[best_idx]
best_fpr = fpr[best_idx]

plt.scatter(best_fpr, best_tpr, c='red', label=f'J={J[best_idx]:.2f}')
# best_threshold kann direkt in adjusted_predict übergeben werden

⚠️ Typische Prüfungsfallen

Diese Fehler passieren häufig – lerne sie auswendig.

⚠️
Accuracy allein reicht nicht – die Accuracy-Falle Wenn du bei 200 Durchgängen (100 Signal, 100 Rauschen) immer „Ja" sagst: 100 % TP, aber 100 % FP. Accuracy = 50 %, Modell wertlos. Lösung: Immer TPR und FPR gemeinsam betrachten.
⚠️
roc_curve() benötigt Scores – nicht Klassen-Labels roc_curve(y_test, model.predict(X_test)) ist falsch. Du brauchst model.decision_function(X_test) oder model.predict_proba(X_test)[:,1]. Nur mit kontinuierlichen Scores kann der Threshold variiert werden.
⚠️
roc_curve() ist auf binäre Klassifikation beschränkt Die Funktion roc_curve() akzeptiert nur binäre Labels. Für Multi-Class: erst label_binarize() + OneVsRestClassifier, dann loop. roc_auc_score() dagegen unterstützt Multi-Class direkt mit multi_class='ovr'.
⚠️
Liberal vs. konservativ verändert den Klassifikator nicht – nur das Kriterium Die ROC-Kurve bleibt dieselbe, egal wie konservativ oder liberal das Kriterium ist. Nur der Punkt auf der Kurve verschiebt sich. Die zugrundeliegende Unterscheidungsfähigkeit (AUC) ändert sich nicht.
⚠️
normalize='true' in confusion_matrix ≠ True Der Parameter heißt normalize und nimmt den String 'true' (nicht den Boolean True). Optionen: 'true' (Zeilen-Normalisierung = TPR pro Klasse), 'pred' (Spalten), 'all' (Gesamt), None (absolut).
⚠️
Threshold senken → mehr TP, aber auch mehr FP (nicht weniger!) Viele denken: „Niedrigerer Threshold = weniger Fehler." Falsch. Niedrigerer Threshold = liberaler = mehr Positive Calls insgesamt → TP ↑ UND FP ↑. Für Krankheitserkennung gut (kein Patient verpasst), für Predictive Maintenance schlecht.
⚠️
PR-Kurve rechts oben ist gut – ROC-Kurve auch, aber anders ROC: links oben ist ideal (FPR=0, TPR=1). PR-Kurve: rechts oben ist ideal (Recall=1, Precision=1). Beide haben AUC-nahe Gütermaße, aber die Interpretation der Ecken unterscheidet sich.
⚠️
Diagonale der ROC = Zufallsklassifikator (AUC = 0.5) Ein Modell, das auf der Diagonale liegt, trifft zufällige Entscheidungen – also TPR = FPR für jeden Threshold. Ein Klassifikator unterhalb der Diagonale wäre sogar schlechter als Zufall (Labels umkehren würde ihn besser machen).

💡 Merksätze zum Einprägen

Kernsätze aus der Vorlesung – direkt prüfungsrelevant.

Merksatz 1 – Signalentdeckungstheorie

„Erst durch die gleichzeitige Betrachtung von TPR und FPR lässt sich die Güte eines Klassifikators wirklich beurteilen." Der Arzt, der nur TP misst, macht einen grundlegenden wissenschaftlichen Fehler.

Merksatz 2 – ROC Kurve

„Die ROC-Kurve beschreibt den Klassifikator – nicht die Entscheidung." Alle Punkte auf der Kurve gehören zum selben Modell, nur mit anderem Threshold. Die Kurve selbst (= AUC) ist das eigentliche Gütekriterium.

Merksatz 3 – Idealer Klassifikator

„Der ideale Klassifikator steigt bei FPR = 0 sofort auf TPR = 1 und bleibt konstant." AUC = 1.0 bedeutet: bei keinem Threshold werden False Positives produziert, solange Positives noch übrig sind.

Merksatz 4 – Decision Threshold

„X ≥ T → Klasse positiv." Der Threshold T verschiebt den Punkt auf der ROC-Kurve. Höheres T = konservativer = weniger FP, aber auch weniger TP.

Merksatz 5 – AUC ist Threshold-invariant

„Die AUC vergleicht Klassifikatoren unabhängig vom gewählten Threshold." Sie integriert über alle möglichen Thresholds – dadurch ist sie ein robusteres Gütekriterium als ein einzelner Accuracy-Wert.

Merksatz 6 – Youden-Index

„J = TPR – FPR maximieren gibt den Punkt auf der ROC-Kurve, der am weitesten von der Diagonale entfernt ist." Pragmatisch und einfach – wenn kein domänenspezifisches Threshold-Ziel bekannt ist.

Merksatz 7 – PR-Kurve

„Nutze die Precision-Recall-Kurve, wenn True Negatives nicht verfügbar oder bedeutungslos sind." Klassisches Beispiel: Object Detection – das Modell findet Objekte in Bildern; „kein Objekt gefunden" ist kein TN.

Merksatz 8 – Sensitivity-Specificity Trade-off

„TPR = Sensitivity, TNR = Specificity – man kann nicht beides gleichzeitig maximieren." Jede Verschiebung des Thresholds erhöht das eine und senkt das andere. Der ROC-Kurve liegt genau dieser Trade-off zugrunde.
