Lernfokus dieser Seite

Du kannst eine Confusion Matrix berechnen, visualisieren und interpretieren – und du weißt, wie AUC berechnet wird und wie der Youden-Index den optimalen Threshold bestimmt.

5.1 Confusion Matrix in sklearn

Die confusion_matrix()-Funktion und ihre Normierungsoption.

confusion_matrix(y_true, y_pred)

  • Gibt eine 2D-Matrix der Form [[TN, FP], [FN, TP]] zurück
  • Für Multi-Class: k×k-Matrix, Zeilen = tatsächliche Klasse, Spalten = vorhergesagte Klasse
  • Parameter normalize='true': normiert die Einträge auf [0, 1] (relativ zur tatsächlichen Klasse)
  • normalize='pred': relativ zur vorhergesagten Klasse
  • normalize='all': relativ zur Gesamtzahl

Heatmap-Visualisierung

Am einfachsten zu lesen als Heatmap mit seaborn:

  • Diagonale: Korrekte Vorhersagen (hell = gut)
  • Außerhalb der Diagonale: Fehler
  • Zeilenbeschriftung = wahre Klasse (yticklabels)
  • Spaltenbeschriftung = vorhergesagte Klasse (xticklabels)
  • Empfohlen: cmap='viridis' oder cmap='Blues'
Python – Confusion Matrix + Heatmap 
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.metrics import confusion_matrix

# Beispiel: Spotify-Klassifikation mit SVM
ytest = np.load('../res/data/spotify_svm_ytest.npy',
                allow_pickle=True)
ypredicted = np.load('../res/data/spotify_svm_ypredicted.npy',
                     allow_pickle=True)

classes = ['baroque', 'classical', 'hip hop', 'pop',
           'progressive bluegrass', 'r&b', 'rap',
           'rock', 'serialism', 'tropical house']

# Normierte Confusion Matrix berechnen
cm = confusion_matrix(ytest, ypredicted, normalize='true')

# Als Heatmap visualisieren
plt.figure(figsize=(8, 6))
sns.heatmap(cm,
            annot=True,           # Werte einblenden
            xticklabels=classes,  # Vorhergesagte Klasse
            yticklabels=classes,  # Wahre Klasse
            cmap='viridis')
plt.ylabel('True Class')
plt.xlabel('Predicted Class')
plt.title('Normierte Confusion Matrix – Spotify SVM')
plt.tight_layout()
Wie liest man die Matrix?
  • Zeile i, Spalte j: Klasse i wurde als Klasse j vorhergesagt
  • Diagonale = 1.0: 100 % korrekt klassifiziert
  • Verwechslungen erkennbar: hip hop oft als rap erkannt
  • baroque und serialism leicht zu verwechseln mit classical
  • normalize='true': Werte pro Zeile summieren sich zu 1
Mehrere Modelle vergleichen: Dieselbe Heatmap lässt sich für verschiedene Modelle erzeugen: bearing (10 Klassen), face recognition, Spotify-SVM, Spotify-RandomForest. So sieht man sofort, welche Klassen welches Modell verwechselt.

5.2 Area Under the Curve (AUC)

Die AUC fasst die Güte einer ROC-Kurve in einem einzigen Skalar zusammen.

Definition

Die AUC ist die Fläche unter der ROC-Kurve – ein Maß für die Gesamtgüte des Klassifikators über alle Threshold-Einstellungen.

\[\text{AUC} = \int_0^1 \text{TPR}(\text{FPR})\, d(\text{FPR})\]
  • AUC = 1.0: Idealer Klassifikator
  • AUC = 0.5: Zufallsklassifikator (Diagonale)
  • AUC < 0.5: Schlechter als Zufall (Modell vertauscht Klassen)

AUC in sklearn

Methode 1: roc_auc_score
from sklearn.metrics import roc_auc_score

# y_score = decision_function() oder predict_proba()
auc = roc_auc_score(y_true, y_score)
print(f"AUC: {auc:.3f}")
Methode 2: direkt im Plot
# RocCurveDisplay zeigt AUC direkt im Plot
RocCurveDisplay.from_estimator(model, X_test, y_test)
Threshold-Invarianz: Die AUC ist unabhängig vom gewählten Threshold. Sie bewertet das Modell über alle möglichen Threshold-Werte. Das macht sie zum wichtigsten einzelnen Gütekriterium für Klassifikatoren.

5.3 Optimaler Threshold – Youden-Index

Wie findet man den Threshold, der TPR und FPR optimal balanciert?

Definition

Der Youden-Index J maximiert die Differenz zwischen TPR und FPR. Er gibt den Punkt auf der ROC-Kurve an, an dem die Balance zwischen Sensitivity und Specificity optimal ist.

\[J = \text{TPR} - \text{FPR}\]

Der optimale Threshold liegt dort, wo J maximal ist. Das entspricht dem Punkt auf der ROC-Kurve mit dem größten vertikalen Abstand zur Diagonalen.

Geometrische Bedeutung

Zeichne auf der ROC-Kurve eine Parallele zur Diagonalen nach oben – der Schnittpunkt mit der höchsten vertikalen Verschiebung ist der Youden-Punkt.

  • Pragmatisch und schnell berechenbar
  • Sinnvoll bei gleichwertigen Fehlern (FP ≈ FN)
  • Für ungleiche Fehlerkosten: Use-Case-spezifischer Threshold (Seite 4)
Python – Youden-Index 
from sklearn.metrics import roc_curve, RocCurveDisplay
import numpy as np
import matplotlib.pyplot as plt

# ROC-Kurve berechnen
fpr, tpr, thresholds = roc_curve(y_test, y_score)

# ROC plotten (zeigt AUC)
RocCurveDisplay.from_estimator(model, X_test, y_test)

# Youden-Index berechnen
J = tpr - fpr                        # J für jeden Threshold
best_threshold_index = np.argmax(J)  # Index des maximalen J

# Optimalen Punkt extrahieren
best_tpr = tpr[best_threshold_index]
best_fpr = fpr[best_threshold_index]
best_threshold = thresholds[best_threshold_index]

print(f"Optimaler Threshold: {best_threshold:.3f}")
print(f"Optimale TPR: {best_tpr:.3f}")
print(f"Optimale FPR: {best_fpr:.3f}")

# Punkt in der ROC-Kurve markieren
plt.scatter(best_fpr, best_tpr,
            c='red', s=100, zorder=5,
            label=f'Youden-Punkt (J={J[best_threshold_index]:.2f})')
plt.legend()

# Kann in adjusted_predict() eingespeist werden:
# ypredicted_new = adjusted_predict(model, Xtest, best_tpr, tp=True)
Erklärung
  • J = tpr - fpr: Vektor der Youden-Werte für alle Thresholds
  • np.argmax(J): Index des optimalen Thresholds
  • Roter Punkt: Maximum des vertikalen Abstands zur Diagonalen
  • Der Threshold kann in adjusted_predict() eingegeben werden

5.4 Praxisbeispiel: „What confuses our models?"

Confusion Matrices für vier verschiedene Modelle – welche Klassen werden wie gut unterschieden?

1

Kugellager – 10 Klassen

Klassen: B007_3, B014_3, B021_3, IR007_3, IR014_3, IR021_3, normal, OR007@6_3, OR014@6_3, OR021@6_3
Erwartete Verwechslungen: unterschiedliche Schadensstufen derselben Fehlerart

2

Gesichtserkennung

Klassen: Ariel Sharon, Colin Powell, Donald Rumsfeld, George W Bush, Gerhard Schroeder, Hugo Chavez, Tony Blair
Personen mit ähnlichen Merkmalen werden öfter verwechselt

3

Spotify – SVM vs. Random Forest

10 Musikgenres: baroque, classical, hip hop, pop, progressive bluegrass, r&b, rap, rock, serialism, tropical house
Hip-Hop / Rap und Baroque / Classical werden oft verwechselt

Python – Alle Modelle vergleichen 
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.metrics import confusion_matrix

# Alle Modelle und ihre Klassen-Labels
models_data = [
    {
        'name': 'Kugellager 10 Klassen',
        'ytest_path': '../res/data/bearing_10classes_ytest.npy',
        'ypred_path': '../res/data/bearing_10classes_ypredicted.npy',
        'classes': ['B007_3', 'B014_3', 'B021_3',
                    'IR007_3', 'IR014_3', 'IR021_3',
                    'normal', 'OR007@6_3', 'OR014@6_3', 'OR021@6_3'],
        'figsize': (10, 8)
    },
    {
        'name': 'Gesichtserkennung',
        'ytest_path': '../res/data/face_recognition_ytest.npy',
        'ypred_path': '../res/data/face_recognition_ypredicted.npy',
        'classes': ['Ariel Sharon', 'Colin Powell',
                    'Donald Rumsfeld', 'George W Bush',
                    'Gerhard Schroeder', 'Hugo Chavez', 'Tony Blair'],
        'figsize': (9, 7)
    },
    {
        'name': 'Spotify SVM',
        'ytest_path': '../res/data/spotify_svm_ytest.npy',
        'ypred_path': '../res/data/spotify_svm_ypredicted.npy',
        'classes': ['baroque', 'classical', 'hip hop', 'pop',
                    'progressive bluegrass', 'r&b', 'rap',
                    'rock', 'serialism', 'tropical house'],
        'figsize': (10, 8)
    },
    {
        'name': 'Spotify Random Forest',
        'ytest_path': '../res/data/spotify_rf_ytest.npy',
        'ypred_path': '../res/data/spotify_rf_ypredicted.npy',
        'classes': ['baroque', 'classical', 'hip hop', 'pop',
                    'progressive bluegrass', 'r&b', 'rap',
                    'rock', 'serialism', 'tropical house'],
        'figsize': (10, 8)
    }
]

for m in models_data:
    ytest = np.load(m['ytest_path'], allow_pickle=True)
    ypredicted = np.load(m['ypred_path'], allow_pickle=True)

    cm = confusion_matrix(ytest, ypredicted, normalize='true')

    plt.figure(figsize=m['figsize'])
    sns.heatmap(cm, annot=True, fmt='.2f',
                xticklabels=m['classes'],
                yticklabels=m['classes'],
                cmap='viridis')
    plt.ylabel('True Class')
    plt.xlabel('Predicted Class')
    plt.title(f"Confusion Matrix – {m['name']}")
    plt.tight_layout()
Beobachtungen
  • Kugellager: Verschiedene Schadensstärken ähnlicher Fehlerart werden leicht verwechselt
  • Gesichter: Häufiger fotografierte Personen werden besser erkannt
  • Spotify SVM: hip hop ↔ rap, baroque ↔ classical
  • RF vs. SVM: Random Forest differenziert oft besser
