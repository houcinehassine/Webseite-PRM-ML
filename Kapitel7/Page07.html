Multi-Class ROC Curves

ROC-Kurven für mehr als zwei Klassen – One-vs-One (OvO), One-vs-Rest (OvR), Micro- und Macro-Averaging. Vollständiges Code-Beispiel mit dem Iris-Datensatz.

OvO OvR Macro-Avg Micro-Avg label_binarize OneVsRestClassifier

Lernfokus dieser Seite

Du weißt, wie das Multi-Class-Problem auf binäre ROC-Probleme reduziert wird (OvO / OvR), und kannst einen OneVsRestClassifier mit mehreren ROC-Kurven implementieren.

7.1 Das Problem: ROC nur für binäre Klassifikation

roc_curve() ist explizit auf binäre Klassifikation beschränkt. Wie geht man bei 3 oder mehr Klassen vor?

Aus der sklearn-Doku: „This implementation is restricted to the binary classification task." → Für Multi-Class muss man das Problem in mehrere binäre Probleme aufteilen.

One-vs-Rest (OvR)

Für jede Klasse \(i\) wird ein eigenes binäres Problem definiert: Klasse \(i\) gegen alle anderen Klassen.

  • Bei k Klassen: k binäre Klassifikatoren
  • Jeder Klassifikator liefert eine ROC-Kurve und eine AUC
  • Aus allen k Kurven kann eine mittlere Kurve berechnet werden
  • sklearn-Implementierung: OneVsRestClassifier

One-vs-One (OvO)

Für jedes Paar von Klassen wird ein eigenes binäres Problem definiert.

  • Bei k Klassen: k·(k−1)/2 binäre Klassifikatoren
  • Beispiel: 3 Klassen → 3 Paare: (0 vs 1), (0 vs 2), (1 vs 2)
  • Rechenintensiver, aber manchmal genauer
  • Analogie zur SVM: Gleiche Strategie wie Multi-Class-SVM

7.2 Micro-Average vs. Macro-Average

Wie kombiniert man die k ROC-Kurven zu einer einzigen mittleren Kurve?

Macro-Average

Berechne für alle k Klassen eine ROC-Kurve, dann mittele die TPR-Werte bei denselben FPR-Punkten.

  • Jede Klasse hat gleiches Gewicht
  • Problematisch bei stark unbalancierten Klassen
  • Interpolation nötig: np.interp()

Verwendung: Wenn alle Klassen gleich wichtig sind

Micro-Average

Aggregiere alle TP, FP, TN, FN über alle Klassen, dann berechne eine einzige ROC-Kurve.

  • Gewichtet nach Klassengröße
  • Bessere Maß bei unbalancierten Klassen
  • Berechnung: roc_curve(y_test.ravel(), y_score.ravel())

Verwendung: Wenn größere Klassen wichtiger sind

Methode Wie berechnet? Gewichtung Wann nutzen?
Macro-Average Mittelwert der k Kurven (nach Interpolation) Alle Klassen gleich Balancierte Klassen
Micro-Average Aggregierte Counts aller Klassen Proportional zur Klassengröße Unbalancierte Klassen

7.3 Vollständiges Code-Beispiel – Iris-Datensatz

Multi-Class ROC mit dem Iris-Datensatz (3 Klassen), One-vs-Rest-Klassifikator und Macro-Average.

Schritt 1 · Imports und Daten vorbereiten

Python 
import numpy as np
import matplotlib.pyplot as plt
from itertools import cycle
from sklearn import svm, datasets
from sklearn.metrics import roc_curve, auc
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import label_binarize
from sklearn.multiclass import OneVsRestClassifier
from sklearn.metrics import roc_auc_score

# Iris-Datensatz laden
iris = datasets.load_iris()
X = iris.data
y = iris.target   # 3 Klassen: 0, 1, 2

# Labels binarisieren (One-Hot): [1,0,0], [0,1,0], [0,0,1]
y = label_binarize(y, classes=[0, 1, 2])
n_classes = y.shape[1]   # = 3

# Schwieriger machen: 200 zufällige Noise-Features hinzufügen
random_state = np.random.RandomState(0)
n_samples, n_features = X.shape
X = np.c_[X, random_state.randn(n_samples, 200 * n_features)]

# Train-Test-Split
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.5, random_state=0)
Erklärung
  • label_binarize(): y = [0,1,2] → One-Hot-Matrix (75, 3)
  • 200× zufällige Noise-Features: macht Klassifikation schwieriger
  • 50/50 Train-Test-Split

Schritt 2 · OneVsRestClassifier trainieren und ROC berechnen

Python 
# OneVsRestClassifier mit linearer SVM
classifier = OneVsRestClassifier(
    svm.SVC(kernel="linear", probability=True,
            random_state=random_state)
)
y_score = classifier.fit(X_train, y_train).decision_function(X_test)

# ROC-Kurven für jede Klasse einzeln berechnen
fpr = dict()
tpr = dict()
roc_auc = dict()

for i in range(n_classes):
    fpr[i], tpr[i], _ = roc_curve(y_test[:, i], y_score[:, i])
    roc_auc[i] = auc(fpr[i], tpr[i])

# Micro-Average berechnen (aggregiert alle Klassen)
fpr["micro"], tpr["micro"], _ = roc_curve(
    y_test.ravel(), y_score.ravel()
)
roc_auc["micro"] = auc(fpr["micro"], tpr["micro"])
Hinweise
  • probability=True: aktiviert Platt-Scaling für Wahrscheinlichkeiten
  • y_score ist nun eine (n_test, 3)-Matrix
  • Schleife über alle 3 Klassen: je eine ROC-Kurve + AUC
  • .ravel(): Matrix in 1D-Array umwandeln für Micro-Avg

Schritt 3 · Macro-Average berechnen

Python 
# Alle einzigartigen FPR-Werte zusammenführen
all_fpr = np.unique(np.concatenate(
    [fpr[i] for i in range(n_classes)]
))

# Alle ROC-Kurven auf diese gemeinsamen FPR-Werte interpolieren
mean_tpr = np.zeros_like(all_fpr)
for i in range(n_classes):
    mean_tpr += np.interp(all_fpr, fpr[i], tpr[i])

# Mitteln über alle Klassen und AUC berechnen
mean_tpr /= n_classes
fpr["macro"] = all_fpr
tpr["macro"] = mean_tpr
roc_auc["macro"] = auc(fpr["macro"], tpr["macro"])
Warum interpolieren?
  • Jede Klassen-ROC hat unterschiedliche FPR-Stützstellen
  • np.unique(np.concatenate(...)): Vereinigung aller FPR-Werte
  • np.interp(): Lineare Interpolation auf das gemeinsame Raster
  • Dann: Mittelwert über alle Klassen

Schritt 4 · Alle ROC-Kurven plotten

Python – Vollständiger Plot 
plt.figure()
lw = 2

# Macro-Average Kurve
plt.plot(fpr["macro"], tpr["macro"],
         label=f"Average ROC (area = {roc_auc['macro']:0.2f})",
         color="navy", linestyle=":", linewidth=4)

# Kurven für jede Klasse
colors = cycle(["aqua", "darkorange", "cornflowerblue"])
for i, color in zip(range(n_classes), colors):
    plt.plot(fpr[i], tpr[i], color=color, lw=lw,
             label=f"ROC class {i} (area = {roc_auc[i]:0.2f})")

# Referenz-Diagonale
plt.plot([0, 1], [0, 1], "k--", lw=lw)

plt.xlim([0.0, 1.0])
plt.ylim([0.0, 1.05])
plt.xlabel("False Positive Rate")
plt.ylabel("True Positive Rate")
plt.title("Multi-Class ROC Curves – Iris")
plt.legend(loc="lower right")
plt.show()
Ergebnis
  • 3 farbige Kurven – je eine pro Klasse
  • Gestrichelte navy-Linie = Macro-Average
  • Gestrichelte schwarze Linie = Zufalls-Klassifikator
  • AUC-Werte im Label sichtbar
  • Klasse 0 (setosa) oft am leichtesten zu klassifizieren

7.4 Zusammenfassung – Multi-Class ROC

1

Labels binarisieren

label_binarize(y, classes=[0,1,2]) → One-Hot-Matrix (n, k)

2

OneVsRestClassifier

OneVsRestClassifier(SVC(probability=True)) → trainiert k binäre Klassifikatoren automatisch

3

Klassen-ROC berechnen

Schleife über alle k Klassen: roc_curve(y_test[:,i], y_score[:,i])

4

Averaging

Micro: roc_curve(y_test.ravel(), y_score.ravel()) | Macro: Interpolation + Mittelwert

5

roc_auc_score für Multi-Class

roc_auc_score(y_true, y_score, multi_class='ovr') oder multi_class='ovo' – direkt in sklearn verfügbar

Wichtig: roc_curve() kann nur binär umgehen. roc_auc_score() hingegen kann direkt mit Multi-Class-Problemen umgehen – mit den Parametern multi_class='ovr' und average='macro'.
