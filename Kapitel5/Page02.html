2.1 Entscheidungsbäume – Grundlagen

Entscheidungsbäume sind die Grundlage eines Random Forest und eine sehr intuitive Methode für Klassifikationen.

Ensemble Learner

  • Random Forests sind sogenannte Ensemble Learner
  • Sie setzen sich aus mehreren schwachen Lernern (Weak Learner) zusammen – den einzelnen Entscheidungsbäumen
  • „Das Ganze ist mehr als die Summe seiner Teile!"
  • Die Klassifikation des gesamten Ensembles ist oft besser als die eines einzelnen Klassifikators

Kern-Idee: Fragen stellen

  • Entscheidungsbäume sind eine intuitive Methode für Klassifikationen
  • Der Kern: geschickte Fragen stellen
  • Im Feature-Raum = Daten durch eine achsenparallele Linie aufteilen
  • Jeder Knoten teilt Daten in zwei Gruppen via eines Cut-off-Wertes entlang einer Feature-Achse auf

💡 Daten generieren & visualisieren

Zuerst erzeugen wir einen Beispieldatensatz mit make_blobs und schauen, wie die Datenpunkte im Feature-Raum verteilt sind.

Python 
import matplotlib.pyplot as plt
import numpy as np
from sklearn.datasets import make_blobs

# 300 Datenpunkte, 4 Cluster
X, y = make_blobs(n_samples=300, centers=4,
                  random_state=0, cluster_std=1.0)

plt.figure(figsize=(5, 4))
plt.scatter(X[:, 0], X[:, 1], c=y, s=50, cmap='viridis')
plt.xlabel('Feature 1')
plt.ylabel('Feature 2')
plt.title('4 Klassen im Feature-Raum')
plt.tight_layout()
plt.show()
❓ Was denken Sie?
Denkaufgabe: Was entspricht „Fragen stellen" im Feature-Raum?
  • Versuche die Daten anhand von Feature 1 zu trennen
  • Dann anhand von Feature 2
  • → Eine Linie parallel zur Achse ist ein Split!
Erkenntnis: Jeder Knoten = eine Frage der Form Feature_i ≤ Schwellenwert?

2.2 Splits im Feature-Raum

Wie werden die Entscheidungsgrenzen mit wachsender Tiefe des Baumes komplexer? Und wie beantwortet der Baum die drei zentralen Fragen?

❓ Drei zentrale Fragen

  • Welche Features werden für den Split gewählt?
    → Quantitatives Qualitätskriterium (Gini Impurity)
  • Welcher Cut-off-Wert wird gewählt?
    → Quantitatives Qualitätskriterium (minimiert Gini)
  • Woher kommt die Färbung der Bereiche?
    → Majoritätsvotum der Datenpunkte im Bereich

📏 Was passiert mit zunehmender Tiefe?

  • depth=1 → 1 Schnitt, sehr grobe Trennung
  • depth=2 → 2 Schnitte, besser
  • depth=3 → 4 Regionen, sinnvoll
  • depth=4 → Grenzen werden feiner
  • Ab depth ≥ 5: Overfitting-Gefahr! Der Baum lernt Rauschen auswendig
Python 
import matplotlib.pyplot as plt
import numpy as np
from sklearn.datasets import make_blobs
from sklearn.tree import DecisionTreeClassifier

def visualize_tree(estimator, X, y, boundaries=True,
                   xlim=None, ylim=None, ax=None):
    ax = ax or plt.gca()
    ax.scatter(X[:, 0], X[:, 1], c=y, s=30,
               cmap='viridis', clim=(y.min(), y.max()),
               zorder=3)
    ax.axis('tight')
    ax.axis('off')
    if xlim is None:
        xlim = ax.get_xlim()
    if ylim is None:
        ylim = ax.get_ylim()
    estimator.fit(X, y)
    xx, yy = np.meshgrid(
        np.linspace(*xlim, num=200),
        np.linspace(*ylim, num=200))
    Z = estimator.predict(
        np.c_[xx.ravel(), yy.ravel()])
    n_classes = len(np.unique(y))
    Z = Z.reshape(xx.shape)
    ax.contourf(xx, yy, Z, alpha=0.3,
                levels=np.arange(n_classes + 1) - 0.5,
                cmap='viridis',
                clim=(y.min(), y.max()), zorder=1)
    ax.set(xlim=xlim, ylim=ylim)

# Datensatz
X, y = make_blobs(n_samples=300, centers=4,
                  random_state=0, cluster_std=1.0)

# Tiefen 1-4 visualisieren
fig, ax = plt.subplots(4, 1, figsize=(4, 9))
fig.subplots_adjust(left=0.02, right=0.98, wspace=0.1)

for axi, depth in zip(ax, range(1, 5)):
    model = DecisionTreeClassifier(max_depth=depth)
    visualize_tree(model, X, y, ax=axi)
    axi.set_title(f'depth = {depth}')

plt.suptitle('Decision Tree – Entscheidungsgrenzen',
             y=1.02, fontsize=12)
plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
Was du siehst:
  • depth=1: Nur 1 achsenparallele Linie → grobe Trennung
  • depth=2: Erste sinnvolle Clusteraufteilung
  • depth=3–4: Grenzen werden feiner und passender
Merksatz: Jeder Split = eine achsenparallele Linie. Feature und Schwellenwert werden durch Minimierung der Gini Impurity gewählt.
Majoritätsvotum: Die Farbe eines Bereichs = die häufigste Klasse der Datenpunkte in diesem Bereich.

2.3 Gini Impurity

Um zu bewerten, ob ein Split „gut" oder „schlecht" ist, brauchen wir ein quantitatives Gütekriterium – die Gini Impurity.

Python – Ausgangsbeispiel: 2 Splits 
import matplotlib.pyplot as plt
import numpy as np
from sklearn.datasets import make_blobs

# 10 Datenpunkte, 2 Klassen
X, y = make_blobs(n_samples=10, centers=2,
                  random_state=42, cluster_std=2.)

fig, axes = plt.subplots(2, 1, figsize=(3, 4))

# Split bei x = 0.9 (schlechter Split)
axes[0].scatter(X[:, 0], X[:, 1], c=y, cmap='viridis',
                s=80, edgecolors='k')
axes[0].axvline(0.9, color='red', linestyle='--',
                linewidth=2, label='Split x=0.9')
axes[0].set_title('Split bei x = 0.9\n(eher unrein)')
axes[0].legend(fontsize=8)

# Split bei x = 3.1 (besserer Split)
axes[1].scatter(X[:, 0], X[:, 1], c=y, cmap='viridis',
                s=80, edgecolors='k')
axes[1].axvline(3.1, color='green', linestyle='--',
                linewidth=2, label='Split x=3.1')
axes[1].set_title('Split bei x = 3.1\n(reiner)')
axes[1].legend(fontsize=8)

plt.suptitle('Welcher Split ist besser?', y=1.02)
plt.tight_layout()
plt.show()
❓ Was denken Sie?
Denkaufgabe: Was ist „more impure"?
  • Split x=0.9: Beide Seiten enthalten gemischte Klassen → hohe Impurity
  • Split x=3.1: Eine Seite ist relativ rein → niedrigere Impurity
Sobald die Zusammensetzung komplizierter wird, verlässt uns die Intuition – wir brauchen ein objektives Maß: die Gini Impurity.

Definition

Die Gini Impurity ist die Wahrscheinlichkeit, einen zufällig gezogenen Datenpunkt falsch zu klassifizieren, falls man die Klassifizierung anhand der Verteilung der Klassen im Datensatz durchführt.

Formel:
Wenn C Klassen vorliegen und p(i) die Wahrscheinlichkeit ist, einen Datenpunkt der Klasse i ∈ C zu ziehen:

\[ G = \sum_{i \in C} p(i)\,(1 - p(i)) \]

Herleitung am Beispiel

Ausgangslage: 2 Klassen, je 50% Anteil

  • Ziehe violett, klassifiziere violett: 25% ✅
  • Ziehe violett, klassifiziere gelb: 25% ❌
  • Ziehe gelb, klassifiziere violett: 25% ❌
  • Ziehe gelb, klassifiziere gelb: 25% ✅
→ 25% + 25% = 50% → Gini Impurity = 0.5 (maximale Unreinheit bei 2 Klassen)

Gewichten nach Anzahl/Anteil der Datenpunkte, aufsummieren → Gütemaß zur Bewertung eines Splits.

So what? Qualitäts- bzw. Gütekriterien sind zentraler Bestandteil des Machine Learnings. Grundgedanke: Falsche Zuordnung ist wahrscheinlicher, je „durchmischter" die Daten sind – ein reiner Knoten hat Gini = 0, maximale Unreinheit = 0.5 (binär).

2.4 Overfitting bei Decision Trees

Mit steigender Tiefe passt sich der Baum immer mehr den Trainingsdaten an – Overfitting ist eine generelle Eigenschaft von Decision Trees.

❓ Was ist hier passiert?
Beobachtung:
  • Ab depth=6 entstehen schmale, isolierte Bereiche
  • Diese entsprechen nicht einer intrinsischen Eigenschaft der Daten
  • Der Baum hat einzelne Ausreißer auswendig gelernt
Fazit: Overfitting ist eine generelle Eigenschaft von Decision Trees – schon nach wenigen Splits!

🔬 Overfitting auf verschiedenen Teilmengen

Dasselbe Modell, trainiert auf verschiedenen Hälften des Datensatzes – zeigt die Instabilität einzelner Decision Trees.

❓ Was also tun?
Beobachtung:
  • Gleiche Modellarchitektur, unterschiedliche Trainingsdaten → sehr unterschiedliche Grenzen
  • In sicheren Bereichen: konsistente Ergebnisse
  • In unsicheren Bereichen: inkonsistente Ergebnisse
So what? → Die Lösung:
  • Viele solcher instabilen Bäume kombinieren
  • Durch Randomisierung dekorrelieren sie
  • Kombination → Random Forest!
  • Wo sicher: konsistente Votes – Unsicherheit mittelt sich heraus

Warum overfittet ein Decision Tree?

  • Kein internes Stoppkriterium – der Baum wächst bis jeder Datenpunkt korrekt klassifiziert ist
  • Bei großer Tiefe: schmale Bereiche für einzelne Ausreißer → kein Generalisierungsvermögen
  • Verschiedene Trainingsmengen → sehr unterschiedliche Bäume → hohe Varianz

Die Ensemble-Lösung

  • Viele Weak Learner kombinieren statt einem starken (überfittenden) Modell
  • Durch Bagging: jeder Baum auf einem anderen Subset trainiert
  • Durch Feature Bagging: Bäume werden dekorreliert
  • Majoritätsvotum über alle Bäume → stabile, generalisierbare Vorhersage
  • Wo Klassifizierung sicher: konsistente Votes – Unsicherheit mittelt sich heraus
