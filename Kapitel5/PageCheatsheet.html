📐 Formeln

Alle Kernformeln des Kapitels – direkt ablesbar.

🌲 Decision Tree – Split-Kriterien

Gini Impurity
\( G = 1 - \displaystyle\sum_k p_k^2 \)
\(p_k\) = Anteil Klasse \(k\) · \(G=0\) → reiner Knoten
MSE (Regression)
\( \text{MSE} = \dfrac{1}{N}\displaystyle\sum_n (y_n - \bar y)^2 \)
\(\bar y\) = Mittelwert im Blatt = Vorhersage
Vorhersage Blatt
Klassifikation → Mehrheitsklasse
Regression → Mittelwert aller Samples

🔀 Permutation Importance

Definition
\( \text{PI}_j = \overline{\text{Acc}}_\text{base} - \overline{\text{Acc}}_{\text{perm}_j} \)
Accuracy-Rückgang wenn Feature \(j\) permutiert wird
Auslesen
result.importances_mean → Mittelwert
result.importances_std → Streuung
result.importances → alle Einzelwerte
PI ≈ 0
Feature trägt nichts bei → weglassen
Wichtig!
Immer auf Testdaten berechnen – niemals auf Trainingsdaten!

⚡ AdaBoost – Alle Formeln

Init
\( w_n^{(1)} = \dfrac{1}{N} \)
Alle Punkte gleichgewichtet
Fehler \(\varepsilon_m\)
\( \varepsilon_m = \dfrac{\sum_n w_n^{(m)}\,\mathbb{I}(y_m \neq t_n)}{\sum_n w_n^{(m)}} \)
\(\varepsilon_m < 0.5\) = besser als Zufall
Gewicht \(\alpha_m\)
\( \alpha_m = \ln\!\left(\dfrac{1-\varepsilon_m}{\varepsilon_m}\right) \)
Kleines \(\varepsilon\) → großes \(\alpha\) → mehr Einfluss
Update
\( w_n^{(m+1)} = w_n^{(m)}\, e^{\,\alpha_m\,\mathbb{I}(y_m \neq t_n)} \)
Falsch klassifiziert → Gewicht ↑
Finale Vorhersage
\( Y_M(x) = \operatorname{sign}\!\left(\displaystyle\sum_{m=1}^M \alpha_m\,y_m(x)\right) \)

✈️ RUL – Crude Approximation

TTF
\( \text{TTF}_n = \text{max\_cycle}_\text{Motor} - \text{cycle}_n \)
Fällt von Anfang an linear → 0
RUL (approx.)
\( \text{RUL}_n = \min(\text{TTF}_n,\;125) \)
Konstant bei 125, dann linear fallend
Code
df['RUL'] = df['TTF'].clip(upper=125)
Warum?
Gesunde Phase bietet keine Lern-Information – Modell soll kritische Phase lernen

🔧 sklearn API

Alle wichtigen Klassen & Methoden – kompakt und ausführbar.

RandomForestClassifier

  • Klassifikation mit Ensemble aus Bäumen
  • .predict() → Klasse
  • .predict_proba() → Wahrscheinlichkeiten
  • .feature_importances_ → Tree-based Importance
  • .score() → Accuracy
  • .estimators_[i] → einzelner Baum
from sklearn.ensemble import RandomForestClassifier
model = RandomForestClassifier(
  n_estimators=100, max_depth=None,
  max_features='sqrt', random_state=42)
model.fit(X_train, y_train)
model.predict_proba(X_test) # [n × K]

RandomForestRegressor

  • Regression → Mittelwert der Baumvorhersagen
  • oob_score=True → gratis Validierung
  • .oob_score_ → \(R^2\) ohne Testset
  • Split-Kriterium: MSE
from sklearn.ensemble import RandomForestRegressor
model = RandomForestRegressor(
  n_estimators=100, max_depth=10,
  oob_score=True, random_state=42)
model.fit(X_train, y_train)
model.oob_score_ # R² kostenlos

permutation_importance()

  • Immer auf Testdaten berechnen
  • n_repeats: 10–30 empfohlen
  • Funktioniert mit jedem sklearn-Modell
  • Robuster als Tree-based Importance
from sklearn.inspection import permutation_importance
result = permutation_importance(
  model, X_test, y_test,
  n_repeats=10, random_state=42)
result.importances_mean # Mittelwert
result.importances_std # Streuung

plot_tree() & Einzelbaum

  • Zeigt: Split-Bedingung, MSE/Gini, Samples, Wert
  • filled=True → Farbe nach Klasse/Reinheit
  • forest.estimators_[i] → Baum Nr. i+1
  • One-Liner: direkt beim Training aufrufen
from sklearn.tree import plot_tree
# Baum Nr. 2 aus Forest:
plot_tree(forest.estimators_[1],
  filled=True,
  feature_names=['math', 'lang'],
  class_names=['A', 'B'])
# One-Liner:
plot_tree(RandomForestClassifier(
  n_estimators=5).fit(X,y)[0], ...)

.predict_proba() – Unsicherheit

  • Gibt für jeden Punkt Wahrscheinlichkeiten zurück
  • max(axis=1) → maximale Konfidenz
  • Unter Schwellenwert → Experten-Review
  • Schwellenwert anwendungsspezifisch: 60–90%
proba = model.predict_proba(X_test)
max_p = proba.max(axis=1)
uncertain = max_p < 0.75 # <75%
plt.scatter(X[:,0], X[:,1],
  c=uncertain) # gelb=unsicher

AdaBoostClassifier

  • Weak Learner: Decision Stumps (max_depth=1)
  • estimator_weights_ → alle \(\alpha_m\)
  • estimator_errors_ → alle \(\varepsilon_m\)
  • 'SAMME': diskret, kein predict_proba nötig
from sklearn.ensemble import AdaBoostClassifier
ada = AdaBoostClassifier(
  estimator=DecisionTreeClassifier(
    max_depth=1),
  n_estimators=50,
  algorithm='SAMME')
ada.fit(X_train, y_train)
ada.estimator_weights_ # α_m

🗺️ Wann welches Modell?

Entscheidungshilfe für Prüfungsaufgaben – drei Fragen reichen meist.

1

Regression oder Klassifikation?

  • Zielwert kontinuierlich (Temperatur, RUL, Preis) → RandomForestRegressor · Split-Kriterium: MSE
  • Zielwert kategorial (Klassen, Labels) → RandomForestClassifier · Split-Kriterium: Gini
2

Bagging oder Boosting?

  • Daten verrauscht / Ausreißer → Random Forest (robust, parallel)
  • Bias reduzieren / schwierige Punkte besonders lernen → AdaBoost (seriell, adaptiv)
  • Parallelisierung wichtig → Random Forest (n_jobs=-1)
3

Feature Importance: Tree-based oder Permutation?

  • Schnell & einfach → model.feature_importances_ (Tree-based)
  • Korrelierte Features vorhanden → permutation_importance() (robuster!)
  • PI ≈ 0 oder negativ → Feature weglassen → Modell schlanker

⚠️ Typische Fallen & Missverständnisse

Die häufigsten Punkte, die zu Fehlern in der Prüfung führen.

⚠️
feature_importances_ bei korrelierten Features Korrelierte Features teilen sich die Importance auf – beide erscheinen weniger wichtig. Lösung: permutation_importance() ist robust dagegen.
⚠️
predict() gibt immer eine Klasse – auch bei 34% Konfidenz [0.34, 0.33, 0.33] → .predict() wählt trotzdem Klasse 0. Immer .predict_proba().max(axis=1) zusätzlich prüfen!
⚠️
Permutation Importance auf Trainingsdaten Trainingsdaten kennt das Modell → Importances verzerrt. Immer X_test, y_test verwenden!
⚠️
RUL ≠ TTF – clip() nicht vergessen TTF fällt von Beginn an. RUL (Crude Approx.) bleibt bei 125 und fällt erst in der kritischen Phase. df['RUL'] = df['TTF'].clip(upper=125)
⚠️
AdaBoost kann overfittten – Random Forest kaum Bei zu vielen Boost-Runden passt AdaBoost sich dem Rauschen an. RF mittelt Fehler heraus – robuster durch Bagging.
⚠️
oob_score braucht bootstrap=True oob_score=True ohne bootstrap=True → ValueError. Default ist True – also nur beim expliziten Deaktivieren aufpassen.
⚠️
Ohne Baseline ist RMSE = 18 nichts sagend Immer konstante Vorhersage + lineare Regression als Baseline. Nur wenn RF signifikant besser ist → Mehraufwand gerechtfertigt.
⚠️
Konstante Sensorspalten vor Training entfernen Spalten mit std ≈ 0 tragen nichts bei. Prüfen mit: [c for c in cols if df[c].std() > 0.01]

💡 Merksätze

Ein Satz pro Konzept – auswendig lernbar.

M 1 · Random Forest

Viele zufällige, schwache Bäume gemeinsam abstimmen lassen → stärker als ein einzelner perfekter Baum.

M 2 · Bagging

Bootstrap + Feature Bagging = Dekorrelation der Bäume = Varianzreduktion.

M 3 · Boosting

Jeder neue Learner lernt aus den Fehlern des letzten – seriell, fokussiert, adaptiv.

M 4 · α_m in AdaBoost

Kleines ε_m → großes α_m: bessere Klassifikatoren dürfen mehr mitreden.

M 5 · Permutation Importance

Wenn das Feature fehlt (permutiert): wie stark sinkt die Accuracy? Das ist seine Importance.

M 6 · predict_proba

Nicht nur die Klasse, sondern wie sicher – Punkte unter 75% an Experten weiterleiten.

M 7 · RUL Crude Approx.

Motor ist lange gesund → RUL konstant bei 125; erst die letzten 125 Zyklen sind lehrreich.

M 8 · Baseline Models

Schlägt ML nicht mal die konstante Vorhersage? Dann lohnt sich der Aufwand nicht.

❓ Typische Prüfungsfragen

Aufklappen → Antwort lesen → Karte zuklappen und selbst formulieren.

Konzept Was ist der wichtigste Unterschied zwischen Bagging und Boosting?
Bagging: Weak Learner werden parallel trainiert, alle Samples gleichgewichtet. Diversität durch Zufälligkeit (Bootstrap + Feature Bagging). Reduziert Varianz.

Boosting: Weak Learner werden seriell trainiert. Falsch klassifizierte Samples erhalten höhere Gewichte → nächster Learner fokussiert sich darauf. Reduziert Bias.

Merke: Bagging = robust gegenüber Ausreißern, Boosting = anfällig dafür.
Konzept Warum funktioniert Random Forest besser als ein einzelner Entscheidungsbaum?
Durch Bagging (Bootstrap Sampling) und Feature Bagging (zufällige Feature-Subsets pro Split) werden die Bäume dekorreliert. Ihr gemeinsames Majority Vote / Mittelwert reduziert die Varianz drastisch – ohne den Bias wesentlich zu erhöhen. Ein einzelner tiefer Baum overfit tet; viele verschiedene Bäume mitteln Fehler heraus.
Konzept Wozu braucht man Baseline Models? Nenne drei Beispiele.
Ohne Baseline ist eine Metrik (z.B. RMSE = 18) nicht bewertbar – gut oder schlecht? Baseline Models zeigen, was das einfachste denkbare Modell leistet. ML muss signifikant besser sein.

Typische Baselines:
  • Konstante Vorhersage (immer Mittelwert / häufigste Klasse)
  • Lineare Regression / Logistische Regression
  • Moving Average / Exponential Smoothing
Formel Was passiert mit α_m wenn ε_m = 0,5? Und wenn ε_m → 0?
\[ \alpha_m = \ln\!\left(\frac{1-\varepsilon_m}{\varepsilon_m}\right) \]
  • \(\varepsilon_m = 0.5\) → \(\alpha_m = \ln(1) = 0\): Zufallsklassifikator trägt nichts bei
  • \(\varepsilon_m \to 0\) → \(\alpha_m \to \infty\): Perfekter Klassifikator dominiert komplett
  • \(\varepsilon_m > 0.5\) → \(\alpha_m < 0\): Klassifikator wird invertiert
Formel Warum verwendet Regression MSE statt Gini Impurity als Split-Kriterium?
Gini Impurity misst Klassen-Reinheit → nur sinnvoll für kategoriale Labels.

MSE misst die Streuung der kontinuierlichen Zielwerte im Knoten. Das Split-Kriterium sucht die Aufteilung, die den MSE der Kinder minimiert. Der vorhergesagte Wert im Blatt ist dann der Mittelwert aller Samples im Blatt.
Formel Warum Crude Approximation statt echte TTF als RUL-Target?
In der gesunden Phase (TTF > 125) verhalten sich alle Sensoren normal – das Modell könnte dort keinen Zusammenhang zwischen Sensordaten und RUL lernen, weil alle Werte ähnlich sind.

Die Crude Approximation hält RUL konstant auf 125 und fokussiert das Modell auf die letzten 125 kritischen Zyklen, wo Verschleiß tatsächlich in den Sensordaten sichtbar wird.

Code: df['RUL'] = df['TTF'].clip(upper=125)

Falle Warum ist feature_importances_ bei korrelierten Features problematisch?
Wenn Feature A und Feature B stark korreliert sind, kann der Baum für einen Split entweder A oder B nutzen. Dadurch wird die Importance auf beide aufgeteilt – beide erscheinen halb so wichtig wie sie wirklich sind.

Lösung: permutation_importance() misst den direkten Einfluss auf die Testgenauigkeit – unabhängig davon, welches Feature der Baum zufällig gewählt hat.
Falle Warum kann AdaBoost bei Ausreißern versagen, Random Forest aber nicht?
AdaBoost: Ausreißer werden wiederholt falsch klassifiziert → ihr Gewicht steigt exponentiell mit \(e^{\alpha_m}\). Das Ensemble versucht, sich auf diesen Ausreißer zu konzentrieren → Overfitting auf Rauschen.

Random Forest: Ausreißer tauchen durch Bootstrap Sampling nur in ~63% der Bäume auf. Es gibt keine Gewichtserhöhung – die anderen 37% der Bäume ignorieren ihn. → Robuster durch Mittelwertbildung.
