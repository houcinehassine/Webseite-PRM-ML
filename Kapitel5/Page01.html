1.1 Wo befinden wir uns?

Random Forests sind sogenannte Ensemble-Methoden – eine generische Klasse von ML-Verfahren, die mehrere Modelle kombinieren.

Einordnung im ML-Kontext

  • Supervised Learning – es gibt Labels
  • Klassifikation & Regression – beides möglich
  • Grundlage: Entscheidungsbäume als Weak Learner
  • Verbesserung durch Randomisierung und Aggregation
  • Zugehörigkeit zum „klassischen" Machine Learning

Was ist eine Ensemble-Methode?

  • Kombination mehrerer Weak Learner → Strong Learner
  • Das Ganze ist mehr als die Summe seiner Teile – Ensemble übertrifft einzelne Modelle
  • Zwei Hauptstrategien: Bagging (parallel) und Boosting (seriell)
  • Random Forest = Bagging auf Entscheidungsbäumen
Merksatz: Clustering-Algorithmen erzeugen diskrete Labels ohne Vorgabe – Random Forests hingegen lernen aus vorgegebenen Labels (Supervised Learning). Wir wechseln hier von Unsupervised zu Supervised Learning.

1.2 Themen-Überblick

Ein erster Blick auf alle Konzepte, die in diesem Kapitel behandelt werden – mit Querverweisen zu den jeweiligen Detailseiten.

🌳

Entscheidungsbäume → Seite 2

Das Basismodell eines Random Forests. Teilen Daten durch achsenparallele Splits im Feature-Raum auf. Tendenz zum Overfitting – genau deshalb brauchen wir Ensembles.

📐

Gini Impurity → Seite 2

Das quantitative Gütekriterium für Splits. Misst die Wahrscheinlichkeit, einen zufällig gezogenen Datenpunkt falsch zu klassifizieren. Wert zwischen 0 (rein) und 0.5 (maximal unrein, binär).

🌲

Random Forest → Seite 3

Kombination vieler randomisierter Entscheidungsbäume via Bagging und Feature Bagging. Reduktion von Overfitting durch Dekorrelation der Bäume. Vorhersage durch Majoritätsvotum.

🔍

Feature Importance → Seite 5

Welche Features haben den größten Impact auf die Vorhersage? Permutation Importance: Randomisierung eines Features → messbarer Performance-Rückgang = Wichtigkeit.

⚡

Boosting → Seite 7

Alternative zu Bagging: Weak Learner werden seriell trainiert. Jeder neue Lerner konzentriert sich auf die Fehler des vorherigen. Wichtigster Vertreter: AdaBoost.

📈

RF Regression → Seite 6.2

Random Forests können auch kontinuierliche Werte vorhersagen. Split-Kriterium: MSE statt Gini Impurity. Vorhersage = Mittelwert aller Samples im Blatt.

1.3 Vorteile von Random Forest

Random Forests gehören zu den stärksten Verfahren im klassischen Machine Learning – aus gutem Grund.

⚡ Performance

  • Schnelles Training: einzelne Decision Trees – Trainingszeit wächst linear mit der Anzahl der Bäume
  • Schnelle Evaluierung: jeder Baum wird einzeln ausgewertet → gut parallelisierbar
  • Sehr effizient bei großen Datenmengen – viele Klassen, viele Samples, viele Features

🛠️ Praktische Stärken

  • .predict_proba() möglich – Klassenwahrscheinlichkeiten statt nur Vorhersagen
  • Kann mit numerischen und kategorialen Variablen umgehen
  • Standardisierung der Features ist nicht nötig – im Gegensatz zu SVM oder kNN
  • Eines der mächtigsten Verfahren aus dem „klassischen" Machine Learning

1.4 Nachteile & Grenzen

Kein Verfahren ist perfekt – ein ehrlicher Blick auf die Schwächen hilft bei der richtigen Modellwahl.

🔲 Black Box

  • Im Vergleich zu einem einzelnen Decision Tree → eher Black-Box
  • Hunderte von Bäumen sind schwer zu interpretieren – Lösung: Feature Importance & Permutation Importance (→ Seite 5)

⚠️ Grenzen der Methode

  • Besteht nur eine lineare Abhängigkeit zwischen Features und Label → RF liefert eventuell keinen Genauigkeitsgewinn gegenüber einem einzelnen Decision Tree
  • Bei mehreren kategorialen Features im Datensatz führt ein Ensemble eventuell auch zu keinem Zugewinn
  • Höherer Speicherbedarf als ein einzelner Baum – viele Bäume müssen gespeichert werden

Direktvergleich: Decision Tree vs. Random Forest

Kriterium Decision Tree Random Forest
Interpretierbarkeit ✅ White-Box ⚠️ Black-Box
Overfitting-Tendenz ❌ Hoch ✅ Gering (durch Bagging)
Trainingsgeschwindigkeit ✅ Sehr schnell ⚠️ Langsamer (viele Bäume)
Vorhersagegenauigkeit ⚠️ Mittel ✅ Hoch
.predict_proba() ✅ Möglich ✅ Möglich (robuster)
Skalierung nötig? ❌ Nein ❌ Nein
Speicherbedarf ✅ Gering ⚠️ Höher
Lineare Abhängigkeiten ⚠️ Kein Vorteil ⚠️ Kein Vorteil gegenüber Tree
