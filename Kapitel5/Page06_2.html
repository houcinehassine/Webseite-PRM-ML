Random Forest Regression

Nicht nur Klassifikation – Random Forests können auch kontinuierliche Werte vorhersagen. Wie unterscheidet sich Regression von Klassifikation, warum nutzen wir MSE statt Gini Impurity – und was passiert bei verschiedenen max_depth-Werten?

Lernfokus dieser Seite

  • Was ändert sich beim Wechsel von Klassifikation zu Regression?
  • Warum wird MSE statt Gini Impurity verwendet?
  • Wie beeinflusst max_depth die Vorhersagekurve?
  • Wie sieht ein Regressions-Entscheidungsbaum aus?

6.2.1 Regression vs. Klassifikation

Decision Trees und Random Forests sind nicht auf Klassifikation beschränkt – mit einem einfachen Wechsel des Gütemaßes wird aus einem Classifier ein Regressor.

Klassifikation – Wiederholung

  • Vorhersage eines kategorialen Labels (z.B. healthy / bad)
  • Aggregation über Bäume: Majority Voting
  • Split-Gütemaß: Gini Impurity oder Entropy
  • sklearn-Klasse: RandomForestClassifier()

Regression – neu

  • Vorhersage eines kontinuierlichen Wertes (z.B. RUL, Temperatur, Preis)
  • Aggregation über Bäume: Mittelwert aller Baumvorhersagen
  • Split-Gütemaß: MSE (Mean Squared Error) – Splits minimieren den Fehler
  • sklearn-Klassen: DecisionTreeRegressor() und RandomForestRegressor()
MSE

Mean Squared Error als Split-Kriterium

Bei jedem Split wird geprüft, welche Aufteilung der Daten den MSE der Zielwerte innerhalb der Bereiche am stärksten reduziert. Der vorhergesagte Wert eines Blatts ist der Mittelwert aller Trainingssamples in diesem Blatt.

⚠️

Overfitting-Risiko bei tiefen Bäumen

Ein Regressionsbaum ohne Tiefenbeschränkung kann jeden Trainingspunkt exakt anpassen – Overfitting. max_depth ist daher der wichtigste Regularisierungsparameter. Der Random Forest mildert dieses Problem durch Ensembling.

6.2.2 Schritt 1 – Datensatz generieren

Wir erzeugen einen einfachen quadratischen Datensatz mit additivem Rauschen – ein klassisches Regressionsproblem, das kein lineares Modell perfekt lösen kann.

100 zufällige \(x\)-Werte im Intervall \([-1, 1]\) werden quadriert und mit einem uniformen Rauschen \(\mathcal{U}(0, 0.3)\) versehen. Das Ergebnis ist eine verrauschte Parabel – das Ziel des Regressors ist es, diese Kurve zu rekonstruieren.

Python – Datensatz generieren & visualisieren 
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)

# Quadratische Daten mit Rauschen
index = np.random.uniform(-1, 1, size=(100, 1))
data  = index ** 2 + np.random.rand(
    len(index), 1) * 0.3

fig, ax = plt.subplots(figsize=(6, 4))
ax.scatter(index, data, color='steelblue',
           s=30, alpha=0.7,
           edgecolors='k', linewidths=0.3,
           label='Datenpunkte')
ax.set_xlabel('$x_1$', fontsize=13)
ax.set_ylabel('$y = x_1^2 + \\epsilon$',
              fontsize=13)
ax.set_title('Quadratischer Datensatz mit Rauschen\n'
             '$y = x^2 + \\mathcal{U}(0, 0.3)$')
ax.legend()
ax.grid(alpha=0.3)
plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
Was du siehst:
  • Typische Parabel-Form: Minimum bei \(x=0\), Anstieg nach beiden Seiten
  • Rauschen sorgt für Streuung um die wahre Kurve – kein perfektes Muster
  • Ein lineares Modell würde hier systematisch versagen – Decision Tree Regression kann die Kurvenform approximieren
Datensatz-Eigenschaften:
  • n_samples = 100
  • \(x \in [-1, 1]\) uniform
  • \(y = x^2 + \epsilon\), \(\epsilon \sim \mathcal{U}(0, 0.3)\)

6.2.3 Schritt 2 – DecisionTreeRegressor trainieren

Ein einzelner DecisionTreeRegressor mit max_depth=2 wird trainiert und der Baum direkt visualisiert – so wird die Struktur der Splits sichtbar.

plot_tree() zeigt für jeden Knoten: den Split-Schwellenwert, den MSE der Samples im Knoten, die Anzahl der Samples und den vorhergesagten Wert. Tiefere Farbe = niedrigerer MSE im Knoten.

Python – DecisionTreeRegressor: Baum visualisieren 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.tree import DecisionTreeRegressor, plot_tree

np.random.seed(42)
index = np.random.uniform(-1, 1, size=(100, 1))
data  = index ** 2 + np.random.rand(
    len(index), 1) * 0.3

# Entscheidungsbaum mit max_depth=2 trainieren
model = DecisionTreeRegressor(
    max_depth=2, random_state=42)
model.fit(index, data)

# Baumstruktur visualisieren
fig, ax = plt.subplots(figsize=(10, 4))
plot_tree(model,
          filled=True,
          feature_names=['x₁'],
          ax=ax,
          rounded=True,
          precision=3)
ax.set_title(
    'DecisionTreeRegressor – max_depth=2\n'
    'Jeder Knoten zeigt: Split | MSE | Samples | Wert',
    fontsize=10)
plt.tight_layout()
plt.show()

# Anzahl der Blätter und Splits ausgeben
print(f"Anzahl Blätter:    {model.get_n_leaves()}")
print(f"Tiefe des Baums:   {model.get_depth()}")
print(f"Feature Importances: "
      f"{model.feature_importances_}")
📊 Ausgabe & Interpretation
Baumstruktur lesen:
  • Jeder Knoten zeigt: Split-Bedingung (z.B. \(x_1 \leq 0.5\)), mse, samples, value
  • value = Mittelwert der Samples in diesem Blatt → wird als Vorhersage genutzt
  • Farbe: je dunkler, desto reiner (niedrigerer MSE)
max_depth=2:
  • 4 Blätter → 4 verschiedene Vorhersagewerte
  • Stufenförmige Approximation der Parabel – noch grob
  • Jede Erhöhung von max_depth verdoppelt die Blattanzahl

6.2.4 Schritt 3 – Vorhersagekurve plotten

Eine dichte Linie von 500 gleichmäßig verteilten \(x\)-Werten wird durch das Modell vorhergesagt – so entsteht die rote Vorhersagekurve über den Datenpunkten.

index_predictions = np.linspace(-1, 1, 500).reshape(-1, 1) erzeugt ein feines Gitter von \(x\)-Werten. Das trainierte Modell gibt für jeden dieser Punkte einen vorhergesagten Wert zurück – die charakteristische Stufenfunktion des Decision Trees wird so sichtbar. Zusätzlich wird die Hilfsfunktion plot_regression_predictions() vorgestellt, die diesen Vorgang vereinfacht.

Python – Vorhersagekurve & Hilfsfunktion 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.tree import DecisionTreeRegressor

np.random.seed(42)
index = np.random.uniform(-1, 1, size=(100, 1))
data  = index ** 2 + np.random.rand(
    len(index), 1) * 0.3

# Modell trainieren
model = DecisionTreeRegressor(
    max_depth=2, random_state=42)
model.fit(index, data)

# Vorhersage auf feinem Gitter
index_predictions = np.linspace(
    -1, 1, 500).reshape(-1, 1)
predictions = model.predict(index_predictions)

# Hilfsfunktion für Regressions-Plots
def plot_regression_predictions(
        tree_reg, X, y,
        axes=[-1, 1, -0.1, 1.3],
        ylabel="$y$", ax=None):
    if ax is None:
        ax = plt.gca()
    x1 = np.linspace(
        axes[0], axes[1], 500).reshape(-1, 1)
    y_pred = tree_reg.predict(x1)
    ax.axis(axes)
    ax.set_xlabel("$x_1$", fontsize=12)
    if ylabel:
        ax.set_ylabel(ylabel, fontsize=12,
                      rotation=0, labelpad=15)
    ax.plot(X, y, "b.", alpha=0.6,
            label='Datenpunkte')
    ax.plot(x1, y_pred, "r-",
            linewidth=2, label=r"$\hat{y}$")
    ax.legend(fontsize=9)
    ax.grid(alpha=0.3)

# Vertikal: direkter Plot + Hilfsfunktion
fig, axes_plot = plt.subplots(
    2, 1, figsize=(7, 9))

# Plot 1: manueller Scatter + Vorhersagekurve
axes_plot[0].scatter(
    index, data, color='steelblue',
    s=30, alpha=0.7,
    edgecolors='k', linewidths=0.3,
    label='Datenpunkte')
axes_plot[0].plot(
    index_predictions, predictions,
    color='red', linewidth=2,
    label=r'$\hat{y}$ (max_depth=2)')
axes_plot[0].set_xlabel('$x_1$', fontsize=12)
axes_plot[0].set_ylabel('$y$', fontsize=12)
axes_plot[0].set_title(
    'Vorhersagekurve – DecisionTreeRegressor\n'
    'max_depth=2 → 4 Stufen sichtbar')
axes_plot[0].legend()
axes_plot[0].grid(alpha=0.3)

# Plot 2: via Hilfsfunktion
plot_regression_predictions(
    model, index, data,
    ax=axes_plot[1])
axes_plot[1].set_title(
    'Gleicher Plot via plot_regression_predictions()')

plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
Vorhersagekurve (rot):
  • Stufenförmig – typisch für Decision Tree Regression
  • Bei max_depth=2: 4 konstante Stufen = 4 Blattknoten
  • Parabelform wird grob approximiert, aber nicht glatt
  • Jede Stufe = Mittelwert aller Trainingssamples im Bereich
Hilfsfunktion: plot_regression_predictions() kapselt den Visualisierungsvorgang – wiederverwendbar für verschiedene max_depth-Werte.

6.2.5 Schritt 4 – Einfluss von max_depth

Sechs verschiedene max_depth-Werte werden direkt verglichen – von zu einfach (Underfitting) bis zu komplex (Overfitting).

Overfitting – zu tief

  • Hohe max_depth (z.B. 10): Baum passt sich jedem Rauschpunkt an
  • Vorhersagekurve springt stark → keine Generalisierung
  • Training-Fehler sehr klein, Test-Fehler groß
  • Random Forest mildert dies – durch Mittelwertbildung

Underfitting – zu flach

  • max_depth=1: nur 1 Split → 2 konstante Vorhersagewerte
  • Kann Parabelstruktur überhaupt nicht erfassen
  • Sowohl Trainings- als auch Test-Fehler hoch
  • Modell zu simpel für die Datenstruktur

Sechs Modelle mit max_depth ∈ {1, 2, 3, 4, 5, 10} werden trainiert und ihre Vorhersagekurven im vertikalen Grid ausgegeben. So wird der Übergang von Underfitting zu Overfitting direkt sichtbar. Das ursprüngliche (2, 3)-Grid wurde auf ein vertikales (6, 1)-Layout umgestellt.

Python – max_depth Vergleich (6 Modelle, vertikal) 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import mean_squared_error

np.random.seed(42)
index = np.random.uniform(-1, 1, size=(100, 1))
data  = index ** 2 + np.random.rand(
    len(index), 1) * 0.3

index_predictions = np.linspace(
    -1, 1, 500).reshape(-1, 1)

max_depth_values = [1, 2, 3, 4, 5, 10]

# Vertikales Layout: 6 Zeilen, 1 Spalte
fig, axes = plt.subplots(
    6, 1, figsize=(7, 24))
fig.subplots_adjust(hspace=0.4)

for ax, max_depth in zip(axes, max_depth_values):
    model = DecisionTreeRegressor(
        max_depth=max_depth, random_state=42)
    model.fit(index, data)

    predictions = model.predict(index_predictions)
    train_mse   = mean_squared_error(
        data, model.predict(index))

    ax.scatter(index, data,
               color='steelblue', s=20,
               alpha=0.6, zorder=2,
               label='Datenpunkte')
    ax.plot(index_predictions, predictions,
            color='red', linewidth=2,
            label=r'$\hat{y}$')
    ax.set_xlim(-1.1, 1.1)
    ax.set_ylim(-0.1, 1.35)
    ax.set_xlabel('$x_1$', fontsize=10)
    ax.set_ylabel('$y$', fontsize=10,
                  rotation=0, labelpad=12)
    ax.set_title(
        f'max_depth = {max_depth}   '
        f'│   Blätter: {model.get_n_leaves()}   '
        f'│   Train-MSE: {train_mse:.4f}',
        fontsize=10)
    ax.legend(fontsize=8, loc='upper center')
    ax.grid(alpha=0.3)

plt.suptitle(
    'DecisionTreeRegressor – '
    'Einfluss von max_depth\n'
    '(oben = Underfitting  →  '
    'unten = Overfitting)',
    fontsize=12, y=1.005)
plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
Von oben nach unten:
  • max_depth=1: Underfitting – 2 Stufen, Parabel nicht erkennbar
  • max_depth=2–3: guter Kompromiss – Parabelform erkennbar, keine Überanpassung
  • max_depth=4–5: feiner, aber erste Rausch-Anpassungen sichtbar
  • max_depth=10: Overfitting – springende Kurve, viele unnötige Stufen
Train-MSE: Fällt monoton mit steigendem max_depth. Das allein ist kein gutes Modell-Kriterium – Cross-Validation-MSE ist entscheidend.

6.2.6 Schritt 5 – RandomForestRegressor

Vom einzelnen Baum zum Ensemble: RandomForestRegressor() kombiniert viele Bäume und liefert eine deutlich glattere und robustere Vorhersagekurve.

RandomForestRegressor

  • Gleiche API wie RandomForestClassifier()
  • Aggregation: Mittelwert aller Baumvorhersagen statt Majority Voting
  • Parameter: n_estimators, max_depth, max_features, bootstrap
  • Vorhersagekurve ist weicher als ein einzelner Baum mit gleicher Tiefe

🔄 Vergleich: Single Tree vs. Forest

  • Single Tree: stufige Kurve, Overfitting bei tiefer Tiefe
  • Random Forest: geglättete Kurve durch Mittelwertbildung – Rauschpunkte werden gemittelt heraus
  • Forest mit vielen flachen Bäumen ≈ Forest mit wenigen tiefen Bäumen – aber die Varianz ist niedriger
  • oob_score=True liefert \(R^2\) ohne separates Testset

Direkter Vergleich: Ein einzelner DecisionTreeRegressor(max_depth=5) gegen einen RandomForestRegressor(n_estimators=100) auf demselben Datensatz. Die Vorhersagekurven werden übereinander geplottet, dazu MSE und \(R^2\) beider Modelle ausgegeben.

Python – RandomForestRegressor vs. Single Tree 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.tree import DecisionTreeRegressor
from sklearn.ensemble import RandomForestRegressor
from sklearn.metrics import (
    mean_squared_error, r2_score)
from sklearn.model_selection import train_test_split

np.random.seed(42)
index = np.random.uniform(-1, 1, size=(100, 1))
data  = index ** 2 + np.random.rand(
    len(index), 1) * 0.3

X_train, X_test, y_train, y_test = train_test_split(
    index, data.ravel(),
    test_size=0.2, random_state=42)

# Modelle
tree = DecisionTreeRegressor(
    max_depth=5, random_state=42)
forest = RandomForestRegressor(
    n_estimators=100, max_depth=5,
    random_state=42, oob_score=True)

tree.fit(X_train, y_train)
forest.fit(X_train, y_train)

# Vorhersagegitter
x_pred = np.linspace(-1, 1, 500).reshape(-1, 1)
y_tree   = tree.predict(x_pred)
y_forest = forest.predict(x_pred)

# Metriken
print("── Single Tree (max_depth=5) ──────────────")
print(f"Test MSE: "
      f"{mean_squared_error(y_test, tree.predict(X_test)):.4f}")
print(f"Test R²:  "
      f"{r2_score(y_test, tree.predict(X_test)):.4f}")
print("\n── Random Forest (n=100, max_depth=5) ────")
print(f"Test MSE: "
      f"{mean_squared_error(y_test, forest.predict(X_test)):.4f}")
print(f"Test R²:  "
      f"{r2_score(y_test, forest.predict(X_test)):.4f}")
print(f"OOB R²:   {forest.oob_score_:.4f}")

# Visualisierung: vertikal (3 × 1)
fig, axes = plt.subplots(3, 1, figsize=(7, 13))

# Plot 1: Single Tree
axes[0].scatter(index, data, color='steelblue',
                s=20, alpha=0.6, label='Daten')
axes[0].plot(x_pred, y_tree, color='red',
             linewidth=2, label='Single Tree')
axes[0].set_title(
    'DecisionTreeRegressor – max_depth=5\n'
    f'Test R² = '
    f'{r2_score(y_test, tree.predict(X_test)):.3f}')
axes[0].set_xlabel('$x_1$')
axes[0].set_ylabel('$y$', rotation=0, labelpad=12)
axes[0].legend(fontsize=9)
axes[0].grid(alpha=0.3)

# Plot 2: Random Forest
axes[1].scatter(index, data, color='steelblue',
                s=20, alpha=0.6, label='Daten')
axes[1].plot(x_pred, y_forest, color='darkgreen',
             linewidth=2, label='Random Forest')
axes[1].set_title(
    'RandomForestRegressor – n=100, max_depth=5\n'
    f'Test R² = '
    f'{r2_score(y_test, forest.predict(X_test)):.3f}  '
    f'│  OOB R² = {forest.oob_score_:.3f}')
axes[1].set_xlabel('$x_1$')
axes[1].set_ylabel('$y$', rotation=0, labelpad=12)
axes[1].legend(fontsize=9)
axes[1].grid(alpha=0.3)

# Plot 3: Direkter Vergleich
axes[2].scatter(index, data, color='steelblue',
                s=20, alpha=0.5, label='Daten',
                zorder=1)
axes[2].plot(x_pred, y_tree, color='red',
             linewidth=2, linestyle='--',
             label='Single Tree (max_depth=5)',
             zorder=2)
axes[2].plot(x_pred, y_forest, color='darkgreen',
             linewidth=2.5,
             label='Random Forest (n=100)',
             zorder=3)
axes[2].set_title(
    'Direkter Vergleich:\n'
    'Single Tree (rot gestrichelt) vs. '
    'Random Forest (grün)')
axes[2].set_xlabel('$x_1$')
axes[2].set_ylabel('$y$', rotation=0, labelpad=12)
axes[2].legend(fontsize=9)
axes[2].grid(alpha=0.3)

plt.suptitle(
    'RandomForestRegressor vs. DecisionTreeRegressor',
    fontsize=12, y=1.005)
plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
Plot 1 vs. Plot 2:
  • Single Tree: stufige, sprungreiche Kurve
  • Random Forest: deutlich glattere Approximation der Parabel
  • Höheres \(R^2\) beim Forest – trotz gleichem max_depth
OOB Score: Ohne separates Testset berechenbar – nutzt die Samples, die nicht im Bootstrap-Sample des jeweiligen Baums waren. Sehr nützlicher kostenloser Validierungsindikator.
Fazit: Random Forest Regression liefert glattere, robustere Kurven – ohne zusätzliches Hyperparameter-Tuning.
