4.1 RandomForestClassifier

In sklearn gibt es für einen Random Forest die Klasse RandomForestClassifier() – mit einer Reihe wichtiger Steuerparameter.

Wichtige Parameter

  • n_estimators – Anzahl der Decision Trees im Wald
    Mehr Bäume = stabiler, aber mehr Rechenzeit. Ab einem Schwellenwert kein Genauigkeitsgewinn mehr
  • criterion – Gütemaß für Splits
    Default: 'gini' · Alternative: 'entropy'
  • max_depth – Maximale Tiefe der Bäume
    None = unbegrenzt (Overfitting-Risiko)
  • bootstrap – Bagging an/aus
    True (Default) = mit Zurücklegen · False = gesamter Datensatz für jeden Baum
  • max_features – Feature Bagging pro Split
    Default: 'sqrt'

❓ Mehr Bäume = mehr Genauigkeit?

  • Grundsätzlich ja – mehr Bäume stabilisieren die Vorhersage durch Mittelung
  • Aber: Ab einem bestimmten Sättigungspunkt bringt jeder zusätzliche Baum kaum noch Genauigkeitsgewinn
  • Gleichzeitig steigt die Trainings- und Vorhersagezeit linear mit n_estimators
  • → Sinnvoll: OOB Score oder Validierungskurve nutzen, um den optimalen Trade-off zu finden
  • Typischer Praxiswert: n_estimators=100–500

Der folgende Code trainiert einen RandomForestClassifier auf einem Blob-Datensatz und visualisiert die Entscheidungsgrenzen des Ensembles. Die Hilfsfunktion visualize_classifier() zeichnet sowohl die Trainingspunkte als auch die farbigen Klassifikationsbereiche.

Python – RandomForestClassifier trainieren & visualisieren 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_blobs
from sklearn.ensemble import RandomForestClassifier

def visualize_classifier(model, X, y,
                         ax=None, cmap='rainbow'):
    ax = ax or plt.gca()
    ax.scatter(X[:, 0], X[:, 1], c=y, s=30,
               cmap=cmap, clim=(y.min(), y.max()),
               zorder=3)
    ax.axis('tight')
    ax.axis('off')
    xlim = ax.get_xlim()
    ylim = ax.get_ylim()
    model.fit(X, y)
    xx, yy = np.meshgrid(
        np.linspace(*xlim, num=200),
        np.linspace(*ylim, num=200))
    Z = model.predict(
        np.c_[xx.ravel(), yy.ravel()]
    ).reshape(xx.shape)
    n_classes = len(np.unique(y))
    ax.contourf(xx, yy, Z, alpha=0.3,
                levels=np.arange(n_classes + 1) - 0.5,
                cmap=cmap, clim=(y.min(), y.max()),
                zorder=1)
    ax.set(xlim=xlim, ylim=ylim)

# Datensatz
X, y = make_blobs(n_samples=300, centers=4,
                  random_state=0, cluster_std=1.0)

# Modell
model = RandomForestClassifier(
    n_estimators=100, random_state=0)

fig, ax = plt.subplots(figsize=(5, 4))
visualize_classifier(model, X, y, ax=ax)
ax.set_title('RandomForest – 100 Bäume\nEntscheidungsgrenzen')
plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
Was du siehst:
  • Weiche, geschwungene Grenzen – anders als ein einzelner Baum
  • Keine schmalen „Ausreißer-Taschen" mehr → Overfitting reduziert
  • Grenzen reflektieren die echte Datenstruktur
So what: Die Kombination der randomisierten Decision Trees führt zu einem Klassifikator, der die zugrundeliegende Struktur der Daten besser modelliert als einzelne Bäume.

Hier wird der Effekt von n_estimators auf die Genauigkeit direkt sichtbar gemacht: Vier verschiedene Waldgrößen werden nebeneinander visualisiert. Die Grenzen werden mit wachsender Baumanzahl stabiler.

Python – n_estimators Vergleich

               zorder=3)
    ax.axis('tight')
    ax.axis('off')
    xlim = ax.get_xlim()
    ylim = ax.get_ylim()
    model.fit(X, y)
    xx, yy = np.meshgrid(
        np.linspace(*xlim, num=200),
        np.linspace(*ylim, num=200))
    Z = model.predict(
        np.c_[xx.ravel(), yy.ravel()]
    ).reshape(xx.shape)
    n_classes = len(np.unique(y))
    ax.contourf(xx, yy, Z, alpha=0.3,
                levels=np.arange(n_classes + 1) - 0.5,

    ax.set(xlim=xlim, ylim=ylim)

X, y = make_blobs(n_samples=300, centers=4,
                  random_state=0, cluster_std=1.0)

n_list = [1, 5, 20, 100]

# Vertikales Layout: 4 Zeilen, 1 Spalte
fig, axes = plt.subplots(4, 1, figsize=(5, 16))
fig.subplots_adjust(hspace=0.3)

for ax, n in zip(axes, n_list):

📊 Ausgabe & Interpretation
Was du siehst (von oben nach unten):
  • n=1: Ein einzelner Baum – instabile, eckige Grenzen
  • n=5: Leichte Verbesserung, aber noch grob
  • n=20: Deutlich glattere Grenzen
  • n=100: Stabile, weiche Grenzen – Sättigungseffekt sichtbar
Fazit: Mehr Bäume ≠ immer mehr Genauigkeit – ab einem Punkt stabilisieren sich die Grenzen und weiteres Training lohnt sich kaum noch.

4.2 .predict_proba()

Neben der reinen Klassenvorhersage kann der RandomForestClassifier auch Klassenwahrscheinlichkeiten ausgeben – ein großer praktischer Vorteil.

Was ist predict_proba()?

  • Gibt für jeden Datenpunkt die Wahrscheinlichkeit zur Zugehörigkeit jeder Klasse aus – nicht nur die wahrscheinlichste Klasse
  • Rückgabe: Matrix der Form [n_samples × n_classes], Zeilensumme = 1.0
  • Berechnung: Anteil der Bäume im Wald, die für Klasse k gestimmt haben
  • Ermöglicht das Ausdrücken von Unsicherheit – z.B. [0.51, 0.49] vs. [0.99, 0.01]

❓ Warum ist das bei RF einfacher?

  • Unser Modell besteht aus vielen kleinen Modellen – jedes macht eine eigenständige Vorhersage
  • Die Verteilung der Stimmen über alle Bäume ergibt natürlich eine Wahrscheinlichkeit
  • Bei einem einzelnen Decision Tree gibt es nur harte 0/1-Entscheidungen – keine natürliche Unsicherheit
  • Anwendungsbeispiele: Risikobewertung, Schwellenwert-Tuning, Anomalie-Erkennung, Ranking

Der Code demonstriert .predict_proba() auf einem Testset: Ausgabe der Wahrscheinlichkeitsmatrix, Vergleich mit .predict() und Visualisierung der Vorhersage-Sicherheit als Heatmap über den Feature-Raum.

Python – predict_proba() anwenden 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_blobs
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split

X, y = make_blobs(n_samples=300, centers=4,
                  random_state=0, cluster_std=1.0)

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42)

model = RandomForestClassifier(
    n_estimators=100, random_state=0)
model.fit(X_train, y_train)

# Wahrscheinlichkeitsmatrix
proba = model.predict_proba(X_test)
pred  = model.predict(X_test)

print("Shape predict_proba:", proba.shape)
print("\nErste 5 Vorhersagen:")
print(f"{'predict':>10} | predict_proba")
print("-" * 45)
for i in range(5):
    p = proba[i]
    prob_str = "  ".join([f"K{j}:{v:.2f}"
                          for j, v in enumerate(p)])
    print(f"{pred[i]:>10} | {prob_str}")

# Confidence = max. Wahrscheinlichkeit
confidence = proba.max(axis=1)

# Visualisierung: Confidence im Feature-Raum
x_min, x_max = X[:, 0].min()-1, X[:, 0].max()+1
y_min, y_max = X[:, 1].min()-1, X[:, 1].max()+1
xx, yy = np.meshgrid(np.linspace(x_min, x_max, 200),
                     np.linspace(y_min, y_max, 200))
proba_grid = model.predict_proba(
    np.c_[xx.ravel(), yy.ravel()])
conf_grid  = proba_grid.max(axis=1).reshape(xx.shape)

fig, axes = plt.subplots(2, 1, figsize=(6, 10))

# Plot 1: Klassifikationsgrenzen
axes[0].contourf(xx, yy, conf_grid, alpha=0.4,
                 cmap='RdYlGn', levels=20)
axes[0].scatter(X_test[:, 0], X_test[:, 1],
                c=pred, cmap='rainbow', s=40,
                edgecolors='k', linewidths=0.5,
                zorder=3)
axes[0].set_title('Vorhersage-Konfidenz im Feature-Raum\n'
                  '(grün=sicher, rot=unsicher)', fontsize=10)
axes[0].axis('off')

# Plot 2: Histogramm der Konfidenz
axes[1].hist(confidence, bins=20, color='steelblue',
             edgecolor='white', alpha=0.85)
axes[1].axvline(confidence.mean(), color='red',
                linestyle='--', linewidth=2,
                label=f'Mittelwert: {confidence.mean():.2f}')
axes[1].set_xlabel('Konfidenz (max. Wahrscheinlichkeit)')
axes[1].set_ylabel('Anzahl Testpunkte')
axes[1].set_title('Verteilung der Vorhersage-Konfidenz')
axes[1].legend()
axes[1].grid(alpha=0.3)

plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
predict_proba() Output:
  • Shape: [n_test × 4] (4 Klassen im Beispiel)
  • Jede Zeile summiert zu 1.0
  • Hohe Konfidenz: z.B. [0.98, 0.01, 0.01, 0.00] → Baum einig
  • Niedrige Konfidenz: z.B. [0.45, 0.40, 0.10, 0.05] → Grenzbereich
Konfidenz-Karte:
  • Grün = sichere Bereiche (alle Bäume einig)
  • Rot = unsichere Grenzbereiche
  • Histogram: Die meisten Punkte haben hohe Konfidenz – Unsicherheit nur an den Grenzen

4.3 Hyperparameter Sweep

„Sophisticated Guesses" – anstatt blind alle Kombinationen zu testen, schränkt man den Suchraum durch Datenwissen und erste Modelle ein.

Warum nicht einfach Grid Search?

  • Grid Search ist Exhaustive Search (Brute Force) im aufgespannten Parameterraum
  • Je mehr Dimensionen, desto exponentiell länger das Training
  • Viele Kombinationen sind offensichtlich sinnlos – z.B. max_depth=1 bei großem Datensatz
  • → Lösung: Suchraum vorab durch Datenwissen einschränken

🔍 Strategien zur Einschränkung

  • Eigenschaften erster Modelle: Wie sehen die Hyperparameterwerte erster Versuche aus? Was lässt sich ableiten?
  • Einschränkung durch Dateneigenschaften: Legen uns Datengröße, Feature-Anzahl oder Klassenverteilung sinnvolle Grenzen auf?
  • Wertebereiche filtern: Für den vorliegenden Use Case: welche Bereiche sind sinnvoll / welche sind es definitiv nicht?
1

Grid Search

Brute Force – alle Kombinationen aus dem definierten Raster werden evaluiert. Garantiert das Optimum im definierten Raster, aber exponentiell teuer bei vielen Parametern. Geeignet nur bei kleinen Parameterräumen.

2

Random Search

Parameter werden zufällig aus Verteilungen gezogen – nicht aus einem festen Raster. Mehr individuelle Werte auf jeder Achse als Grid Search. Oft besser als Grid Search bei gleichem Budget.

3

Bayesian Optimization

Baut iterativ ein Surrogate-Modell der Kostenfunktion auf. Jede neue Evaluation wird intelligent gewählt – basierend auf bisherigen Ergebnissen. Weniger Schritte, bessere Resultate.

4.4 Random Search in sklearn

RandomizedSearchCV ersetzt den Brute-Force-Ansatz durch zufälliges Sampling aus den Parameterverteilungen.

Wichtige Parameter

  • estimator – das zu optimierende Modell
  • param_distributions – Parameterintervalle als Dictionary (Verteilungen oder Listen)
  • n_iter – Anzahl der gezogenen Parameterkombinationen
  • cv – Cross-Validation Strategie (Integer = Anzahl Splits)
  • scoring, n_jobs=-1 (Parallelisierung), random_state

⚡ Vorteil gegenüber Grid Search

  • Mehr individuelle Werte auf den einzelnen Parameterachsen – bessere Abdeckung des Raums
  • Bei gleichem n_iter-Budget oft bessere Ergebnisse als Grid Search
  • Parameterverteilungen (z.B. scipy.stats.randint) erlauben kontinuierliches Sampling statt diskreter Gitterpunkte
  • Einfach skalierbar: n_iter erhöhen = mehr Suche

Der Code führt einen RandomizedSearchCV über vier wichtige Hyperparameter des RandomForestClassifier durch und visualisiert die besten gefundenen Kombinationen nach Score.

Python – RandomizedSearchCV 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_classification
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import (
    RandomizedSearchCV, train_test_split)
from scipy.stats import randint

# Datensatz
X, y = make_classification(
    n_samples=600, n_features=10,
    n_informative=6, n_redundant=2,
    random_state=42)

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42)

# Parameterraum
param_dist = {
    'n_estimators': randint(50, 300),
    'max_depth':    randint(2, 20),
    'max_features': randint(1, 10),
    'min_samples_split': randint(2, 20)
}

# Random Search
rs = RandomizedSearchCV(
    estimator=RandomForestClassifier(random_state=42),
    param_distributions=param_dist,
    n_iter=50,          # 50 zufällige Kombinationen
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    random_state=42
)
rs.fit(X_train, y_train)

# Ergebnisse
print("Beste Parameter:")
for k, v in rs.best_params_.items():
    print(f"  {k:25s}: {v}")
print(f"\nBester CV-Score:  {rs.best_score_:.4f}")
print(f"Test Accuracy:    "
      f"{rs.best_estimator_.score(X_test, y_test):.4f}")

# Top-20 Ergebnisse visualisieren
results = rs.cv_results_
top_idx = np.argsort(
    results['mean_test_score'])[::-1][:20]
top_scores = results['mean_test_score'][top_idx]
top_n = [results['params'][i]['n_estimators']
         for i in top_idx]

fig, axes = plt.subplots(2, 1, figsize=(8, 8))

# Score der Top-20 Kombinationen
axes[0].bar(range(20), top_scores,
            color='steelblue', edgecolor='white',
            alpha=0.85)
axes[0].axhline(rs.best_score_, color='red',
                linestyle='--', linewidth=2,
                label=f'Best: {rs.best_score_:.4f}')
axes[0].set_xlabel('Rang (beste Kombination links)')
axes[0].set_ylabel('CV Accuracy')
axes[0].set_title('Top-20 Random Search Kombinationen')
axes[0].legend()
axes[0].grid(axis='y', alpha=0.3)

# n_estimators der Top-20
axes[1].scatter(top_n, top_scores,
                color='darkorange', s=60,
                edgecolors='k', linewidths=0.5,
                zorder=3)
axes[1].set_xlabel('n_estimators (Top-20)')
axes[1].set_ylabel('CV Accuracy')
axes[1].set_title('n_estimators vs. Score (Top-20)')
axes[1].grid(alpha=0.3)

plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
Was der Code macht:
  • 50 zufällige Kombinationen aus dem Parameterraum werden gezogen
  • Jede Kombination wird mit 5-Fold CV bewertet
  • Die beste Kombination wird auf dem Testset evaluiert
Interessant: Im Vergleich zum Grid Search resultiert Random Search in einer höheren Anzahl individueller Werte auf den einzelnen Parameterachsen – bei gleichem Rechenbudget.
Tipp: scipy.stats.randint(a, b) zieht ganzzahlig aus [a, b). Für kontinuierliche Parameter: scipy.stats.uniform(loc, scale)

4.5 Bayesian Optimization

Der intelligenteste Ansatz: statt blind zu suchen, wird iterativ ein Modell der Kostenfunktion aufgebaut und zur gezielten Suche genutzt.

Wie funktioniert es?

  • Bayesian Optimization erzeugt iterativ ein Surrogate-Modell der Kostenfunktion (z.B. Gaussian Process)
  • Basierend auf diesem Modell werden vielversprechende Parameterkombinationen als nächstes evaluiert
  • Nach jeder Evaluation wird das Surrogate-Modell aktualisiert
  • Dadurch werden intelligent Schritte in Richtung Optimum gegangen – kein blinder Zufall

📊 Vergleich der Methoden

  • Grid Search: Exhaustive, garantiert Raster-Optimum, teuer
  • Random Search: Zufällig, besser als Grid bei gleichem Budget, keine Intelligenz
  • Bayesian Opt.: Iterativ intelligent, weniger Schritte, beste Resultate
  • Bayesian Optimization führt in weniger Schritten zu besseren Ergebnissen als Grid und Random Search

Der Code simuliert den Vorteil von Bayesian Optimization gegenüber Random Search: Der beste gefundene Score wird über die Anzahl der Evaluationen geplottet – Bayesian Optimization konvergiert schneller zum Optimum. Verwendet wird das Paket scikit-optimize (skopt).

Python – Bayesian Optimization vs. Random Search 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_classification
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score
from skopt import BayesSearchCV
from skopt.space import Integer
from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint

# Installation falls nötig:
# pip install scikit-optimize

X, y = make_classification(
    n_samples=600, n_features=10,
    n_informative=6, n_redundant=2,
    random_state=42)

# ── Bayesian Optimization ──────────────────────────
search_space = {
    'n_estimators':      Integer(50, 300),
    'max_depth':         Integer(2, 20),
    'max_features':      Integer(1, 10),
    'min_samples_split': Integer(2, 20)
}

bayes_search = BayesSearchCV(
    estimator=RandomForestClassifier(random_state=42),
    search_spaces=search_space,
    n_iter=30,
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    random_state=42
)
bayes_search.fit(X, y)

bayes_scores = [
    max(bayes_search.cv_results_['mean_test_score'][:i+1])
    for i in range(
        len(bayes_search.cv_results_['mean_test_score']))]

# ── Random Search (gleiche Iterationen) ───────────
param_dist = {
    'n_estimators':      randint(50, 300),
    'max_depth':         randint(2, 20),
    'max_features':      randint(1, 10),
    'min_samples_split': randint(2, 20)
}

rand_search = RandomizedSearchCV(
    estimator=RandomForestClassifier(random_state=42),
    param_distributions=param_dist,
    n_iter=30,
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    random_state=42
)
rand_search.fit(X, y)

rand_scores = [
    max(rand_search.cv_results_['mean_test_score'][:i+1])
    for i in range(
        len(rand_search.cv_results_['mean_test_score']))]

# ── Ergebnisse ausgeben ────────────────────────────
print(f"Bayesian Best Score: {bayes_search.best_score_:.4f}")
print(f"Random   Best Score: {rand_search.best_score_:.4f}")
print("\nBayesian Best Params:")
for k, v in bayes_search.best_params_.items():
    print(f"  {k:25s}: {v}")

# ── Konvergenzplot ─────────────────────────────────
fig, axes = plt.subplots(2, 1, figsize=(7, 8))

# Plot 1: Konvergenz
iters = range(1, 31)
axes[0].plot(iters, bayes_scores,
             color='purple', linewidth=2.5,
             marker='o', markersize=4,
             label='Bayesian Optimization')
axes[0].plot(iters, rand_scores,
             color='darkorange', linewidth=2.5,
             linestyle='--', marker='s', markersize=4,
             label='Random Search')
axes[0].set_xlabel('Anzahl Evaluationen')
axes[0].set_ylabel('Bester CV Score (kumulativ)')
axes[0].set_title('Konvergenz: Bayesian vs. Random Search')
axes[0].legend()
axes[0].grid(alpha=0.3)

# Plot 2: Finale Score-Verteilung
all_bayes = bayes_search.cv_results_['mean_test_score']
all_rand  = rand_search.cv_results_['mean_test_score']

axes[1].hist(all_rand, bins=15, alpha=0.6,
             color='darkorange', label='Random Search',
             edgecolor='white')
axes[1].hist(all_bayes, bins=15, alpha=0.6,
             color='purple', label='Bayesian Opt.',
             edgecolor='white')
axes[1].axvline(rand_search.best_score_,
                color='darkorange', linestyle='--',
                linewidth=2)
axes[1].axvline(bayes_search.best_score_,
                color='purple', linestyle='--',
                linewidth=2)
axes[1].set_xlabel('CV Score')
axes[1].set_ylabel('Häufigkeit')
axes[1].set_title('Verteilung aller evaluierten Scores')
axes[1].legend()
axes[1].grid(alpha=0.3)

plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
Konvergenzplot (oben):
  • Bayesian erreicht das Optimum früher – weniger Evaluationen nötig
  • Random konvergiert langsamer und unregelmäßiger
  • Bei kleinem Evaluationsbudget: Bayesian klar überlegen
Score-Verteilung (unten):
  • Bayesian wertet mehr Punkte nahe am Optimum aus
  • Random Search verschwendet Evaluationen in schlechten Regionen
Hinweis: skopt installieren via pip install scikit-optimize
