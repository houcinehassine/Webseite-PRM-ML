Servo Fault Classification

Ein realistisches Industriebeispiel: 300 Servomotoren, zwei Sensoren – und ein Random Forest, der nicht nur klassifiziert, sondern auch seine eigene Unsicherheit quantifiziert.

⚙️ Servomotoren 🌡️ Temperatur & Strom 🎲 predict_proba() ❓ Unsicherheit 🔍 Experten-Review

Lernfokus dieser Seite

  • Wie modelliert man ein 3-Klassen-Klassifikationsproblem mit überlappenden Klassen?
  • Was leistet .predict_proba() bei schwierig trennbaren Klassen?
  • Wie definiert man einen Unsicherheits-Schwellenwert und visualisiert ihn?
  • Wie nutzt man ML als Entscheidungsunterstützung für Fachexperten?

6.1.0 Szenario: Maschinenbauunternehmen

Das Ziel ist nicht nur eine Klassifikation – sondern eine quantitative Aussage darüber, wie sicher diese Klassifikation ist.

📋 Ausgangslage

  • 300 Servomotoren einer hochautomatisierten Produktionsanlage
  • Features: Temperatur & Strom – Wochenmittelwerte (keine Zeitreihendaten)
  • 3 bekannte Zustände: healthy, do_inspection, bad
  • Aus Erfahrung: Zustände sind oft schwierig voneinander zu unterscheiden

🎯 Aufgabe

  • Random Forest auf den Datensatz trainieren
  • .predict_proba() nutzen, um Klassenwahrscheinlichkeiten zu berechnen
  • Unsicherheit quantifizieren: Punkte mit max. Konfidenz < 75% hervorheben
  • Unsichere Punkte → Fachexperten zur Nachbeurteilung weiterleiten
Warum predict_proba() hier besonders wertvoll ist:
  • Ein normales .predict() liefert immer eine Klasse – auch wenn der Classifier nur 51% Konfidenz hat
  • .predict_proba() macht die Unsicherheit sichtbar: ein Punkt mit [0.34, 0.33, 0.33] ist faktisch unentschieden
  • Nur möglich, weil ein Ensemble aus vielen Bäumen natürlicherweise eine Stimmverteilung liefert

Schritt 1 – Datensatz erstellen & speichern

Der Datensatz wird mit make_blobs() synthetisch erzeugt – mit bewusst großen, überlappenden Clustern (cluster_std skaliert mit Faktor 1.7), um die realistische Schwierigkeit der Trennung zu simulieren. Die erste Spalte wird als Temperatursensor skaliert.

Python – Datensatz generieren & speichern 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_blobs

# Überlappende Cluster simulieren
cluster_std = np.array([1, 1.5, 2.5]) * 1.7
X, y = make_blobs(
    n_samples=300,
    n_features=2,
    centers=[[3, 3], [7, 5], [13, 15]],
    cluster_std=cluster_std,
    random_state=22)

# Feature-Skalierung: Spalte 0 = Temperatur
X[:, 0] = np.abs(X[:, 0]) * 5  # Temperatur (°C)

# Rohdaten visualisieren
fig, ax = plt.subplots(figsize=(6, 4))
scatter = ax.scatter(X[:, 0], X[:, 1],
                     c=y, cmap='viridis',
                     edgecolors='k',
                     linewidths=0.4, s=40)
plt.colorbar(scatter, ax=ax,
             label='Klasse (0=healthy, '
                   '1=do_inspection, 2=bad)')
ax.set_xlabel('Temperatur (skaliert)')
ax.set_ylabel('Strom (skaliert)')
ax.set_title('Servo-Motoren: Rohdaten\n'
             '(Überlappende Klassen sichtbar)')
ax.grid(alpha=0.3)
plt.tight_layout()
plt.show()

# Datensatz speichern
np.save('servo_engines.npy',
        np.concatenate(
            (X, y.reshape(-1, 1)), axis=1))
print("Datensatz gespeichert: servo_engines.npy")
print(f"Shape: {X.shape}, Klassen: {np.unique(y)}")
📊 Ausgabe & Interpretation
Was du siehst:
  • 3 Cluster – healthy, do_inspection, bad
  • Cluster überlappen stark – besonders zwischen healthy und do_inspection
  • cluster_std × 1.7 erzeugt bewusst unscharf trennbare Bereiche
Praxisbezug: Reale Sensordaten zeigen genau dieses Bild – Grenzfälle sind häufig und schwierig eindeutig zuzuordnen.

Schritt 2 – Laden, Splitten & Trainieren

Der gespeicherte Datensatz wird geladen, in Trainings- und Testset aufgeteilt und ein RandomForestClassifier trainiert. Anschließend werden die Klassenwahrscheinlichkeiten mit .predict_proba() berechnet.

Python – Laden, Splitten & Modell trainieren 
import numpy as np
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split

# Datensatz laden
X_y = np.load('servo_engines.npy')

# Aufteilen
X_train, X_test, y_train, y_test = train_test_split(
    X_y[:, :-1], X_y[:, -1],
    test_size=0.25, random_state=42)

# Modell trainieren
model = RandomForestClassifier(
    n_estimators=100, random_state=0)
model.fit(X_train, y_train)

acc = model.score(X_test, y_test)
print(f"Test Accuracy: {acc:.4f}")
print(f"Klassen: {model.classes_}")

# Klassenwahrscheinlichkeiten
prediction_probabilities = model.predict_proba(X_test)
print(f"\nShape predict_proba: "
      f"{prediction_probabilities.shape}")
print("\nErste 5 Zeilen (Wahrscheinlichkeiten):")
for i in range(5):
    p = prediction_probabilities[i]
    print(f"  {['healthy','do_insp','bad'][int(y_test[i])]:<12}"
          f" → h:{p[0]:.2f}  d:{p[1]:.2f}  b:{p[2]:.2f}"
          f"  max={p.max():.2f}")
📊 Ausgabe & Interpretation
predict_proba() Ausgabe:
  • Matrix: [75 × 3] (75 Testsamples, 3 Klassen)
  • Jede Zeile summiert zu 1.0
  • max(axis=1) = maximale Konfidenz des Modells für diesen Punkt
Beispiel: [0.40, 0.38, 0.22] → Modell tendiert zu healthy, aber mit nur 40% Konfidenz → unsicherer Punkt!

Schritt 3 – Unsicherheit analysieren & visualisieren

Die maximale Klassenwahrscheinlichkeit pro Datenpunkt gibt direkt an, wie sicher das Modell ist. Alles unter 75% gilt als unsicherer Grenzfall – ideal für einen Experten-Review.

Unsicherheits-Schwellenwert

  • max_prob = predict_proba.max(axis=1)
  • uncertain = max_prob < 0.75 → Boolean-Maske
  • Punkte mit max. Konfidenz < 75% = unsicher
  • Grenzwert ist frei wählbar: je nach Domäne und Fehlertoleranz z.B. 60%, 80%, 90%
  • Höherer Grenzwert → mehr Punkte zum Review → mehr Sicherheit, aber mehr Aufwand

🔍 Experten-Review Workflow

  • ML klassifiziert alle Punkte
  • Unsichere Punkte werden markiert & gefiltert
  • Fachexperten beurteilen nur diese Grenzfälle neu – spart Zeit und Ressourcen
  • Optional: Neubeurteilungen können als Trainingsdaten für das nächste Modell genutzt werden (Active Learning)

Der Code berechnet die maximale Konfidenz pro Testpunkt, zeigt deren Verteilung als Histogramm und hebt unsichere Punkte (max. Konfidenz < 75%) im Scatter-Plot farblich hervor – getrennt nach echter Klasse.

Python – Konfidenz-Histogramm & Unsicherheits-Scatter 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split

# Datensatz laden
X_y = np.load('servo_engines.npy')
X_train, X_test, y_train, y_test = train_test_split(
    X_y[:, :-1], X_y[:, -1],
    test_size=0.25, random_state=42)

model = RandomForestClassifier(
    n_estimators=100, random_state=0)
model.fit(X_train, y_train)
prediction_probabilities = model.predict_proba(X_test)

# Maximale Konfidenz pro Punkt
max_prediction_probabilities = (
    prediction_probabilities.max(axis=1))

THRESHOLD = 0.75
uncertain = max_prediction_probabilities < THRESHOLD

print(f"Unsichere Punkte: "
      f"{uncertain.sum()} / {len(uncertain)}"
      f" ({uncertain.mean()*100:.1f}%)")

# Visualisierung: vertikal (2 × 1)
fig, axes = plt.subplots(2, 1, figsize=(7, 10))

# Plot 1: Histogramm der max. Konfidenz
axes[0].hist(max_prediction_probabilities,
             bins=20, color='steelblue',
             edgecolor='white', alpha=0.85)
axes[0].axvline(THRESHOLD, color='red',
                linestyle='--', linewidth=2,
                label=f'Schwellenwert = {THRESHOLD}')
axes[0].set_xlabel('Maximale Klassenwahrscheinlichkeit')
axes[0].set_ylabel('Anzahl Testpunkte')
axes[0].set_title(
    'Verteilung der Vorhersage-Konfidenz\n'
    '(links der roten Linie = unsicher)')
axes[0].legend()
axes[0].grid(alpha=0.3)

# Plot 2: Scatter – unsichere Punkte hervorgehoben
colors_map = {False: 'steelblue', True: 'red'}
labels_map = {False: 'Sicher (≥75%)',
              True:  'Unsicher (<75%)'}
sizes_map  = {False: 30, True: 80}
markers_map = {False: 'o', True: 'X'}

for flag in [False, True]:
    mask = uncertain == flag
    axes[1].scatter(
        X_test[mask, 0],
        X_test[mask, 1],
        c=colors_map[flag],
        s=sizes_map[flag],
        marker=markers_map[flag],
        edgecolors='k',
        linewidths=0.4,
        alpha=0.85,
        label=labels_map[flag],
        zorder=3 if flag else 2)

axes[1].set_xlabel('Temperatur (skaliert)')
axes[1].set_ylabel('Strom (skaliert)')
axes[1].set_title(
    'Unsichere Testpunkte im Feature-Raum\n'
    '(rot ✕ = Konfidenz < 75% → Experten-Review)')
axes[1].legend()
axes[1].grid(alpha=0.3)

plt.tight_layout()
plt.show()
📊 Ausgabe & Interpretation
Histogramm (oben):
  • Viele Punkte rechts der roten Linie → hohe Konfidenz, klare Fälle
  • Punkte links der Linie → Grenzfälle
  • Peak nahe 1.0: Die meisten Punkte werden sehr sicher klassifiziert
Scatter-Plot (unten):
  • Blau ● = sicher klassifiziert
  • Rot ✕ = unsicher – befinden sich zwischen den Clustern
  • Unsichere Punkte häufen sich erwartungsgemäß in den Überlappungsbereichen

Schritt 4 – Vollständige Diagnose-Übersicht

Alle Perspektiven kombiniert: Entscheidungsgrenzen, Konfidenz-Heatmap, wahrer Zustand und Unsicherheitsverteilung in einem zusammenhängenden Plot.

Dieser Code erzeugt eine vollständige Diagnosegrafik: Die Entscheidungsgrenzen des Modells werden als Konfidenz-Heatmap über den Feature-Raum gelegt (Helligkeit = Sicherheit des Modells). Darüber werden alle Testpunkte mit ihrer wahren Klasse geplottet, unsichere Punkte zusätzlich mit einem roten Ring markiert.

Python – Diagnose-Übersicht mit Konfidenz-Heatmap 
import numpy as np
import matplotlib.pyplot as plt
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split

X_y = np.load('servo_engines.npy')
X_train, X_test, y_train, y_test = train_test_split(
    X_y[:, :-1], X_y[:, -1],
    test_size=0.25, random_state=42)

model = RandomForestClassifier(
    n_estimators=100, random_state=0)
model.fit(X_train, y_train)

proba = model.predict_proba(X_test)
max_proba = proba.max(axis=1)
uncertain  = max_proba < 0.75

# Grid für Konfidenz-Heatmap
x0_min, x0_max = X_y[:, 0].min()-2, X_y[:, 0].max()+2
x1_min, x1_max = X_y[:, 1].min()-2, X_y[:, 1].max()+2
xx, yy = np.meshgrid(
    np.linspace(x0_min, x0_max, 300),
    np.linspace(x1_min, x1_max, 300))
grid_proba = model.predict_proba(
    np.c_[xx.ravel(), yy.ravel()])
conf_grid  = grid_proba.max(axis=1).reshape(xx.shape)
pred_grid  = model.predict(
    np.c_[xx.ravel(), yy.ravel()]
).reshape(xx.shape)

class_labels = ['healthy', 'do_inspection', 'bad']
colors_cls   = ['#2ecc71', '#f39c12', '#e74c3c']

fig, axes = plt.subplots(2, 1, figsize=(7, 11))

# Plot 1: Konfidenz-Heatmap + Testpunkte
axes[0].contourf(xx, yy, conf_grid,
                 levels=20, cmap='Blues',
                 alpha=0.55)
axes[0].contour(xx, yy, pred_grid,
                levels=[0.5, 1.5],
                colors='gray', linewidths=0.8,
                linestyles='--', alpha=0.6)
for cls_idx, (label, col) in enumerate(
        zip(class_labels, colors_cls)):
    mask = y_test == cls_idx
    axes[0].scatter(
        X_test[mask, 0], X_test[mask, 1],
        color=col, s=40, edgecolors='k',
        linewidths=0.4, label=label, zorder=3)
# Unsichere Punkte mit rotem Ring
axes[0].scatter(
    X_test[uncertain, 0],
    X_test[uncertain, 1],
    s=130, facecolors='none',
    edgecolors='red', linewidths=2,
    label=f'Unsicher (<75%)', zorder=4)
axes[0].set_xlabel('Temperatur (skaliert)')
axes[0].set_ylabel('Strom (skaliert)')
axes[0].set_title(
    'Entscheidungsgrenzen & Konfidenz-Heatmap\n'
    '(heller = sicherer | roter Ring = unsicher)')
axes[0].legend(fontsize=8, loc='upper left')
axes[0].grid(alpha=0.2)

# Plot 2: Konfidenz pro Klasse (gestapeltes Bar)
cls_uncertain = [
    ((y_test == i) & uncertain).sum()
    for i in range(3)]
cls_certain = [
    ((y_test == i) & ~uncertain).sum()
    for i in range(3)]

x = np.arange(3)
w = 0.4
axes[1].bar(x - w/2, cls_certain,
            width=w, color=colors_cls,
            alpha=0.85, edgecolor='white',
            label='Sicher')
axes[1].bar(x + w/2, cls_uncertain,
            width=w, color=colors_cls,
            alpha=0.4, edgecolor='red',
            linewidth=1.5, label='Unsicher')
axes[1].set_xticks(x)
axes[1].set_xticklabels(class_labels)
axes[1].set_ylabel('Anzahl Testpunkte')
axes[1].set_title(
    'Unsichere Punkte pro Klasse\n'
    '(transparent = unsicher, voll = sicher)')
axes[1].legend()
axes[1].grid(axis='y', alpha=0.3)

plt.tight_layout()
plt.show()

# Zusammenfassung
print("\n── Diagnose-Report ──────────────────────")
for i, label in enumerate(class_labels):
    total   = (y_test == i).sum()
    unc_n   = cls_uncertain[i]
    print(f"  {label:<15}: "
          f"{total:3d} Punkte | "
          f"{unc_n:2d} unsicher "
          f"({unc_n/total*100:.0f}%)")
📊 Ausgabe & Interpretation
Konfidenz-Heatmap (oben):
  • Hellblau = Modell ist sicher (weit weg von Grenzen)
  • Weiß/hellgrau = Grenzbereich, Modell unsicher
  • Rote Ringe liegen erwartungsgemäß genau in diesen Grenzbereichen
Balkendiagramm (unten):
  • do_inspection hat typischerweise die meisten unsicheren Punkte → liegt zwischen healthy und bad
  • Zeigt, welche Klasse am härtesten trennbar ist → Handlungsbedarf für Experten
Diagnose-Report: Prozentualer Anteil unsicherer Punkte pro Klasse – direkt als Entscheidungsgrundlage für den Review-Aufwand nutzbar.
So what – Mehrwert von predict_proba() in der Praxis:
  • ML klassifiziert automatisch – ohne Expertenwissen für klare Fälle nötig
  • Unsichere Fälle werden gezielt herausgefiltert und an Experten weitergeleitet
  • Ergebnis: optimales Mensch-Maschine-Zusammenspiel
  • Der Schwellenwert von 75% ist anwendungsspezifisch: bei sicherheitskritischen Maschinen eher 90%+ wählen
