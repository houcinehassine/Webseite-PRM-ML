Preprocessing & Pipelines

Bevor ein Modell trainiert wird, müssen Daten vorbereitet werden: skalieren, kodieren, Features auswählen, Dimension reduzieren. Pipelines verketten das sauber und verhindern Data Leakage.

StandardScaler OneHotEncoder PCA make_pipeline fit/transform

Worum geht's?

Alle Vorverarbeitungs-Werkzeuge des Kurses — plus die zentrale fit/transform-Regel gegen Data Leakage.

4.1 Skalierung

Features auf vergleichbare Größenordnungen bringen.

StandardScaler sklearn.preprocessing

Standardisiert Features auf Mittelwert 0 und Std 1. Wichtig für distanzbasierte Verfahren (SVM, KNN, KMeans).

from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)   # nur transform!
Im Kurs: Vor SVM/KNN/Clustering. fit nur auf Trainingsdaten — sonst Data Leakage.
MinMaxScaler sklearn.preprocessing

Skaliert Features linear in den Bereich [0, 1].

from sklearn.preprocessing import MinMaxScaler
X_s = MinMaxScaler().fit_transform(X)
Im Kurs: Skalierungseinfluss auf Clustering zeigen (Blatt 04), Inertia normalisieren.
PolynomialFeatures / PowerTransformer sklearn.preprocessing

PolynomialFeatures erzeugt Polynom-/Interaktionsterme; PowerTransformer macht Verteilungen normaler.

from sklearn.preprocessing import PolynomialFeatures
X_poly = PolynomialFeatures(degree=2).fit_transform(X)
Im Kurs: Nichtlineare Regression, Feature-Engineering.

4.2 Encoding

Kategoriale Daten in Zahlen umwandeln.

LabelEncoder sklearn.preprocessing

Wandelt kategoriale Labels in ganze Zahlen um (z.B. 'Hund'→0). Achtung: erzeugt eine künstliche Ordnung.

from sklearn.preprocessing import LabelEncoder
enc = LabelEncoder()
y_enc = enc.fit_transform(['a', 'b', 'a', 'c'])  # [0 1 0 2]
Im Kurs: Zielvariablen kodieren (Kapitel 1).
OneHotEncoder sklearn.preprocessing

Wandelt jede Kategorie in eine eigene 0/1-Spalte — ohne künstliche Ordnung.

from sklearn.preprocessing import OneHotEncoder
ohe = OneHotEncoder(sparse_output=False)
X_ohe = ohe.fit_transform(kategorien.reshape(-1, 1))
Im Kurs: Kategoriale Features ohne Reihenfolge (Kapitel 1), bevorzugt gegenüber LabelEncoder.
label_binarize sklearn.preprocessing

Wandelt Mehrklassen-Labels in eine binäre One-vs-Rest-Matrix — Basis für Multi-Class-ROC.

from sklearn.preprocessing import label_binarize
Y = label_binarize(y, classes=[0, 1, 2])
Im Kurs: Multi-Class ROC-Kurven (Kapitel 7).

4.3 Feature-Auswahl & -Extraktion

Die richtigen Features behalten.

VarianceThreshold sklearn.feature_selection

Entfernt Features mit zu geringer Varianz (z.B. fast konstante Spalten).

from sklearn.feature_selection import VarianceThreshold
X_sel = VarianceThreshold(threshold=0.01).fit_transform(X)
Im Kurs: Uninformative Features entfernen.
SelectKBest + mutual_info_classif sklearn.feature_selection

Wählt die k Features mit dem höchsten Zusammenhang zum Ziel (hier: Transinformation).

from sklearn.feature_selection import SelectKBest, mutual_info_classif
X_best = SelectKBest(mutual_info_classif, k=10).fit_transform(X, y)
Im Kurs: Feature-Auswahl vor dem Training.
CountVectorizer / FeatureHasher sklearn.feature_extraction

Wandeln Text/kategoriale Daten in numerische Vektoren (Bag-of-Words bzw. Hashing-Trick).

from sklearn.feature_extraction.text import CountVectorizer
X = CountVectorizer().fit_transform(texte)
Im Kurs: Text-Features (Bag-of-Words, Kapitel 1).

4.4 Dimensionsreduktion

Viele Features auf wenige verdichten.

PCA sklearn.decomposition

Hauptkomponentenanalyse — reduziert die Dimension, indem Richtungen größter Varianz behalten werden. whiten=True normiert die Komponenten.

from sklearn.decomposition import PCA
pca = PCA(n_components=0.95, whiten=True)   # 95% Varianz
X_pca = pca.fit_transform(X)
print(pca.n_components_)
X_back = pca.inverse_transform(X_pca)       # zurück in Pixelraum
Im Kurs: Dimensionsreduktion vor GMM-Bildgenerator (Blatt 06), Gesichtserkennung.
TSNE sklearn.manifold

Nichtlineare Dimensionsreduktion auf 2D — gut zur Visualisierung von Clustern (nur zum Plotten, nicht zum Trainieren).

from sklearn.manifold import TSNE
X_2d = TSNE(n_components=2).fit_transform(X)
plt.scatter(X_2d[:,0], X_2d[:,1], c=y)
Im Kurs: Hochdimensionale Daten visualisieren (Kapitel 1).

4.5 Pipelines & die fit/transform-Regel

Sauber verketten, Leakage vermeiden.

make_pipeline / Pipeline sklearn.pipeline

Verkettet Vorverarbeitung + Modell zu einem Objekt. Bei Cross-Validation wird der Scaler in jedem Fold neu gefittet → verhindert Data Leakage.

from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVC
pipe = make_pipeline(StandardScaler(), SVC())
pipe.fit(X_train, y_train)
# benannt:
Pipeline([('scaler', StandardScaler()), ('svc', SVC())])
Im Kurs: Scaler+Modell sauber kombinieren (Blatt 03/05), GridSearch mit svc__C.
fit vs. transform vs. fit_transform Konzept

fit lernt Parameter (z.B. Mittelwert), transform wendet sie an, fit_transform macht beides. Test-Daten nur transformieren!

scaler.fit(X_train)          # lernt mean/std aus Train
X_tr = scaler.transform(X_train)
X_te = scaler.transform(X_test)   # gleiche mean/std
# Kurzform für Training:
X_tr = scaler.fit_transform(X_train)
Im Kurs: Die wichtigste Regel gegen Data Leakage im ganzen Kurs.
