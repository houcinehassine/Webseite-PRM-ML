7.1 Splits & Cross-Validation

Daten fair aufteilen und robust bewerten.

train_test_split sklearn.model_selection

Teilt Daten zufällig in Trainings- und Testset. stratify=y erhält die Klassenverteilung.

  • test_size — Anteil Testdaten (z.B. 0.3)
  • stratify — Klassenverteilung erhalten
  • random_state — reproduzierbar
from sklearn.model_selection import train_test_split
X_tr, X_te, y_tr, y_te = train_test_split(
    X, y, test_size=0.3, stratify=y, random_state=42)
Im Kurs: Standard-Aufteilung in jedem Klassifikationsbeispiel.
cross_val_score + KFold sklearn.model_selection

Bewertet ein Modell über k Folds — robuster als ein einzelner Split. KFold steuert die Aufteilung.

from sklearn.model_selection import cross_val_score, KFold
scores = cross_val_score(model, X, y, cv=5)
print(scores.mean(), scores.std())
kf = KFold(n_splits=5, shuffle=True, random_state=42)
Im Kurs: Robuste Modellbewertung (Blatt 02/05), eigene CV implementieren.
LeaveOneOut sklearn.model_selection

Extremfall der Cross-Validation: jeder einzelne Punkt ist einmal das Testset.

from sklearn.model_selection import LeaveOneOut
cross_val_score(model, X, y, cv=LeaveOneOut())

7.2 Hyperparameter-Suche

Die beste Konfiguration finden.

GridSearchCV sklearn.model_selection

Probiert systematisch alle Hyperparameter-Kombinationen per Cross-Validation und merkt sich die beste.

  • param_grid — Dict der zu testenden Werte
  • best_params_ — beste Kombination
  • cv — Anzahl Folds
  • n_jobs=-1 — alle CPU-Kerne
from sklearn.model_selection import GridSearchCV
par = {'C': [1, 10, 100], 'gamma': [0.01, 0.1]}
grid = GridSearchCV(SVC(), par, cv=5, n_jobs=-1, verbose=1)
grid.fit(X_train, y_train)
print(grid.best_params_, grid.best_score_)
Im Kurs: Hyperparameter-Optimierung (Kapitel 3, Blatt 03/05).
RandomizedSearchCV sklearn.model_selection

Wie GridSearch, testet aber nur eine zufällige Auswahl — schneller bei großen Suchräumen.

from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint
rs = RandomizedSearchCV(rf, {'n_estimators': randint(50, 300)},
                        n_iter=20, cv=5)
rs.fit(X, y)
Im Kurs: Effiziente Suche, kombiniert mit Verteilungen aus scipy.stats.
BayesSearchCV skopt

Bayes'sche Optimierung — wählt die nächsten Kombinationen klug statt zufällig.

from skopt import BayesSearchCV
from skopt.space import Integer
opt = BayesSearchCV(rf, {'n_estimators': Integer(50, 300)})

7.3 Validierungs- & Lernkurven

Over-/Underfitting sichtbar machen.

validation_curve sklearn.model_selection

Zeigt Train- vs. Validierungs-Score als Funktion EINES Hyperparameters — deckt Over-/Underfitting auf.

from sklearn.model_selection import validation_curve
tr, val = validation_curve(SVC(), X, y,
    param_name='C', param_range=[0.1, 1, 10, 100], cv=5)
Im Kurs: Wirkung eines Hyperparameters analysieren.
learning_curve sklearn.model_selection

Zeigt Score als Funktion der Trainingsgröße — zeigt, ob mehr Daten helfen würden.

from sklearn.model_selection import learning_curve
sizes, tr, val = learning_curve(model, X, y,
    train_sizes=np.linspace(0.1, 1.0, 9), cv=5)
