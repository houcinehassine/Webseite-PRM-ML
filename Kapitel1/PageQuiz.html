Frage 1

Welche drei Eigenschaften müssen gute Features laut Vorlesung haben?

✅ Richtig! I-D-U: Informativ (trägt Information über das Phänomen), Diskriminativ (erklärt Unterschiede zwischen Klassen), Unabhängig (geringe Korrelation untereinander → kein Redundanz). Merksatz: I-D-U.
❌ Falsch — Richtige Antwort: B Die drei Eigenschaften sind Informativ, Diskriminativ und Unabhängig (Merksatz: I-D-U). Fehlt eine davon, leidet die Modellgüte. → Seite 2 nachschlagen
Frage 2

Was bedeutet die Schreibweise X ∈ ℝᴺˣᴷ für die Feature-Matrix?

✅ Richtig! Jede Zeile = ein Sample (Beobachtungspunkt), jede Spalte = ein Feature. X.shape → (N, K). Typischer Fehler: N und K verwechseln.
❌ Falsch — Richtige Antwort: B N = Anzahl Samples (Zeilen), K = Anzahl Features (Spalten). Nicht umgekehrt! → Seite 2 nachschlagen
Frage 3

Was ist die korrekte Aussage über den Feedback-Loop zwischen Features und Modell?

✅ Richtig! Feature Engineering ist ein iterativer Prozess: Features → Modell trainieren → Analyse → neue/bessere Features → ... Es ist kein einmaliger Schritt!
❌ Falsch — Richtige Antwort: C Der Feedback-Loop bedeutet: beide beeinflussen sich. Man optimiert nicht nur das Modell, sondern auch die Features. Feature Engineering ist iterativ. → Seite 2 nachschlagen
Frage 4

Welche formale Notation beschreibt Feature Extraction aus Zeitserien korrekt?

✅ Richtig! Die Eingabe ist eine Zeitserie mit T Messpunkten (ℝᵀ), die Ausgabe ein kompaktes Feature-Vektor mit K Skalaren (ℝᴷ). Ziel: K ≪ T (stark komprimiert).
❌ Falsch — Richtige Antwort: B f : ℝᵀ → ℝᴷ — Eingabe T Zeitschritte, Ausgabe K Features. Das Ziel ist Kompression (K ≪ T). → Seite 4 nachschlagen
Frage 5

Wofür sollte t-SNE laut Vorlesung nicht verwendet werden?

✅ Richtig! t-SNE ist ein reines Visualisierungswerkzeug. Die projizierten 2D-Koordinaten sind nicht für das Modelltraining geeignet — sie sind nicht deterministisch und die globalen Abstände sind nicht interpretierbar.
❌ Falsch — Richtige Antwort: C t-SNE = NUR Visualisierung! Die Ausgabe darf nicht als Feature für Modelle verwendet werden. Für Feature Extraction: PCA oder Autoencoder. → Seite 3 nachschlagen
🔤 Encoding Encoding & Text Features Fragen 6–8
Frage 6

Was gibt folgender Code aus?

from sklearn.preprocessing import LabelEncoder import numpy as np categories = np.array(['apple', 'milk', 'banana', 'apple']) enc = LabelEncoder() print(enc.fit_transform(categories))
✅ Richtig! LabelEncoder sortiert alphabetisch: apple=0, banana=1, milk=2. Daher: apple→0, milk→2, banana→1, apple→0. Genau das Problem: das Modell liest apple < banana < milk als falsche Ordnung!
❌ Falsch — Richtige Antwort: B LabelEncoder sortiert alphabetisch: apple=0, banana=1, milk=2. Ergebnis: [0, 2, 1, 0]. Nicht nach Reihenfolge im Array! → Seite 5 nachschlagen
Frage 7

Zwei Dokumente werden mit CountVectorizer kodiert:
Doc 1: "the dog bites the man"
Doc 2: "the man bites the dog"
Was ist korrekt?

✅ Richtig! Bag-of-Words ignoriert die Wortstellung komplett. Beide Sätze enthalten exakt die gleichen Wörter in den gleichen Häufigkeiten → identische Vektoren. Das ist die zentrale Schwäche von BoW.
❌ Falsch — Richtige Antwort: B Bag-of-Words ignoriert Wortstellung! "dog bites man" = "man bites dog" → gleicher Vektor. Für reihenfolge-sensiblen Text: Word Embeddings oder Transformer. → Seite 5 nachschlagen
Frage 8

Du hast eine Spalte "Farbe" mit Werten: rot, grün, blau (keine natürliche Ordnung). Welcher Encoder ist korrekt?

✅ Richtig! Bei nominalen Variablen (keine Ordnung) immer OneHotEncoder. LabelEncoder würde blau < grün < rot implizieren — was falsch ist.
❌ Falsch — Richtige Antwort: C OneHotEncoder für nominale Kategorien ohne Ordnung. LabelEncoder ist nur bei echten ordinalen Variablen (klein/mittel/groß) korrekt. → Seite 5 nachschlagen
⚙️ sklearn API sklearn API & Data Leakage Fragen 9–11
Frage 9

Was ist an folgendem Code falsch?

scaler = StandardScaler() X_train_s = scaler.fit_transform(X_train) X_test_s = scaler.fit_transform(X_test) # ← Zeile 3
✅ Richtig! Data Leakage! fit_transform(X_test) berechnet Mittelwert und Standardabweichung der Testdaten und verwendet diese für die Transformation. Das Modell "sieht" Testdaten indirekt. Korrekt: scaler.transform(X_test)
❌ Falsch — Richtige Antwort: C Data Leakage: fit_transform(X_test) lernt von Testdaten. Richtig: scaler.transform(X_test) — nur transformieren, nicht neu lernen! → Seite 5 nachschlagen
Frage 10

Welche Aussage über .fit(), .transform() und .fit_transform() ist korrekt?

✅ Richtig! fit() = lernen (nur Training), transform() = anwenden (Training + Test), fit_transform() = beides kombiniert aber NUR auf Trainingsdaten!
❌ Falsch — Richtige Antwort: C fit() lernt, transform() wendet an, fit_transform() = beides — aber nur Training! → Seite 5 nachschlagen
Frage 11

Jemand skaliert den gesamten Datensatz vor dem Train/Test-Split. Was ist das Problem?

✅ Richtig! Wenn vor dem Split skaliert wird, berechnet fit() Statistiken über alle Daten inklusive Testdaten. Das ist Data Leakage. Richtig: erst splitten, dann fit() nur auf X_train.
❌ Falsch — Richtige Antwort: C Immer erst splitten, dann skalieren! Skalierung vor dem Split ist Data Leakage. → Cheatsheet: Typische Fallen
🎯 Selection Feature Selection Fragen 12–15
Frage 12

Was beschreibt der „Curse of Dimensionality" am besten?

✅ Richtig! Mit jedem Feature wächst der Raum exponentiell. Für gleiche Datendichte braucht man 10ᴷ Samples. Algorithmen wie k-NN versagen, weil alle Abstände gleich werden.
❌ Falsch — Richtige Antwort: C Mit steigendem K wächst der Raum exponentiell → Daten werden spärlich (Sparsity). k-NN leidet besonders. → Seite 7 nachschlagen
Frage 13

Was ist X_new.shape nach diesem Code?

from sklearn.datasets import load_digits from sklearn.feature_selection import SelectKBest, mutual_info_classif X, y = load_digits(return_X_y=True) sel = SelectKBest(mutual_info_classif, k=20) X_new = sel.fit_transform(X, y) print(X_new.shape)
✅ Richtig! SelectKBest reduziert Spalten (Features), nicht Zeilen (Samples). Digits: 1797 Samples × 64 Features → 1797 × 20. Die 20 informativsten Pixel-Positionen werden behalten.
❌ Falsch — Richtige Antwort: C SelectKBest entfernt Features (Spalten): 64 → 20. Samples bleiben alle: 1797. Ergebnis: (1797, 20). → Seite 7 nachschlagen
Frage 14

Welche SelectKBest Scoring-Funktion ist die Standard-Wahl für Klassifikation, wenn der Zusammenhang auch nicht-linear sein könnte?

✅ Richtig! mutual_info_classif ist die sicherste Wahl — es erkennt auch nicht-lineare Abhängigkeiten. f_classif ist schneller aber nur linear. chi2 nur für nicht-negative Features!
❌ Falsch — Richtige Antwort: C mutual_info_classif = Standard-Wahl. Erkennt auch nicht-lineare Zusammenhänge, anders als f_classif (nur linear) oder chi2 (nur positive Features). → Seite 7 nachschlagen
Frage 15

X.var(axis=0) gibt [0.139, 0.222, 0.250] zurück. Was macht VarianceThreshold(threshold=0.15).fit_transform(X)?

✅ Richtig! Feature 1 hat Varianz 0.139 < threshold 0.15 → wird entfernt. Features 2 (0.222) und 3 (0.250) bleiben. Aus (N×3) wird (N×2).
❌ Falsch — Richtige Antwort: B Feature 1 (var=0.139) liegt unter threshold=0.15 → wird entfernt. Ergebnis: 2 statt 3 Spalten. → Seite 7 nachschlagen
⚖️ Scaling Feature Scaling & Normalisierung Fragen 16–18
Frage 16

Für welche Algorithmen ist Feature Scaling in der Regel nicht erforderlich?

✅ Richtig! Baum-basierte Algorithmen (Decision Tree, Random Forest, XGBoost) sind nicht skalensensitiv — sie teilen Daten anhand von Schwellwerten, nicht Abständen.
❌ Falsch — Richtige Antwort: D Baum-basierte Algorithmen brauchen kein Scaling. k-NN, SVM, PCA, Neuronale Netze und Gradient-Descent-Modelle sind skalensensitiv! → Seite 8 nachschlagen
Frage 17

Wann sollte PowerTransformer(method='box-cox') nicht verwendet werden?

✅ Richtig! Box-Cox funktioniert nur für strikt positive Werte (X > 0). Für Daten mit X ≤ 0: method='yeo-johnson' verwenden.
❌ Falsch — Richtige Antwort: C Box-Cox nur für X > 0! Bei negativen Werten oder Null: yeo-johnson. → Seite 8 nachschlagen
Frage 18

Was sind nach Anwenden von StandardScaler die Eigenschaften jedes Features (aus dem Notebook)?

✅ Richtig! StandardScaler transformiert auf μ≈0, σ=1. Im Notebook: Vorher μ=0.473, σ=0.285 → Nachher μ=1.42e-16 (≈0), σ=1.0. Formel: Z = (X−μ)/σ.
❌ Falsch — Richtige Antwort: C Z-Score: μ=0, σ=1. Im Notebook: Vorher 0.473/0.285, Nachher ≈0/1.0. → Seite 8 nachschlagen
📐 Formeln Rechenaufgaben Fragen 19–20
Frage 19

Z-Score Berechnung: X = 11, μ = 8, σ = 3.
Formel: Z = (X − μ) / σ

✅ Richtig! Z = (11 − 8) / 3 = 1.0. X=11 liegt genau 1 Standardabweichung über dem Mittelwert μ=8. Positiver Z-Score = über dem Durchschnitt.
❌ Falsch — Richtige Antwort: B Z = (X − μ) / σ = (11 − 8) / 3 = 1.0. Reihenfolge: zuerst X − μ, dann durch σ dividieren. → Seite 8 nachschlagen
Frage 20

MinMax-Transformation: X = 4, X_min = 0, X_max = 10, Zielbereich [0, 1].

✅ Richtig! X' = (X − X_min) / (X_max − X_min) = (4−0)/(10−0) = 4/10 = 0.4. Vorsicht: Option A gibt zufällig denselben Wert, hat aber die falsche Formel (X/X_max statt (X−Xmin)/(Xmax−Xmin)).
❌ Falsch — Richtige Antwort: B Korrekte Formel: (X − X_min) / (X_max − X_min) = (4−0)/(10−0) = 0.4. Option C ist (X_max − X) / Bereich — das ist die falsche Richtung! → Seite 8 nachschlagen
