Feature-Raum & Visualisierung

Eine Feature-Matrix ist mehr als eine Tabelle – sie ist ein geometrischer Raum. Mit t-SNE lassen sich selbst 64-dimensionale Daten auf 2D projizieren, ohne die lokale Struktur zu verlieren.

Feature-Raum K-dimensionaler Raum 3D-Beispiel t-SNE Digits-Datensatz Dimensionsreduktion TF Projector

Lernfokus dieser Seite

Du verstehst, dass jeder Datenpunkt ein Punkt im K-dimensionalen Feature-Raum ist, kennst die Intuition hinter t-SNE und kannst erklären, warum die Digits (8×8 Pixel = 64 Features) mit t-SNE in 2D sichtbar geclustert werden.

3.1 Von der Feature-Matrix in den Feature-Raum

Die gleichen Daten – einmal als Matrix, einmal als geometrischer Raum. Beide Darstellungen sind äquivalent.

1D 1 Feature → Zahlenstrahl
z. B. Temperatur
→
2D 2 Features → Koordinatensystem
z. B. Temp + Druck
→
3D 3 Features → Raum mit 3 Achsen
→ Beispiel unten ↓
→
K-dim K Features → Nicht mehr visualisierbar
→ t-SNE hilft ↓

Datenpunkt = Vektor

Jede Zeile der Feature-Matrix wird zu einem Punkt (Vektor) im K-dimensionalen Raum. Die K Feature-Werte sind die Koordinaten dieses Punktes.

Warum ist das nützlich?

Im Feature-Raum sehen wir Cluster, Abstände und Strukturen, die in der rohen Tabelle unsichtbar sind. Machine Learning Algorithmen operieren in diesem Raum – z. B. findet k-Means Cluster, SVM trennt Klassen durch Hyperebenen.

Kernidee:   Feature Engineering = den Feature-Raum so formen, dass ähnliche Datenpunkte nahe beieinander liegen und verschiedene Klassen trennbar werden.

3.2 Konkretes Beispiel: Datenpunkt im 3D-Raum

Drei Features – drei Achsen. Ein Datenpunkt ist ein Punkt im 3-dimensionalen Feature-Raum.

Datenpunkt x⃗ = (6, 3.5, 1)
x₁ Temperatur (°C) 6.0
x₂ Luftdruck (hPa, normiert) 3.5
x₃ Windgeschwindigkeit (m/s) 1.0
Relative Ausprägung der Features
x₁ · Temperatur
6.0
x₂ · Luftdruck
3.5
x₃ · Windgeschw.
1.0

Ein zweiter Datenpunkt (Messung) hätte andere Koordinaten und würde an einem anderen Ort im 3D-Raum liegen.

Verallgemeinerung:   Mit K Features hat man einen K-dimensionalen Raum. Bei K > 3 ist er nicht mehr direkt visualisierbar – aber geometrische Konzepte wie Abstände, Cluster und Hyperebenen gelten weiterhin. Genau hier hilft t-SNE ↓

3.3 t-SNE – Hochdimensional sichtbar machen

t-distributed Stochastic Neighbor Embedding – eine nicht-lineare Methode, die hochdimensionale Daten auf 2D oder 3D projiziert, ohne die lokale Nachbarschaftsstruktur zu zerstören.

1

Ähnlichkeiten im Hochdim.

t-SNE berechnet für jedes Paar von Datenpunkten im hochdimensionalen Raum eine Ähnlichkeitswahrscheinlichkeit (Gaußsche Verteilung der Abstände).

2

Projektion in 2D / 3D

Im niedrigdimensionalen Zielraum werden die Punkte zufällig platziert und iterativ so verschoben, dass die Ähnlichkeitsstruktur erhalten bleibt (t-Verteilung für Abstände im Zielraum).

3

Lokale Struktur ≫ globale

Nahe Nachbarn im hochdim. Raum bleiben nahe Nachbarn in 2D. Globale Abstände zwischen Clustern sind in der t-SNE-Visualisierung nicht direkt interpretierbar.

4

Nur für Visualisierung

t-SNE ist ein Visualisierungswerkzeug, keine Feature Extraction für Modelle. Die projizierten 2D-Koordinaten sind nicht für Training geeignet.

Hyperparameter: Perplexity

Der wichtigste Parameter in t-SNE. Er steuert, wie viele Nachbarn pro Punkt berücksichtigt werden. Typischer Bereich: 5 – 50. Kleinere Werte → feine lokale Struktur, größere Werte → grobe globale Struktur.

Achtung: Nicht-deterministisch

Verschiedene Läufe erzeugen verschiedene Visualisierungen (deshalb random_state setzen). Cluster-Formen und -Größen in t-SNE sind bedeutungslos – nur die Gruppierung zählt.

3.4 Beispiel: Embedding Digits (64-dim → 2D)

Der Digits-Datensatz aus sklearn enthält handgeschriebene Ziffern (0–9) als 8×8 Pixel-Bilder – jedes Bild hat 64 Features.

8 × 8 Pixel = 64 Features
🖼️
Eingabe: 8 × 8 Pixel-Bild
Jedes Bild = eine Ziffer (0–9). Graustufenwerte von 0–16.
8 × 8
➡️
Flatten: Matrix → Vektor
8 × 8 Pixel-Matrix wird zu einem 64-dimensionalen Vektor.
= 64 dim
🧭
t-SNE: 64-dim → 2D
t-SNE projiziert alle 1797 Vektoren in 2D. Gleiche Ziffern clustern zusammen.
→ 2D
📍
Ergebnis: 10 sichtbare Cluster
Jede Ziffer (0–9) bildet im 2D-Raum eine eigene Gruppe. Strukturen werden sichtbar.
10 Cluster

Code aus 01_data_preparation_core.ipynb

Python – Digits laden 
from sklearn.datasets import load_digits

# Datensatz laden
X, y = load_digits(return_X_y=True)

# Dimensionen prüfen
print(X.shape)   # (N_samples, N_features)
print(y.shape)   # Labels: Ziffern 0–9
Output 
(1797, 64)
(1797,)
Python – t-SNE anwenden & visualisieren 
from sklearn.manifold import TSNE
import matplotlib.pyplot as plt

# t-SNE: 64-dim → 2D
tsne  = TSNE(n_components=2, random_state=42, perplexity=30)
X_2d  = tsne.fit_transform(X)

print(X_2d.shape)   # (1797, 2) – jetzt 2D

# Visualisierung: Jede Ziffer bekommt eine Farbe
plt.figure(figsize=(8, 6))
scatter = plt.scatter(X_2d[:, 0], X_2d[:, 1],
                      c=y, cmap='tab10', alpha=0.7, s=10)
plt.colorbar(scatter, label='Ziffer (0–9)')
plt.title('t-SNE der Digits – 64-dim → 2D')
plt.xlabel('t-SNE Achse 1')
plt.ylabel('t-SNE Achse 2')
plt.show()
Ergebnis:   Im 2D-Plot bildet jede Ziffer einen eigenen Cluster – obwohl das Modell nur 64 Graustufenwerte sieht. Das zeigt: Die Feature-Matrix enthält bereits genug Struktur, um die 10 Klassen zu trennen.
