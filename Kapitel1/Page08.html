8.2 MinMax-Transformation

Skaliert alle Werte auf einen fest definierten Wertebereich [min, max] – standardmäßig [0, 1].

MinMax-Formel
X' = (X − Xmin) / (Xmax − Xmin) × (max − min) + min
X Originalwert
Xmin, Xmax Min/Max der Trainingsdaten
min, max Ziel-Wertebereich
X' Skalierter Wert
Python – MinMax From Scratch + MinMaxScaler (aus Notebook) 
import numpy as np
from sklearn.preprocessing import MinMaxScaler

# Zufällige Daten (10 Samples, 5 Features)
X = np.random.randn(10, 5)

# ── Variante A: From Scratch ───────────────────────
max_value = 5
min_value = -3

# Schritt 1: Auf [0, 1] normieren
X_std = (X - X.min(axis=0)) / (X.max(axis=0) - X.min(axis=0))

# Schritt 2: Auf [min_value, max_value] skalieren
X_scaled = X_std * (max_value - min_value) + min_value

# ── Variante B: MinMaxScaler ───────────────────────
scaler = MinMaxScaler(feature_range=(min_value, max_value))
scaler.fit_transform(X)   # Gleiche Ausgabe wie X_scaled
Output (Ausschnitt) 
array([[-2.87,  4.03,  1.72, -1.84, -1.54],
       [ 5.  ,  1.86, -0.50,  0.61, -1.34],
       [ 0.88,  0.67,  0.79,  3.79,  1.05],
       [ 2.41,  0.27,  5.  , -0.58,  5.  ],
       ...
       [-3.  , -3.  ,  2.32, -1.53, -3.  ],
       [ 0.39,  0.51,  1.38, -1.67, -2.94]])

# Jede Spalte liegt nun exakt in [-3, 5]

Vorteil

Klarer, interpretierbarer Wertebereich. Gut für Algorithmen, die Eingaben in [0, 1] erwarten (z. B. Neuronale Netze mit Sigmoid, k-NN wenn gleiche Gewichtung gewünscht).

Nachteil: Ausreißer

Ein einzelner extremer Ausreißer „streckt" die Skala – alle anderen Werte werden auf einen engen Bereich gedrückt. Lösung: RobustScaler (skaliert anhand von Quartilen).

8.3 Standardisierung (Z-Score)

Transformiert Features auf Mittelwert 0 und Standardabweichung 1 – die robusteste und meistgenutzte Methode.

Z-Score Formel
Z = (X − μ) / σ
X Originalwert
μ Mittelwert (Trainingsdaten)
σ Standardabweichung (Training)
Z Standardisierter Wert
"Standardization of datasets is a common requirement for many machine learning estimators implemented in scikit-learn; they might behave badly if the individual features do not more or less look like standard normally distributed data: Gaussian with zero mean and unit variance."

Python – StandardScaler (aus Notebook) 
from sklearn.preprocessing import StandardScaler

# Daten (200 Samples, 1 Feature) – uniform verteilt
X = np.random.rand(200, 1)

# Vor der Skalierung
print(X.mean())   # 0.4732...
print(X.std())    # 0.2849...

# Skalieren
scaler = StandardScaler()
X = scaler.fit_transform(X)

# Nach der Skalierung
print(X.mean())   # ≈ 0 (1.42e-16)
print(X.std())    # 1.0
Output

8.4 Power Transformations (Box-Cox)

Für Daten mit stark schiefer Verteilung – transformiert in eine normalverteilungsähnliche Form, damit Algorithmen, die Normalverteilung voraussetzen, besser funktionieren.

Vor Box-Cox
Log-Normal – rechtsschiefe Verteilung
z. B. Gehälter, Hauspreise
→ Box-Cox
λ ≈ 0.006
Nach Box-Cox
Annähernd normalverteilt
nach PowerTransformer(Box-Cox)
Python – PowerTransformer Box-Cox (aus Notebook) 
from sklearn.preprocessing import PowerTransformer

# Log-normal verteilte Daten (typisch schief)
X = np.random.lognormal(size=(2000, 1))

# Vor der Transformation: stark rechtsschiefe Verteilung
plt.hist(X, bins=100); plt.show()

# Box-Cox Transformation anwenden
pt = PowerTransformer(method='box-cox')
X_transformed = pt.fit_transform(X)

# Nach der Transformation: annähernd normalverteilt
plt.hist(X_transformed, bins=100); plt.show()

# Gelernter Box-Cox Parameter λ
print(pt.lambdas_)
Output 
# Box-Cox Parameter λ:
array([0.00583342])

# λ ≈ 0 entspricht einer
# Log-Transformation:
# X' ≈ log(X)
# (Box-Cox ist eine Verallgemeinerung)
Box-Cox Bedingung:   Box-Cox funktioniert nur mit positiven Werten (X > 0). Für Daten mit negativen Werten: PowerTransformer(method='yeo-johnson') (funktioniert auch mit Null und negativen Werten).

Bonus aus dem Notebook: Quantisierung

Diskretisierung kontinuierlicher Werte in Klassen – feste oder adaptive Klassengrenzen.

Feste Klassengrößen

Gleichmäßige Klassengrenzen auf einer logarithmischen oder linearen Skala. Geeignet für log-normalverteilte Daten (z. B. Einkommensverteilungen, Hauspreise). 

# Klassen: 10, 100, 1000, 10.000
plt.vlines([10, 100, 1000, 10_000],
           ymin=0, ymax=1.5e4, color='r')

Adaptive Klassengrößen (Dezile)

Quantile-basierte Klassengrenzen – jede Klasse enthält gleich viele Datenpunkte. Robust gegenüber Ausreißern und skewed distributions. 

# 10. bis 90. Perzentile als Grenzen
dezile = df.quantile(
    [.1, .2, .3, .4, .5, .6, .7, .8, .9]
)

8.5 Vergleich: Wann welche Methode?

Entscheidungshilfe für die Praxis.

Methode Wertebereich Ausreißer Verteilung Wann einsetzen?
MinMaxScaler [min, max]
fest definiert		 Sensibel Beliebig Wenn fester Ausgabebereich nötig (z. B. [0, 1] für Neural Nets), keine starken Ausreißer
StandardScaler Unbegrenzt
μ=0, σ=1		 Robuster Bevorzugt Normalvert. Standard-Wahl. Für SVM, PCA, Lineare Modelle, k-NN. Wenn Verteilung unbekannt.
RobustScaler Unbegrenzt
median=0		 Sehr robust Beliebig Wenn Ausreißer vorhanden und nicht entfernt werden können. Skalierung über IQR.
PowerTransformer Unbegrenzt
≈ Normalvert.		 Sensibel Schief → Normal Wenn Daten stark rechtsschiefe Verteilung haben. Box-Cox nur für positive Werte.
Keine Original — — Tree-based Algorithmen (Random Forest, Gradient Boosting, Decision Tree).
Goldene Regel:   .fit() immer nur auf den Trainingsdaten. Dann .transform() auf Training- und Testdaten separat. Nie .fit_transform() auf Testdaten → Data Leakage (gilt für alle sklearn-Scaler, genau wie für Encoder auf Seite 5).
