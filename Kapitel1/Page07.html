Feature Selection

Nicht alle Features sind nützlich. Feature Selection entscheidet, welche Features in das Modell eingehen – mit drei Methodenfamilien: Varianz, Korrelation und statistischen Tests.

7.1 Warum Feature Selection?

Drei Gründe, warum mehr Features nicht automatisch besser ist.

🌌 Curse of Dimensionality

Mit steigender Anzahl Features wächst der Feature-Raum exponentiell. Gleich viele Datenpunkte werden immer spärlicher verteilt – das Modell kann keine verlässlichen Muster lernen.

🗑️ Garbage In / Garbage Out

Irrelevante oder rauschbehaftete Features schaden dem Modell aktiv. Das Modell lernt Muster in Zufallsdaten – es passt sich an das Rauschen an (Overfitting).

🪒 Occam's Razor

So wenige Features wie nötig, so viele wie erforderlich. Einfachere Modelle generalisieren besser auf ungesehene Daten und sind interpretierbarer.

⚡ Effizienz

Weniger Features = kürzere Trainingszeiten, geringerer Speicherbedarf und schnellere Inferenz. Besonders wichtig bei großen Datensätzen.

7.3 Occam's Razor & Garbage In / Garbage Out

Zwei Prinzipien, die Feature Selection begründen.

🪒 Occam's Razor

„Wenn zwei Modelle gleich gut erklären, bevorzuge das einfachere." In ML: das Modell mit den wenigsten relevanten Features generalisiert besser auf neue Daten und ist weniger anfällig für Overfitting.

🗑️ Garbage In / Garbage Out

Kein Algorithmus kann aus schlechten Features gute Vorhersagen machen. Irrelevante Features erzeugen Rauschen – das Modell lernt falsche Korrelationen und überanpasst sich an den Trainingssatz.

Ziel der Selection

Die minimale Menge von Features, die maximalen Informationsgehalt über das Zielmerkmal (Label) trägt, ohne redundante oder irrelevante Features zu enthalten.

7.4 Einfache Feature Selection: VarianceThreshold

Entfernt Features, deren Varianz unter einem Schwellwert liegt – ohne das Label zu berücksichtigen.

1

Missing Values

Features mit sehr vielen fehlenden Werten tragen kaum Information. Vor VarianceThreshold: fehlende Werte auffüllen oder Feature direkt entfernen.

NaN-Anteil ≥ Schwelle → entfernen
2

Constant Values

Features, die immer denselben Wert haben (Varianz = 0), enthalten keinerlei Information und können ohne Informationsverlust entfernt werden.

threshold = 0.0
3

Low Variance

Features mit sehr geringer, aber nicht-null Varianz sind fast konstant und tragen wenig Diskriminationskraft. Threshold nach Datenlage wählen.

threshold = 0.15 (Beispiel)

Python – VarianceThreshold (aus Notebook) 
import numpy as np
from sklearn.feature_selection import VarianceThreshold

# Eingabe-Daten (6 Samples, 3 Features)
X = np.array([[0, 0, 1],
              [0, 1, 0],
              [1, 0, 0],
              [0, 1, 1],
              [0, 1, 0],
              [0, 1, 1]])

# Schritt 1: Varianz prüfen
X.var(axis=0)

# Schritt 2: Threshold anwenden
sel = VarianceThreshold(threshold=.15)
sel.fit_transform(X)

7.5 Korrelations-basierte Feature Selection

Stark korrelierende Features liefern dieselbe Information doppelt – eines davon kann entfernt werden.

Features untereinander

Wenn zwei Features hoch miteinander korrelieren (z. B. r > 0.9), sind sie redundant. Eines der beiden kann ohne Informationsverlust entfernt werden.

Methode: Paarweise Korrelationsmatrix berechnen, dann Paare mit Korrelation > Schwelle identifizieren und das Feature mit geringerer Target-Korrelation entfernen.

Features mit Target

Features, die kaum mit dem Target-Label korrelieren, tragen wenig Vorhersagekraft. Methode: Korrelation Feature ↔ Target berechnen und Top-N Features auswählen.

Aber Vorsicht: Korrelation misst nur lineare Zusammenhänge. Für nicht-lineare: mutual_info (→ SelectKBest).

7.6 SelectKBest

Wählt automatisch die K besten Features nach einem statistischen Test aus – unter Berücksichtigung des Labels.

Python – SelectKBest (aus Notebook) 
from sklearn.datasets import load_digits
from sklearn.feature_selection import SelectKBest, mutual_info_classif

# Digits laden: 1797 Samples × 64 Features
X, y = load_digits(return_X_y=True)
print(X.shape)        # (1797, 64)

# Die 20 besten Features nach mutual information wählen
sel   = SelectKBest(mutual_info_classif, k=20)
X_new = sel.fit_transform(X, y)

print(X_new.shape)    # (1797, 20)

Alle Scoring-Funktionen für SelectKBest

🏷️ Klassifikation
mutual_info_classif Misst die gegenseitige Information zwischen Feature und Zielvariable. Erkennt auch nicht-lineare Abhängigkeiten. → Standard-Wahl für Klassifikation
f_classif ANOVA F-Test. Misst die Varianz zwischen den Klassen relativ zur Varianz innerhalb der Klassen. Nur für lineare Abhängigkeiten. → Gut wenn lineare Trennbarkeit bekannt ist
chi2 Chi-Quadrat-Test. Nur für nicht-negative Features (z. B. Häufigkeiten, Bag-of-Words). Testet Unabhängigkeit von Feature und Label. → Nur für ganzzahlige, nicht-negative Features
📈 Regression
mutual_info_regression Misst die gegenseitige Information für kontinuierliche Zielvariablen. Erkennt auch nicht-lineare Abhängigkeiten. → Standard-Wahl für Regression
f_regression Pearson-Korrelationskoeffizient zwischen jedem Feature und dem kontinuierlichen Zielmerkmal. Nur für lineare Zusammenhänge geeignet. → Gut wenn linearer Zusammenhang erwartet
Merksatz:   mutual_info_* ist fast immer die sicherste Wahl – es erkennt auch nicht-lineare Abhängigkeiten. f_classif / f_regression sind schneller, aber nur für lineare Zusammenhänge geeignet. chi2 nur wenn Features garantiert nicht-negativ sind.
