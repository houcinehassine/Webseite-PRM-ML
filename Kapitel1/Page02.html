Lernfokus dieser Seite

Du kennst die präzise Definition eines Features, weißt welche 3 Eigenschaften gute Features ausmachen, verstehst die N×K-Feature-Matrix und erkennst, warum Feature Engineering und Modellierung ein untrennbarer iterativer Prozess sind.

2.1 Was ist ein Feature?

Von der intuitiven Idee zur formalen Definition.

"In machine learning and pattern recognition, a feature is an individual measurable property or characteristic of a phenomenon being observed."
Wikipedia – Feature (machine learning)

Operationalisierung

Features sind Merkmale – etwas, das zusammenfassend und charakterisierend bezüglich der zugrundeliegenden Daten ist. Feature Engineering bildet eine messbare Eigenschaft auf einen Skalar ab.

Features können sowohl die Daten an sich sein, als auch davon abgeleitete Größen. Meist spricht man von Features, wenn die Daten auf eine bestimmte Weise aggregiert wurden.

Warum Features?

Ein Machine Learning Modell nutzt Features, um die sogenannten Labels vorherzusagen. Features sind im Grunde nichts anderes als Arrays bzw. DataFrames.

Strikt formuliert: Feature Engineering bildet eine messbare Eigenschaft auf einen Skalar ab. Durch diesen Prozess verringert man in der Regel bereits die Dimensionalität des Problems.

So what?   Der Begriff Feature ist oft nicht scharf abzugrenzen – aus den Rohdaten erzeugt man im Prozess des Feature Engineerings eine Feature-Matrix mit sauberem Input für das Modell.

2.2 Welche Eigenschaften müssen Features haben?

Drei notwendige Bedingungen für gute Features.

📡
1. Informativ

Features müssen informativ sein und Informationen über das zugrundeliegende Phänomen tragen. Ein Feature, das keine Auskunft über das gibt, was wir vorhersagen wollen, nützt dem Modell nichts.

Trägt Information über das Phänomen
🔬
2. Diskriminativ

Features müssen diskriminativ sein und somit Unterschiede bzw. Strukturen in den Daten erklären können. Ein Feature, das für alle Klassen denselben Wert liefert, kann das Modell bei der Unterscheidung nicht unterstützen.

Erklärt Unterschiede in den Daten
🔗
3. Unabhängig

Features dürfen nicht zu stark miteinander korrelieren, um Redundanz zu vermeiden. Stark korrelierte Features liefern dieselbe Information doppelt – das erhöht die Dimensionalität ohne Mehrwert.

Geringe Korrelation untereinander
Merksatz:   Gute Features sind informativ, diskriminativ und (weitgehend) unabhängig voneinander. Fehlt eine dieser Eigenschaften, leidet die Modellgüte.

2.3 Die Feature-Matrix N × K

Wie Features formal als Matrix repräsentiert werden.

N – Samples

N ist die Anzahl der Samples (Beobachtungen). Jede Zeile der Feature-Matrix entspricht einem einzelnen Datenpunkt – z. B. einer Messung, einem Bild oder einem Dokument.

K – Features

K ist die Anzahl der Features. Jede Spalte der Feature-Matrix entspricht einem Feature – z. B. Temperatur, Frequenz, Pixel-Intensität oder Worthäufigkeit.

Kreativität im Prozess

In der Data Science kann man an vielen Stellen kreativ werden – ein zentraler Aspekt ist das Finden und Erzeugen geeigneter Features. Im Erzeugen von Features steckt Domänenwissen. Wir unterstützen das Modell durch unser Wissen.

Der Aufwand

Den meisten Aufwand im Data Science Prozess hat man mit Datenbereinigung und Feature Engineering – Modellierung macht gefühlte 20 % aus. Feature Engineering ist kein Vorschritt, sondern der Kern der Arbeit.

2.5 Iterativer Prozess: Modell vs. Features

Features und Modell beeinflussen sich gegenseitig – man optimiert nie nur das eine, ohne das andere zu berücksichtigen.

✨ Features Geeignete Features hängen
vom Phänomen und vom
gewählten Modell ab
Features fließen ins Modell →
← Modell zeigt, welche Features gut sind
🤖 Modell Die Güte des Modells hängt
maßgeblich von den Features
und deren Anzahl ab
1

Features engineeren

Basierend auf Domänenwissen und den Rohdaten werden erste Features erzeugt. Die Wahl ist zunächst eine informierte Hypothese.

2

Modell trainieren

Das Modell wird auf den erzeugten Features trainiert. Feature-Qualität und Modellgüte sind direkt verknüpft.

3

Erkenntnisse ziehen

Das Modell kann zeigen, welche Features gut funktionieren und welche nicht – z. B. über Feature Importance.

4

Features neu engineeren → Loop

Basierend auf diesen Informationen werden neue Features engineered, die wieder ins Modell fließen. Der Kreis schließt sich.

Kernaussage:   Man optimiert nicht nur das Modell, sondern auch die Features. Es ist ein Feedback-Loop zwischen beiden – Modell und Features beeinflussen sich gegenseitig.

2.6 Das Balance-Problem: Zu wenig vs. zu viele Features

Auch die Anzahl der verwendeten Features ist entscheidend – zu wenige und zu viele haben beide negative Auswirkungen.

⬇️ Zu wenig Features

Das Modell kann das Phänomen nicht ausreichend beschreiben. Wichtige Strukturen in den Daten bleiben unerkannt. Das Modell underfittet – es lernt nicht genug.

⚖️ vs.

⬆️ Zu viele Features

Das Modell ist schwierig anzulernen. Irrelevante oder redundante Features rauschen das Signal aus. → Curse of Dimensionality (mehr auf Seite 7).

Das Ziel:   Die minimale Menge an informativen, diskriminativen und unabhängigen Features, die das Phänomen vollständig beschreiben – nicht mehr und nicht weniger. Der Feedback-Loop hilft dabei.
