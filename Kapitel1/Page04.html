4.1 Das Spektrum: Domänengetrieben ↔ Datengetrieben

Feature Extraction ist kein einzelnes Verfahren – es ist ein Spektrum zwischen zwei Polen.

← Domänengetrieben Datengetrieben →
Domäne

Statistische Features

Mittelwert, Varianz, Min, Max, RMS – klassische Signal-Kenngrößen aus dem Domänenwissen.

Domäne

Frequenz-Features

FFT, Spektralleistung, Dominant-Frequenz – erfordern Wissen über physikalische Prozesse.

Hybrid

tsfresh / TSFEL

Bibliotheken, die hunderte Features automatisch berechnen – Domänenwissen hilft bei der Auswahl.

Daten

Autoencoder

Neuronales Netz, das komprimierte Features im Bottleneck lernt. Keine manuellen Features nötig.

Daten

Deep Learning End-to-End

CNN / RNN lernen Features direkt aus den Rohdaten. Maximale Flexibilität, aber weniger Interpretierbarkeit.

Faustregel:   Je mehr Domänenwissen vorhanden ist, desto weiter links (domänengetrieben) kann man ansetzen und interpretierbarere Features erzeugen. Je komplexer die Daten und je weniger Domänenwissen, desto mehr lohnt sich der datengetriebene Ansatz.

4.2 Das Zeitserien-Szenario: Motorausfall-Erkennung

Ein konkretes Beispiel – von der Messung zum Feature-Vektor.

Das Problem

Ein Elektromotor soll überwacht werden. Ziel: Motorausfall erkennen bevor er passiert, um ungeplante Stillstandszeiten zu vermeiden.

Verfügbare Daten: Stromverbrauch-Zeitserien (Messung des Stroms in Ampere über die Zeit).

Die Lösung

Aus dem Rohsignal (T Messpunkte) werden K Features extrahiert, die einen bevorstehenden Ausfall anzeigen – z. B. steigende Grundfrequenz, veränderte Harmonische oder zunehmende Varianz.

Diese Features werden dann an ein Klassifikationsmodell übergeben (z. B. Random Forest).

Stilisiertes Stromsignal eines Motors (Normalbetrieb → Anomalie)

4.3 Event-triggered vs. Kontinuierliche Zeitserien

Nicht alle Zeitserien sind gleich – die Art der Aufzeichnung bestimmt die Feature-Extraction-Strategie.

⚡ Event-triggered

Die Messung wird durch ein Ereignis ausgelöst (z. B. Motor startet, Taste wird gedrückt). Alle Zeitserien haben dann dieselbe Länge und sind am Ereignis ausgerichtet.

Länge: Fest (T immer gleich)
Ausrichtung: Am Trigger-Zeitpunkt
Vergleich: Direkt möglich
Beispiel: Motoranlauf, Schlagzeug-Hit
🔄 Kontinuierlich

Die Messung läuft ununterbrochen. Kein klares Start-/Endereignis. Für die Feature Extraction werden Fenster (Windows) über das Signal geschoben.

Länge: Variabel (Fenstergröße wählbar)
Ausrichtung: Sliding Window
Vergleich: Über Fenstergröße normierbar
Beispiel: Langzeit-EKG, Windmessung

4.4 Mathematische Formulierung

Feature Extraction aus Zeitserien als formale Abbildungsfunktion.

Die Feature-Extraction-Funktion
f :  ℝᵀ → ℝᴷ
Eine Zeitserien-Eingabe wird auf einen
K-dimensionalen Feature-Vektor abgebildet.
Ziel: In der Regel gilt K ≪ T
(Feature-Vektor viel kleiner als Rohsignal)
ℝᵀ Eingang: T Messpunkte
Die Zeitserien-Eingabe als T-dimensionaler Vektor. T = Anzahl der Zeitschritte (kann sehr groß sein – z. B. 10.000 Messpunkte bei 10 kHz über 1 Sekunde).
ℝᴷ Ausgang: K Features
Der kompakte Feature-Vektor. K = Anzahl der extrahierten Merkmale. Typisch: K ≪ T (z. B. K = 20 Features aus T = 10.000 Punkten).
f Die Abbildungsfunktion
Kann statistisch (Mittelwert, Varianz), frequenzbasiert (FFT), wavelet-basiert oder neuronalbasiert (Autoencoder) sein.
Wichtig:   Die Funktion f muss nicht differenzierbar oder parametrisch sein. Sie kann eine einfache Mittelwert-Berechnung oder ein komplexes Deep Learning Modell sein – solange sie ℝᵀ auf ℝᴷ abbildet, ist es Feature Extraction.

4.5 Exkurs: Feature-Based Time Series Analysis

Eine empirische Studie im großen Maßstab – was passiert, wenn man alles systematisch misst?

Deep Dive
Feature-Based Time Series Analysis Systematische Untersuchung von Features über reale Zeitserien aus verschiedenen Domänen
38.190
Zeitserien
9.613
Methoden
6
Kernfragen

Die Idee

Statt Features intuitiv zu wählen: alle verfügbaren Feature-Methoden auf einem riesigen, domänenübergreifenden Zeitserien-Datensatz berechnen und systematisch analysieren, welche Features tatsächlich nützlich sind.

Die Skala

38.190 reale Zeitserien aus Geophysik, Medizin, Finanzmarkt, Industrie, u.v.m. × 9.613 Feature-Berechnungsmethoden = eine Riesige Feature-Matrix, die dann analysiert wurde.

Die 6 Kernfragen (a – f)

a
Welche Features hängen zusammen?
Korrelationsanalyse und Cluster-Analyse der Feature-Methoden – welche Features messen im Wesentlichen dasselbe?
b
Welche Features sind informativ?
Welche Feature-Methoden liefern für Klassifikationsaufgaben den größten Informationsgewinn?
c
Was ist das minimale Feature-Set?
Wie viele Features braucht man wirklich? Gibt es ein kompaktes Set, das gut genug ist (Redundanzreduktion)?
d
Wie universell sind Features?
Wie gut verhalten sich dieselben Features über verschiedene Domänen (Medizin vs. Geophysik vs. Industrie)?
e
Was ist übertragbar?
Können Features, die auf einem Datensatz gut funktionieren, auf einen anderen übertragen werden? (Verbindung zu Transfer Learning)
f
Neue Features entdecken?
Können aus dem systematischen Vergleich neue Feature-Konzepte entstehen, die bisher nicht bekannt waren?
Erkenntnis:   Viele Feature-Methoden sind hoch-korreliert und damit redundant. Ein kleines Set gut gewählter Features übertrifft oft ein riesiges Set zufällig kombinierter Features. Genau das motiviert Feature Selection (Seite 7).
