01 🧱 Kernkonzepte
3 Eigenschaften guter Features

1. Informativ — trägt Information über das Phänomen
2. Diskriminativ — erklärt Unterschiede / Klassen
3. Unabhängig — geringe Korrelation untereinander

Merksatz: I-D-U
Feature-Matrix

X ∈ ℝᴺˣᴷ
N = Anzahl Samples (Zeilen)
K = Anzahl Features (Spalten)
Jeder Datenpunkt = Punkt im K-dim. Raum

X.shape → (N, K)

Die Pipeline
🗄️Quellen
⚙️Extraction
🎯Selection
⚖️Scaling
🤖Modell

← alles Teal = Feature Engineering (Kap. 1)

Aufwand-Realität

Feature Engineering = ~80% des Aufwands
Modellierung = gefühlte 20%
„Modellierung macht gefühlte 20 % aus"

Iterativer Prozess

Features ↔ Modell beeinflussen sich gegenseitig.
Nicht nur Modell optimieren – auch Features!
Feedback-Loop: Features → Train → Analyse → neue Features

02 📐 Formelsammlung
MinMax X' = (X − Xmin) / (Xmax − Xmin) × (max − min) + min Standard: min=0, max=1
Z-Score Z = (X − μ) / σ Ergebnis: μ=0, σ=1
Feature Extr. f : ℝᵀ → ℝᴷ T Zeitschritte → K Features (K ≪ T)
Haar Feature Haar = Σ(weiß) − Σ(schwarz) Ideal: 255−0=255 · Real: 229−8.75=220.25
Varianz Var(X) = Σ(xᵢ − x̄)² / N VarianceThreshold prüft pro Feature

04 ⚙️ sklearn API – Die 3 Methoden
.fit(X_train)

Lernt Parameter aus Training.
Transformiert NICHT.

enc.fit(X_train)
.transform(X)

Wendet gelernte Parameter an.
Für Train UND Test verwendbar.

enc.transform(X_test)
.fit_transform(X)

fit + transform in einem.
⚠️ NUR auf Trainingsdaten!

enc.fit_transform(X_train)
⚠️ Data Leakage – Richtig vs. Falsch 
# ✅ RICHTIG: fit nur auf Training
scaler.fit(X_train)
X_train_s = scaler.transform(X_train)
X_test_s  = scaler.transform(X_test)    # gleiche Scaler-Parameter!

# ✅ AUCH RICHTIG (Kurzform für Training):
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)

# ❌ FALSCH – Data Leakage!
X_test_s = scaler.fit_transform(X_test)  # lernt von Testdaten!
05 🎯 Feature Selection Code
VarianceThreshold
from sklearn.feature_selection import VarianceThreshold

# Varianz prüfen (vor dem Threshold)
X.var(axis=0)
# array([0.139, 0.222, 0.250])

# threshold=0.0 → nur konstante entfernen
# threshold=.15 → Feature 1 entfernt (0.139 < .15)
sel = VarianceThreshold(threshold=.15)
X_new = sel.fit_transform(X)
# (6,3) → (6,2): Feature 1 weg
SelectKBest (Digits: 64 → 20)
from sklearn.datasets import load_digits
from sklearn.feature_selection import (
    SelectKBest, mutual_info_classif
)

X, y = load_digits(return_X_y=True)
# X.shape → (1797, 64)

sel   = SelectKBest(mutual_info_classif, k=20)
X_new = sel.fit_transform(X, y)
# X_new.shape → (1797, 20)
Korrelation (Clustermap)
import seaborn as sns
corr = pd.DataFrame(X).corr()
sns.clustermap(corr, cmap='coolwarm')
# Stark korrelierende Features (r > 0.9) → eines entfernen
06 📊 SelectKBest – Alle Scoring-Funktionen
🏷️ Klassifikation
mutual_info_classif Gegenseitige Information. Auch nicht-linear. ← Standard-Wahl
f_classif ANOVA F-Test. Nur linear. Schneller.
chi2 Chi-Quadrat. Nur für nicht-negative Features! (z.B. Worthäufigkeiten)

08 🗺️ Entscheidungsguide – Wann was?
Situation Methode Begründung
Kategoriale Variable, keine Ordnung OneHotEncoder Kein künstliches Ordnungsproblem
Kategoriale Variable, Ordnung vorhanden LabelEncoder klein=0, mittel=1, groß=2 – korrekt
Sehr viele Kategorien (>1000) FeatureHasher Feste Ausgabe-Dimension, kein riesiges OHE
Text zu Features CountVectorizer Bag-of-Words, Worthäufigkeiten
Feature mit Varianz = 0 VarianceThreshold(0.0) Konstante Features entfernen
K beste Features nach Label SelectKBest(mutual_info_classif) Standard-Wahl, auch nicht-linear
Scaling, kein Ausreißer, Bereich nötig MinMaxScaler Fester Ausgabebereich [0,1]
Scaling, allgemein robust StandardScaler ← Standard μ=0, σ=1, robuster als MinMax
Viele Ausreißer vorhanden RobustScaler Skaliert über IQR statt Min/Max
Schiefe (rechtslastige) Verteilung PowerTransformer(box-cox) Nur positive Werte! → Yeo-Johnson sonst
Tree-based Algorithmen (RF, XGB) Kein Scaler nötig Nicht skalensensitiv
SVM, k-NN, PCA, Neural Nets Immer skalieren! Stark skalensensitiv
09 ⚠️ Typische Fallen & Fehler
💧
Data Leakage .fit_transform(X_test) — Teststatistiken fließen ins Training. Ergebnis zu optimistisch. ✓ Fix: fit() nur auf X_train, transform() auf X_test
🔢
Künstliche Ordnung LabelEncoder für nominale Variablen → apple=0 < banana=1 < coffee=2 (falsch!) ✓ Fix: OneHotEncoder für nominale Kategorien
📈
Curse of Dimensionality Zu viele Features → Raum wächst exponentiell, Daten werden spärlich, k-NN versagt ✓ Fix: Feature Selection (VarianceThreshold, SelectKBest)
🗑️
Garbage In / Garbage Out Irrelevante Features schaden aktiv – Modell lernt Rauschen (Overfitting) ✓ Fix: Feature Selection, Domänenwissen anwenden
🔢
Box-Cox mit negativen Werten PowerTransformer(box-cox) wirft Fehler wenn X ≤ 0 ✓ Fix: method='yeo-johnson' für alle Werte
🧭
t-SNE für Feature Extraction t-SNE-Ausgabe nicht für Training verwenden – nur Visualisierung! ✓ Fix: PCA oder Autoencoder für Feature Extraction
📏
Skalieren vor Train/Test-Split Wenn man vor dem Split skaliert, kennt der Scaler alle Testdaten → Data Leakage ✓ Fix: Erst splitten, dann fit() auf X_train
📊
chi2 mit negativen Werten chi2-Scoring erfordert nicht-negative Features (z.B. Häufigkeiten, TF-IDF) ✓ Fix: mutual_info_classif oder f_classif verwenden

11 ❓ Typische Prüfungsfragen (aufklappen)

Klicke auf eine Frage um die Antwort zu sehen.

F1 Nenne und erkläre die 3 Eigenschaften, die gute Features haben müssen. ▼
1. Informativ: Features müssen Informationen über das zugrundeliegende Phänomen tragen. Ein nicht-informatives Feature hat keinen Zusammenhang mit dem, was wir vorhersagen wollen.

2. Diskriminativ: Features müssen Unterschiede zwischen den Klassen/Strukturen in den Daten erklären können. Ein Feature, das für alle Klassen denselben Wert liefert, hilft dem Modell nicht.

3. Unabhängig: Features dürfen nicht zu stark miteinander korrelieren (Redundanz vermeiden). Hoch korrelierende Features liefern dieselbe Information doppelt ohne Mehrwert.
F2 Was ist das Problem mit Label-Encoding bei nominalen Variablen? Wie löst man es? ▼
Problem: LabelEncoder weist alphabetisch Zahlen zu (apple=0, banana=1, coffee=2, milk=3). Das Modell interpretiert diese als Reihenfolge: milk(3) > coffee(2) > banana(1) > apple(0). Diese Ordnung existiert nicht — es ist eine künstliche Ordnung.

Lösung: OneHotEncoder — erzeugt für jede Kategorie eine eigene Binär-Spalte ohne implizite Reihenfolge. Label-Encoding ist nur bei ordinalen Variablen korrekt (z.B. klein/mittel/groß).
F3 Erkläre den Unterschied zwischen .fit(), .transform() und .fit_transform(). Was ist Data Leakage? ▼
.fit(X): Lernt die Parameter aus den Daten (z.B. Min/Max für MinMaxScaler). Transformiert nicht.
.transform(X): Wendet die gelernten Parameter an. Nur für Trainingsdaten nötig, aber auf beide anwendbar.
.fit_transform(X): fit + transform in einem Schritt. Nur auf Trainingsdaten!

Data Leakage: Wenn man .fit_transform(X_test) verwendet, lernt der Scaler die Statistiken (Min, Max, μ, σ) der Testdaten — diese fließen dann indirekt ins Modelltraining ein. Das Modell "sieht" die Testdaten also schon beim Training → Ergebnisse sind zu optimistisch und nicht realistisch.
F4 Was ist der Curse of Dimensionality? Welche Algorithmen leiden besonders? ▼
Mit steigender Anzahl an Features (K) wächst der Feature-Raum exponentiell. Für gleiche Datendichte benötigt man 10ᴷ Samples. Dieselben N Datenpunkte werden im hochdimensionalen Raum immer spärlicher verteilt (Sparsity).

Besonders betroffen: k-NN (Abstände werden gleichförmig, "nächster Nachbar" bedeutungslos), SVM, k-Means. Tree-based Algorithmen leiden weniger.

Lösung: Feature Selection (VarianceThreshold, SelectKBest) oder Dimensionsreduktion (PCA, Autoencoder).
F5 Berechne den Z-Score für X=8, wenn μ=5 und σ=3. ▼
Formel: Z = (X − μ) / σ

Rechnung: Z = (8 − 5) / 3 = 3 / 3 = 1.0

Das bedeutet: X=8 liegt genau 1 Standardabweichung über dem Mittelwert.
F6 Welche Scoring-Funktion verwendest du für SelectKBest bei einem Klassifikationsproblem, wenn du nicht weißt ob der Zusammenhang linear ist? ▼
mutual_info_classif — misst die gegenseitige Information (Mutual Information) zwischen Feature und Zielvariable.

Im Gegensatz zu f_classif (ANOVA F-Test, nur linear) erkennt mutual_info_classif auch nicht-lineare Abhängigkeiten.

chi2 wäre falsch, wenn Features negative Werte haben können.
F7 Wann verwendest du MinMaxScaler, wann StandardScaler? ▼
MinMaxScaler: Wenn ein fester Ausgabebereich benötigt wird (z.B. [0,1] für Sigmoid-Aktivierungen in Neural Nets), und keine starken Ausreißer vorhanden sind. Sensibel gegenüber Ausreißern.

StandardScaler: Standard-Wahl für die meisten Fälle. Robuster als MinMax. Gut für SVM, PCA, Lineare Modelle, k-NN. Setzt keine Normalverteilung voraus, funktioniert aber besonders gut bei annähernd normalverteilten Daten.

Faustregel: StandardScaler wenn unsicher. MinMaxScaler wenn expliziter Wertebereich nötig und keine Ausreißer.
F8 Was macht VarianceThreshold(threshold=0.15) mit diesem Array? X.var(axis=0) = [0.139, 0.222, 0.250] ▼
Antwort: Feature 1 (Varianz = 0.139) liegt unter dem Schwellwert 0.15 und wird entfernt. Features 2 (0.222) und 3 (0.250) überleben.

Eine (6×3)-Matrix wird zu einer (6×2)-Matrix — Feature 1 (erste Spalte) wird entfernt.
